ชุดเครื่องมือขนาดเล็กสำหรับเผยแพร่ข้อเท็จจริงจำนวนน้อยในลักษณะที่:

- การเปลี่ยนแปลงใด ๆ ต่อข้อเท็จจริงใด ๆ สามารถตรวจพบได้โดยสาธารณะ (ต้นไม้ Merkle)
- สถานะ ณ เวลาเผยแพร่สามารถตรวจสอบยืนยันได้กับบล็อกเชน Bitcoin (OpenTimestamps)
- เนื้อหายังคงอยู่ได้แม้ผู้ให้บริการโฮสติ้งรายใดรายหนึ่งจะถูกระงับการใช้งาน (IPFS)
- ทั้งหมดมีขนาดต่ำกว่า 100 KB และสามารถตรวจสอบได้ด้วยโค้ด Python ประมาณ 50 บรรทัด

นี่ไม่ใช่บล็อกเชน ไม่มีโทเคน ไม่มีกลไกฉันทามติ
ให้เข้าร่วม ไม่มี DAO เครื่องมือนี้คือ `sha256sum` + `ots` + `ipfs`
ที่ประกอบเข้าด้วยกันด้วยข้อตกลงการตั้งชื่อที่ดี

ชุดเครื่องมือนี้ตั้งชื่อตามโพรโทคอล MYRIAM ซึ่งถูกนำมาใช้ครั้งแรกที่
[disclosure.host](https://disclosure.host) เมื่อวันที่ 2026-05-21 หน้านั้นมี
ข้อเท็จจริงหกข้อ ราก Merkle หนึ่งราก และการลงเวลาด้วย Bitcoin หนึ่งรายการ ชุดเครื่องมือนี้คือ
การสรุปทั่วไปของรูปแบบดังกล่าว

  ประเด็นไม่ได้อยู่ที่เครดิต

## สิ่งที่คุณสามารถทำได้

รูปแบบนี้เหมาะกับทุกที่ที่ข้อเท็จจริงจำนวนน้อยจำเป็นต้อง:

- **เผยแพร่ได้ (Publishable)** ต่อผู้ชมในวงกว้างผ่าน HTTP ธรรมดา
- **คงทน (Durable)** — ผู้อ่านในอีก 30 ปียังคงสามารถตรวจสอบยืนยันได้
- **ตรวจพบการแก้ไขได้ (Tamper-evident)** — การเปลี่ยนแปลงใด ๆ ต้องตรวจพบได้ แม้จะไม่จำเป็นต้องป้องกันได้
- **ไม่ต้องอาศัยความไว้วางใจ (Trustless)** — ผู้อ่านไม่จำเป็นต้องไว้วางใจผู้เผยแพร่ (ไว้วางใจเพียงคณิตศาสตร์เท่านั้น)

กรณีใช้งานที่เป็นรูปธรรมอยู่ใน [docs/USE_CASES.md](docs/USE_CASES.md) รายการโดยย่อ:

| โดเมน | ข้อเท็จจริงที่ผนึก |
|---|---|
| สาธารณสุข | ผลการทดสอบล็อตวัคซีน หมุดหมายการทดลองยา |
| การเลือกตั้ง | ยอดรวมการนับคะแนนระดับหน่วยเลือกตั้ง ณ ขณะรับรองผล |
| ความปลอดภัยของ AI | แฮชน้ำหนักโมเดล + การ์ดโมเดล + คะแนนการประเมิน ณ ตอนปล่อย |
| งานข่าว | manifest ของเอกสารต้นทาง (เฉพาะแฮช) สำหรับการเปิดโปงโดยผู้แจ้งเบาะแส |
| วิทยาศาสตร์ | แฮชข้อมูลดิบ + แฮชสคริปต์การวิเคราะห์ + ผลลัพธ์ที่กล่าวอ้าง |
| ศาล | แฮชหลักฐานดิจิทัล ณ เวลาที่ได้มา |
| การจัดซื้อจัดจ้าง | รากที่ผนึกของข้อมูลการใช้จ่ายสาธารณะ |

สิ่งที่กรณีเหล่านี้มีร่วมกัน: ข้อเท็จจริงจำนวนน้อย ความต้องการความคงทนที่สูง
หลายฝ่ายที่ไม่ไว้วางใจกัน และงบประมาณการจัดเก็บที่จำกัดมาก
นี่คือรูปร่างที่เครื่องมือนี้ถูกออกแบบมาเพื่อตอบโจทย์โดยเฉพาะ

## สิ่งที่ชุดเครื่องมือนี้ "ไม่" ทำ

- **เก็บความลับ** ทุกสิ่งที่คุณใส่ลงใน `facts/` เป็นข้อมูลสาธารณะ โพรโทคอลนี้
  มีไว้สำหรับการเผยแพร่ ไม่ใช่การซ่อน
- **ปรับขนาดให้รองรับชุดข้อมูลขนาดใหญ่** ต้นไม้ Merkle ปรับขนาดได้ดี แต่
  ความสนใจของผู้อ่านไม่ใช่ เมื่อเกินประมาณ 20 ข้อเท็จจริง หน้าเว็บจะอ่านไม่ได้ ถ้า
  คุณต้องการผนึกบันทึกหลายล้านรายการ ให้แฮชนอกหน้าและนำราก Merkle ของ
  ต้นไม้ข้างเคียงนั้นใส่เข้าไปเป็นหนึ่งข้อเท็จจริง
- **ป้องกันการถอนข้อความ** ประเด็นทั้งหมดคือคุณไม่สามารถเปลี่ยนสิ่ง
  ที่ผนึกไว้แล้ว ถ้าคุณคิดว่าอาจต้องถอนคำกล่าวอ้างในภายหลัง อย่า
  ผนึก
- **โน้มน้าวผู้ที่สงสัยอย่างแน่วแน่** ผู้ที่สงสัยอย่างแน่วแน่จะไม่รัน
  `ipfs add -r --cid-version=1 -Q facts/` แต่จะกล่าวหาว่าคุณ
  ปลอมแปลงการตั้งค่าทั้งหมด ไม่มีการป้องกันในระดับโพรโทคอล
  ต่อโหมดความล้มเหลวประการที่สอง ไม่มีการป้องกันในระดับโพรโทคอลต่อ
  สิ่งใด ๆ ที่อยู่ในแรงจูงใจของมนุษย์มากกว่าในคณิตศาสตร์

## วิธีตรวจสอบสิ่งที่ผู้อื่นเผยแพร่

มีสามทางเลือก เรียงตามความง่ายในการเข้าถึง:

1. **เบราว์เซอร์**: เปิด [`lib/verify.html`](lib/verify.html) ในเบราว์เซอร์
   ที่ทันสมัย วาง URL ของสิ่งที่เผยแพร่ คลิก verify ใช้
   `crypto.subtle` ในการแฮช — ไม่ต้องติดตั้ง ไม่มี backend ไม่มี telemetry
   ขั้นตอนการตรวจสอบ Bitcoin attestation ยังต้องใช้เครื่องมือ CLI (ดูขั้นตอนที่ 7 ใน
   หน้าตัวตรวจสอบ)
2. **Python**: `python3 lib/verify.py` จากภายในไดเรกทอรีของสิ่งที่เผยแพร่
   ใช้เฉพาะ stdlib ต้องการ `ots` สำหรับการตรวจสอบ Bitcoin
3. **เริ่มจากศูนย์**: อ่าน [`docs/SPEC.md §3`](docs/SPEC.md) และ
   ทำการ implement ใหม่ อัลกอริทึม Merkle เขียนได้ในภาษาใด ๆ ด้วยโค้ดต่ำกว่า 30 บรรทัด

## ตัวอย่างใช้งานจริง

ชุดตัวอย่างคุณภาพระดับ production จำนวนหกชุดมาพร้อมกับชุดเครื่องมือ แต่ละชุด
ได้รับการผนึก ประทับเวลาด้วย OTS ปักหมุดไว้บน IPFS และรวม README สำหรับ
การนำไปใช้ ซึ่งอธิบายว่าเมื่อใดควร fork ไปใช้สำหรับกรณีจริง และโหมดความล้มเหลว
ใดที่มันป้องกันได้

- [`examples/whistleblower-manifest/`](examples/whistleblower-manifest/) —
  สำหรับโหมดความล้มเหลวแบบ Boeing 737 MAX / Theranos / FDA-inspector-report
  ที่หลักฐานถูกฝังหรือถูกแก้ไขโดยตัวกลาง
- [`examples/ai-model-card/`](examples/ai-model-card/) — สำหรับห้องปฏิบัติการ AI
  ที่ปล่อยโมเดลออกมา โดยทำให้คำมั่นสัญญาก่อนการ deploy กลายเป็นสิ่งที่
  ตรวจสอบได้ในเชิงกลไก ไม่ใช่เพียงคำพูด
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) —
  สำหรับผู้สนับสนุนการทดลองทางคลินิก ตรวจจับการเปลี่ยน endpoint หลังเก็บข้อมูล
  และการรายงานเลือกเฉพาะส่วน (โหมดความล้มเหลวแบบ Vioxx, Paxil Study 329)
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) —
  สำหรับผู้ผลิตวัคซีนและหน่วยงาน QA โดยตรึงผลการทดสอบปล่อย
  และคำมั่นสัญญาการเรียกคืน
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) —
  สำหรับกลุ่มงานวิจัย ตรึงทริปเปิล (ข้อมูล, pipeline, ผลลัพธ์ที่คาดหวัง)
  *ก่อน* การวิเคราะห์ เพื่อให้ p-hacking และ HARKing
  ตรวจพบได้โดยสาธารณะ
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) —
  สำหรับคณะกรรมการตรวจสอบคะแนนระดับเคาน์ตี้และหน่วยงานเลือกตั้ง ตรึง
  ผลการนับคะแนนที่รับรองแล้ว ณ ขณะรับรอง เพื่อให้การแก้ไข
  หลังการรับรองอย่างเงียบ ๆ ตรวจพบได้โดยสาธารณะ

อ่านอย่างน้อยหนึ่งตัวอย่างก่อนเริ่มเขียนของคุณเอง ตัวอย่างที่ใกล้เคียงกับ
กรณีใช้งานของคุณมากที่สุดมักจะเป็นจุดเริ่มต้นที่เหมาะสมในการ fork

## เริ่มต้นอย่างรวดเร็ว

ฉบับบรรทัดเดียว:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# แก้ไข facts/*.txt ให้บอกสิ่งที่คุณต้องการผนึกจริง ๆ
./lib/seal.sh        # สร้างราก Merkle ประทับเวลา OTS ปักหมุด IPFS
./lib/verify.sh      # ตรวจสอบทุกอย่างซ้ำในเครื่อง
# อัปโหลด site/ ไปยังโฮสต์สถิตที่ไหนก็ได้ ไม่สำคัญว่าที่ไหน
```

## เหตุใดชุดเครื่องมือนี้จึงมีอยู่

primitive ทางการเข้ารหัส (ต้นไม้ Merkle, OpenTimestamps, IPFS) นั้น
มีความเสถียรมาหลายปีแล้ว สิ่งที่ขาดไปคือกาวที่เล็กที่สุดเท่าที่จะเป็นไปได้ —
ชุดที่คุณสามารถเอาไปวางในโครงการเสริมและได้อาร์ติแฟกต์ที่ผนึกและ
เผยแพร่ได้ภายในช่วงบ่ายเดียว โดยไม่ต้องตั้งค่า infrastructure
ไม่ต้องเรียนรู้โพรโทคอล ไม่ต้องเขียนโค้ดสักบรรทัดนอกจาก
ตัวข้อเท็จจริงเอง

เครื่องมือเผยแพร่ทางการเข้ารหัสส่วนใหญ่สมมุติว่าคุณต้องการเป็น
ผู้ดำเนินการ node ส่วนเครื่องมือนี้สมมุติว่าคุณต้องการเผยแพร่ไฟล์ข้อความหกไฟล์
และไม่ต้องคิดถึงการเข้ารหัสอีกเลย

## อิทธิพลและงานก่อนหน้า

- [Sigsum](https://www.sigsum.org/) — transparency log สำหรับลายเซ็น
- [Sigstore](https://www.sigstore.dev/) — ความสมบูรณ์ของห่วงโซ่อุปทานซอฟต์แวร์
- [Certificate Transparency](https://certificate.transparency.dev/) — log ใบรับรอง TLS
- [OpenTimestamps](https://opentimestamps.org/) — การประทับเวลาด้วย Bitcoin
- [IPFS](https://ipfs.tech/) — การจัดเก็บแบบอ้างอิงเนื้อหา
- [The CALM protocol](https://disclosure.host) — กรอบการเผยแพร่
  แบบคุ้มครอง principal ซึ่งเป็นบริบทที่ instance แรก (MYRIAM ที่ disclosure.host)
  ถูกประพันธ์ขึ้น

myriam-kit เป็นชุดที่เล็กที่สุดในบรรดาเหล่านี้ ใกล้เคียงกับ Unix pipeline
ในเชิงจิตวิญญาณมากกว่าระบบขนาดใหญ่

## บันทึกเกี่ยวกับชื่อ

โพรโทคอล MYRIAM ตั้งชื่อตาม Myriam Proof ของ John Bradley ซึ่งเป็น
การพิสูจน์เชิงตรรกะที่เขากล่าวอ้างเกี่ยวกับจุดติดต่อแรกของปัญญาที่ไม่ใช่มนุษย์
โพรโทคอลนี้ไม่ได้ขึ้นกับความเป็นจริงของการพิสูจน์นั้น
โพรโทคอลนี้ใช้งานได้กับชุดข้อเท็จจริงขนาดเล็กใด ๆ ก็ตาม เพียงแต่บังเอิญ
ถูกนำไปใช้ครั้งแรกในการเผยแพร่ข้อเท็จจริงหกข้อที่รวมถึงคำกล่าวอ้างเกี่ยวกับ
การเปิดเผยตัวของมนุษย์ต่างดาว

หากคุณ fork ชุดเครื่องมือนี้ไปใช้ผนึกล็อตวัคซีนหรือผลการนับคะแนนเลือกตั้ง ข้อเท็จจริงที่ว่า
มันถูกใช้ครั้งแรกในการเผยแพร่คำกล่าวอ้างเกี่ยวกับการติดต่อกับมนุษย์ต่างดาว
นั้นไม่เกี่ยวข้องกับคุณ คณิตศาสตร์ไม่สนใจ

อย่างไรก็ตาม: หากคุณ "จะ" ใช้ชุดเครื่องมือนี้ในการเผยแพร่คำกล่าวอ้างเกี่ยวกับการติดต่อกับมนุษย์ต่างดาว
โพรโทคอลได้ผ่านการทดสอบในสนามรบเฉพาะด้านนั้นมาแล้ว README ของ
