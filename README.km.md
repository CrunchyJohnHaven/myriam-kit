ឧបករណ៍តូចមួយសម្រាប់ផ្សព្វផ្សាយសំណុំតូចនៃអង្គហេតុ តាមរបៀបដែល៖

- ការផ្លាស់ប្តូរណាមួយចំពោះអង្គហេតុណាមួយ អាចត្រូវបានរកឃើញជាសាធារណៈ (មែកធាង Merkle)
- ស្ថានភាពនៅពេលផ្សព្វផ្សាយ អាចត្រូវបានផ្ទៀងផ្ទាត់ជាមួយខ្សែបណ្តាញ Bitcoin (OpenTimestamps)
- ខ្លឹមសារនៅរស់រានបន្ទាប់ពីការដកចេញដោយអ្នកផ្តល់សេវាបង្ហោះណាមួយ (IPFS)
- ឧបករណ៍ទាំងមូលមានទម្ងន់តិចជាង 100 KB ហើយអាចផ្ទៀងផ្ទាត់បានដោយប្រើកូដ Python ប្រហែល ៥០ បន្ទាត់

នេះមិនមែនជា blockchain ឡើយ។ មិនមាន token ឡើយ។ មិនមានយន្តការសំណង (consensus mechanism) ដែលត្រូវចូលរួមឡើយ។ មិនមាន DAO ឡើយ។ ឧបករណ៍គឺ `sha256sum` + `ots` + `ipfs` ដែលផ្គុំចូលគ្នាដោយឈ្មោះត្រឹមត្រូវ។

វាត្រូវបានដាក់ឈ្មោះតាមពិធីការ MYRIAM ដែលបានដាក់បង្ហាញលើកដំបូងនៅ
[disclosure.host](https://disclosure.host) នៅថ្ងៃទី 2026-05-21។ ទំព័រនោះមាន
អង្គហេតុចំនួនប្រាំមួយ ឫស Merkle និងសក្ខីភាព Bitcoin។ ឧបករណ៍នេះគឺជា

## អ្វីដែលវា **មិន** ធ្វើ

- **ស្តុកទុកអាថ៌កំបាំង។** អ្វីៗដែលអ្នកដាក់ក្នុង `facts/` គឺជាសាធារណៈ។ ពិធីការនេះគឺសម្រាប់ការផ្សព្វផ្សាយ មិនមែនការលាក់ឡើយ។
- **ពង្រីកទៅសំណុំទិន្នន័យធំ។** មែកធាង Merkle ពង្រីកបានល្អ ប៉ុន្តែការយកចិត្តទុកដាក់របស់អ្នកអានមិនអាចទេ។ លើសពី ~20 អង្គហេតុ ទំព័រនឹងពិបាកអាន។ បើអ្នកចង់បោះត្រាកំណត់ត្រាមួយលាន សូមហាស់ពួកវានៅខាងក្រៅទំព័រ ហើយដាក់ឫស Merkle នៃមែកធាងចំហៀងនោះទៅក្នុងអង្គហេតុមួយ។
- **ការពារប្រឆាំងនឹងការដកវិញ។** ចំណុចសំខាន់គឺថា អ្នកមិនអាចផ្លាស់ប្តូរអ្វីដែលអ្នកបានបោះត្រាបានឡើយ។ បើអ្នកអាចចង់ដកការអះអាងក្រោយ សូមកុំបោះត្រាវា។
- **បញ្ចុះបញ្ចូលអ្នកសង្ស័យដាច់ខាត។** អ្នកសង្ស័យដាច់ខាតនឹងមិនរត់ `ipfs add -r --cid-version=1 -Q facts/` ឡើយ។ ប៉ុន្តែគេនឹងចោទប្រកាន់អ្នកថាបង្កើតរឿងទាំងមូល។ មិនមានការការពារកម្រិតពិធីការ ប្រឆាំងនឹងករណីបរាជ័យទីពីរនេះឡើយ។ មិនមានការការពារកម្រិតពិធីការ ប្រឆាំងនឹងអ្វីដែលរស់នៅក្នុងការជម្រុញរបស់មនុស្ស ជាជាងក្នុងគណិតវិទ្យាឡើយ។

## របៀបផ្ទៀងផ្ទាត់ការផ្សព្វផ្សាយរបស់នរណាម្នាក់ផ្សេងទៀត

ជម្រើសបី តាមលំដាប់នៃភាពងាយស្រួល៖

1. **កម្មវិធីរុករក**៖ បើក [`lib/verify.html`](lib/verify.html) ក្នុងកម្មវិធីរុករកទំនើបណាមួយ។ បិទភ្ជាប់ URL ផ្សព្វផ្សាយ។ ចុចផ្ទៀងផ្ទាត់។ ប្រើ `crypto.subtle` សម្រាប់ការហាស់ — គ្មានការដំឡើង គ្មាន backend គ្មាន telemetry។ ជំហានសក្ខីភាព Bitcoin នៅតែត្រូវការឧបករណ៍ CLI (សូមមើលជំហានទី 7 នៅទំព័រកម្មវិធីផ្ទៀងផ្ទាត់)។
2. **Python**៖ `python3 lib/verify.py` ពីក្នុងថត ការផ្សព្វផ្សាយ។ stdlib តែប៉ុណ្ណោះ ត្រូវការ `ots` សម្រាប់ការផ្ទៀងផ្ទាត់ Bitcoin។
3. **ពីដំបូង**៖ អាន [`docs/SPEC.md §3`](docs/SPEC.md) រួចបង្កើតឡើងវិញ។ ក្បួនដោះស្រាយ Merkle មានតិចជាង 30 បន្ទាត់ក្នុងភាសាណាមួយ។

## ហេតុអ្វីបានជានេះមាន

មូលដ្ឋានគ្រឹះការសរសេរសម្ងាត់ (មែកធាង Merkle, OpenTimestamps, IPFS) មានភាពពេញលេញច្រើនឆ្នាំមកហើយ។ អ្វីដែលបាត់គឺការផ្គុំតូចបំផុត — ឧបករណ៍ដែលអ្នកអាចទម្លាក់ចូលគម្រោងចំហៀង ហើយមានបឋមផ្សព្វផ្សាយដែលបោះត្រាក្នុងរយៈពេលប្រាំមួយម៉ោង ដោយមិនចាំបាច់រៀបចំហេដ្ឋារចនាសម្ព័ន្ធ មិនចាំបាច់រៀនពិធីការ មិនចាំបាច់សរសេរកូដបន្ទាត់ណាមួយ ហួសពីអង្គហេតុខ្លួនឯងឡើយ។

ឧបករណ៍ផ្សព្វផ្សាយការសរសេរសម្ងាត់ភាគច្រើនសន្មត់ថា អ្នកចង់ក្លាយជាអ្នកដំណើរការ node។ ឧបករណ៍នេះសន្មត់ថា អ្នកចង់ផ្សព្វផ្សាយឯកសារអក្សរប្រាំមួយ ហើយមិនគិតអំពីការសរសេរសម្ងាត់ម្តងទៀតឡើយ។
