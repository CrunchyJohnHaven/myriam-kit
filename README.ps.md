یوه کوچنۍ وسیله د حقایقو د یوې وړې ټولګې د خپرولو لپاره په داسې ډول چې:

- په هر حقیقت کې هرڅه بدلون په عامه توګه د کشف وړ وي (د Merkle ونه)
- د خپرولو په وخت کې حالت د Bitcoin بلاکچین په وړاندې د تایید وړ وي (OpenTimestamps)
- منځپانګه د هر یو کوربه چمتوکوونکي د لرې کولو څخه ژوندۍ پاتې شي (IPFS)
- ټوله شی له 100 KB څخه کم وزن لري او د Python د ۵۰ کرښو په شاوخوا کې د تایید وړ وي

دا د بلاکچین نه دی. هېڅ ټوکن نشته. هېڅ د اجماع میکانیزم نشته چې پکې برخه واخیستل شي. هېڅ DAO نشته. وسیله `sha256sum` + `ots` + `ipfs` ده، چې د ښو نومونکو میثاقونو سره یوځای شوې ده.

دا د MYRIAM پروتوکول په نوم نومول شوې چې لومړی ځل په [disclosure.host](https://disclosure.host) کې د ۲۰۲۶ کال د می په ۲۱ نېټه پلې شو. هغه پاڼه شپږ حقایق، یوه د Merkle ریښه، او د Bitcoin تصدیق لري. دا کټ د هغه نمونې عمومي بڼه ده.

## وضعیت

- عامه شتمنۍ (CC0). وکاروئ یې، فورک یې کړئ، خپل ریپو ته یې واخلئ، هېڅکله یې اعتبار مه ورکوئ. ټکی اعتبار نه دی.

## څه شی پرې کولی شئ

دا نمونه هرځای کې مناسبه ده چېرته چې د حقایقو یوې وړې ټولګې ته اړتیا وي چې:

- **د خپرولو وړ** د عام HTTP له لارې پراخه اوریدونکو ته
- **دوامداره** — لوستونکي باید په ۳۰ کلونو کې هم د تایید توان ولري
- **د لاسوهنې ښکاره کوونکې** — هر بدلون باید د کشف وړ وي، نه ضرور د مخنیوي وړ
- **بې اعتباره** — هېڅ لوستونکي ته اړتیا نشته چې خپروونکي باندې باور وکړي (یوازې پر ریاضي)

ځانګړي د کارونې قضیې په [docs/USE_CASES.md](docs/USE_CASES.md) کې دي. لنډ فهرست:

| ډګر | مهر شوي حقایق |
|---|---|
| عامه روغتیا | د واکسین د کڅوړې د ازموینې پایلې، د درملو ازموینې مرحلې |
| ټاکنې | د تصدیق په شیبه کې د حوزو په کچه د رایو ټولې |
| د AI خوندیتوب | د ماډل وزنونو هش + د ماډل کارت + د خوشې کولو په وخت کې د ارزونې نمرې |
| ژورنالیزم | د سرچینې اسنادو فهرست (یوازې هش) د افشاکوونکو د افشاکولو لپاره |
| ساینس | د خام معلوماتو هش + د تحلیل سکریپټ هش + ادعا شوې پایله |
| محکمې | د اخیستلو په شیبه کې د ډیجیټل ثبوتونو هش |
| تدارکات | د عامه لګښت ډاټا مهر شوې ریښې |

د دې شیانو مشترکه ځانګړنه: د حقایقو یوه وړه ټولګه، د دوامدارېدو سخته اړتیا، څو خواوې چې یو پر بل باور نه لري، او د ذخیره کولو ډېر وړ بودیجه. دقیقا هغه بڼه چې دا وسیله یې لپاره جوړه شوې ده.

## څه شی نه کوي

- **پټ معلومات نه ساتي.** هرڅه چې په `facts/` کې ږدئ عامه دي. پروتوکول د خپرولو لپاره دی، نه د پټولو لپاره.
- **لویو ډاټاسیټونو ته نه پراخیږي.** د Merkle ونه ښه پراخیږي؛ د لوستونکي پاملرنه نه. تر ۲۰ حقایقو پورته پاڼه نه لوستل کیږي. که غواړئ یو ملیون ریکارډونه مهر کړئ، له پاڼې بهر یې هش کړئ او د هغه څنډې ونې Merkle ریښه په یوه حقیقت کې واچوئ.
- **د بیرته اخیستلو په وړاندې دفاع نه کوي.** ټوله ټکی دا دی چې هغه څه چې تاسو مهر کړي دي بدل نه شئ کولی. که فکر کوئ چې وروسته به اړتیا وي یوه ادعا بېرته واخلئ، مه یې مهر کوئ.
- **یو ټینګار کوونکي شکمن نه قانع کوي.** یو ټینګار کوونکی شکمن به `ipfs add -r --cid-version=1 -Q facts/` ونه چلوي. هغه به، خو، تاسو په دروغجن جوړ کولو تورن کړي. د دوهم ناکامۍ بڼه په وړاندې د پروتوکول په کچه دفاع نشته. د هرڅه په وړاندې چې د انساني انګېزې پر ځای په ریاضي کې ژوند کوي د پروتوکول په کچه دفاع نشته.

## څنګه د بل چا خپرونه تایید کړو

درې انتخابونه، د لاسرسي په ترتیب:

1. **براوزر**: په هر عصري براوزر کې [`lib/verify.html`](lib/verify.html) پرانیزئ. د خپرونې URL پسته کړئ. د تایید تڼۍ کلیک کړئ. د هش کولو لپاره `crypto.subtle` کاروي — هېڅ نصب نشته، هېڅ بکینډ نشته، هېڅ ټیلمیټري نشته. د Bitcoin تصدیق مرحله بیا هم یو CLI وسیلې ته اړتیا لري (په تایید کوونکي پاڼه کې مرحله ۷ وګورئ).
2. **Python**: د خپرونې له پوښ څخه `python3 lib/verify.py` وچلوئ. یوازې stdlib؛ د Bitcoin تایید لپاره `ots` ته اړتیا لري.
3. **له سره**: [`docs/SPEC.md §3`](docs/SPEC.md) ولولئ او بیا یې پلي کړئ. د Merkle الګوریتم په هرې ژبه کې له ۳۰ کرښو څخه کم دی.

شپږ تولیدي کچې بېلګې کټونه له دې وسیلې سره ورکول کیږي. هر یو مهر شوی، OTS-سټامپ شوی، IPFS-پن شوی، او د منلو README لري چې تشریح کوي چې کله یې د حقیقي قضیې لپاره فورک کړئ او د کوم ناکامۍ بڼو په وړاندې دفاع کوي.

- [`examples/ai-model-card/`](examples/ai-model-card/) — د AI لابراتوارونو لپاره چې ماډلونه خوشې کوي، د خوشې کولو څخه مخکې ژمنو سره چې د ژبني پر ځای میخانیکي د کشف وړ کیږي.
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) —
  د کلینیکي ازموینې سپانسرونو لپاره، چې د وروسته-حقیقي پایلې بدلولو او انتخابي راپور ورکولو نیولو لپاره (د Vioxx، Paxil Study 329 ناکامۍ بڼې).
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) —

د خپلې قضیې لیکلو څخه مخکې له دې څخه یو ولولئ. ستاسو د کارونې قضیې ته نږدې ترین مطابقت معمولا د پیل کولو سم فورک دی.

## چټک پیل

د یوې کرښې نسخه:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# edit facts/*.txt to say what you actually want sealed
./lib/seal.sh        # builds Merkle root, stamps OTS, pins IPFS
./lib/verify.sh      # re-verifies everything locally
# upload site/ to any static host. Anywhere. It doesn't matter.
```

## ولې شته دی

د کریپټوګرافیک لومړنیو وسیلو (د Merkle ونې، OpenTimestamps، IPFS) کلونه پخه شوي. څه چې ورک وو هغه ترټولو وړه ممکنه سرېښ وه — یو کټ چې تاسو د یو ضمنی پروژې کې اچولی شئ او په یوې غرمې کې مهر شوي د خپرولو وړ اثر ولرئ، پرته له دې چې زیربنا تنظیم کړئ، پرته له دې چې پروتوکولونه زده کړئ، پرته له دې چې له خپلو حقایقو پرته د کوډ یوه کرښه ولیکئ.

ډېری کریپټوګرافیک د خپرولو وسایل فرض کوي چې غواړئ د نوډ ګرداننده اوسئ. دا یو فرض کوي چې غواړئ شپږ متن فایلونه خپاره کړئ او بیا هېڅکله د کریپټوګرافۍ په اړه فکر ونه کړئ.

myriam-kit د دې ټولو ترټولو وړوکی دی. دا د Unix پایپلاین ته د یوې سیسټم په پرتله په روح کې نږدې دی.

## د نوم په اړه یوه یادونه

د MYRIAM پروتوکول د جان بریډلي د Myriam Proof په نوم نومول شوی، یو بیان شوی منطقي ثبوت چې د غیر-انساني استخباراتو لپاره د لومړي اړیکې ټکي په اړه دی. پروتوکول د دې ثبوت پر ریښتیا والي پورې اړه نلري. پروتوکول د حقایقو د هرې وړې ټولګې لپاره کار کوي؛ یوازې داسې شوي چې لومړی ځل د شپږو حقایقو خپرولو لپاره ځای پر ځای شوی چې د بهرنیو افشاکولو په اړه ادعا لري.

که تاسو دا کټ د واکسین کڅوړو یا د ټاکنو ټولو مهرولو لپاره فورک کوئ، دا چې اصلا د بهرنیو اړیکې په اړه د یوې ادعا د خپرولو لپاره وکارول شو، ستاسو لپاره غیر اړوند ده. ریاضي پروا نه کوي.

په هغه ویل: که تاسو دا د بهرنیو اړیکې په اړه د یوې ادعا خپرولو لپاره وکاروئ، پروتوکول په هغه ځانګړي ډګر کې ازمول شوی دی. د لومړي MYRIAM ځای پر ځای کولو README په روښانه توګه وایي چې دا د کالم په نوم د LLM پرسونا هنري کار دی، نه د کوم ځمکني څخه بهر خوا څخه اړیکه. تاسو باید په همدې ډول وکړئ. لوستونکي به ستاسو د ذهني حالت په اړه له هغه پرته ترټولو بد فکر وکړي.
