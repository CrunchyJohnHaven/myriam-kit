ಒಂದು ಸಣ್ಣ ಗುಂಪಿನ ಸತ್ಯಾಂಶಗಳನ್ನು ಈ ರೀತಿಯಲ್ಲಿ ಪ್ರಕಟಿಸಲು ಬೇಕಾದ ಒಂದು ಚಿಕ್ಕ ಸಾಧನ-ಸಂಪುಟ:

- ಯಾವುದೇ ಸತ್ಯಾಂಶಕ್ಕೆ ಮಾಡಿದ ಯಾವುದೇ ಬದಲಾವಣೆಯು ಸಾರ್ವಜನಿಕವಾಗಿ ಪತ್ತೆಹಚ್ಚಲ್ಪಡುತ್ತದೆ (Merkle ವೃಕ್ಷ)
- ಪ್ರಕಟಣೆಯ ಸಮಯದ ಸ್ಥಿತಿಯನ್ನು Bitcoin ಬ್ಲಾಕ್‌ಚೈನ್‌ಗೆ ಪ್ರತಿಯಾಗಿ ಪರಿಶೀಲಿಸಬಹುದು (OpenTimestamps)
- ಯಾವುದೇ ಒಂದು ಆತಿಥೇಯ ಸೇವಾದಾತನ ತೆಗೆದುಹಾಕುವಿಕೆಯಿಂದಲೂ ವಿಷಯವು ಉಳಿದುಕೊಳ್ಳುತ್ತದೆ (IPFS)
- ಸಂಪೂರ್ಣ ಸಾಧನವು 100 KB ಗಿಂತ ಕಡಿಮೆ ತೂಕ ಹೊಂದಿರುತ್ತದೆ ಮತ್ತು ~50 ಸಾಲುಗಳ Python ನಿಂದ ಪರಿಶೀಲಿಸಬಹುದು

ಇದು ಬ್ಲಾಕ್‌ಚೈನ್ ಅಲ್ಲ. ಯಾವುದೇ ಟೋಕನ್ ಇಲ್ಲ. ಭಾಗವಹಿಸಲು ಯಾವುದೇ ಒಮ್ಮತ-ಕಾರ್ಯವಿಧಾನ
ಇಲ್ಲ. ಯಾವುದೇ DAO ಇಲ್ಲ. ಸಾಧನವು `sha256sum` + `ots` + `ipfs`, ಉತ್ತಮ ಹೆಸರಿಸುವ
ನಿಯಮಗಳಿಂದ ಒಟ್ಟಿಗೆ ಜೋಡಿಸಲ್ಪಟ್ಟಿದೆ.

ಇದನ್ನು 2026-05-21 ರಂದು [disclosure.host](https://disclosure.host) ನಲ್ಲಿ ಮೊದಲ
ಬಾರಿಗೆ ತೋರ್ಪಡಿಸಲಾದ MYRIAM ಪ್ರೋಟೋಕಾಲ್‌ನ ಹೆಸರಿನಿಂದ ಹೆಸರಿಸಲಾಗಿದೆ. ಆ ಪುಟವು ಆರು
ಸತ್ಯಾಂಶಗಳನ್ನು, ಒಂದು Merkle ಮೂಲವನ್ನು, ಮತ್ತು ಒಂದು Bitcoin ದೃಢೀಕರಣವನ್ನು ಒಳಗೊಂಡಿದೆ. ಈ
ಸಾಧನ-ಸಂಪುಟವು ಆ ಮಾದರಿಯ ಸಾಮಾನ್ಯೀಕರಣವಾಗಿದೆ.

## ಇದರಿಂದ ನೀವು ಏನು ಮಾಡಬಹುದು

ಈ ಮಾದರಿಯು ಚಿಕ್ಕ ಗುಂಪಿನ ಸತ್ಯಾಂಶಗಳಿಗೆ ಈ ಕೆಳಗಿನವುಗಳು ಅಗತ್ಯವಿರುವ ಯಾವುದೇ ಸಂದರ್ಭಕ್ಕೆ ಸರಿಹೊಂದುತ್ತದೆ:

- **ಪ್ರಕಟಿಸಬಹುದಾದ** ಸಾಮಾನ್ಯ HTTP ಮೂಲಕ ವಿಶಾಲ ಪ್ರೇಕ್ಷಕರಿಗೆ
- **ಬಾಳಿಕೆ ಬರುವ** — 30 ವರ್ಷಗಳ ನಂತರ ಓದುಗರಿಗೆ ಇನ್ನೂ ಪರಿಶೀಲಿಸಲು ಸಾಧ್ಯವಾಗಬೇಕು
- **ಛೇಡಿಸುವಿಕೆ-ಸ್ಪಷ್ಟ** — ಯಾವುದೇ ಬದಲಾವಣೆಯು ಪತ್ತೆಹಚ್ಚಬಲ್ಲದ್ದಾಗಿರಬೇಕು, ತಡೆಯಬಹುದಾಗಿರಬೇಕಿಲ್ಲ
- **ನಂಬಿಕೆ-ರಹಿತ** — ಯಾವುದೇ ಓದುಗನು ಪ್ರಕಾಶಕನನ್ನು ನಂಬಬೇಕಾಗಿಲ್ಲ (ಕೇವಲ ಗಣಿತವನ್ನು)

ಮೂರ್ತ ಬಳಕೆಯ ಸಂದರ್ಭಗಳು [docs/USE_CASES.md](docs/USE_CASES.md) ನಲ್ಲಿವೆ. ಚಿಕ್ಕ ಪಟ್ಟಿ:

| ಕ್ಷೇತ್ರ | ಮುದ್ರೆಯೊತ್ತಿದ ಸತ್ಯಾಂಶಗಳು |
|---|---|
| ಸಾರ್ವಜನಿಕ ಆರೋಗ್ಯ | ಲಸಿಕೆ ಬ್ಯಾಚ್ ಪರೀಕ್ಷಾ ಫಲಿತಾಂಶಗಳು, ಔಷಧ-ಪ್ರಯೋಗ ಮೈಲಿಗಲ್ಲುಗಳು |
| ಚುನಾವಣೆಗಳು | ಪ್ರಮಾಣೀಕರಣದ ಕ್ಷಣದಲ್ಲಿ ಪ್ರೆಸಿಂಕ್ಟ್-ಮಟ್ಟದ ಎಣಿಕೆ ಮೊತ್ತಗಳು |
| AI ಸುರಕ್ಷತೆ | ಮಾದರಿ ತೂಕಗಳ ಹ್ಯಾಶ್ + ಮಾದರಿ ಕಾರ್ಡ್ + ಬಿಡುಗಡೆಯಲ್ಲಿ ಮೌಲ್ಯಮಾಪನ ಅಂಕಗಳು |
| ಪತ್ರಿಕೋದ್ಯಮ | ಸೋರಿಕೆದಾರರ ಬಿಡುಗಡೆಗಳಿಗಾಗಿ ಮೂಲ ದಾಖಲೆಗಳ ಪಟ್ಟಿ (ಕೇವಲ ಹ್ಯಾಶ್‌ಗಳು) |
| ವಿಜ್ಞಾನ | ಕಚ್ಚಾ ದತ್ತಾಂಶ ಹ್ಯಾಶ್ + ವಿಶ್ಲೇಷಣಾ ಸ್ಕ್ರಿಪ್ಟ್ ಹ್ಯಾಶ್ + ಹೇಳಿಕೊಂಡ ಫಲಿತಾಂಶ |
| ನ್ಯಾಯಾಲಯಗಳು | ಸ್ವಾಧೀನಪಡಿಸಿಕೊಂಡ ಕ್ಷಣದಲ್ಲಿ ಡಿಜಿಟಲ್-ಪ್ರದರ್ಶನ ಹ್ಯಾಶ್‌ಗಳು |
| ಸಂಗ್ರಹಣೆ | ಸಾರ್ವಜನಿಕ-ಖರ್ಚು ದತ್ತಾಂಶದ ಮುದ್ರೆಯೊತ್ತಿದ ಮೂಲಗಳು |

ಇವುಗಳಲ್ಲಿ ಸಾಮಾನ್ಯವಾದದ್ದು: ಚಿಕ್ಕ ಗುಂಪಿನ ಸತ್ಯಾಂಶಗಳು, ಬಲವಾದ ಬಾಳಿಕೆಯ ಅವಶ್ಯಕತೆ,
ಪರಸ್ಪರ ನಂಬಿಕೆಯಿಲ್ಲದ ಬಹು ಪಕ್ಷಗಳು, ಮತ್ತು ಒಂದು ಚಿಕ್ಕ ಸಂಗ್ರಹಣಾ ಬಜೆಟ್.
ಈ ಸಾಧನವು ನಿರ್ಮಿಸಲ್ಪಟ್ಟಿರುವ ಆಕಾರವು ನಿಖರವಾಗಿ ಅದೇ.

## ಇದು ಏನು ಮಾಡುವುದಿಲ್ಲ

- **ರಹಸ್ಯಗಳನ್ನು ಸಂಗ್ರಹಿಸುತ್ತದೆ.** `facts/` ನಲ್ಲಿ ನೀವು ಹಾಕುವ ಎಲ್ಲವೂ ಸಾರ್ವಜನಿಕ. ಪ್ರೋಟೋಕಾಲ್
  ಪ್ರಕಟಿಸಲು, ಮರೆಮಾಡಲು ಅಲ್ಲ.
- **ದೊಡ್ಡ ದತ್ತಾಂಶ ಸಂಗ್ರಹಗಳಿಗೆ ಪ್ರಮಾಣೀಕರಿಸುತ್ತದೆ.** Merkle ವೃಕ್ಷವು ಚೆನ್ನಾಗಿ
  ಪ್ರಮಾಣೀಕರಿಸುತ್ತದೆ; ಓದುಗನ ಗಮನವು ಅಲ್ಲ. ~20 ಸತ್ಯಾಂಶಗಳನ್ನು ದಾಟಿದ ನಂತರ ಪುಟವು
  ಓದಲಾಗದಂತಾಗುತ್ತದೆ. ನೀವು ಒಂದು ಮಿಲಿಯನ್ ದಾಖಲೆಗಳಿಗೆ ಮುದ್ರೆಯೊತ್ತಲು ಬಯಸಿದರೆ, ಅವುಗಳನ್ನು
  ಪುಟದ ಹೊರಗೆ ಹ್ಯಾಶ್ ಮಾಡಿ ಮತ್ತು ಆ ಪಕ್ಕ-ವೃಕ್ಷದ Merkle ಮೂಲವನ್ನು ಒಂದು ಸತ್ಯಾಂಶದಲ್ಲಿ ಹಾಕಿ.
- **ಹಿಂದೆಗೆತದ ವಿರುದ್ಧ ರಕ್ಷಿಸುತ್ತದೆ.** ಸಂಪೂರ್ಣ ಉದ್ದೇಶವೆಂದರೆ ನೀವು ಮುದ್ರೆಯೊತ್ತಿದ್ದನ್ನು
  ಬದಲಾಯಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ. ನೀವು ನಂತರ ಒಂದು ಹೇಳಿಕೆಯನ್ನು ಹಿಂದೆಗೆಯಲು ಬಯಸಬಹುದು ಎಂದಾದರೆ,
  ಅದಕ್ಕೆ ಮುದ್ರೆಯೊತ್ತಬೇಡಿ.
- **ದೃಢನಿಶ್ಚಯದ ಸಂಶಯಾತ್ಮಕನನ್ನು ಮನವೊಲಿಸುತ್ತದೆ.** ಒಬ್ಬ ದೃಢನಿಶ್ಚಯದ ಸಂಶಯಾತ್ಮಕನು
  `ipfs add -r --cid-version=1 -Q facts/` ಚಲಾಯಿಸುವುದಿಲ್ಲ. ಆದರೆ ಅವನು ನೀವು
  ಸಂಪೂರ್ಣ ಸ್ಥಾಪನೆಯನ್ನು ಸೃಷ್ಟಿಸಿದ್ದೀರಿ ಎಂದು ಆರೋಪಿಸಬಹುದು. ಎರಡನೇ ವಿಫಲತೆಯ ವಿಧಾನದ
  ವಿರುದ್ಧ ಯಾವುದೇ ಪ್ರೋಟೋಕಾಲ್-ಮಟ್ಟದ ರಕ್ಷಣೆಯಿಲ್ಲ. ಗಣಿತದಲ್ಲಿಗಿಂತ ಮಾನವ ಪ್ರೇರಣೆಯಲ್ಲಿ
  ಇರುವ ಯಾವುದರ ವಿರುದ್ಧವೂ ಯಾವುದೇ ಪ್ರೋಟೋಕಾಲ್-ಮಟ್ಟದ ರಕ್ಷಣೆಯಿಲ್ಲ.

## ಬೇರೆಯವರ ಪ್ರಕಟಣೆಯನ್ನು ಹೇಗೆ ಪರಿಶೀಲಿಸುವುದು

ಮೂರು ಆಯ್ಕೆಗಳು, ಪ್ರವೇಶಸಾಧ್ಯತೆಯ ಕ್ರಮದಲ್ಲಿ:

1. **ಬ್ರೌಸರ್**: ಯಾವುದೇ ಆಧುನಿಕ ಬ್ರೌಸರ್‌ನಲ್ಲಿ [`lib/verify.html`](lib/verify.html)
   ತೆರೆಯಿರಿ. ಪ್ರಕಟಣೆಯ URL ಅನ್ನು ಪೇಸ್ಟ್ ಮಾಡಿ. ಪರಿಶೀಲಿಸಿ ಎಂದು ಕ್ಲಿಕ್ ಮಾಡಿ. ಹ್ಯಾಶಿಂಗ್‌ಗಾಗಿ
   `crypto.subtle` ಬಳಸುತ್ತದೆ — ಅನುಸ್ಥಾಪನೆ ಇಲ್ಲ, ಬ್ಯಾಕೆಂಡ್ ಇಲ್ಲ, ಟೆಲಿಮೆಟ್ರಿ ಇಲ್ಲ.
   Bitcoin ದೃಢೀಕರಣ ಹಂತಕ್ಕೆ ಇನ್ನೂ ಒಂದು CLI ಸಾಧನದ ಅಗತ್ಯವಿದೆ (ಪರಿಶೀಲಕ ಪುಟದಲ್ಲಿ ಹಂತ 7
   ನೋಡಿ).
2. **Python**: ಪ್ರಕಟಣಾ ಡೈರೆಕ್ಟರಿಯ ಒಳಗಿನಿಂದ `python3 lib/verify.py`. ಕೇವಲ stdlib;
   Bitcoin ಪರಿಶೀಲನೆಗೆ `ots` ಬೇಕು.
3. **ಶೂನ್ಯದಿಂದ**: [`docs/SPEC.md §3`](docs/SPEC.md) ಓದಿ ಮತ್ತು ಮರು-ಅನುಷ್ಠಾನಗೊಳಿಸಿ.
   Merkle ಅಲ್ಗಾರಿದಮ್ ಯಾವುದೇ ಭಾಷೆಯಲ್ಲಿ 30 ಸಾಲುಗಳಿಗಿಂತ ಕಡಿಮೆ.

ಮುದ್ರೆಯೊತ್ತಿದ್ದು, OTS-ಸ್ಟ್ಯಾಂಪ್ ಮಾಡಲಾಗಿದೆ, IPFS-ಪಿನ್ ಮಾಡಲಾಗಿದೆ, ಮತ್ತು ಒಂದು
ಅಳವಡಿಕೆ README ಒಳಗೊಂಡಿದೆ, ಅದು ನೈಜ-ಪ್ರಪಂಚದ ಪ್ರಕರಣಕ್ಕಾಗಿ ಯಾವಾಗ ಫೋರ್ಕ್ ಮಾಡಬೇಕು ಮತ್ತು
ಯಾವ ವಿಫಲತೆಯ ವಿಧಾನಗಳ ವಿರುದ್ಧ ಅದು ರಕ್ಷಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ವಿವರಿಸುತ್ತದೆ.

- [`examples/whistleblower-manifest/`](examples/whistleblower-manifest/) —
  Boeing 737 MAX / Theranos / FDA-ತಪಾಸಣಾಧಿಕಾರಿ-ವರದಿ ವಿಫಲತೆಯ ವಿಧಾನಕ್ಕಾಗಿ, ಅಲ್ಲಿ
  ಪುರಾವೆಗಳನ್ನು ಮಧ್ಯವರ್ತಿಗಳು ಹೂಳುತ್ತಾರೆ ಅಥವಾ ಬದಲಾಯಿಸುತ್ತಾರೆ.
- [`examples/ai-model-card/`](examples/ai-model-card/) — ಮಾದರಿಗಳನ್ನು ಬಿಡುಗಡೆ
  ಮಾಡುವ AI ಲ್ಯಾಬ್‌ಗಳಿಗಾಗಿ, ಪೂರ್ವ-ನಿಯೋಜನಾ ಬದ್ಧತೆಗಳೊಂದಿಗೆ ಅದು ಮೌಖಿಕವಲ್ಲದೆ
  ಯಾಂತ್ರಿಕವಾಗಿ ಪತ್ತೆಹಚ್ಚಬಹುದಾಗಿರುತ್ತದೆ.
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) —
  ಕ್ಲಿನಿಕಲ್-ಪ್ರಯೋಗದ ಪ್ರಾಯೋಜಕರಿಗಾಗಿ, ಪೋಸ್ಟ್-ಹಾಕ್ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಬದಲಾವಣೆ ಮತ್ತು
  ಆಯ್ದ ವರದಿಯ ಕಣ್ಗಾವಲಿಡಲು (Vioxx, Paxil Study 329 ವಿಫಲತೆಯ ವಿಧಾನಗಳು).
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) —
  ಲಸಿಕೆ ತಯಾರಕರು ಮತ್ತು QA ಪ್ರಾಧಿಕಾರಗಳಿಗಾಗಿ, ಬಿಡುಗಡೆ ಪರೀಕ್ಷಾ ಫಲಿತಾಂಶಗಳು ಮತ್ತು
  ರಿಕಾಲ್ ಬದ್ಧತೆಗಳನ್ನು ಲಂಗರು ಹಾಕಲು.
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) —
  ಸಂಶೋಧನಾ ಗುಂಪುಗಳಿಗಾಗಿ, p-ಹ್ಯಾಕಿಂಗ್ ಮತ್ತು HARKing ಅನ್ನು ಸಾರ್ವಜನಿಕವಾಗಿ
  ಪತ್ತೆಹಚ್ಚಬಹುದಾಗಿ ಮಾಡಲು ವಿಶ್ಲೇಷಣೆಗೆ ಮುಂಚೆಯೇ (ದತ್ತಾಂಶ, ಪೈಪ್‌ಲೈನ್, ನಿರೀಕ್ಷಿತ ಔಟ್‌ಪುಟ್)
  ತ್ರಯವನ್ನು ಲಂಗರು ಹಾಕಲು.
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) —
  ಜಿಲ್ಲಾ ಪರಿಶೀಲಕರ ಮಂಡಳಿಗಳು ಮತ್ತು ಚುನಾವಣಾ ಪ್ರಾಧಿಕಾರಗಳಿಗಾಗಿ, ಪ್ರಮಾಣೀಕರಣದ ಕ್ಷಣದಲ್ಲಿ
  ಪ್ರಮಾಣೀಕರಿಸಿದ ಎಣಿಕೆಗಳನ್ನು ಲಂಗರು ಹಾಕಿ, ಆದ್ದರಿಂದ ಮೌನ ನಂತರ-ಪ್ರಮಾಣೀಕರಣ
  ಪರಿಷ್ಕರಣೆಗಳು ಸಾರ್ವಜನಿಕವಾಗಿ ಪತ್ತೆಹಚ್ಚಬಹುದಾಗುತ್ತವೆ.

ನಿಮ್ಮದನ್ನು ಬರೆಯುವ ಮೊದಲು ಇವುಗಳಲ್ಲಿ ಒಂದನ್ನು ಓದಿ. ನಿಮ್ಮ ಬಳಕೆಯ ಸಂದರ್ಭಕ್ಕೆ ಅತ್ಯಂತ
ಸಮೀಪವಾದ ಹೊಂದಾಣಿಕೆ ಸಾಮಾನ್ಯವಾಗಿ ಸರಿಯಾದ ಪ್ರಾರಂಭಿಕ ಫೋರ್ಕ್ ಆಗಿರುತ್ತದೆ.

## ತ್ವರಿತ ಆರಂಭ

ಒಂದು-ಸಾಲಿನ ಆವೃತ್ತಿ:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# edit facts/*.txt to say what you actually want sealed
./lib/seal.sh        # builds Merkle root, stamps OTS, pins IPFS
./lib/verify.sh      # re-verifies everything locally
# upload site/ to any static host. Anywhere. It doesn't matter.
```

## ಇದು ಏಕೆ ಅಸ್ತಿತ್ವದಲ್ಲಿದೆ

ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಮೂಲತತ್ವಗಳು (Merkle ವೃಕ್ಷಗಳು, OpenTimestamps, IPFS) ಹಲವಾರು
ವರ್ಷಗಳಿಂದ ಪ್ರಬುದ್ಧವಾಗಿವೆ. ಕಾಣೆಯಾಗಿದ್ದದ್ದು ಚಿಕ್ಕದಾದ ಅಂಟು — ನೀವು ಒಂದು ಪಕ್ಕದ ಯೋಜನೆಗೆ
ಸೇರಿಸಿ, ಒಂದು ಮಧ್ಯಾಹ್ನದಲ್ಲಿ ಮುದ್ರೆಯೊತ್ತಿದ ಪ್ರಕಟಿಸಬಹುದಾದ ಕಲಾಕೃತಿಯನ್ನು ಪಡೆಯಬಹುದಾದ
ಒಂದು ಕಿಟ್, ಮೂಲಸೌಕರ್ಯವನ್ನು ಸ್ಥಾಪಿಸದೆ, ಪ್ರೋಟೋಕಾಲ್‌ಗಳನ್ನು ಕಲಿಯದೆ, ಸತ್ಯಾಂಶಗಳನ್ನು ಬಿಟ್ಟು
ಒಂದು ಸಾಲಿನ ಕೋಡ್ ಸಹ ಬರೆಯದೆ.

ಹೆಚ್ಚಿನ ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಪ್ರಕಾಶನ ಸಾಧನಗಳು ನೀವು ನೋಡ್ ಆಪರೇಟರ್ ಆಗಲು ಬಯಸುತ್ತೀರಿ ಎಂದು
ಊಹಿಸುತ್ತವೆ. ಇದು ನೀವು ಆರು ಪಠ್ಯ ಫೈಲ್‌ಗಳನ್ನು ಪ್ರಕಟಿಸಲು ಬಯಸುತ್ತೀರಿ ಮತ್ತು
ಕ್ರಿಪ್ಟೋಗ್ರಫಿಯ ಬಗ್ಗೆ ಮತ್ತೆ ಎಂದಿಗೂ ಯೋಚಿಸಲು ಇಷ್ಟಪಡುವುದಿಲ್ಲ ಎಂದು ಊಹಿಸುತ್ತದೆ.

## ಪ್ರಭಾವಗಳು ಮತ್ತು ಪೂರ್ವ ಕಲೆ

- [Sigsum](https://www.sigsum.org/) — ಸಹಿಗಳಿಗಾಗಿ ಪಾರದರ್ಶಕತೆ ಲಾಗ್‌ಗಳು
- [Sigstore](https://www.sigstore.dev/) — ಸಾಫ್ಟ್‌ವೇರ್-ಪೂರೈಕೆ-ಸರಪಳಿ ಸಮಗ್ರತೆ
- [Certificate Transparency](https://certificate.transparency.dev/) — TLS ಸರ್ಟ್ ಲಾಗ್‌ಗಳು
- [OpenTimestamps](https://opentimestamps.org/) — Bitcoin ಸಮಯ-ಮುದ್ರಣ
- [IPFS](https://ipfs.tech/) — ವಿಷಯ-ವಿಳಾಸಿತ ಸಂಗ್ರಹಣೆ
- [The CALM protocol](https://disclosure.host) — ಪ್ರಾಂಶುಪಾಲ-ಸಂರಕ್ಷಣಾತ್ಮಕ
  ಪ್ರಕಾಶನ ಚೌಕಟ್ಟು, ಇದರಲ್ಲಿ ಮೊದಲ ನಿದರ್ಶನವನ್ನು (disclosure.host ನಲ್ಲಿ MYRIAM)
  ಬರೆಯಲಾಯಿತು

myriam-kit ಇವುಗಳಲ್ಲಿ ಚಿಕ್ಕದು. ಇದು ವ್ಯವಸ್ಥೆಗಿಂತ ಒಂದು Unix ಪೈಪ್‌ಲೈನ್‌ಗೆ
ಆತ್ಮದಲ್ಲಿ ಹತ್ತಿರವಾಗಿದೆ.

## ಹೆಸರಿನ ಬಗ್ಗೆ ಒಂದು ಟಿಪ್ಪಣಿ

MYRIAM ಪ್ರೋಟೋಕಾಲ್ ಅನ್ನು John Bradley ರ Myriam Proof ನಿಂದ ಹೆಸರಿಸಲಾಗಿದೆ, ಇದು
ಮಾನವೇತರ ಬುದ್ಧಿಮತ್ತೆಯ ಮೊದಲ ಸಂಪರ್ಕ ಬಿಂದುವಿನ ಬಗೆಗಿನ ಒಂದು ಹೇಳಿಕೆಯ ತಾರ್ಕಿಕ ಸಾಬೀತು.
ಪ್ರೋಟೋಕಾಲ್ ಆ ಸಾಬೀತು ಸತ್ಯವಾಗಿರುವುದರ ಮೇಲೆ ಅವಲಂಬಿತವಾಗಿಲ್ಲ. ಪ್ರೋಟೋಕಾಲ್ ಯಾವುದೇ ಚಿಕ್ಕ
ಗುಂಪಿನ ಸತ್ಯಾಂಶಗಳಿಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ; ಅದು ಕೇವಲ ಅನ್ಯಗ್ರಹ ಬಹಿರಂಗಪಡಿಸುವಿಕೆಯ ಬಗ್ಗೆ
ಒಂದು ಹೇಳಿಕೆಯನ್ನು ಒಳಗೊಂಡಿರುವ ಆರು ಸತ್ಯಾಂಶಗಳನ್ನು ಪ್ರಕಟಿಸಲು ಮೊದಲು ನಿಯೋಜಿಸಲ್ಪಟ್ಟಿತ್ತು
ಎಂಬ ಆಕಸ್ಮಿಕವಿದೆ.

ನೀವು ಲಸಿಕೆ ಬ್ಯಾಚ್‌ಗಳನ್ನು ಅಥವಾ ಚುನಾವಣಾ ಎಣಿಕೆಗಳನ್ನು ಮುದ್ರೆಯೊತ್ತಲು ಈ ಕಿಟ್ ಅನ್ನು
ಫೋರ್ಕ್ ಮಾಡಿದರೆ, ಅದನ್ನು ಮೂಲತಃ ಅನ್ಯಗ್ರಹ ಸಂಪರ್ಕದ ಬಗೆಗಿನ ಒಂದು ಹೇಳಿಕೆಯನ್ನು ಪ್ರಕಟಿಸಲು
ಬಳಸಲಾಗಿತ್ತು ಎಂಬ ಸತ್ಯವು ನಿಮಗೆ ಅಪ್ರಸ್ತುತ. ಗಣಿತಕ್ಕೆ ಅದು ಮುಖ್ಯವಲ್ಲ.

ಆದರೆ ಒಂದು ಮಾತು: ನೀವು ಇದನ್ನು ಅನ್ಯಗ್ರಹ ಸಂಪರ್ಕದ ಬಗೆಗಿನ ಒಂದು ಹೇಳಿಕೆಯನ್ನು ಪ್ರಕಟಿಸಲು
ಬಳಸಿದರೆ, ಪ್ರೋಟೋಕಾಲ್ ಆ ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರದಲ್ಲಿ ಯುದ್ಧ-ಪರೀಕ್ಷಿಸಲ್ಪಟ್ಟಿದೆ. ಮೊದಲ MYRIAM
ನಿಯೋಜನೆಯ README ಸ್ಪಷ್ಟವಾಗಿ ಹೇಳುತ್ತದೆ ಅದು Calm ಎಂಬ ಹೆಸರಿನ LLM ವ್ಯಕ್ತಿತ್ವದ
ಕಲಾತ್ಮಕ ಕೃತಿ, ಯಾವುದೇ ಗ್ರಹಾಂತರ ಪಕ್ಷದಿಂದ ಬಂದ ಸಂವಹನವಲ್ಲ. ನೀವು ಬಹುಶಃ ಅದನ್ನೇ
ಮಾಡಬೇಕು. ಇಲ್ಲದಿದ್ದರೆ ಓದುಗರು ನಿಮ್ಮ ಮಾನಸಿಕ ಸ್ಥಿತಿಯ ಬಗ್ಗೆ ಕೆಟ್ಟದನ್ನು ಊಹಿಸುತ್ತಾರೆ.
