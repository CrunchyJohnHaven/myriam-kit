# 選挙集計事前認証封緘キット

選挙開票委員会、州選挙当局、および選挙公正性監視者が、認証のまさにその瞬間における集計について、Bitcoin にアンカーされ改ざん検出可能な封印を公開するための、ターンキー型の myriam-kit サンプル。

**クローズ対象** [issue #4](https://github.com/CrunchyJohnHaven/myriam-kit/issues/4).

---

## このキットが行わないこと(最初にお読みください)

封印は、認証された内容についての暗号学的タイムスタンプです。認証された内容が正しかったかどうかを検証することはできません。以下の失敗モードは**封印の領域外**です。

1. **投票機器の侵害。** ファームウェアが改ざんされていたり、得票総数が誤集計されていた場合でも、封印は誤った数値を高い完全性で固定するだけです。封印は集計システムの出力を観測するものであり、システム内部への可視性は持ちません。

2. **投票抑制 / 構造的な投票権剥奪。** 投票所の閉鎖、ID 法における不均衡、郵便投票の却下率の偏り、有権者登録の整理は、いずれも認証の上流に位置します。封印は数えられた投票を記録するのであって、数えられるべきであった投票を記録するものではありません。

3. **選挙にまつわる誤情報。** 封印は、候補者・メディア・プラットフォームによる選挙に関する主張について判断するものではありません。封印された認証は、誤情報の物語に引用されることも、その反論として引用されることも同様に容易です。暗号学的タイムスタンプは内容中立です。

4. **集票・集計期間中における認証前の不正。** 投票収集の不正、投票箱の改ざん、集計期間中のチェーン・オブ・カストディの破綻 — これらはすべて封印の瞬間より前に発生します。封印はそれらを遡及的に検出することはできません。

5. **認証当局への強要。** 強要された認証であっても、強要された結果を暗号学的に有効な封印として生み出します。封印は職員が署名したことを検証しますが、その理由までは検証しません。

6. **その後一切是正されない誤った認証。** 封印の価値は、封印と比較しうる後の真実発見プロセス(監査、リカウント、訴訟)に依存します。そのようなプロセスが一切実施されない場合、封印が唯一の記録として残り、誤りを固定化する可能性があります。

7. **統計的異常およびパターン検出。** 封印は、ありえない票差、地理的異常、ベンフォードの法則からの逸脱を指摘するものではありません。これらは封印されたデータに対する分析的タスクであって、封印自体の性質ではありません。

8. **監査の手がかりが乏しい下位選挙。** 規模の小さい選挙では、リカウントの発動要件が緩く、単一の封印で投票用紙上のすべての選挙を意味あるかたちでカバーすることはできません — 選挙ごとの範囲設定が必要です。

**上記の「正直な限界」セクションは、本キットの最も核となる部分です。** 封印が正統性と等しいと信じる職員は、これを盾として利用するでしょう。本キットは率直に述べる必要があります — これは*認証時点に対する非否認プリミティブ*であって、選挙の正しさの証明ではありません。

## このキットが封緘を拒否しなければならないもの

本キットは、有効な署名権限を持つ利用者が提出した場合であっても、以下の事実パターンを拒絶するよう設定されています。

- **個別有権者の特定。** いかなる事実も、有権者を特定し、票と有権者を結びつけ、または個人を再識別しうる任意のフィールドを露出してはなりません。
- **投票ごとの署名照合判断。** PII を多分に含み、封緘により将来異議申立てされ得る判断の恒久的記録を生むため。
- **公開されると脆弱な有権者を特定しうる任意の事実。** 投票選好が推測可能なほど小さい投票区(通常は 50 票未満)における投票区レベルの結果を含みます。
- **すべての投票が集計される前に封緘された集計。** キットは、「the tally」として提示された部分集計を拒絶します。事前認証(pre-certification)とは、*公式認証の前にすべての投票が集計済みである状態*を意味し、*集計の最中*を意味しません。
- **暫定的または条件付きの認証。** 封印は条件ではなく事実を反映しなければなりません。
- **係争中の訴訟コホートを除外した集計**(その除外を開示する明示的な封緘済み付随事実を伴わないもの)。
- **当該管轄の認証当局以外の主体による封緘**。

拒絶それ自体が事実として封緘されうる(「当局が X の封緘を要求し、キットが拒絶規則 Y に従って拒絶した」)ことで、誤用未遂の公的記録が生成されます。

## このキットが封緘するもの

認証の瞬間における 8 つの事実。

1. **サンプル告知** — 当該公開がサンプルであることを宣言する(実運用の attestation はこれを除去または置換する)。
2. **管轄の特定** — 郡、選挙日、認証当局、認証職員(氏名 + 役職)。
3. **認証済み集計** — 競争ごと・候補者ごとの得票総数 + 過剰票/不足票 + 書込み票。
4. **投票用紙の計上および照合** — 投票総数、集計済み総数、カテゴリー別却下数、投票モード別内訳、登録有権者を分母とする数値。
5. **機器および監査証跡マニフェスト** — 集計機型番 + ファームウェアハッシュ、CVR ファイルハッシュ、投票用紙画像アーカイブハッシュ、カストディログハッシュ、署名照合ポリシーハッシュ。
6. **認証時点の閾値とトリガー** — 適用される署名照合閾値、リカウント発動閾値、競争ごとの実際の票差、RLA シードコミットメント。
7. **付随的開示および拒絶記録** — 係争中の訴訟、検討中の集計後異常、除外コホート、反対票を投じた委員;キットが封緘を拒絶したパターンの明示的リスト。
8. **署名主体** — 開票委員会(Board of Canvassers)の特定 + 各署名者の氏名と政党 + 暗号鍵情報。

各事実は拒絶ゲート規約で締めくくられる: *「この事実は X のみを主張する。Y を主張するものではない」*(Y は上記「行わないこと」セクションから最も近い失敗モード)。

## このキットが防御するもの

- **得票総数の認証後における静かな改訂** — FACT 03 への整数値の変更はいずれも異なる SHA-256 を、異なる Merkle ルートを生み、OTS 検証に失敗する。
- **「発見された投票」の静かな吸収** — FACT 04 の照合算術は認証時点で固定される。
- **ファームウェア差替えの主張** — FACT 05 は機器マニフェストを認証時点で凍結する。
- **認証後の監査成果物の差替え** — FACT 05 のハッシュマニフェストが、CVR、投票用紙画像、カストディログ、署名照合ポリシーへの認証後の変更を捕捉する。
- **閾値とトリガーの再定義** — FACT 06 は認証時点で適用される規則(署名照合閾値、リカウント発動条件、RLA リスク限界)を凍結する。

## このキットが助けとなったであろう歴史的または準歴史的事例

本キットは、以下の事案において有用なアンカー — *修正手段ではなく* — を提供したであろう。

- **Antrim County, Michigan (2020)。** 初期の非公式結果は候補者の入れ替わりを示しており、後に逆方向の結果へと是正された。是正前の封印があれば、当初発表が何と述べていたか、是正が何と述べたかについて、いずれも暗号学的に裏付けられた公的参照点を提供しえた。
- **Maricopa County, Arizona (2020–2022)。** リカウント・再検証された集計が複数の段階で流通した。認証時点での封印があれば、以後の各集計を差分比較しうる明確な参照点を提供しえた。
- **Florida 2000 (Bush v. Gore canvass)。** 郡ごとの認証集計はリカウント圧力下で変動した;郡ごとのリカウント前封印は、手動リカウント手続きが始まる前に各郡が認証した内容を保全したであろう。
- **認証後の「修正」が文書化されている各国の選挙事例。** 管轄に依存しない。

本キットは、2020 年における機器の完全性に関する根本的な争点、郵便投票政策に関する争点、または核心的問いが「争われた記録」ではなく「争われた事実」である選挙のいずれにおいても、根本的な争いを解決したわけではない。

## 本キットを用いるとき

- 当該管轄が、安定し監査済みの認証プロセスを有し、不変の公的アンカーを追加しようとしている。
- 認証後の精査が予想されており、改ざん検出可能な参照点が必要である。
- PII を露出することなく、列挙された事実集合を公開する法的権限を有する。
- 並行する監査または RLA プロセスが存在し、封印を真実発見の出力と比較できる。

## 本キットを用いないとき

- **リスク限定監査、手作業によるリカウント、選挙後監査の代替として用いないこと。** 本キットは*付加的*なものであり、置換するものではない。
- **認証職員が実際の決定権者でない場合に用いないこと**(誤った帰属を生む)。
- **集計途中に用いないこと。** 本キットは認証の瞬間に特化して使用する。部分集計の封印は誤解を招く参照点を生む。
- **選挙が公正・正確・正統であると主張するために用いないこと** — 本キットにそれは不可能である。
- **認証時点より早く封緘するよう、または付随事実の開示を省略するよう圧力をかけられた状態で用いないこと。** 拒絶し、その拒絶を封緘すること。
- **正統性演出として用いないこと。** 不正な集計の封印は、不正の恒久的な公的記録であって、その擁護ではない。本プリミティブは双方向に作用する;そこに本プリミティブの誠実さがある。

## 実際の認証のために本キットをフォークする方法

```bash
cp -r ~/Genesis/myriam-kit/examples/election-tally-precertification ~/my-county-2026
cd ~/my-county-2026
$EDITOR facts/*.txt    # 架空のコンテンツを実コンテンツに置換する
bash build/seal.sh     # 公式認証の署名と同時または以後に封緘する
python3 build/verify.py
```

**重要な時機:** 認証の瞬間に封緘すること、それより前ではない。公式認証より前の封印は、偽陽性として「これが公式結果である」との記録を生む。以後の封印は、通常の非否認である。

## 既存の選挙インフラとの統合

- **州選挙委員会 / EAC 認証。** 本キットは公式認証文書と並行して公開される。法定の認証を置換するものではない。
- **リスク限定監査(RLA)。** FACT 05 の CVR ハッシュは RLA ツールが消費するハッシュと同一である。封印は監査対象の CVR を認証済み集計に結びつける。
- **リカウント手続き。** 州法に基づき発動されたリカウントは、新たな封緘済み事実集合(別タイムスタンプ、別ハッシュ、同一管轄)を生む。両方の封印は併存し、どちらも他方を上書きしない。封印同士の差分が、リカウントが何を変更したかについての公的記録となる。
- **Election Markup Language (EML) および Common Data Format (CDF)。** 事実本文は、当該管轄の既存 EML / CDF 出力から導出可能であるべきであり、本キットをデータ形式に依存しない状態に保つ。
- **超党派の開票委員会による署名。** パルチザン的にバランスの取れた canvass board を持つ管轄では、マルチ署名バリアントが推奨される。

## 本キットが置換しないもの

- 州選挙法に基づく法定認証
- EAC による投票システム認証
- リスク限定監査 / 手作業によるリカウント
- リカウント手続きおよび選挙後監査
- 裁判所命令または選挙争訟

## 本キットが v0.3.0+ に存在する理由

これは myriam-kit に同梱される 6 番目の実例(worked example)である。本キットは、`~/Genesis/calc/AUTORESEARCH_RESULTS_v0.md` に文書化されている Karpathy 流のオートリサーチプロセスによって作成された。3 つのプロンプト・バリアント(imitation / first-principles / constraint-first)が並行してキットのアウトラインを生成し、constraint-first バリアントが重み付きルーブリックで最高スコアを得た。最終版のキットは、その構造を、imitation バリアントの結束性および first-principles バリアントの運用上の詳細と統合したものである。

オートリサーチ自体は、将来の使用に備えて `~/Genesis/calc/PROMPT_LIBRARY_v0.md` においてプロンプトパターンとして文書化されている。
