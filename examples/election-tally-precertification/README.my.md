# ရွေးကောက်ပွဲ ရေတွက်ချက် ကြိုတင်အသိအမှတ်ပြု တံဆိပ်ခတ် ကိရိယာ

အရေအတွက် ရေတွက်ရေး ခရိုင်ဘုတ်အဖွဲ့များ၊ ပြည်နယ်ရွေးကောက်ပွဲ အာဏာပိုင်များ နှင့် ရွေးကောက်ပွဲ ဖြူစင်မှု လေ့လာသူများအတွက် Bitcoin-anchored၊ ပြောင်းလဲမှု ပေါ်လွင်စေသော ရေတွက်ချက် တံဆိပ်ကို တိကျသော အသိအမှတ်ပြုသည့် အချိန်တွင် ထုတ်ဝေလိုသူများအတွက် အသင့်သုံး myriam-kit ဥပမာ။

## ဤကိရိယာက လုပ်ဆောင်**မပေး**သည်များ (ဦးစွာဖတ်ပါ)

တံဆိပ်သည် အသိအမှတ်ပြုခဲ့သည်များ၏ cryptographic timestamp ဖြစ်သည်။ ၎င်းသည် အသိအမှတ်ပြုခဲ့သည်များ မှန်ကန်ခြင်း ရှိ/မရှိ စစ်ဆေးပေး**မ**နိုင်ပါ။ အောက်ပါ ပျက်ကွက်မှု ပုံစံများသည် **တံဆိပ်၏ နယ်ပယ်ပြင်ပ**တွင် ရှိသည်—

1. **မဲပေးစက် အပေးအယူ။** Firmware ပြောင်းလဲခြင်း သို့မဟုတ် မဲစုစုပေါင်း လွဲမှားစွာ ရေတွက်ခြင်း ဖြစ်ပါက တံဆိပ်က မှားယွင်းသော ကိန်းကို မြင့်မားသော ဖြူစင်မှုဖြင့် ထောက်ပေးသည်။ တံဆိပ်က ရေတွက်ရေး စနစ်၏ output များကို လေ့လာသည်; စနစ်ထဲတွင် ရှိသည်များကို မမြင်နိုင်ပါ။

2. **မဲဖျောက်ဖျက်ခြင်း / ဖွဲ့စည်းပုံအရ ဆုံးရှုံးခြင်း။** မဲရုံ ပိတ်ခြင်း၊ ID ဥပဒေ ခွဲခြားဆက်ဆံမှု၊ စာတိုက်မဲ ပယ်နှုတ်နှုန်း ဘက်လိုက်မှု၊ မှတ်ပုံတင်ခြင်း ဖျက်သိမ်းခြင်းတို့သည် အသိအမှတ်ပြုခြင်း၏ အထက်ပိုင်းတွင် ရှိသည်။ တံဆိပ်က ရေတွက်ခဲ့သော မဲများကို မှတ်တမ်းတင်သည်၊ ရေတွက်သင့်ခဲ့သော မဲများကို မဟုတ်။

3. **ရွေးကောက်ပွဲ မှားယွင်းသော သတင်းအချက်အလက်။** ကိုယ်စားလှယ်လောင်းများ၊ မီဒီယာ သို့မဟုတ် ပလက်ဖောင်းများက ပြုလုပ်သော ရွေးကောက်ပွဲ တောင်းဆိုချက်များကို တံဆိပ်က ဆုံးဖြတ်မပေး။ တံဆိပ်ခတ်ထားသော အသိအမှတ်ပြုမှုကို ကာကွယ်ရေး ဇာတ်လမ်းတွင် လည်းကောင်း၊ မှားယွင်းသော သတင်းအချက်အလက် ဇာတ်လမ်းတွင် လည်းကောင်း တူညီသော လွယ်ကူမှုဖြင့် ကိုးကားနိုင်သည်။ Cryptographic timestamps သည် အကြောင်းအရာ ဘက်မလိုက်ပါ။

4. **မဲစုဆောင်းရေး သို့မဟုတ် ရေတွက်ရေး အတွင်း အသိအမှတ်ပြုခြင်းမတိုင်မီ လိမ်လည်မှု။** မဲရိတ်သိမ်းမှု ပုံမှန်မဟုတ်မှု၊ drop-box အပြောင်းအလဲ၊ ရေတွက်ရေး ဝင်းတန်းတွင် ထိန်းသိမ်းမှု ပြတ်တောက်ခြင်း — အားလုံးသည် တံဆိပ်ခတ်ချိန် မတိုင်မီ ဖြစ်ပျက်သည်။ တံဆိပ်က ၎င်းတို့ကို နောက်ပြန် ရှာဖွေ မပေးနိုင်ပါ။

5. **အသိအမှတ်ပြုသူ အာဏာပိုင်အား ဖိအားပေးခြင်း။** ဖိအားပေးခံရသော အသိအမှတ်ပြုမှု တစ်ခုက ဖိအားပေးခံရသော ရလဒ်၏ cryptographic အရ မှန်ကန်သော တံဆိပ်ကို ထုတ်လုပ်သည်။ တံဆိပ်က မှူးထမ်းတစ်ဦး လက်မှတ်ထိုးခဲ့ကြောင်း အတည်ပြုသည်၊ အဘယ်ကြောင့်ဆိုသည်ကို မဟုတ်။

6. **နောက်ပိုင်းတွင် မပြင်ဆင်ဖြစ်သော မှားယွင်းသော အသိအမှတ်ပြုခြင်းများ။** တံဆိပ်၏ တန်ဖိုးသည် ၎င်းကို နှိုင်းယှဉ်နိုင်သော နောက်ပိုင်း အမှန်တရား-ရှာဖွေရေး လုပ်ငန်းစဉ် (audit၊ ပြန်ရေတွက်ခြင်း၊ တရားစွဲဆိုခြင်း) အပေါ် မူတည်သည်။ ထိုသို့သော လုပ်ငန်းစဉ် ဘယ်တော့မှ မလုပ်ဖြစ်ပါက တံဆိပ်သည် တစ်ခုတည်းသော မှတ်တမ်းအဖြစ် ရပ်တည်နေပြီး အမှားကို ခိုင်မာစေနိုင်သည်။

7. **စာရင်းအင်း ပုံမှန်မဟုတ်မှုနှင့် ပုံစံ ထောက်လှမ်းမှု။** တံဆိပ်က မဖြစ်နိုင်သော margin များ၊ ပထဝီဝင် ပုံမှန်မဟုတ်မှုများ သို့မဟုတ် Benford-law သွေဖည်မှုများကို အလံ မထူပါ။ ၎င်းတို့သည် တံဆိပ်ခတ်ထားသော ဒေတာအပေါ် သုံးသပ်ရေး လုပ်ငန်းများ ဖြစ်ပြီး တံဆိပ်၏ ဂုဏ်သတ္တိများ မဟုတ်ပါ။

8. **Audit trail ပါးလွှာသော down-ballot ပြိုင်ပွဲများ။** သေးငယ်သော ပြိုင်ပွဲများတွင် recount trigger အားနည်းသည်; တံဆိပ်တစ်ခုတည်းသည် ပြိုင်ပွဲတစ်ခုစီ scoping မရှိဘဲ မဲပုံစံအတွင်း ပြိုင်ပွဲအားလုံးကို အဓိပ္ပါယ်ပြည့်ဝစွာ မလွှမ်းခြုံနိုင်ပါ။

**အထက်ပါ ရိုးသား-ကန့်သတ်ချက် ကဏ္ဍသည် ဤကိရိယာ၏ အဓိက ဝန်ထမ်းခြင်း အပိုင်းဖြစ်သည်။** တံဆိပ်ခတ်ခြင်း = တရားဝင်မှု ဟု ယုံကြည်သော အရာရှိများသည် ၎င်းကို အကာအကွယ်အဖြစ် အသုံးပြုကြလိမ့်မည်။ ကိရိယာက ရှင်းရှင်းလင်းလင်း ပြောရမည် — ၎င်းသည် *အသိအမှတ်ပြုသည့် အချိန်အတွက် non-repudiation primitive တစ်ခု* ဖြစ်သည်၊ ရွေးကောက်ပွဲ မှန်ကန်မှု၏ သက်သေ မဟုတ်ပါ။

## ဤကိရိယာက တံဆိပ်ခတ်ရန် **ငြင်းပယ်ရမည်** ဖြစ်သည်များ

တရားဝင် လက်မှတ်ထိုးခွင့်ရှိသော အသုံးပြုသူက အောက်ပါ အချက်အလက် ပုံစံများကို တင်သွင်းသော်လည်း ကိရိယာက ၎င်းတို့ကို ငြင်းပယ်ရန် ပြင်ဆင်ထားသည် —

- **ပုဂ္ဂိုလ်တစ်ဦးချင်း မဲဆန္ဒရှင် ဖော်ထုတ်ခြင်း။** မည်သည့် fact မှ မဲဆန္ဒရှင် အမည် ပြောခြင်း၊ မဲကို မဲဆန္ဒရှင်နှင့် ချိတ်ဆက်ခြင်း သို့မဟုတ် လူပုဂ္ဂိုလ်တစ်ဦးကို ပြန်လည် ဖော်ထုတ်နိုင်သော မည်သည့် field ကိုမှ ဖော်ပြခြင်း မပြုရ။
- **မဲတစ်ခုစီ၏ လက်မှတ်-ကိုက်ညီခြင်း ဆုံးဖြတ်ချက်များ။** PII ပြည့်နှက်နေသည်; တံဆိပ်ခတ်ခြင်းသည် နောက်ပိုင်းတွင် စိန်ခေါ်ခံရနိုင်သော ဆုံးဖြတ်ချက်များ၏ အမြဲတမ်း မှတ်တမ်းများကို ဖန်တီးသည်။
- **ထုတ်ပြန်ပါက အန္တရာယ်ရှိသော မဲဆန္ဒရှင်ကို ဖော်ထုတ်နိုင်သော မည်သည့် fact မဆို။** မဲ ရွေးချယ်မှု ခန့်မှန်းနိုင်လောက်အောင် သေးငယ်သော precinct များ (များသောအားဖြင့် <50 မဲ) ၏ precinct အဆင့် ရလဒ်များ ပါဝင်သည်။
- **မဲအားလုံး မရေတွက်ရသေးခင် တံဆိပ်ခတ်ထားသော ရေတွက်ချက်များ။** ကိရိယာက "ရေတွက်ချက်" အဖြစ် တင်ပြထားသော တစ်စိတ်တစ်ပိုင်း အရေအတွက်များကို ငြင်းပယ်သည်။ ကြိုတင်အသိအမှတ်ပြုခြင်း ဆိုသည်မှာ *မဲအားလုံး ရေတွက်ပြီး၊ တရားဝင် အသိအမှတ်ပြုခြင်း မပြုမီ* — *ရေတွက်နေစဉ်* မဟုတ်ပါ။
- **ယာယီ သို့မဟုတ် အခြေအနေအလိုက် အသိအမှတ်ပြုခြင်းများ။** တံဆိပ်က အခြေအနေတစ်ခုကို မဟုတ်ဘဲ အချက်တစ်ခုကို ထင်ဟပ်ရမည်။
- **တရားစွဲဆိုဆဲ ဖြစ်နေသော အုပ်စုများကို ဖယ်ထုတ်ထားသော ရေတွက်ချက်များ** — ဖယ်ထုတ်မှုကို ဖော်ပြသော တံဆိပ်ခတ်ထားသော companion-fact မရှိဘဲ။
- **ထိုနယ်နိမိတ်အတွက် အသိအမှတ်ပြုသူ အာဏာပိုင်မှ မဟုတ်သော တစ်စုံတစ်ဦးက တံဆိပ်ခတ်ခြင်း။**

ငြင်းပယ်ခြင်း ကိုယ်တိုင်ကို fact တစ်ခုအဖြစ် တံဆိပ်ခတ်နိုင်သည် ("ဤအာဏာပိုင်က X ၏ တံဆိပ်ခတ်ခြင်းကို တောင်းဆိုခဲ့သည်; ကိရိယာက ငြင်းပယ်-ဥပဒေ Y အရ ငြင်းပယ်ခဲ့သည်") — အလွဲသုံးစားပြုရန် ကြိုးပမ်းမှု၏ အများပြည်သူ မှတ်တမ်းကို ဖန်တီးခြင်း။

## ဤကိရိယာက တံဆိပ်ခတ်**ပေး**သည်များ

အသိအမှတ်ပြုသည့် အချိန်တွင် fact 8 ခု —

1. **ဥပမာ အသိပေးချက်** — ထုတ်ဝေမှုကို ဥပမာအဖြစ် ကြေငြာသည် (အမှန်တကယ် attestations များက ဤအရာကို ဖယ်ထုတ် သို့မဟုတ် အစားထိုးသည်)။
2. **နယ်နိမိတ် ဖော်ထုတ်ခြင်း** — ခရိုင်၊ ရွေးကောက်ပွဲ နေ့စွဲ၊ အသိအမှတ်ပြုသူ အာဏာပိုင်၊ အသိအမှတ်ပြုသူ မှူးထမ်းများ၏ အမည် + အခန်းကဏ္ဍ။
3. **အသိအမှတ်ပြုထားသော ရေတွက်ချက်** — ပြိုင်ပွဲတစ်ခုစီ၊ ကိုယ်စားလှယ်လောင်းတစ်ဦးစီ၏ မဲ စုစုပေါင်း + over/under-votes + write-ins။
4. **မဲ စာရင်းအင်းနှင့် ပြန်လည် ညှိနှိုင်းခြင်း** — ပေးခဲ့သော စုစုပေါင်း၊ ရေတွက်ခဲ့သော စုစုပေါင်း၊ အမျိုးအစားအလိုက် ပယ်နှုတ်ခဲ့သော စုစုပေါင်း၊ နည်းလမ်း ခွဲခြားမှု၊ မှတ်ပုံတင်ထားသော မဲဆန္ဒရှင် ပိုင်းခြားကိန်း။
5. **ပစ္စည်းကိရိယာနှင့် audit-trail manifest** — tabulator မော်ဒယ် + firmware hash၊ CVR ဖိုင် hash၊ မဲပုံ archive hash၊ ထိန်းသိမ်းမှု-log hash၊ လက်မှတ်-ကိုက်ညီခြင်း မူဝါဒ hash။
6. **အသိအမှတ်ပြုသည့်အခါ ကန့်သတ်ချက်များနှင့် trigger များ** — အကျိုးသက်ရောက်နေသော လက်မှတ်-ကိုက်ညီခြင်း ကန့်သတ်ချက်၊ recount-trigger ကန့်သတ်ချက်များ၊ ပြိုင်ပွဲတစ်ခုစီ၏ အမှန်တကယ် margin များ၊ RLA seed commitment။
7. **Companion ဖော်ပြချက်များနှင့် ငြင်းပယ်ခြင်း မှတ်တမ်း** — တရားစွဲဆိုဆဲ ဖြစ်နေသော အရာများ၊ ပြန်လည် သုံးသပ်နေသော ရေတွက်ပြီးနောက် ပုံမှန်မဟုတ်မှုများ၊ ဖယ်ထုတ်ထားသော အုပ်စုများ၊ သဘောမတူသော ဘုတ်အဖွဲ့ဝင်များ; ကိရိယာက တံဆိပ်ခတ်ရန် ငြင်းပယ်ခဲ့သော ပုံစံများ၏ ထင်ရှားသော စာရင်း။
8. **လက်မှတ်ထိုး ပင်ရင်း** — Canvassers ဘုတ်အဖွဲ့ ဖော်ထုတ်ခြင်း + လက်မှတ်ထိုးသူ တစ်ဦးစီ၏ အမည် + ပါတီ + cryptographic key အချက်အလက်။

Fact တစ်ခုစီသည် ငြင်းပယ်-ဂိတ်အလေ့အထဖြင့် အဆုံးသတ်သည် — *"ဤ fact က X ကိုသာ အာဆုံအဆုံပြုသည်။ ၎င်းသည် Y ကို မအာဆုံအဆုံပြုပါ"* (Y သည် "DOES NOT do" ကဏ္ဍမှ အနီးဆုံး ပျက်ကွက်မှု ပုံစံ ဖြစ်သည်)။

## ဤကိရိယာက ကာကွယ်**ပေး**သည်များ

- **အသိအမှတ်ပြုပြီးနောက် မဲစုစုပေါင်းကို တိတ်တိတ်ဆိတ်ဆိတ် ပြန်လည် ပြင်ဆင်ခြင်း** — FACT 03 ၏ မည်သည့် integer ပြောင်းလဲမှုမှ မတူသော SHA-256၊ မတူသော Merkle root ကို ထုတ်ပေးကာ OTS verification ပျက်ကွက်စေသည်။
- **"တွေ့ရှိသော မဲများ" ကို တိတ်တိတ်ဆိတ်ဆိတ် ထည့်သွင်းခြင်း** — FACT 04 ပြန်လည် ညှိနှိုင်းခြင်း ဂဏန်းသင်္ချာသည် အသိအမှတ်ပြုသည့်အခါ ပုံသေဖြစ်သည်။
- **Firmware-အစားထိုးခြင်း တောင်းဆိုချက်များ** — FACT 05 က ပစ္စည်းကိရိယာ manifest ကို အသိအမှတ်ပြုသည့်အခါ အေးခဲစေသည်။
- **အသိအမှတ်ပြုပြီးနောက် audit-artifact အစားထိုးခြင်း** — FACT 05 ၏ hash manifest က CVR၊ မဲပုံများ၊ ထိန်းသိမ်းမှု log များ သို့မဟုတ် လက်မှတ်-ကိုက်ညီခြင်း မူဝါဒ၏ post-cert ပြောင်းလဲမှု မှန်သမျှကို ဖမ်းမိသည်။
- **ကန့်သတ်ချက်နှင့် trigger ပြန်လည် သတ်မှတ်ခြင်း** — FACT 06 က အသိအမှတ်ပြုသည့်အခါ အကျိုးသက်ရောက်နေသော စည်းမျဉ်းများ (လက်မှတ်-ကိုက်ညီခြင်း ကန့်သတ်ချက်၊ recount triggers၊ RLA risk limit) ကို အေးခဲစေသည်။

## ဤကိရိယာက အထောက်အကူပြုနိုင်ခဲ့မည့် သမိုင်း သို့မဟုတ် နီးစပ်-သမိုင်း ဖြစ်ရပ်များ

ဤကိရိယာက အသုံးဝင်သော anchor တစ်ခုကို ထုတ်ပေးနိုင်ခဲ့မည် — *ပြုပြင်ချက် မဟုတ်* — ဖြစ်ရပ်များ —

- **Antrim County, Michigan (2020)။** ပထမဦးဆုံး ရှေ့ပြေး မဟုတ်သေးသော ရလဒ်များက ကိုယ်စားလှယ်လောင်း-ဖလှယ်မှု ပြသခဲ့ပြီး၊ နောက်ပိုင်းတွင် ပြောင်းပြန် ရလဒ်သို့ ပြင်ဆင်ခဲ့သည်။ ပြင်ဆင်ခြင်းမတိုင်မီ တံဆိပ်ခတ်ထားပါက မူရင်း ကြေငြာချက်က မည်သို့ဆိုခဲ့သည်နှင့် ပြင်ဆင်ချက်က မည်သို့ဆိုခဲ့သည်ကို နှိုင်းယှဉ်ရန် အများပြည်သူ ကိုးကားချက် တစ်ခု ပေးနိုင်ခဲ့မည် — ပြောင်းပြန် နှစ်ဖက်စလုံးက cryptographically အာဆုံအာဆုံပြုထားသည်ဖြစ်၍။
- **Maricopa County, Arizona (2020–2022)။** ပြန်ရေတွက်ထား/ပြန်စစ်ဆေးထားသော စုစုပေါင်းများ၏ ပြန်ပြောင်းမှု များစွာ လည်ပတ်ခဲ့သည်။ အသိအမှတ်ပြုသည့်အချိန် တံဆိပ်ခတ်ထားပါက နောက်ဆက်တွဲ ရေတွက်မှု တစ်ခုစီကို diff လုပ်နိုင်သော မရှင်းမလင်း ကိုးကားချက် တစ်ခု ပေးနိုင်ခဲ့မည်။
- **Florida 2000 (Bush v. Gore canvass)။** ခရိုင်တစ်ခုစီ၏ အသိအမှတ်ပြုထားသော စုစုပေါင်းများသည် ပြန်ရေတွက်ခြင်း ဖိအားအောက်တွင် ပြောင်းလဲခဲ့သည်; ပြန်ရေတွက်ခြင်း မတိုင်မီ၊ ခရိုင်တစ်ခုစီ တံဆိပ်ခတ်ထားပါက ခရိုင်တစ်ခုစီ၏ ပြန်ရေတွက်ခြင်း လုပ်ထုံးလုပ်နည်းများ မစတင်မီ အသိအမှတ်ပြုခဲ့သည်များကို ထိန်းသိမ်းနိုင်ခဲ့မည်။
- **အသိအမှတ်ပြုပြီးနောက် "ပြင်ဆင်ချက်များ" မှတ်တမ်းတင်ထားသော ပြည်ပ ရွေးကောက်ပွဲ အမျိုးမျိုး။** နယ်နိမိတ်-ဘက်မလိုက်။

ဤကိရိယာက 2020 ၏ စက်-ဖြူစင်မှု အကြောင်း မူရင်း အငြင်းပွားမှုများ၊ စာတိုက်မဲ မူဝါဒ အငြင်းပွားမှုများ သို့မဟုတ် မဲစုစုပေါင်း ပြန်လည် မှတ်တမ်းတင်ထားသော အရာ မဟုတ်ဘဲ မူရင်း မေးခွန်းသည် မှန်ကန်မှု အငြင်းပွားခြင်း ဖြစ်သော မည်သည့် ရွေးကောက်ပွဲ၏ မူရင်း အငြင်းပွားမှုများကို ဖြေရှင်းပေး**မ**နိုင်ပါ။

## ဤကိရိယာကို မည်သည့်အခါ သုံးသင့်သနည်း

- သင်၏ နယ်နိမိတ်တွင် တည်ငြိမ်ပြီး audit လုပ်ထားသော အသိအမှတ်ပြုခြင်း လုပ်ငန်းစဉ် ရှိပြီး၊ ပြောင်းလဲမှု မရှိနိုင်သော အများပြည်သူ anchor ထည့်လိုသည်။
- အသိအမှတ်ပြုပြီးနောက် စိစစ်မှု မျှော်လင့်ပြီး၊ ပြောင်းလဲမှု ပေါ်လွင်စေသော ကိုးကားချက် တစ်ခု လိုလားသည်။
- PII မဖော်ထုတ်ဘဲ ဖော်ပြထားသော fact set ကို ထုတ်ဝေရန် ဥပဒေအရ အာဏာ ရှိသည်။
- တံဆိပ်ကို အမှန်တရား-ရှာဖွေရေး output နှင့် နှိုင်းယှဉ်နိုင်ရန် ပြိုင်တူ audit သို့မဟုတ် RLA လုပ်ငန်းစဉ် ရှိသည်။

## ဤကိရိယာကို မည်သည့်အခါ မသုံးသင့်သနည်း

- **Risk-limiting audits၊ လက်ဖြင့် ပြန်ရေတွက်ခြင်း သို့မဟုတ် ရွေးကောက်ပွဲ ပြီးပြီးနောက် audits များ၏ အစားထိုးအဖြစ် မသုံးပါနှင့်။** ဤကိရိယာသည် *အပိုဆောင်း* ဖြစ်ပြီး အစားထိုး မဟုတ်ပါ။
- **အသိအမှတ်ပြုသူ မှူးထမ်းသည် အမှန်တကယ် ဆုံးဖြတ်သူ မဟုတ်ပါက မသုံးပါနှင့်** (မှားယွင်းသော အာဆုံအာဆုံပြုခြင်း ဖန်တီးသည်)။
- **ရေတွက်နေစဉ် မသုံးပါနှင့်။** ကိရိယာသည် အသိအမှတ်ပြုသည့်အချိန် အတွက်သာ ဖြစ်သည်။ တစ်စိတ်တစ်ပိုင်း ရေတွက်ထားသော တံဆိပ်များက လှည့်စားသော ကိုးကားချက်များကို ဖန်တီးသည်။
- **ရွေးကောက်ပွဲသည် တရားမျှတ၊ တိကျမှန်ကန် သို့မဟုတ် တရားဝင်ကြောင်း တောင်းဆိုရန် မသုံးပါနှင့်** — တံဆိပ်က ထိုသို့ မပြုနိုင်ပါ။
- **အသိအမှတ်ပြုသည့်အချိန် မတိုင်မီ တံဆိပ်ခတ်ရန် သို့မဟုတ် companion-fact ဖော်ပြခြင်းကို ဖယ်ထုတ်ရန် ဖိအားအောက်တွင် မသုံးပါနှင့်။** ငြင်းပယ်ပြီး ငြင်းပယ်ခြင်းကို တံဆိပ်ခတ်ပါ။
- **တရားဝင်မှု ပြခန်းအဖြစ် မသုံးပါနှင့်။** လိမ်လည်သော ရေတွက်ချက်တစ်ခု၏ တံဆိပ်သည် လိမ်လည်မှု၏ အမြဲတမ်း အများပြည်သူ မှတ်တမ်း တစ်ခု ဖြစ်သည်၊ ၎င်း၏ ကာကွယ်ရေး မဟုတ်ပါ။ Primitive သည် နှစ်ဖက်စလုံးကို ဖြတ်တောက်သည်; ၎င်းသည် ၎င်း၏ ရိုးသားမှု ဖြစ်သည်။

## အမှန်တကယ် အသိအမှတ်ပြုခြင်းအတွက် ဤကိရိယာကို မည်သို့ fork လုပ်မည်နည်း

```bash
cp -r ~/Genesis/myriam-kit/examples/election-tally-precertification ~/my-county-2026
cd ~/my-county-2026
$EDITOR facts/*.txt    # replace fictional content with real
bash build/seal.sh     # seal AT or after the official certification signature
python3 build/verify.py
```

**အရေးပါသော အချိန်ကိုက်ခြင်း—** အသိအမှတ်ပြုသည့်အချိန် AT ဖြင့် တံဆိပ်ခတ်ပါ၊ မတိုင်မီ မဟုတ်ပါ။ တရားဝင် အသိအမှတ်ပြုခြင်း မတိုင်မီ တံဆိပ်ခတ်ခြင်းသည် မှားယွင်းသော "ဤသည် တရားဝင် ရလဒ်" မှတ်တမ်းတစ်ခုကို ဖန်တီးသည်။ နောက်ပိုင်း တံဆိပ်ခတ်ခြင်းသည် သာမန် non-repudiation ဖြစ်သည်။

## ရှိနေပြီးသား ရွေးကောက်ပွဲ အခြေခံအဆောက်အအုံများနှင့် ပေါင်းစပ်ခြင်း

- **ပြည်နယ် ရွေးကောက်ပွဲ ဘုတ်အဖွဲ့များ / EAC အသိအမှတ်ပြုခြင်း။** ကိရိယာက တရားဝင် အသိအမှတ်ပြုခြင်း စာရွက်စာတမ်းနှင့်အတူ ထုတ်ဝေသည်။ ၎င်းသည် ဥပဒေအရ အသိအမှတ်ပြုခြင်းကို မအစားထိုးပါ။
- **Risk-limiting audits။** FACT 05 ၏ CVR hash သည် RLA tool က ဝါးရှူသော hash နှင့် တူသည်။ တံဆိပ်က audit လုပ်ထားသော CVR ကို အသိအမှတ်ပြုထားသော စုစုပေါင်းနှင့် ချည်နှောင်ထားသည်။
- **ပြန်ရေတွက်ခြင်း လုပ်ထုံးလုပ်နည်းများ။** ပြည်နယ် ဥပဒေအောက်တွင် trigger ဖြစ်သော ပြန်ရေတွက်ခြင်းသည် တံဆိပ်ခတ်ထားသော fact-set အသစ်တစ်ခု (မတူသော timestamp၊ မတူသော hashes၊ တူသော နယ်နိမိတ်) ကို ထုတ်လုပ်သည်။ တံဆိပ် နှစ်ခုစလုံး အတူ ရှိနေသည်; တစ်ခု မ ထပ်နေပါ။ တံဆိပ်များ အကြား diff သည် ပြန်ရေတွက်ခြင်းက မည်သို့ ပြောင်းလဲခဲ့သည်ကို အများပြည်သူ မှတ်တမ်း ဖြစ်သည်။
- **Election Markup Language (EML) နှင့် Common Data Format (CDF)။** Fact body များကို နယ်နိမိတ်၏ ရှိနေပြီးသား EML/CDF output များမှ ထုတ်နုတ်နိုင်သင့်ပြီး ကိရိယာကို data-format-agnostic အဖြစ် ထိန်းသိမ်းရန် ဖြစ်သည်။
- **ပါတီ နှစ်ဖက်စလုံး Board-of-canvassers လက်မှတ်ထိုးခြင်း။** ပါတီ-မျှတ canvass ဘုတ်အဖွဲ့များရှိ နယ်နိမိတ်များအတွက် Multi-signature ပုံစံကို အကြံပြုသည်။

## ဤကိရိယာက အစား**မ**ထိုးသည်များ

- ပြည်နယ် ရွေးကောက်ပွဲ ဥပဒေအောက်တွင် ဥပဒေအရ အသိအမှတ်ပြုခြင်း
- EAC မဲပေးစက် အသိအမှတ်ပြုခြင်း
- Risk-limiting audits / လက်ဖြင့် ပြန်ရေတွက်ခြင်း
- ပြန်ရေတွက်ခြင်း လုပ်ထုံးလုပ်နည်းများနှင့် ရွေးကောက်ပွဲ-အပြီး audits
- တရားရုံး အမိန့်များ သို့မဟုတ် ရွေးကောက်ပွဲ-ပြိုင်ပွဲ တရားစွဲဆိုခြင်း

## ဤကိရိယာက v0.3.0+ တွင် အဘယ်ကြောင့် ရှိနေသနည်း

ဤသည် myriam-kit နှင့်အတူ ပို့ထားသော 6th worked example ဖြစ်သည်။ ကိရိယာကို `~/Genesis/calc/AUTORESEARCH_RESULTS_v0.md` တွင် မှတ်တမ်းတင်ထားသော Karpathy-style autoresearch လုပ်ငန်းစဉ်ဖြင့် ထုတ်လုပ်ခဲ့သည်။ Prompt အမျိုးအစား သုံးခု (imitation / first-principles / constraint-first) သည် ပြိုင်တူ kit outlines များ ထုတ်ပေးခဲ့သည်; constraint-first အမျိုးအစားက အလေးပေး rubric တွင် အမြင့်ဆုံး အမှတ်ရရှိခဲ့ပြီး၊ နောက်ဆုံး kit သည် ၎င်း၏ ဖွဲ့စည်းပုံကို imitation အမျိုးအစား၏ ပူးပေါင်းမှုနှင့် first-principles အမျိုးအစား၏ လုပ်ဆောင်ရေး အသေးစိတ်နှင့် ပေါင်းစပ်ထားသည်။

Autoresearch ကိုယ်တိုင်ကို နောက်ပိုင်း သုံးစွဲရန် `~/Genesis/calc/PROMPT_LIBRARY_v0.md` တွင် prompt pattern အဖြစ် မှတ်တမ်းတင်ထားသည်။
