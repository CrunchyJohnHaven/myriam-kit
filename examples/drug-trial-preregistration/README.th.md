# Drug Trial Pre-registration Kit

*คำแปลภาษาไทย — สำหรับเวอร์ชันต้นฉบับภาษาอังกฤษ ดู [README.md](README.md)*

ตัวอย่าง myriam-kit แบบสำเร็จรูปสำหรับผู้สนับสนุนการทดลองทางคลินิกและ
นักวิจัยทางวิชาการที่ต้องการลงทะเบียนการทดลองล่วงหน้าในแบบที่ตรวจพบ
การแก้ไขได้และตรึงไว้กับ Bitcoin **เพิ่มเติมจาก** การลงทะเบียนที่
ClinicalTrials.gov, EU CTR หรือ registry ของเขตอำนาจอื่น ๆ

## เหตุใดควรลงทะเบียนล่วงหน้า *เพิ่มเติม* ภายใต้ MYRIAM?

registry การทดลองที่มีอยู่ (ClinicalTrials.gov, EU CTR, ISRCTN) มีจุด
อ่อนด้านความถูกต้องสมบูรณ์ที่ได้รับการบันทึก:

- ClinicalTrials.gov อนุญาตให้ผู้สนับสนุนอัปเดต endpoint การคำนวณ
  ขนาดตัวอย่าง และแผนการวิเคราะห์ **หลังจาก** การลงทะเบียนต้นฉบับ
  โดยวันที่แก้ไขมองเห็นได้ใน registry แต่ *เนื้อหา* ที่เปลี่ยนแปลง
  ไม่ได้ถูกตรึงเชิงการเข้ารหัสเสมอไป
- การศึกษาขนาดใหญ่หลายชิ้น (Goldacre et al., 2019; Mathieu et al., 2009)
  ได้บันทึกการสลับ endpoint หลังเหตุการณ์อย่างเป็นระบบทั่วทั้งสาขา
- การบังคับใช้ของบรรณาธิการโดยวารสาร (CONSORT, ICMJE) จับบางกรณี
  หลายกรณีลอดผ่าน

ตราผนึก MYRIAM ปิดช่องว่างนี้ รากของ Merkle + ใบรับ Bitcoin OTS
ทำให้การเปลี่ยนแปลงหลังเหตุการณ์ใด ๆ ต่อ endpoint หลักที่ลงทะเบียน
ล่วงหน้า แผนการวิเคราะห์ทางสถิติ หรือรายการ subgroup สามารถตรวจพบ
เชิงการเข้ารหัสได้ ตราผนึกเป็นเรื่องยากกว่าทาง forensic ที่จะ retcon
เทียบกับ registry เอง

ชุดเครื่องมือนี้เป็น **ส่วนเสริมของ** ไม่ใช่การทดแทน registry ของ
เขตอำนาจ การทดลองจริงควรลงทะเบียนใน registry ที่เหมาะสมและผนึก
ภายใต้ MYRIAM

## สิ่งที่ชุดเครื่องมือนี้ป้องกัน

คดีการเสียชีวิตทางประวัติศาสตร์ที่ชุดเครื่องมือนี้จะช่วยได้:

- **Vioxx (Merck, 2004)**: เหตุการณ์ทางหัวใจและหลอดเลือดเป็น endpoint
  ที่ลงทะเบียนล่วงหน้าในการทดลอง APPROVe แต่การวิเคราะห์ที่เผยแพร่ใช้
  cutoff ที่ลดการมองเห็นอันตราย ตราผนึก cutoff ที่ลงทะเบียนล่วงหน้าจะ
  ทำให้การเปลี่ยน cutoff หลังเหตุการณ์ตรวจพบได้สาธารณะก่อน paper
  ที่เผยแพร่
- **Paxil/Study 329 (GSK/SmithKline, 2001)**: ข้อมูลการทดลองเชิงลบ
  ในการฆ่าตัวตายของวัยรุ่นถูกใส่กรอบใหม่ผ่านการวิเคราะห์หลังเหตุการณ์
  การลงทะเบียนล่วงหน้าที่ผนึกของ endpoint จะทำให้การใส่กรอบใหม่ตรวจพบ
  ได้สาธารณะ
- **การทดลองมะเร็งหลายครั้ง** ที่ endpoint การมีชีวิตรอดถูกสลับเป็น
  endpoint surrogate (progression-free survival, response rate)
  เมื่อประโยชน์การมีชีวิตรอดล้มเหลวที่จะปรากฏ

**โหมดความล้มเหลวที่ตราผนึกจับโดยตรง:**

- **การสลับ endpoint**: การเลื่อน endpoint รอง "หลักเหตุการณ์" เป็น
  "หลัก" เมื่อหลักจริงล้มเหลว FACT 05 ของตราผนึกบรรจุ endpoint หลักที่
  ลงทะเบียนล่วงหน้า endpoint หลักที่เผยแพร่ต้องตรงกัน
- **การเลือก outcome cherry-picking**: การรายงานเฉพาะการวิเคราะห์ที่
  ได้ผลเท่านั้น FACT 06 บรรจุแผนการวิเคราะห์ที่ลงทะเบียนล่วงหน้า
  ทั้งหมด การวิเคราะห์ที่เผยแพร่ต้องรวมทุกอันที่ลงทะเบียนล่วงหน้า
- **การขุด subgroup หลังเหตุการณ์**: การกล่าวอ้าง "ยานี้ใช้ได้กับ
  subgroup X" หลังจากเห็นข้อมูล FACT 07 บรรจุรายการ subgroup ที่
  ลงทะเบียนล่วงหน้า subgroup อื่นใดต้องถูกระบุว่าเป็นหลังเหตุการณ์
- **การปรับขนาดตัวอย่าง**: การเพิ่มขนาดตัวอย่างหลังจากเห็นแนวโน้ม
  ระหว่างทาง FACT 06 บรรจุขนาดตัวอย่างที่ลงทะเบียนล่วงหน้าและการคำนวณ
  พลัง
- **การลบ endpoint รองที่ไม่สะดวกแบบเงียบ ๆ**: FACT 05 จัดเรียง
  endpoint รองล่วงหน้าสำหรับการทดสอบลำดับชั้น การจัดเรียงใหม่หรือการ
  ลบสามารถตรวจพบได้
- **การปราบปรามผลลัพธ์เชิงลบ**: FACT 08 บรรจุ commitment การเผยแพร่
  ความล้มเหลวที่จะเผยแพร่ภายในหน้าต่างที่ commit ไว้สามารถมองเห็นได้
  สาธารณะ

**โหมดความล้มเหลวที่ไม่ครอบคลุม:**

- การทุจริตในการดำเนินการทดลองเอง (ข้อมูลปลอม การลงทะเบียนที่ฉ้อโกง)
  ตราผนึกตรึง *แผน* ไม่ใช่ *การดำเนินการ*
- การรายงานเชิงเลือกในการเผยแพร่ที่รวมการวิเคราะห์ที่ลงทะเบียนล่วงหน้า
  ทั้งหมดแต่ตีความในทางที่ดี ตราผนึกจับสิ่งที่ขาดหายไป มันไม่สามารถจับ
  ความลำเอียงในการตีความ
- อันตรายต่อผู้ป่วยระหว่างการทดลองเอง ตราผนึกเป็นกฎระเบียบและพยานหลักฐาน
  มันไม่ลดความเสี่ยงในการดำเนินการทดลอง

## เมื่อใดควรใช้ชุดเครื่องมือนี้

คุณควรพิจารณาผนึกการลงทะเบียนล่วงหน้าหาก:

- คุณกำลังสนับสนุนหรือดำเนินการทดลองทางคลินิก — Phase 1 ถึง 4 —
  ที่จะถูกใช้เพื่อแจ้งการตัดสินใจทางการแพทย์หรือกฎระเบียบ
- คุณยินดีที่จะถูกผูกพันสาธารณะกับการออกแบบที่ลงทะเบียนล่วงหน้า
  endpoint และแผนการวิเคราะห์
- คุณสามารถจัดการกับผลที่ตามมา: หากการทดลองล้มเหลวบน endpoint ที่
  ลงทะเบียนล่วงหน้า คุณไม่สามารถ retcon เงียบ ๆ ไปยัง endpoint อื่นที่
  ได้ผล
- คุณเข้าร่วมในวัฒนธรรมการวิจัย (เวชศาสตร์วิชาการ กลุ่มสหกรณ์ การวิจัย
  ที่ได้รับทุน NIH) ที่ความถูกต้องสมบูรณ์ของการลงทะเบียนล่วงหน้าถูก
  เรียกร้องมากขึ้น

ชุดเครื่องมือนี้ *มีค่าเป็นพิเศษ* สำหรับ:

- การทดลองที่นักวิจัยริเริ่มในสถาบันวิชาการ (ที่แรงกดดันของผู้สนับสนุน
  ในการเปลี่ยน endpoint หลังเหตุการณ์เป็นเรื่องจริง แต่นักวิจัยทาง
  วิชาการมีแรงจูงใจเชิงชื่อเสียงที่จะต่อต้าน)
- การทดลองวิทยาศาสตร์เปิดและการศึกษา citizen science
- การทดลองการใช้ยาซ้ำในไบโอเทคขนาดเล็กที่ไม่มีประวัติการดำเนินการ
  ทดลองที่ยาวนาน
- การทดลองใด ๆ ที่มีเจตนาแจ้งการยื่นเรื่องกฎระเบียบที่ข้อโต้แย้ง
  "เราไม่เคยพูดอย่างนั้น" อาจถูกทำในอนาคต

## เมื่อใด "ไม่" ควรใช้ชุดเครื่องมือนี้

อย่าใช้ชุดเครื่องมือนี้หาก:

- โปรโตคอลการทดลองยังเปลี่ยนแปลงอย่างแท้จริง ลงทะเบียนล่วงหน้าเมื่อ
  โปรโตคอลเป็นที่สิ้นสุด ไม่ใช่ก่อน
- คุณไม่สามารถ commit ที่จะเผยแพร่ผลลัพธ์โดยไม่คำนึงถึงทิศทาง ตราผนึก
  ทำให้ความล้มเหลวที่จะเผยแพร่ตรวจพบได้สาธารณะ
- การทดลองเกี่ยวข้องกับความลับทางอุตสาหกรรมที่แข่งขันได้ซึ่งป้องกันคุณ
  จากการทำให้แผนการวิเคราะห์เป็นสาธารณะ ตราผนึกทำให้แผนการวิเคราะห์เป็น
  บันทึกสาธารณะถาวร ไม่มีวิธีที่จะผนึกสิ่งใดเป็นการส่วนตัว
- คุณไม่มีการอนุมัติทางกฎหมายและจริยธรรมในโปรโตคอล ตราผนึกบันทึก
  โปรโตคอลตามที่คุณผนึก มันไม่สามารถถูกรับรองย้อนหลังโดย IRB หรือ
  คณะกรรมการจริยธรรม

## วิธี fork ชุดเครื่องมือนี้สำหรับการลงทะเบียนล่วงหน้าจริง

```bash
# 1. คัดลอกตัวอย่างนี้
cp -r ~/Genesis/myriam-kit/examples/drug-trial-preregistration ~/my-trial
cd ~/my-trial

# 2. แทนที่เนื้อหาสมมุติด้วยเนื้อหาโปรโตคอลการทดลองจริง
# แต่ละข้อเท็จจริงสอดคล้องกับส่วนของการลงทะเบียนล่วงหน้ามาตรฐาน:
$EDITOR facts/01_example_notice.txt   # REPLACE with real cover statement
$EDITOR facts/02_trial_identification.txt
$EDITOR facts/03_hypothesis.txt
$EDITOR facts/04_study_design.txt
$EDITOR facts/05_endpoints.txt       # PRIMARY + SECONDARY + EXPLORATORY
$EDITOR facts/06_statistical_analysis_plan.txt
$EDITOR facts/07_subgroups_and_exclusions.txt
$EDITOR facts/08_commitments_and_principal.txt

# 3. อัปเดต site/index.html เพื่อสะท้อนการทดลองจริง

# 4. ผนึก (ก่อนการลงทะเบียนเริ่ม ก่อน database lock):
bash build/seal.sh

# 5. ตรวจสอบยืนยันในเครื่อง:
python3 build/verify.py

# 6. ปรับใช้:
#    - เผยแพร่ URL ตราผนึกบนเว็บไซต์ผู้สนับสนุนการทดลอง และ
#    - เพิ่ม CID IPFS ไปยังฟิลด์ "Description" หรือ "References" ของ
#      บันทึก ClinicalTrials.gov ของการทดลอง เพื่อให้บันทึก registry
#      อ้างอิงข้ามตราผนึกเชิงการเข้ารหัส
```

**เวลาที่สำคัญยิ่ง**: ตราผนึกต้องถูกสร้างก่อนการลงทะเบียนเริ่ม การ
ลงทะเบียนล่วงหน้าที่ผนึกหลังจากผู้ป่วยรายแรกถูกลงทะเบียนนั้นอ่อนแอกว่ามาก
ทาง forensic อย่างเหมาะสมที่สุด ผนึกในขณะของการอนุมัติ IRB

## การบูรณาการกับโครงสร้างพื้นฐานการลงทะเบียนล่วงหน้าที่มีอยู่

ชุดเครื่องมือถูกออกแบบให้ประกอบ ไม่ทดแทน:

- **ClinicalTrials.gov / EU CTR / ISRCTN**: ลงทะเบียนการทดลองใน
  registry ของเขตอำนาจตามที่กำหนด จากนั้นในฟิลด์ "Description" หรือ
  "References" ของ registry ลิงก์ไปยังการเผยแพร่ที่ผนึก MYRIAM
- **OSF (Open Science Framework)**: OSF รองรับการลงทะเบียนล่วงหน้า
  พร้อมการประทับวันที่แต่ไม่มีการตรึงกับ Bitcoin ตราผนึก MYRIAM
  สามารถแนบเป็นไฟล์เสริมในหน้าการลงทะเบียน OSF ให้ทั้งการประทับวันที่
  และการตรึงกับ Bitcoin
- **AsPredicted.org**: เช่นเดียวกัน — ลิงก์ตราผนึก MYRIAM เป็น URL
  เสริม
- **วารสาร**: วารสารบางฉบับ (เช่น Trials, BMJ Open) ยอมรับ URL การ
  ลงทะเบียนล่วงหน้าเป็นส่วนหนึ่งของการยื่นเรื่อง รวม URL ตราผนึก
  MYRIAM ควบคู่กับ URL registry

## สิ่งที่ผู้รับ (หน่วยงานกำกับดูแล ผู้ทบทวนวารสาร นักวิเคราะห์ meta-)
ได้รับ

การทดลองที่ลงทะเบียนล่วงหน้าภายใต้ MYRIAM ให้ผู้ทบทวน:

1. **การพิสูจน์เชิงการเข้ารหัสว่าโปรโตคอลเสร็จสิ้น ณ เวลาที่เฉพาะเจาะจง**
   ผู้ทบทวนสามารถยืนยันเวลาผนึกผ่านใบรับ OTS
2. **ความเสถียรของไบต์ของข้อความโปรโตคอล** ผู้ทบทวนที่เปรียบเทียบส่วน
   methods ที่เผยแพร่กับการออกแบบที่ลงทะเบียนล่วงหน้าสามารถทำได้ทีละ
   อักขระ
3. **การตรวจจับการสลับ endpoint** diff ง่าย ๆ ระหว่าง FACT 05 ที่ผนึก
   กับ endpoint หลักที่เผยแพร่เปิดเผยการแทนที่ใด ๆ
4. **การตรวจจับการเบี่ยงเบนของแผนการวิเคราะห์** diff ง่าย ๆ ระหว่าง
   FACT 06 ที่ผนึกกับวิธีการทางสถิติที่เผยแพร่เปิดเผยการแก้ไข
5. **การตรวจจับการรายงาน subgroup เชิงเลือก** FACT 07 ที่ผนึกบรรจุ
   subgroup ที่ลงทะเบียนล่วงหน้าทั้งหมด อันใดในการเผยแพร่ที่ไม่อยู่
   ในตราผนึกต้องถูกระบุว่าเป็นหลังเหตุการณ์

สิ่งนี้ทำให้การทบทวนเชิงระบบและการวิเคราะห์ meta- ง่ายขึ้นอย่างมาก
เทียบกับการปฏิบัติปัจจุบัน

## สิ่งที่ชุดเครื่องมือนี้ "ไม่ใช่"

- ไม่ใช่สิ่งทดแทนการอนุมัติ IRB / คณะกรรมการจริยธรรม
- ไม่ใช่สิ่งทดแทนการลงทะเบียน registry ของเขตอำนาจ
- ไม่ใช่การรับประกันว่าการทดลองจะถูกดำเนินการตามแผน
- ไม่ใช่วิธีที่จะตรึงความยินยอมของผู้ป่วยหรือเอกสารจริยธรรมอื่น ๆ
  (สิ่งเหล่านั้นต้องการกลไกการผนึกของตัวเอง)
- ไม่ใช่การป้องกันการวิจารณ์การออกแบบการทดลองหลังการเผยแพร่

## หมายเหตุเกี่ยวกับการเผยแพร่ผลลัพธ์เชิงลบ

Commitment #1 ใน FACT 08 ("การเผยแพร่โดยไม่คำนึงถึงทิศทาง") เป็น
commitment เดียวที่มีผลกระทบต่อการเสียชีวิตสูงสุดในตราผนึก การปราบปราม
ผลลัพธ์การทดลองเชิงลบในอดีต (โดยเฉพาะสำหรับยาทางจิตเวชในวัยรุ่น ยา
มะเร็งในประชากรสูงวัย และยาหัวใจและหลอดเลือดในผู้หญิง) ได้มีส่วนสาเหตุ
ที่เป็นไปได้ต่อการเสียชีวิตที่ป้องกันได้นับพันคนตลอดหลายทศวรรษผ่าน
การจ่ายยามากเกินไปตามการประมาณประสิทธิผลที่ลำเอียง

commitment ที่ผนึกที่จะเผยแพร่ผลลัพธ์เชิงลบโดยตัวมันเองเป็นหนึ่งใน
การเคลื่อนไหวด้านสาธารณสุขที่มี leverage สูงสุดที่ผู้สนับสนุนสามารถทำได้
ตราผนึกให้ความรับผิดชอบภายนอกสำหรับ commitment นั้นในแบบที่นโยบายการ
เผยแพร่ที่ตนเองควบคุมไม่ทำ

## การปิดท้าย

การลงทะเบียนล่วงหน้าการทดลองยาภายใต้ MYRIAM เป็นงานเพิ่มเติมเล็กน้อย
สำหรับผู้สนับสนุน: ~30 นาทีในการกรอกข้อเท็จจริง ~10 วินาทีในการผนึก
ความรับผิดชอบที่เพิ่มเข้ามาเป็นแบบไม่สมมาตร: ต้นทุนเล็กสำหรับการดำเนิน
การทดลองอย่างซื่อสัตย์ ข้อจำกัดขนาดใหญ่ต่อการเคลื่อนไหวหลังเหตุการณ์ที่
มีส่วนสาเหตุต่อความลำเอียงในประวัติศาสตร์ของฐานพยานหลักฐาน

หากคุณเป็นผู้สนับสนุนหรือนักวิจัยหลักที่กำลังพิจารณาชุดเครื่องมือนี้
ชุดเครื่องมือเป็นสาธารณสมบัติ (CC0) ใช้ แก้ไข ปรับปรุง

## ใบอนุญาต

สาธารณสมบัติ (CC0) ไม่ต้องระบุที่มา
