# 药物临床试验预注册套件

myriam-kit 的一个开箱即用范例,面向临床试验赞助方与学术研究者,用于以防篡改、锚定于 Bitcoin 的方式预注册试验,**作为对** ClinicalTrials.gov、EU CTR 或其他司法管辖区注册库注册之外的补充。

## 为什么要在 MYRIAM 下*额外*预注册?

现有的试验注册库(ClinicalTrials.gov、EU CTR、ISRCTN)存在已被记录在案的完整性弱点:

- ClinicalTrials.gov 允许赞助方在原始注册**之后**更新终点、样本量计算和分析计划,虽然修改日期在注册库中可见,但*内容*的变更并非总是被加密锚定。
- 多项大规模研究(Goldacre 等,2019;Mathieu 等,2009)记录了整个领域系统性的事后终点切换。
- 期刊的编辑性执法(CONSORT、ICMJE)能捕获部分案例;许多仍漏网而过。

一个 MYRIAM 密封封闭了这一缺口。Merkle 根 + Bitcoin OTS 收据使得对*预注册的*主要终点、统计分析计划或亚组列表的任何事后改动都可被加密检测。从司法取证角度看,密封比注册库本身更难被追溯篡改(retcon)。

本套件是司法管辖区注册库的**补充**,而非替代。真实试验应当在适用的注册库中注册**并且**在 MYRIAM 下密封。

## 本套件防御什么

历史上曾造成死亡后果、本机制本可有所帮助的案例:

- **Vioxx(默克,2004)**:心血管事件曾是 APPROVe 试验中预注册的终点,但已发表的分析使用了一个让表观危害最小化的截止时间。对预注册截止时间的密封,本可使事后的截止时间变更在论文发表之前就被公开检测到。
- **Paxil/Study 329(GSK/SmithKline,2001)**:关于青少年自杀倾向的阴性试验数据被经由事后分析重新包装。对终点的密封式预注册,本可使该重新包装被公开检测到。
- **多项肿瘤学试验**中,当生存获益未能兑现时,生存终点被替换为替代终点(无进展生存期、缓解率)。

**密封可直接捕获的失败模式:**

- **终点切换**:当实际的主要终点失败时,事后将某个次要终点擢升为"主要"。密封的 FACT 05 包含预注册的主要终点;发表的主要终点必须与之吻合。
- **结果挑拣(cherry-picking)**:选择性地仅报告奏效的分析。FACT 06 包含完整的预注册分析计划;发表的分析必须包含全部预注册分析。
- **事后亚组挖掘**:在看到数据之后才声称"药物对亚组 X 有效"。FACT 07 包含预注册的亚组列表;任何其他亚组都必须标注为事后亚组。
- **样本量修订**:在看到中期趋势之后增加样本量。FACT 06 包含预注册的样本量与功效计算。
- **悄然剔除不便的次要终点**:FACT 05 已为分层检验预先排定次要终点的顺序;任何重新排序或剔除都可被检测。
- **阴性结果压制**:FACT 08 包含发表承诺;在所承诺的窗口期内未能发表,公开可见。

**未覆盖的失败模式:**

- 试验执行本身中的不当行为(伪造数据、欺诈性入组)。密封锚定的是*计划*,不是*执行*。
- 在发表中包含了全部预注册分析、却进行有利方向解读的选择性报告。密封能捕获缺失的内容;无法捕获解释性偏倚。
- 试验期间的患者伤害。密封是监管性与证据性的;它不会降低试验执行风险。

## 何时使用本套件

如果出现以下情形,您应考虑密封一份预注册:

- 您正在赞助或开展一项临床试验 —— 从 Phase 1 到 Phase 4 —— 该试验将被用于支持医学或监管决策。
- 您愿意被公开约束于预注册的设计、终点和分析计划。
- 您能够承担其后果:如果试验在其预注册终点上失败,您不能悄然追溯切换(retcon)到另一个奏效的终点。
- 您身处一种研究文化(学术医学、协作研究组、NIH 资助研究)之中,这种文化对预注册的完整性提出了日益提高的要求。

本套件*尤其*适用于:

- 在学术机构内部由研究者发起的试验(在此情境下,赞助方要求事后变更终点的压力是真实存在的,而学术研究者具有抵制该压力的声誉激励)。
- 开放科学试验与公民科学研究。
- 小型生物科技公司开展的药物再利用试验,这类公司尚未具备长期的试验执行记录。
- 任何旨在支持监管申报、未来可能出现"我们从未那么说过"式辩解的试验。

## 何时**不要**使用本套件

请**不要**在以下情形使用本套件:

- 试验方案确实仍在变动中。请在方案最终定稿后再预注册,不要在此之前。
- 您无法承诺无论结果方向如何都予以发表。密封会使"未能发表"这一事实公开可检测。
- 试验涉及具有竞争性的工业秘密,使您无法公开分析计划。密封会将分析计划永久公开存档;没有任何方式可以私密地进行密封。
- 您尚未取得对方案的法律与伦理批准。密封记录的是**您**所密封的方案;它不能被 IRB 或伦理委员会追溯地认可。

## 如何派生本套件用于真实预注册

```bash
# 1. Copy this example
cp -r ~/Genesis/myriam-kit/examples/drug-trial-preregistration ~/my-trial
cd ~/my-trial

# 2. Replace fictional content with the real trial protocol content.
# Each fact corresponds to a section of a standard pre-registration:
$EDITOR facts/01_example_notice.txt   # REPLACE with real cover statement
$EDITOR facts/02_trial_identification.txt
$EDITOR facts/03_hypothesis.txt
$EDITOR facts/04_study_design.txt
$EDITOR facts/05_endpoints.txt       # PRIMARY + SECONDARY + EXPLORATORY
$EDITOR facts/06_statistical_analysis_plan.txt
$EDITOR facts/07_subgroups_and_exclusions.txt
$EDITOR facts/08_commitments_and_principal.txt

# 3. Update site/index.html to reflect the real trial.

# 4. Seal (BEFORE enrollment begins, BEFORE database lock):
bash build/seal.sh

# 5. Verify locally:
python3 build/verify.py

# 6. Deploy:
#    - Publish the seal URL to your trial sponsor's website AND
#    - Add the IPFS CID to the trial's ClinicalTrials.gov record's
#      "Description" or "References" field, so the registry record
#      cross-references the cryptographic seal.
```

**关键时机**:密封必须**在入组开始之前**创建。在第一位患者入组之后才创建的预注册密封,从司法取证角度看其效力大为削弱。最理想的做法是在 IRB 批准的当刻进行密封。

## 与既有预注册基础设施的整合

本套件的设计目的是与既有基础设施组合,而非替代:

- **ClinicalTrials.gov / EU CTR / ISRCTN**:按规定在司法管辖区注册库中注册试验。然后,在该注册记录的 "Description" 或 "References" 字段中,链接到 MYRIAM 密封的发布。
- **OSF(Open Science Framework)**:OSF 支持带日期戳的预注册,但不支持 Bitcoin 锚定。MYRIAM 密封可作为补充文件附加在 OSF 注册页面上,同时获得日期戳和 Bitcoin 锚定。
- **AsPredicted.org**:类似 —— 将 MYRIAM 密封作为补充 URL 链接进来。
- **期刊**:部分期刊(例如 Trials、BMJ Open)接受预注册 URL 作为投稿材料的一部分。请将 MYRIAM 密封 URL 与注册库 URL 一并提交。

## 接收方(监管机构、期刊评审、Meta 分析者)所获得的内容

在 MYRIAM 下预注册的试验,可为评审者提供:

1. **方案在特定时刻定稿的加密证明。** 评审者可通过 OTS 收据验证密封时间。
2. **方案文本的字节级稳定性。** 将发表方法学章节与预注册设计相对照的评审者,可逐字符进行比对。
3. **终点切换的检测。** 在密封的 FACT 05 与发表的主要终点之间做一次简单 diff,即可揭示任何替换。
4. **分析计划偏离的检测。** 在密封的 FACT 06 与发表的统计方法之间做一次简单 diff,即可揭示任何修改。
5. **亚组选择性报告的检测。** 密封的 FACT 07 包含全部预注册亚组;发表中任何不在密封内的亚组都必须标注为事后亚组。

这使得系统综述与 Meta 分析较当前实践显著更易开展。

## 本套件**不**是什么

- 不是 IRB / 伦理委员会批准的替代品。
- 不是司法管辖区注册库注册的替代品。
- 不是对试验将按计划开展的保证。
- 不是用以锚定患者知情同意或其他伦理文件的途径(那些需要其自身的密封机制)。
- 不是对试验设计在发表后受到批评的防护。

## 关于阴性结果发表的说明

FACT 08 中的承诺 #1("无论结果方向都予以发表")是整个密封中对死亡率影响最高的单一承诺。历史上对阴性试验结果的压制(尤其是针对青少年的精神类药物、针对老年群体的肿瘤学药物以及针对女性的心血管药物)在合理推断下,经由基于偏倚疗效估计的过度处方,数十年间已导致数千例本可避免的死亡。

仅就其本身而言,密封式承诺发表阴性结果,就是赞助方可以采取的具有最高杠杆效应的公共卫生举措之一。密封以一种自我监督的发表政策所无法企及的方式,为该承诺提供外部问责。

## 结语

在 MYRIAM 下预注册药物临床试验,对赞助方而言只是极小的额外工作:约 30 分钟填充事实,约 10 秒完成密封。它所增加的问责是非对称的:对诚实开展的试验只构成微小成本,对那些助长了历史上证据基础偏倚的事后操作则构成巨大约束。
