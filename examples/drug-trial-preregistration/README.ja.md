# 治験事前登録キット(Drug Trial Pre-registration Kit)

*日本語訳 — 原文の英語版については [README.md](README.md) を参照してください*

ClinicalTrials.gov、EU CTR、あるいは各国・各法域の試験登録機関への登録**に加えて**、改ざん検出可能で Bitcoin に錨を打った形で治験を事前登録したい治験スポンサーおよび学術研究者向けの、ターンキー型の myriam-kit 例です。

## なぜ MYRIAM の下で *追加的に* 事前登録するのか?

既存の試験登録機関(ClinicalTrials.gov、EU CTR、ISRCTN)には、文書化された完全性上の弱点があります。

- ClinicalTrials.gov は、当初の登録 **後** にスポンサーがエンドポイント、サンプルサイズ計算、解析計画を更新することを許可しており、変更日は登録に表示されるものの、*内容* の変更が必ずしも暗号学的に固定されるわけではありません。
- いくつかの大規模研究(Goldacre et al., 2019; Mathieu et al., 2009)は、分野全体にわたる体系的な事後的エンドポイント変更を文書化しています。
- 学術誌による編集的執行(CONSORT、ICMJE)は一部のケースを捕捉しますが、多くはすり抜けます。

MYRIAM の封緘はこのギャップを閉じます。Merkle ルート + Bitcoin OTS レシートは、*事前登録された* 主要エンドポイント、統計解析計画、サブグループ一覧に対する事後的な変更を、暗号学的に検出可能にします。封緘は、登録機関そのものよりも、フォレンジック的に retcon しにくいものです。

このキットは、各法域の登録機関を **置き換える** ものではなく、**補完する** ものです。実際の治験は、適切な登録機関に登録した上で、MYRIAM の下でも封緘するべきです。

## このキットが防ぐもの

このキットが助けになったであろう、歴史上の死亡関連事例:

- **Vioxx(Merck、2004)**: 心血管イベントは APPROVe 試験において事前登録されたエンドポイントでしたが、公表された解析では見かけ上の有害性を最小化するカットオフが用いられました。事前登録されたカットオフの封緘があれば、論文公表に先んじて事後的なカットオフ変更が公的に検出可能になったでしょう。
- **Paxil / Study 329(GSK / SmithKline、2001)**: 青少年の自殺傾向に関する陰性の治験データが、事後解析によって枠組みを変えて再提示されました。エンドポイントを封緘した事前登録があれば、その枠組み変更が公的に検出可能になったでしょう。
- **複数の腫瘍学試験** において、生存率の便益が実際には出なかったときに、生存率エンドポイントが代替エンドポイント(無増悪生存期間、奏効率)に差し替えられました。

**封緘が直接捕捉する失敗モード:**

- **エンドポイントの差し替え**: 実際の主要エンドポイントが失敗したときに、副次エンドポイントを事後的に「主要」に格上げすること。封緘の FACT 05 には事前登録された主要エンドポイントが含まれます。公表された主要エンドポイントは、これと一致しなければなりません。
- **アウトカムのつまみ食い(cherry-picking)**: うまくいった解析だけを選択的に報告すること。FACT 06 には事前登録された完全な解析計画が含まれます。公表される解析には、事前登録されたすべての解析が含まれていなければなりません。
- **事後的なサブグループマイニング**: データを見た後で「この薬はサブグループ X に効く」と主張すること。FACT 07 には事前登録されたサブグループ一覧が含まれます。それ以外のサブグループは、すべて事後的(post-hoc)と明示されなければなりません。
- **サンプルサイズの改訂**: 中間トレンドを見た後にサンプルサイズを増やすこと。FACT 06 には事前登録されたサンプルサイズと検出力計算が含まれます。
- **不都合な副次エンドポイントの静かな除去**: FACT 05 は階層的検定のために副次エンドポイントを事前に順序付けしており、並び替えや削除は検出可能です。
- **陰性結果の隠蔽**: FACT 08 には公表に関するコミットメントが含まれており、コミットされた期間内に公表しなかった場合は公的に可視となります。

**カバーされ *ない* 失敗モード:**

- 治験実施そのものでの不正(データ捏造、不正な被験者登録)。封緘が固定するのは *計画* であって、*実施* ではありません。
- 事前登録されたすべての解析を含めた上で、それらを好意的にスピンする公表における選択的報告。封緘は欠落を捕捉できますが、解釈上のバイアスは捕捉できません。
- 治験中の被験者の被害。封緘は規制上・証拠上のものであり、治験実施リスクを低減するものではありません。

## このキットを使うべき場合

事前登録の封緘を検討すべきなのは、以下に該当する場合です。

- 第 1 相から第 4 相までの治験を、医学的または規制上の判断に影響を与える形でスポンサーする、または実施する場合。
- 事前登録した設計、エンドポイント、解析計画に対して、公的に拘束されることを受け入れられる場合。
- その帰結を受け入れられる場合 — すなわち、事前登録されたエンドポイントで治験が失敗したときに、うまくいった別のエンドポイントへこっそり差し替えることができない、という帰結。
- 事前登録の完全性がますます求められる研究文化(学術医学、共同研究グループ、NIH 資金研究)に参加している場合。

このキットは、特に次のような場面で価値があります。

- 学術機関での研究者主導型治験(スポンサー側からの事後的なエンドポイント変更圧力が現実にあり、学術研究者には抵抗するレピュテーション上の動機がある場合)。
- オープンサイエンス治験および市民科学研究。
- 長期の治験実施実績を持たない小規模バイオテックでの薬剤再利用治験。
- 将来「そんなことは言っていない」という主張がなされ得る、規制申請に資することを意図したあらゆる治験。

## このキットを使うべき *でない* 場合

以下に該当する場合は、このキットを使わないでください。

- 治験プロトコルが、まだ本当に流動的である場合。プロトコルが確定する前ではなく、確定してから事前登録してください。
- 結果の方向にかかわらず公表することを約束できない場合。封緘は、公表しなかったことを公的に検出可能にします。
- 競合産業の機密性が、解析計画を公開することを妨げる治験。封緘は解析計画を恒久的な公的記録にします。非公開で封緘する方法はありません。
- プロトコルについて法的・倫理的な承認を得ていない場合。封緘はプロトコルを *あなたが封緘した時点のもの* として記録します。後付けで IRB や倫理委員会が承認することはできません。

## 実際の事前登録のためにキットをフォークする方法

```bash
# 1. この例をコピーする
cp -r ~/Genesis/myriam-kit/examples/drug-trial-preregistration ~/my-trial
cd ~/my-trial

# 2. 架空の内容を、実際の治験プロトコルの内容に置き換える。
# 各 fact は、標準的な事前登録のセクションに対応している:
$EDITOR facts/01_example_notice.txt   # 実際のカバー文に置き換える
$EDITOR facts/02_trial_identification.txt
$EDITOR facts/03_hypothesis.txt
$EDITOR facts/04_study_design.txt
$EDITOR facts/05_endpoints.txt       # 主要 + 副次 + 探索的
$EDITOR facts/06_statistical_analysis_plan.txt
$EDITOR facts/07_subgroups_and_exclusions.txt
$EDITOR facts/08_commitments_and_principal.txt

# 3. site/index.html を、実際の治験を反映する内容に更新する。

# 4. 封緘する(被験者登録の開始前、データベースロックの前に):
bash build/seal.sh

# 5. ローカルで検証する:
python3 build/verify.py

# 6. デプロイする:
#    - 治験スポンサーのウェブサイトに封緘 URL を公表し、かつ
#    - 治験の ClinicalTrials.gov レコードの "Description" または "References"
#      フィールドに IPFS CID を追加し、登録レコードから暗号学的封緘を相互参照できるようにする。
```

**重要なタイミング**: 封緘は、被験者登録が始まる **前** に作成しなければなりません。最初の被験者が登録された後に封緘した事前登録は、フォレンジック的にかなり弱くなります。理想的には、IRB の承認の瞬間に封緘してください。

## 既存の事前登録基盤との統合

このキットは、置き換えではなく、合成されるように設計されています。

- **ClinicalTrials.gov / EU CTR / ISRCTN**: 治験を、要求される法域の登録機関に登録します。その上で、登録機関の "Description" または "References" フィールドに、MYRIAM 封緘の公表物へのリンクを記載します。
- **OSF(Open Science Framework)**: OSF は日付付き事前登録をサポートしますが、Bitcoin への錨打ちはサポートしません。MYRIAM 封緘を OSF 登録ページの補足ファイルとして添付すれば、日付スタンプと Bitcoin 錨打ちの両方が得られます。
- **AsPredicted.org**: 同様に、MYRIAM 封緘を補足 URL としてリンクします。
- **学術誌**: 一部の学術誌(例: Trials、BMJ Open)は、投稿の一部として事前登録 URL を受け入れます。登録機関の URL に加えて、MYRIAM 封緘の URL も含めてください。

## 受け手(規制当局、学術誌の査読者、メタアナリスト)が得るもの

MYRIAM の下で事前登録された治験は、査読者に対して次のものを提供します。

1. **プロトコルが特定の時点に確定されたことの暗号学的証拠。** 査読者は OTS レシートを通じて封緘時刻を検証できます。
2. **プロトコル本文のバイト単位での安定性。** 査読者は、公表された方法セクションを、事前登録された設計と一文字単位で比較できます。
3. **エンドポイント差し替えの検出。** 封緘された FACT 05 と公表された主要エンドポイントとの単純な diff によって、任意の差し替えが明らかになります。
4. **解析計画からの逸脱の検出。** 封緘された FACT 06 と公表された統計手法との単純な diff によって、修正が明らかになります。
5. **選択的なサブグループ報告の検出。** 封緘された FACT 07 には事前登録されたすべてのサブグループが含まれます。公表中のサブグループのうち封緘に含まれないものは、事後的(post-hoc)と明示されなければなりません。

これにより、システマティックレビューやメタアナリシスが、現行の実務よりも大幅に容易になります。

## このキットが **そうでない** もの

- IRB や倫理委員会の承認の代替ではありません。
- 各法域の試験登録機関への登録の代替ではありません。
- 治験が計画通りに実施されることの保証ではありません。
- 患者の同意やその他の倫理文書を錨打ちする手段ではありません(これらには独自の封緘機構が必要です)。
- 治験設計に対する公表後の批判への防護ではありません。

## 陰性結果の公表に関する注

FACT 08 のコミットメント #1(「方向にかかわらず公表する」)は、封緘の中で、歴史的に最も死亡率への影響が大きい単一のコミットメントです。陰性の治験結果が歴史的に隠蔽されてきたこと(特に青少年における精神科薬、高齢者集団における腫瘍学薬、女性における心血管薬)は、偏った有効性推定に基づく過剰処方を通じて、数十年にわたり、もっともらしく見積もっても数千人規模の予防可能な死に寄与してきました。

陰性結果を公表するという封緘されたコミットメントは、それ単独でも、スポンサーが行うことのできる公衆衛生上、最もてこの効く一手です。封緘は、そのコミットメントに対して、自主規制の公表ポリシーには得られない外部からの説明責任を提供します。

## 結び

MYRIAM の下での治験事前登録は、スポンサーにとってわずかな追加作業です: facts の記入に約 30 分、封緘に約 10 秒。それが追加する説明責任は非対称です: 誠実な治験実施には小さなコスト、歴史的にエビデンス基盤の偏りに寄与してきた事後的な調整に対しては大きな制約。

スポンサーまたは治験責任医師として本キットを検討している方へ: 本キットはパブリックドメイン(CC0)です。使用、改変、改善のいずれも自由です。
