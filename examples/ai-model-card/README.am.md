# የ AI ሞዴል ባህሪ መቆለፊያ ስብስብ

*የአማርኛ ትርጉም — ለመጀመሪያው የእንግሊዝኛ ስሪት፣ ይመልከቱ [README.md](README.md)*

ለ AI ላቦራቶሪዎች (የንግድ ወይም ክፍት-ምንጭ) የተዘጋጀ ዝግጁ የሆነ myriam-kit ምሳሌ
ሲሆን ሞዴል የሚለቁ እና የመለቀቅን ይዘቶች — የ weights ማንነት፣ ችሎታዎች፣
የግምገማ ውጤቶች፣ የደህንነት ግምገማ፣ የቅድመ-ማስፈር ቃላቶች —
**ክሪፕቶግራፊክ በሆነ መንገድ ተሰክተው እና በሕዝብ ሊረጋገጡ የሚችሉ**
ለማድረግ የሚፈልጉ ናቸው።

ይህ ስብስብ የተተከሉ ሞዴሎች በዝምታ ሲንሳፈፉ፣ የግምገማ ውጤቶች በኋላ ሲቀየሩ፣ ወይም
የቅድመ-ማስፈር ይዘቶች ዓመታት ዘግይተው "እንደዚያ አላልንም" በሚል ሲከራከሩ
የሚገጥመውን ልዩ የውድቀት ሁኔታ ይከላከላል።

## ይህ ስብስብ ምን ያመነጫል

ለአንድ ሞዴል መለቀቅ ክሪፕቶግራፊክ የታሸገ ማኒፌስት የሚከተሉትን የያዘ፦

1. የሞዴል ምልክት (ስም፣ architecture፣ የ parameter ብዛት፣ ፍቃድ)
2. **የ weights ፋይል SHA-256** — የቀኖናዊ artifact hash
3. የ model card ጽሑፍ (ችሎታዎች፣ የታሰበ አጠቃቀም፣ ከወሰን-ውጭ
   አጠቃቀም፣ የታወቁ የውድቀት ሁነታዎች)
4. በመለቀቅ ጊዜ የግምገማ ውጤቶች (የችሎታ ግምገማዎች፣ የደህንነት ግምገማዎች፣
   calibration)
5. የቅድመ-ማስፈር ቃላቶች (ከመለቀቅ በኋላ በዝምታ ላለመለወጥ ቃል የሚገቡባቸው ነገሮች)
6. ፈራሚ ዋና ኃላፊ (ኃላፊነቱን የሚወስደው ሰው + ድርጅት)

ማኅተሙ እያንዳንዱን ከእነዚህ መካከል byte-stable ያደርጋል፣ በ OpenTimestamps
በኩል ወደ Bitcoin ጊዜ-የተሰካ ያደርጋል፣ እና ከማንኛውም አንድ የማስተናገጃ ወገን
ራሱን ችሎ በ IPFS በኩል ሊገኝ የሚችል ያደርጋል።

የተሠራውን ምናባዊ ምሳሌ ይመልከቱ፦ [facts/](facts/) እና ቀርቦ የተሠራው
[site/index.html](site/index.html)። ምሳሌው ምናባዊ ሞዴል ("Calm-mini-v0")
ይጠቀማል፤ ትክክለኛ መለቀቅ የምሳሌውን ይዘት ይተካል።

## ይህ ስብስብ ከምን ይከላከላል

**ማኅተሙ በቀጥታ የሚይዛቸው የውድቀት ሁነታዎች፦**

- **የ weights መታለል** — የተለቀቀው የ weights ፋይል SHA-256 በ FACT 03 አለ።
  ተጠቃሚ የሚያወርዳቸው byte ላይ ከመለቀቅ-በኋላ ማንኛውም ለውጥ እንደገና በ hashing
  ሊታወቅ ይችላል።
- **የዝምታ የ eval retcons** — በመለቀቅ ጊዜ የነበሩ የግምገማ ውጤቶች በ FACT 05 አሉ።
  የወደፊት "የግምገማ ውጤት ሁልጊዜ X ነበር" ክርክሮች የሕዝብ መሰካቻ አላቸው።
- **የችሎታ ይዘት መንሳፈፍ** — በ FACT 04 ያለው የ model card ጽሑፍ byte-stable
  ነው። የወደፊት "ችሎታ X ን አላልንም" ክርክሮች በታሸገው ጽሑፍ ላይ በሕዝብ
  ሊጣሩ ይችላሉ።
- **የዝምታ fine-tune እንደገና ማከፋፈል** — በ FACT 06 (በተሠራው ምሳሌ)
  የቅድመ-ማስፈር ቃል #1 "በተመሳሳይ ስም ምንም የዝምታ fine-tuning የለም" የሚል
  ነው። ማወቅ፦ ማንኛውም አዲስ weight hash ማለት አዲስ የሞዴል ስም እና አዲስ
  የታሸገ ህትመት ማለት ነው፣ ካልሆነ ቃሉ ተጥሷል ማለት ነው።
- **የተደበቁ የውድቀት ሁነታዎች** — FACT 04 ለሚታወቁ-የውድቀት-ሁነታዎች ዝርዝር
  ቃል ይሰጣል። ዝርዝሩ monotonic ነው (ለቀዘቀዘ checkpoint ሊጨምር ይችላል፣
  ሊቀንስ አይችልም)። ላቦራቶሪው ውስጥ-ለውስጥ ስለሚያውቀው ግን ስላልታተመው የውድቀት
  ሁነታ የወደፊት ግኝቶች ሊከራከሩ የሚችሉ ማስረጃ ይሆናሉ።

**የማይሸፈኑ የውድቀት ሁነታዎች፦**

- ሞዴሎች በዳውን-ስትሪም ተጠቃሚዎች መጥፎ መጠቀም (ማኅተሙ የተለቀቀውን ያረጋግጣል፣
  ተጠቃሚዎች ምን እንደሚያደርጉበት አይደለም)።
- በሦስተኛ ወገኖች የተደረገ ተቃራኒ fine-tuning (የሦስተኛ-ወገን fine-tune የተለየ
  weight hash ያመነጫል፣ ስለዚህ ኦሪጅናሉ ማኅተም ሳይነካ ይቆያል፣ ግን
  የሦስተኛ ወገን ሞዴል ከማኅተሙ ጥበቃ ውጭ ነው)።
- በማስፈር-አካባቢ ለውጥ ምክንያት የባህሪ ለውጦች (የተለያዩ system prompts፣
  RAG configurations፣ ወዘተ)።
- በመለቀቅ ጊዜ ላቦራቶሪው በትክክል ያላወቃቸው ውድቀቶች። ማኅተሙ ላቦራቶሪውን
  *ለሚያውቀው* ያስገድዳል፤ ላላወቀው መሰካት አይችልም።

## ይህን ስብስብ መቼ መጠቀም

የሚከተሉት ሁኔታዎች ካሉ model card ማሸግን ማሰብ አለብዎት፦

- ሞዴል weights በሕዝብ የሚለቁ ከሆነ ወይም ዳውን-ስትሪም ተጠቃሚዎች የባህሪ
  መረጋጋት የሚያስፈልጋቸው ሁኔታ ውስጥ ሞዴል እያስፈሩ ከሆነ።
- መለቀቁ የወደፊት "እንዲህ አላልንም" ክርክሮች ሊከሰቱ የሚችሉበት ያህል ጉልህ ከሆነ
  (በትልቅ ሚዛን የተተከለ ማንኛውም ሞዴል፣ ማንኛውም ለደህንነት-ወሳኝ የሆነ ሞዴል፣
  የችሎታ ይዘቶች ጋር አብሮ የተለቀቀ ማንኛውም መለቀቅ)።
- ድርጅትዎ የወደፊት ድርጊቶችን የሚገድቡ የቅድመ-ማስፈር ቃላቶችን ለመስጠት ፈቃደኛ
  ከሆነ (ይህ ጭነቱን የሚሸከመው ነው — የሚቀጥለውን ክፍል ይመልከቱ)።
- በመለቀቅ ጊዜ የ weights ፋይል SHA-256 ማስላት የሚችሉ ከሆነ።

## ይህን ስብስብ መቼ መጠቀም **የለብዎትም**

ይህን ስብስብ **አይጠቀሙ** ከሆነ፦

- በምሳሌው FACT 06 ውስጥ ባሉ የቅድመ-ማስፈር ቃላቶች ለመታሰር ፈቃደኛ ካልሆኑ።
  ማኅተሙ ቃላቶችን በሜካኒካዊ መንገድ ሊታወቁ የሚችሉ ያደርጋል፤ ለዚያ ኃላፊነት
  ካልተዘጋጁ፣ አያትሙ።
- የ weights ፋይል በተደጋጋሚ የሚቀየር ከሆነ (ለምሳሌ፣ ቀጣይነት ያለው fine-tuning
  pipelines)። እያንዳንዱ የ weight ለውጥ አዲስ የታሸገ መለቀቅ ይፈልጋል። በመለቀቅ
  መታሸግ ከባድ ነው፤ ይህ ስብስብ ለ*ቀዘቀዙ* checkpoints ነው።
- የደህንነት ግምገማውን ገና ካላደረጉ። ማኅተሙ የደህንነት ግምገማውን ግኝቶች
  ቃል ይገባል። ያልተሟላ ግምገማ ካተሙ፣ ያልተሟላ ግምገማ ቋሚ የሕዝብ መዝገብ ይኖራዎታል።
- በኋላ ላይ ማንኛውንም የተወሰነ የግምገማ ውጤት መመለስ የሚፈልጉ ከሆነ። ማኅተሙ
  ቋሚ ነው። አዳዲስ የታሸጉ ህትመቶች ሊተኩ ይችላሉ፣ ግን ኦሪጅናሉ ሊቀር አይችልም።

## ጭነቱን የሚሸከሙ የቅድመ-ማስፈር ቃላቶች

የምሳሌው FACT 06 አታሚውን ለሚከተሉት ያስገድዳል፦

1. **በተመሳሳይ ስም ምንም የዝምታ fine-tuning የለም።** ማንኛውም fine-tune አዲስ
   የሞዴል ስም፣ አዲስ የታሸገ ህትመት፣ አዲስ weight hash ያመነጫል። ቀዳሚው
   ቀኖናዊ በሆነ መንገድ የተሰየመው ስሪት ሆኖ ይቀራል።

2. **ምንም የዝምታ የግምገማ ዝመናዎች የሉም።** ድጋሚ-ግምገማዎች ኦሪጅናሉን በ Merkle
   root የሚጠቅሱ አዳዲስ የታሸጉ ህትመቶችን ያገኛሉ። ስሪቶች ይከታተላሉ፣
   አይታደሱም።

3. **የታወቁ የውድቀት ሁነታዎች ሊጨምሩ ይችላሉ፣ ፈጽሞ ሊቀንሱ አይችሉም።** ቀዝቀዘ
   checkpoint የውድቀት ሁነታዎች ሊጠገኑ አይችሉም። ስለዚህ የታተመው ዝርዝር
   በውጤታማነት monotonic ነው።

4. **የማስወገድ አያያዝ።** ሞዴሉን ከስርጭት ለማስወገድ የተገደደ ከሆነ፣ ላቦራቶሪው
   የመጨረሻ የታሸገ የማስወገድ መግለጫ ለማተም ቃል ይሰጣል።

ይህን ስብስብ የሚቀበል ላቦራቶሪ ከእነዚያ ቃላቶች ውስጥ የትኞቹን በትክክል ማክበር
እንደሚችል መወሰን አለበት። አራቱንም ማክበር ካልቻሉ፣ ከማሸግ *በፊት* FACT 06 ን
ሊያከብሩት የሚችሉትን ለማንፀባረቅ ይቀይሩ፣ እና የተፈጠረውን የተዳከመ ጥበቃ

## የ weights hash ማስላት

ለአንድ safetensors ፋይል፦

```bash
shasum -a 256 model.safetensors
```

ለ multi-file checkpoints (የ GGUF shards፣ የተሸረሸሩ safetensors)፦

```bash
# Hashes በተደረደረ ቅደም ተከተል ይያያዙ፣ ከዚያ እንደገና hash ያድርጉ
for f in $(ls model-*.safetensors | sort); do
  shasum -a 256 "$f"
done > weights_manifest.txt
shasum -a 256 weights_manifest.txt
```

የመጨረሻው hash ወደ FACT 03 የሚገባው ነው። ተጠቃሚዎች በሚያወርዱበት ጊዜ
የተናጥል shards ማረጋገጥ እንዲችሉ የእያንዳንዱን-shard hashes እና የመጨረሻውን
የተጠቃለለ hash ሁለቱንም ያካትቱ።

## ተቀባዮች (የእርስዎ ሞዴል ተጠቃሚዎች) ምን ማወቅ አለባቸው

ከ MYRIAM-ታሸገ ህትመት ጋር የተደረገ የሞዴል መለቀቅ ተጠቃሚውን ያስችላል፦

1. **ያወረዱትን weights ያረጋግጡ** እንደገና በ hashing እና ከ FACT 03 ጋር
   በማነጻጸር።
2. **Model card retcon እንዳልሆነ ያረጋግጡ** በታሸገው FACT 04 በላቦራቶሪው
   አሁን ካተመው ስሪት ጋር በማነጻጸር።
3. **የግምገማ ውጤቶቹ በመለቀቅ ጊዜ የነበሩት መሆናቸውን ያረጋግጡ** በላቦራቶሪው
   የአሁን የግምገማ ይዘቶችን በታሸገው FACT 05 ጋር በማነጻጸር።
4. **ላቦራቶሪውን በቅድመ-ማስፈር ቃላቶቹ ላይ ኃላፊነት ያስጠይቁ** ትክክለኛውን
   ባህሪ በታሸገው FACT 06 ላይ በመፈተሽ።

ልዩነቶችን የሚያገኝ ተጠቃሚ የልዩነቱ ክሪፕቶግራፊክ ማስረጃ አለው። ያን ማስረጃ
ሊያትሙ ይችላሉ፤ ኦሪጅናሉ የታሸገ ህትመት መሰካቻ ነው።

## ይህ ስብስብ ምን **አይደለም**

- የደህንነት ግምገማን በትክክል ለማድረግ ምትክ አይደለም። ማኅተሙ የደህንነት ግምገማ
  *መዝገብ* ነው፣ ግምገማው ራሱ አይደለም።
- ለትክክለኛ ግምገማዎች ምትክ አይደለም። ማኅተሙ የሚያመነጩትን ውጤቶች ይሰካል፣
  ስልቱን አያረጋግጥም።
- ሞዴሉ ደህንነቱ የተጠበቀ መሆኑን ዋስትና አይደለም። ማኅተሙ ላቦራቶሪው ስለ ሞዴሉ
  በመለቀቅ ጊዜ የሚለውን ይመዘግባል፤ እነዚያን ይዞዎች አያረጋግጥም።
- በዳውን-ስትሪም ተጠቃሚዎች ከሚደረግ የሞዴል መጥፎ መጠቀም ጥበቃ አይደለም።

## ስለ ተቃራኒ fine-tuning ማስታወሻ

የተለመደ የ AI-ደህንነት ስጋት፦ አንድ ሰው ሞዴልዎን የደህንነት ባህሪዎችን ለማስወገድ
fine-tune ያደርግና እንደገና ያከፋፍላል። ማኅተሙ ይህን ሊከላከል **አይችልም**።
ማኅተሙ ሊያደርግ የሚችለው፦

- *ኦሪጅናሉ* ሞዴል ምን ማድረግ እንደነበረበት መሰካት (FACT 04)።
- ኦሪጅናሉ የግምገማ ውጤቶች ምን እንደነበሩ መሰካት (FACT 05)።
- ተቃራኒ fine-tunes ላይ ለመለካት የሚያስችል ማመሳከሪያ ነጥብ ማቅረብ።

የሦስተኛ-ወገን እንደገና ማከፋፈል የሚገጥመው ተጠቃሚ weights ከኦሪጅናሉ የታሸገ
hash ጋር የሚስማማ መሆኑን ማረጋገጥ ይችላል። ካልሆነ፣ ተጠቃሚው ኦሪጅናል ያልሆነ
ሞዴል እንዳለው እና ለላቦራቶሪው የተሰጡ ማንኛውም የደህንነት ይዞዎች የማይተገበሩ
መሆናቸውን ያውቃል።

## መዝጊያ

የሞዴል መለቀቅ ማሸግ ለላቦራቶሪ ትንሽ ተጨማሪ ሥራ ነው፦ hashes ለማስላት እና
የማሸግ ስክሪፕት ለማስኬድ ~10 ደቂቃ፣ ከዚያ የትኞቹ የቅድመ-ማስፈር ቃላት እንደሚካተቱ
30-ሰከንድ ውሳኔ። የሚጨምረው ኃላፊነት ሚዛን የጎደለው ነው፦ ለታማኝ ላቦራቶሪዎች
ትንሽ ወጪ፣ ይዞዎቻቸውን ካልሆነ በዝምታ ለሚንሳፉ ላቦራቶሪዎች ትልቅ ገደብ።

ስለመቀበል እያሰበ ያለ AI ላቦራቶሪ ከሆኑ፣ ስብስቡ የሕዝብ ጎራ (CC0) ነው። ይቅረጹት።
