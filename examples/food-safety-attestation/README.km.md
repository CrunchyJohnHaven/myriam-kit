  ហ៊ុនផលិតទេ។ Principal ចុះហត្ថលេខាក្នុង FACT 08 គឺជាអ្នកចុះហត្ថ
  លេខាត្រឹមត្រូវតែមួយគត់សម្រាប់ការនោះ។ (R8)

ការបដិសេធខ្លួនវាផ្ទាល់អាចត្រូវបានបិទត្រាជាការពិត ("ក្រុមហ៊ុនផលិតនេះ
ស្នើការបិទត្រានៃ X; ឧបករណ៍បដិសេធតាមច្បាប់បដិសេធ Y") បង្កើតកំណត់ត្រា
សាធារណៈនៃការប៉ុនប៉ងប្រើប្រាស់ខុស។

## អ្វីដែលឧបករណ៍នេះបិទត្រា

8 ការពិតនៅគ្រាននៃការបញ្ចេញ៖

1. **ការជូនដំណឹងឧទាហរណ៍** — ប្រកាសការបោះផ្សាយជាឧទាហរណ៍ (ការ
   បញ្ជាក់ពិតប្រាកដដកចេញ ឬជំនួសវា)។
2. **ការកំណត់អត្តសញ្ញាណផលិតផល និងបាច់** — ឈ្មោះផលិតផល ប្រភេទ
   ក្រុមហ៊ុនផលិត តំបន់រោងចក្រ (មិនមែនទីតាំងច្បាស់លាស់) កំណែផែនការ
   HACCP + hash, lot ID, កាលបរិច្ឆេទផលិត / បំពាក់ / ប្រើដោយ, ចំនួន
   គ្រឿងផ្សំអ្នកប្រើប្រាស់, តម្រូវការទុក, ចំណុចត្រួតពិនិត្យសំខាន់បី
   ជាមួយ hash កំណត់ត្រាដែលបានកត់ត្រា (cook lethality, cooling
   stabilization, post-lethality RTE environment)។
3. **លទ្ធផលនៃការសាកល្បងបញ្ចេញ** — ការសាកល្បង microbiological ផលិតផល
   ដែលបញ្ចប់ (Listeria, Salmonella, STEC, APC, Enterobacteriaceae)
   ការសាកល្បងគីមី / កាកសំណល់ (nitrite, កាកសំណល់ veterinary, ដែកធ្ងន់)
   ការផ្ទៀងផ្ទាត់ការប៉ះពាល់ឆ្លងនៃ allergen ការតាមដានបរិស្ថានសង្ខេប
   នីមួយៗជាមួយ SHA-256 នៃ PDF របាយការណ៍ពេញលេញ និងការរឹតបន្តឹងការ
   បកស្រាយ sample-vs-bulk ច្បាស់លាស់។
4. **វិធីសាស្ត្រ និងគុណវុឌ្ឍិមន្ទីរពិសោធន៍** — វិធីសាស្ត្រណា (និងកំណែ
   ឯកសារវិធីសាស្ត្រណា) ត្រូវបានប្រើតាមជួរ មន្ទីរពិសោធន៍ណាដំណើរការការ
   សាកល្បងណា ស្ថានភាពទទួលស្គាល់ ISO/IEC 17025:2017 និង hash វិញ្ញាបនបត្រ
   លទ្ធផល proficiency-test ថ្មីៗ សេចក្តីថ្លែងការណ៍ឯករាជ្យ កំណត់ត្រា
   ការប្រែប្រួល / retest / OOS សម្រាប់ lot នេះ។
5. **ការសម្រេចចិត្តបញ្ចេញ** — កាលបរិច្ឆេទបញ្ចេញ អាជ្ញាធរបញ្ចេញកម្រិត
   តួនាទី + ការចុះហត្ថលេខាបញ្ច្រាស សង្ខេបកំណត់ត្រាគាំទ្រការសម្រេច
   ចិត្ត គោលដៅចែកចាយកម្រិតខ្ពស់ កាតព្វកិច្ចក្រោយការបញ្ចេញដែលបាន
   ទទួលស្គាល់នៅពេលបញ្ចេញ ការតាំងសច្ចានៃពេលវេលា-និង-ភាពមិនអាច
   ត្រឡប់ក្រោយ។
6. **ការតាំងសច្ចានៃការដកវិល និងការតាមដាន post-market** — ការប្រៀប
   ធៀបសុវត្ថិភាពម្ហូបអាហារនៃ pharmacovigilance: តម្លាភាពសញ្ញាជំងឺ
   ការតាំងសច្ចានៃការប្រតិបត្តិការដកវិល (24h / 72h / 98% ដែនកំណត់)
   ការឆ្លើយតបការចម្លងរោគខ្សែសង្វាក់ផ្គត់ផ្គង់ ការតាំងសច្ចា no-silent-
   retesting ការតាំងសច្ចា no-post-hoc-specification-changes ការចែករំ
   លែកសញ្ញាសុវត្ថិភាព aggregate។
7. **ការបង្ហាញ companion និងកំណត់ត្រាបដិសេធ** — hash នៃ log ស៊ើបអង្កេត
   ការបន្តសកម្ម ប្រវត្តិការត្រួតពិនិត្យថ្មីៗ ប្រវត្តិដកវិល បញ្ហាបើក
   ដែលដឹងនៅក្នុងវិធីសាស្ត្រសាកល្បងដែលប្រើ ការផ្សារភ្ជាប់តួនាទី-ទៅ-
   បុគ្គល (បិទត្រាដាច់ដោយឡែក hash នៅទីនេះ) ការត្រៀមខ្លួន traceback
   អ្នកផ្គត់ផ្គង់ បញ្ជីច្បាស់នៃច្បាប់បដិសេធ R1-R8 ជាមួយការបដិសេធណាមួយ
   ដែលធ្វើនៅក្នុងការបញ្ចេញនេះ។
8. **Principal ចុះហត្ថលេខា** — ក្រុមហ៊ុនផលិត + អ្នកចុះហត្ថលេខាកម្រិត
   តួនាទី (QA Director) + ការចុះហត្ថលេខាបញ្ច្រាសកម្រិតតួនាទី (អ្នក
   គ្រប់គ្រងរោងចក្រ) + ព័ត៌មាន cryptographic key; មូលហេតុនៃការបោះ
   ផ្សាយកម្រិតតួនាទី។

ការពិតនីមួយៗបញ្ចប់ដោយអនុសញ្ញា refusal-gate: *"ការពិតនេះអះអាងតែ X
ប៉ុណ្ណោះ។ វាមិនអះអាង Y ទេ"* (ដែល Y ជារបៀបនៃការបរាជ័យដែលជិតបំផុតពី
ផ្នែក "DOES NOT do" ខាងលើ)។

## អ្វីដែលឧបករណ៍នេះការពារ

- **ការកែប្រែស្ងាត់ៗនៃលទ្ធផលនៃការសាកល្បងបញ្ចេញ។** PDF របាយការណ៍សាកល្បង
  ទាំងអស់ត្រូវ hash។ Re-hashing ក្រោយការបញ្ចេញបង្ហាញការកែប្រែណាមួយ។
- **ជម្លោះអំពីវិធីសាស្ត្រណា កំណែឯកសារវិធីសាស្ត្រណា និងមន្ទីរពិសោធន៍
  ណាបានប្រើ។** FACT 04 បង្កកវិធីសាស្ត្រ និងស្ថានភាពទទួលស្គាល់មន្ទីរ
  ពិសោធន៍នៅពេលបញ្ចេញ។ ការអះអាងពេលក្រោយ "យើងពិតជាបានប្រើ MLG 4.13
  មិនមែន 4.12" អាចបដិសេធដោយសាធារណៈ។
- **ការ re-spec ក្រោយការសាកល្បងសម្រាប់ batch ដែលនៅក្រៅ spec។** ប្រសិន
  បើលទ្ធផលនៃការសាកល្បងបញ្ចេញពិតប្រាកដជា PASS នៅគែម ហើយត្រូវបានចាត់
  ថ្នាក់ឡើងវិញជា FAIL ក្រោយការបញ្ចេញ hash នៃរបាយការណ៍ដែលបានបិទត្រា
  បង្ហាញពីអ្វីដែល PDF លទ្ធផលដើមបាននិយាយ។ ស៊ីមេទ្រី OOS នៅគែមដែលត្រូវ
  ស្ងាត់ៗ retest ទៅ PASS ដោយគ្មានការបង្ហាញអាចរកឃើញប្រសិនបើការតាំង
  សច្ចា no-silent-retesting របស់ FACT 06 ត្រូវបានប្រៀបធៀបនៅពេលក្រោយ
  ប្រឆាំងនឹងការកែប្រែដែលមិនបានបិទត្រា។
- **ការកាត់បន្ថយវិសាលភាពនៃការដកវិលដោយស្ងាត់ៗ។** ការតាំងសច្ចានៃការដក
  វិលនៅក្នុង FACT 06 (24h notification, 72h initiation, 98%
  recoverability) ត្រូវបានបិទត្រា; ការមិនគោរពពួកវាអាចមើលឃើញដោយ
  សាធារណៈ។
- **ការនិទានឡើងវិញ post-hoc នៃប្រវត្តិសកម្មភាពកែតម្រូវ។** Hash នៃ log
  សកម្មភាពកែតម្រូវរបស់ FACT 07 បង្កកអ្វីដែលក្រុមហ៊ុនបាននិយាយថាខ្លួន
  បានធ្វើបន្ទាប់ពី EMP presumptive; ការអះអាងពេលក្រោយ "យើងបានធ្វើ
  ច្រើនជាង/តិចជាង" អាចបដិសេធដោយសាធារណៈ។
- **ការដកការបង្ហាញ companion ដោយជ្រើសរើស។** FACT 07 រាប់ប្រភេទនៃ
  ការបង្ហាញដែលរំពឹង (ប្រវត្តិការត្រួតពិនិត្យ ប្រវត្តិដកវិល ការពិនិត្យ
  ការប្រាប់វិធីសាស្ត្រ ការត្រៀមខ្លួន traceback); ប្រភេទទទេត្រូវតែ
  បញ្ជាក់យ៉ាងច្បាស់ ("គ្មាននៅពេលបញ្ចេញ") ដូច្នេះការនិយាយពេលក្រោយ
  "យើងមិនបានដឹងថាវាពាក់ព័ន្ធ" អាចបដិសេធដោយសាធារណៈ។
- **ការជំនួសស្ថានភាពទទួលស្គាល់ផ្សេងគ្នានៃមន្ទីរពិសោធន៍។** FACT 04 បង្កក
  វិសាលភាព ISO 17025 និង hash វិញ្ញាបនបត្ររបស់មន្ទីរពិសោធន៍នីមួយៗនៅ
  ពេលបញ្ចេញ; ការជំនួសពេលក្រោយនៃមន្ទីរពិសោធន៍ដែលមិនមានការទទួលស្គាល់
  អាចរកឃើញដោយសាធារណៈ។

ឧបករណ៍ **មិន** ការពារប្រឆាំងនឹងអំពើពុករលួយក្នុងពេលជាមួយ — ក្រុមដែល
