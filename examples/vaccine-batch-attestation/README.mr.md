# लस बॅच साक्षांकन किट

Bitcoin-नांगरलेले, छेडछाड-स्पष्ट बॅच रिलीज सील प्रकाशित करू इच्छिणाऱ्या लस उत्पादक, गुणवत्ता आश्वासन प्राधिकरण आणि सार्वजनिक आरोग्य संस्थांसाठी एक तयार-वापर myriam-kit उदाहरण.

## हे किट कशापासून संरक्षण करते

**सीलला पकडता येणारे अपयश प्रकार:**

- **रिलीज चाचणी निकालांचे शांत पूर्वलक्षी सुधारणा.** उदाहरणामधील सर्व 8 रिलीज-चाचणी PDFs hash केल्या आहेत. रिलीजनंतर पुन्हा-hashing कोणताही बदल उघड करते.
- **रिलीजच्या वेळी कोणती तपशीलवार माहिती लागू होती याबद्दलचे वाद.** सीलबंद FACT 02 मध्ये कॅनोनिकल बॅच ओळख आणि उत्पादन-रेकॉर्ड hash आहे. भविष्यातील "स्पेक नेहमी X होते" वादांना सार्वजनिक नांगर आहे.
- **शांत रिकॉल-व्याप्ती कमी करणे.** FACT 05 मधील फार्माकोव्हिजिलन्स वचनबद्धता सीलबंद आहेत; त्यांचे पालन न करणे सार्वजनिकपणे पाहता येण्यासारखे आहे.
- **स्पेक-बाहेरील बॅचसाठी पूर्वलक्षी पुनःस्पेक.** जर रिलीज-चाचणीचा निकाल खरोखर सीमारेषेवरील PASS होता तो रिलीजनंतर FAIL म्हणून पुनःवर्गीकृत केला गेला, तर सीलबंद अहवाल hashes मूळ निकाल PDF काय म्हणाला हे सिद्ध करतात.

**समाविष्ट नसलेले अपयश प्रकार:**

- उत्पादन प्रक्रियेतच फसवणूक (सील चाचणी निकालांना नांगरते, उत्पादन सचोटीला नाही).
- अन्यथा-उत्तीर्ण बॅचच्या जैविक परिणामांमुळे प्रतिकूल घटना (सील क्लिनिकल सुरक्षा प्रमाणित करत नाही, फक्त दस्तऐवज सचोटी).
- नियामक तपशीलाची कोणती आवृत्ती अधिकृत आहे याबद्दलचे वाद.

## हे किट कधी वापरावे

बॅच रिलीज सील करण्याचा विचार करा जर:

- तुम्ही लस उत्पादक आहात जो नियामकांच्या आवश्यकतेपेक्षा जास्त रिलीज रेकॉर्डची फॉरेन्सिक सचोटी दाखवू इच्छितो
- तुम्ही "आम्ही जे साक्षांकित करतो ते प्रकाशित करा" अभ्यास स्वीकारणारी QA टीम आहात
- तुम्ही दुर्बल नियामक पायाभूत सुविधा असलेल्या अधिकारक्षेत्रातील लहान/राष्ट्रीय-प्रमाणाचे उत्पादक आहात, जागतिक सत्यापकाविरुद्ध बॅच रेकॉर्ड नांगरू इच्छिता
- तुम्ही दुर्लक्षित रोगांसाठी अव्यावसायिक लस उत्पादन कार्यक्रम चालवणारी सार्वजनिक-आरोग्य संस्था किंवा NGO आहात

## ऐतिहासिक मृत्यू पूर्वगामी

- **Heparin दूषितपणा (2008, China):** heparin बॅचेसमध्ये बनावट oversulfated chondroitin sulfate. एक सीलबंद रिलीज-चाचणी मॅनिफेस्ट चाचणी रेकॉर्डचा रिलीज-नंतरचा शांत बदल क्रिप्टोग्राफिकदृष्ट्या शोधण्यायोग्य बनवला असता.
- **Tylenol cyanide (1982):** छेडछाडीच्या उत्पत्ती बिंदूबद्दल पूर्वलक्षी वाद. एक सीलबंद उत्पादन-रेकॉर्ड hash वितरण-पूर्व स्थितीला नांगरते.
- **विविध लस बॅच रिकॉल** जिथे "ही बॅच कधी योग्य प्रकारे चाचणी केली गेली होती का" हा प्रश्न वर्षांनंतर वादग्रस्त होतो.

हे किट यापैकी कोणत्याही गोष्टीसाठी कडक अर्थाने उपाय नाही — परंतु प्रत्येक प्रकरणात, सीलबंद बॅच साक्षांकनांनी अधिक स्वच्छ फॉरेन्सिक कालक्रम दिला असता.

## वास्तविक बॅचसाठी हे किट fork कसे करावे

```bash
cp -r ~/Genesis/myriam-kit/examples/vaccine-batch-attestation ~/release-FBP-XXXX
cd ~/release-FBP-XXXX
$EDITOR facts/*.txt       # काल्पनिक सामग्री वास्तविक बॅचने बदला
bash build/seal.sh        # वितरण सुरू होण्यापूर्वी SEAL करा
python3 build/verify.py   # पुष्टी करा
```

**महत्त्वाची वेळ**: बॅच वितरित होण्यापूर्वी, QA रिलीजच्या क्षणी सील करा. वितरणानंतरचा सील फॉरेन्सिकदृष्ट्या कमजोर असतो.

## विद्यमान फार्मास्युटिकल पायाभूत सुविधांसह एकत्रीकरण

- **eCTD सबमिशन (FDA / EMA):** सीलबंद प्रकाशन URL किंवा IPFS CID Module 3 (गुणवत्ता) विभागामध्ये संदर्भित केले जाऊ शकते
- **WHO Prequalification:** सीलबंद मॅनिफेस्ट FDA/EMA-नसलेल्या अधिकारक्षेत्रांना रिलीज-पूर्व चाचणी साखळी दर्शविते
- **फार्माकोव्हिजिलन्स अभिप्राय:** FACT 05 उत्पादकाला 6 आणि 12 महिन्यांत एकूण AE संख्यांच्या सीलबंद अहवालासाठी बांधते
- **रिकॉल प्रणाली (RxConnect, MedWatch):** जर रिकॉलची गरज असेल, तर रिकॉलची व्याप्ती स्वतःच MYRIAM अंतर्गत सील केली जाते, सर्व वितरित लॉट संबोधित केले जात आहेत याचा सार्वजनिक पडताळणी प्रदान करते
