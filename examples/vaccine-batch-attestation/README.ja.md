# ワクチンバッチアテステーションキット

*日本語訳 — 原文の英語版については [README.md](README.md) を参照してください*

バッチリリースの Bitcoin にアンカーされた改ざん検出可能なシールを
公表したいワクチン製造業者、品質保証当局、および公衆衛生機関のための
ターンキー型 myriam-kit サンプルです。

## 本キットが防御する対象

**シールが捕捉する失敗様態:**

- **リリース試験結果の遡及的な秘密の改変。** 本サンプルでは 8 件の
  リリース試験 PDF すべてがハッシュ化されています。リリース後に
  再ハッシュ化することで、いかなる改変も明らかになります。
- **リリース時にどの仕様が適用されていたかについての紛争。**
  封緘された FACT 02 には正規のバッチ識別と製造記録ハッシュが
  含まれています。将来の「仕様は常に X だった」という紛争には
  公的なアンカーが存在することになります。
- **リコール範囲の秘密の最小化。** FACT 05 のファーマコビジランス
  公約は封緘されており、それらを履行しないことは公的に観察可能
  です。
- **規格外バッチに対する遡及的な仕様の書き換え。** リリース試験
  結果が実際にはぎりぎりの合格(PASS)だったものをリリース後に
  不合格(FAIL)として再分類した場合、封緘された報告書ハッシュが
  元の結果 PDF に何が記載されていたかを証明します。

**シールが対象としない失敗様態:**

- 製造プロセス自体の不正(シールは試験結果をアンカーするものであり、
  製造工程の完全性をアンカーするものではありません)。
- 他の点では合格バッチの生物学的帰結としての有害事象(シールは
  臨床的安全性を証明するものではなく、文書の完全性のみを証明
  します)。
- 規制仕様のどのバージョンが権威あるものかについての紛争。

## 本キットを使用すべきとき

以下の場合、バッチリリースの封緘を検討してください:

- 規制当局が要求する以上のリリース記録の法医学的完全性を実証
  したいワクチン製造業者である
- 「証明したものを公表する」慣行を採用している QA チームである
- 規制インフラの弱い管轄区域における小規模・国家規模の製造業者で
  あり、世界的検証者に対してバッチ記録をアンカーしたい
- 非商業的なワクチン製造プログラム(例: 顧みられない病気のための)
  を運営している公衆衛生機関または NGO である

## 歴史的な死亡率の前例

- **ヘパリン汚染(2008 年、中国):** ヘパリンバッチに含まれていた
  偽造の過硫酸化コンドロイチン硫酸。封緘されたリリース試験
  マニフェストがあれば、試験記録のリリース後の秘密の改変は
  暗号学的に検出可能だったでしょう。
- **タイレノールのシアン化物事件(1982):** 改ざん発生点に関する
  遡及的な紛争。封緘された製造記録ハッシュは流通前の状態を
  アンカーします。
- **複数のワクチンバッチリコール**: 「このバッチが本当に適切に
  試験されていたか」という問いが数年後に争われるようになる事例。

本キットは厳密な意味でこれらの**いずれも修復するものではありません**。
しかしいずれの場合も、封緘されたバッチアテステーションは、より
清浄な法医学的タイムラインを提供したはずです。

## 実際のバッチに対する本キットのフォーク方法

```bash
cp -r ~/Genesis/myriam-kit/examples/vaccine-batch-attestation ~/release-FBP-XXXX
cd ~/release-FBP-XXXX
$EDITOR facts/*.txt       # 架空の内容を実際のバッチ情報に置き換える
bash build/seal.sh        # 配布開始の前に封緘する
python3 build/verify.py   # 確認
```

**重要なタイミング**: QA リリースの時点で、バッチが流通する前に
封緘してください。流通後の封緘は法医学的に弱くなります。

## 既存の医薬品インフラとの統合

- **eCTD 提出(FDA / EMA):** 封緘された公表 URL または IPFS CID を
  Module 3(品質)セクションで参照することができます
- **WHO Prequalification:** 封緘されたマニフェストは FDA/EMA 以外の
  管轄区域に対してリリース前試験のチェーンを実証します
- **ファーマコビジランスのフィードバック:** FACT 05 は製造業者を
  6 か月および 12 か月での集計 AE 件数の封緘された報告に対して
  公約させます
- **リコールシステム(RxConnect, MedWatch):** リコールが必要な場合、
  リコール範囲自体が MYRIAM の下で封緘され、流通したすべてのロットが
  対応されていることの公的な検証を提供します

## 本キットが代替しないもの

- FDA / EMA / WHO への規制提出
- GMP 監査証跡(シールは結果をアンカーするものであり、生のプロセス
  記録をアンカーするものではありません)
- ファーマコビジランスデータベース(VAERS / EudraVigilance)
- リコール調整システム

**重要な区別**: 封緘された請求は否認防止(non-repudiation)であり、
安全性の証明ではありません。シールは封緘の時点で何が文書化されて
いたかを証明するものであり、ワクチン自体が安全であることや、製造
プロセスが GMP に準拠していたことを証明するものではありません。
