# ಲಸಿಕೆ ಬ್ಯಾಚ್ ದೃಢೀಕರಣ ಕಿಟ್

Bitcoin-ಲಂಗರು ಹಾಕಿದ, ತಿರುಚುವಿಕೆ-ಸಾಕ್ಷ್ಯವಾದ ಬ್ಯಾಚ್ ಬಿಡುಗಡೆಯ ಮುದ್ರೆಯನ್ನು ಪ್ರಕಟಿಸಲು ಬಯಸುವ ಲಸಿಕೆ ತಯಾರಕರು, ಗುಣಮಟ್ಟ ಭರವಸೆ ಅಧಿಕಾರಿಗಳು ಮತ್ತು ಸಾರ್ವಜನಿಕ ಆರೋಗ್ಯ ಸಂಸ್ಥೆಗಳಿಗಾಗಿ ಒಂದು ಸಿದ್ಧ-ಬಳಕೆಯ myriam-kit ಉದಾಹರಣೆ.

## ಈ ಕಿಟ್ ಏನನ್ನು ರಕ್ಷಿಸುತ್ತದೆ

**ಮುದ್ರೆ ಹಿಡಿಯುವ ವೈಫಲ್ಯ ವಿಧಾನಗಳು:**

- **ಬಿಡುಗಡೆ ಪರೀಕ್ಷಾ ಫಲಿತಾಂಶಗಳ ಮೌನವಾದ ಪೂರ್ವಾನ್ವಯಿ ಮಾರ್ಪಾಡು.** ಉದಾಹರಣೆಯಲ್ಲಿನ ಎಲ್ಲಾ 8 ಬಿಡುಗಡೆ-ಪರೀಕ್ಷಾ PDFಗಳನ್ನು hash ಮಾಡಲಾಗಿದೆ. ಬಿಡುಗಡೆಯ ನಂತರ ಮರು-hashing ಯಾವುದೇ ಮಾರ್ಪಾಡನ್ನು ಬಹಿರಂಗಪಡಿಸುತ್ತದೆ.
- **ಬಿಡುಗಡೆಯ ಸಮಯದಲ್ಲಿ ಯಾವ ವಿವರಣೆಗಳು ಅನ್ವಯವಾಗಿದ್ದವು ಎಂಬ ವಿವಾದಗಳು.** ಮುದ್ರಿತ FACT 02 ಅಂಗೀಕೃತ ಬ್ಯಾಚ್ ಗುರುತಿಸುವಿಕೆ ಮತ್ತು ತಯಾರಿಕಾ-ದಾಖಲೆ hashಅನ್ನು ಒಳಗೊಂಡಿದೆ. ಭವಿಷ್ಯದ "ವಿವರಣೆ ಯಾವಾಗಲೂ X ಆಗಿತ್ತು" ವಿವಾದಗಳಿಗೆ ಸಾರ್ವಜನಿಕ ಲಂಗರು ಇದೆ.
- **ಮೌನವಾದ ಮರುಪಡೆಯುವಿಕೆ-ವ್ಯಾಪ್ತಿ ಕನಿಷ್ಠೀಕರಣ.** FACT 05 ರಲ್ಲಿನ ಫಾರ್ಮಾಕೋವಿಜಿಲೆನ್ಸ್ ಬದ್ಧತೆಗಳು ಮುದ್ರಿತವಾಗಿವೆ; ಅವುಗಳನ್ನು ಗೌರವಿಸದಿರುವುದು ಸಾರ್ವಜನಿಕವಾಗಿ ಗಮನಿಸಬಹುದು.
- **ವಿವರಣೆ-ಹೊರಗಿನ ಬ್ಯಾಚ್‌ಗಾಗಿ ಪೂರ್ವಾನ್ವಯಿ ಮರು-ವಿವರಣೆ.** ಒಂದು ಬಿಡುಗಡೆ-ಪರೀಕ್ಷಾ ಫಲಿತಾಂಶವು ವಾಸ್ತವವಾಗಿ ಗಡಿರೇಖೆಯ PASS ಆಗಿದ್ದು ಅದನ್ನು ಬಿಡುಗಡೆಯ ನಂತರ FAIL ಎಂದು ಮರು-ವರ್ಗೀಕರಿಸಿದರೆ, ಮುದ್ರಿತ ವರದಿ hashಗಳು ಮೂಲ ಫಲಿತಾಂಶ PDF ಏನು ಹೇಳಿತು ಎಂಬುದನ್ನು ಸಾಬೀತುಪಡಿಸುತ್ತವೆ.

**ಒಳಗೊಳ್ಳದ ವೈಫಲ್ಯ ವಿಧಾನಗಳು:**

- ತಯಾರಿಕಾ ಪ್ರಕ್ರಿಯೆಯಲ್ಲಿಯೇ ಮೋಸ (ಮುದ್ರೆ ಪರೀಕ್ಷಾ ಫಲಿತಾಂಶಗಳನ್ನು ಲಂಗರು ಹಾಕುತ್ತದೆ, ತಯಾರಿಕಾ ಸಮಗ್ರತೆಯನ್ನಲ್ಲ).
- ಇಲ್ಲದಿದ್ದರೆ-ಉತ್ತೀರ್ಣ ಬ್ಯಾಚ್‌ನ ಜೈವಿಕ ಪರಿಣಾಮಗಳಾದ ಪ್ರತಿಕೂಲ ಘಟನೆಗಳು (ಮುದ್ರೆ ಕ್ಲಿನಿಕಲ್ ಸುರಕ್ಷತೆಯನ್ನು ಪ್ರಮಾಣೀಕರಿಸುವುದಿಲ್ಲ, ಕೇವಲ ದಸ್ತಾವೇಜು ಸಮಗ್ರತೆಯನ್ನು ಮಾತ್ರ).
- ನಿಯಂತ್ರಕ ವಿವರಣೆಯ ಯಾವ ಆವೃತ್ತಿ ಅಧಿಕೃತ ಎಂಬ ವಿವಾದಗಳು.

## ಈ ಕಿಟ್ ಅನ್ನು ಯಾವಾಗ ಬಳಸಬೇಕು

ಒಂದು ಬ್ಯಾಚ್ ಬಿಡುಗಡೆಯನ್ನು ಮುದ್ರಿಸುವುದನ್ನು ಪರಿಗಣಿಸಿ:

- ನಿಯಂತ್ರಕರು ಬೇಡಿಕೆಯಿಡುವುದಕ್ಕಿಂತ ಹೆಚ್ಚಿನ ಬಿಡುಗಡೆ ದಾಖಲೆಗಳ ನ್ಯಾಯವೈದ್ಯಕೀಯ ಸಮಗ್ರತೆಯನ್ನು ಪ್ರದರ್ಶಿಸಲು ಬಯಸುವ ಲಸಿಕೆ ತಯಾರಕರಾಗಿದ್ದರೆ
- "ನಾವು ದೃಢೀಕರಿಸುವದನ್ನು ಪ್ರಕಟಿಸಿ" ಅಭ್ಯಾಸವನ್ನು ಅಳವಡಿಸಿಕೊಳ್ಳುತ್ತಿರುವ QA ತಂಡ
- ದುರ್ಬಲ ನಿಯಂತ್ರಕ ಮೂಲಸೌಕರ್ಯ ಹೊಂದಿರುವ ನ್ಯಾಯವ್ಯಾಪ್ತಿಯಲ್ಲಿನ ಸಣ್ಣ/ರಾಷ್ಟ್ರೀಯ-ಪ್ರಮಾಣದ ತಯಾರಕರು, ಜಾಗತಿಕ ಪರಿಶೀಲಕರ ವಿರುದ್ಧ ಬ್ಯಾಚ್ ದಾಖಲೆಗಳನ್ನು ಲಂಗರು ಹಾಕಲು ಬಯಸಿದರೆ
- ನಿರ್ಲಕ್ಷಿತ ರೋಗಗಳಿಗಾಗಿ ವಾಣಿಜ್ಯೇತರ ಲಸಿಕೆ ಉತ್ಪಾದನಾ ಕಾರ್ಯಕ್ರಮವನ್ನು ನಡೆಸುತ್ತಿರುವ ಸಾರ್ವಜನಿಕ-ಆರೋಗ್ಯ ಸಂಸ್ಥೆ ಅಥವಾ NGO ಆಗಿದ್ದರೆ

## ಐತಿಹಾಸಿಕ ಮರಣ ಪೂರ್ವನಿದರ್ಶನ

- **Heparin ಮಾಲಿನ್ಯ (2008, China):** heparin ಬ್ಯಾಚ್‌ಗಳಲ್ಲಿ ನಕಲಿ oversulfated chondroitin sulfate. ಮುದ್ರಿತ ಬಿಡುಗಡೆ-ಪರೀಕ್ಷಾ ಪಟ್ಟಿಯು ಪರೀಕ್ಷಾ ದಾಖಲೆಯ ಬಿಡುಗಡೆಯ ನಂತರದ ಮೌನ ಮಾರ್ಪಾಡನ್ನು ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕವಾಗಿ ಪತ್ತೆಹಚ್ಚಬಹುದಾಗಿರಿಸುತ್ತಿತ್ತು.
- **Tylenol cyanide (1982):** ತಿರುಚುವಿಕೆ ಮೂಲದ ಬಗ್ಗೆ ಪೂರ್ವಾನ್ವಯಿ ವಿವಾದಗಳು. ಮುದ್ರಿತ ತಯಾರಿಕಾ-ದಾಖಲೆ hash ವಿತರಣಾ-ಪೂರ್ವ ಸ್ಥಿತಿಯನ್ನು ಲಂಗರು ಹಾಕುತ್ತದೆ.
- **ವಿವಿಧ ಲಸಿಕೆ ಬ್ಯಾಚ್ ಮರುಪಡೆಯುವಿಕೆಗಳು** ಅಲ್ಲಿ "ಈ ಬ್ಯಾಚ್ ಎಂದಾದರೂ ಸರಿಯಾಗಿ ಪರೀಕ್ಷಿಸಲ್ಪಟ್ಟಿತ್ತೆ" ಎಂಬ ಪ್ರಶ್ನೆ ವರ್ಷಗಳ ನಂತರ ವಿವಾದಾಸ್ಪದವಾಗುತ್ತದೆ.

ಕಿಟ್ ಯಾವುದಕ್ಕೂ ಕಟ್ಟುನಿಟ್ಟಾದ ಅರ್ಥದಲ್ಲಿ ಪರಿಹಾರವಲ್ಲ — ಆದರೆ ಪ್ರತಿಯೊಂದು ಸಂದರ್ಭದಲ್ಲೂ, ಮುದ್ರಿತ ಬ್ಯಾಚ್ ದೃಢೀಕರಣಗಳು ಸ್ವಚ್ಛವಾದ ನ್ಯಾಯವೈದ್ಯಕೀಯ ಕಾಲರೇಖೆಯನ್ನು ಒದಗಿಸುತ್ತಿತ್ತು.

## ನಿಜವಾದ ಬ್ಯಾಚ್‌ಗಾಗಿ ಈ ಕಿಟ್ ಅನ್ನು fork ಮಾಡುವುದು ಹೇಗೆ

```bash
cp -r ~/Genesis/myriam-kit/examples/vaccine-batch-attestation ~/release-FBP-XXXX
cd ~/release-FBP-XXXX
$EDITOR facts/*.txt       # ಕಾಲ್ಪನಿಕ ವಿಷಯವನ್ನು ನಿಜವಾದ ಬ್ಯಾಚ್‌ನಿಂದ ಬದಲಾಯಿಸಿ
bash build/seal.sh        # ವಿತರಣೆ ಪ್ರಾರಂಭವಾಗುವ ಮೊದಲು SEAL ಮಾಡಿ
python3 build/verify.py   # ದೃಢೀಕರಿಸಿ
```

**ನಿರ್ಣಾಯಕ ಸಮಯ**: ಬ್ಯಾಚ್ ವಿತರಣೆಗೊಳ್ಳುವ ಮೊದಲು, QA ಬಿಡುಗಡೆಯ ಕ್ಷಣದಲ್ಲಿ ಮುದ್ರಿಸಿ. ವಿತರಣೆಯ ನಂತರದ ಮುದ್ರೆಯು ನ್ಯಾಯವೈದ್ಯಕೀಯವಾಗಿ ದುರ್ಬಲವಾಗಿರುತ್ತದೆ.

## ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ಔಷಧೀಯ ಮೂಲಸೌಕರ್ಯದೊಂದಿಗೆ ಏಕೀಕರಣ

- **eCTD ಸಲ್ಲಿಕೆಗಳು (FDA / EMA):** ಮುದ್ರಿತ ಪ್ರಕಟಣೆ URL ಅಥವಾ IPFS CID ಅನ್ನು Module 3 (ಗುಣಮಟ್ಟ) ವಿಭಾಗದಲ್ಲಿ ಉಲ್ಲೇಖಿಸಬಹುದು
- **WHO Prequalification:** ಮುದ್ರಿತ ಪಟ್ಟಿಯು FDA/EMA-ಅಲ್ಲದ ನ್ಯಾಯವ್ಯಾಪ್ತಿಗಳಿಗೆ ಬಿಡುಗಡೆ-ಪೂರ್ವ ಪರೀಕ್ಷಾ ಸರಪಳಿಯನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತದೆ
- **ಫಾರ್ಮಾಕೋವಿಜಿಲೆನ್ಸ್ ಪ್ರತಿಕ್ರಿಯೆ:** FACT 05 ತಯಾರಕರನ್ನು 6 ಮತ್ತು 12 ತಿಂಗಳಲ್ಲಿ ಒಟ್ಟು AE ಸಂಖ್ಯೆಗಳ ಮುದ್ರಿತ ವರದಿಗೆ ಬದ್ಧಗೊಳಿಸುತ್ತದೆ
- **ಮರುಪಡೆಯುವಿಕೆ ವ್ಯವಸ್ಥೆಗಳು (RxConnect, MedWatch):** ಮರುಪಡೆಯುವಿಕೆ ಅಗತ್ಯವಿದ್ದರೆ, ಮರುಪಡೆಯುವಿಕೆ ವ್ಯಾಪ್ತಿಯನ್ನು MYRIAM ಅಡಿಯಲ್ಲಿ ಮುದ್ರಿಸಲಾಗುತ್ತದೆ, ಎಲ್ಲಾ ವಿತರಿತ ಬ್ಯಾಚ್‌ಗಳನ್ನು ಪರಿಹರಿಸಲಾಗುತ್ತಿದೆ ಎಂಬ ಸಾರ್ವಜನಿಕ ಪರಿಶೀಲನೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ

## ಈ ಕಿಟ್ ಬದಲಾಯಿಸದಿರುವುದು

- FDA / EMA / WHO ನಿಯಂತ್ರಕ ಸಲ್ಲಿಕೆಗಳು
- GMP ಲೆಕ್ಕಪರಿಶೋಧನಾ ದಾರಿಗಳು (ಮುದ್ರೆ ಫಲಿತಾಂಶಗಳನ್ನು ಲಂಗರು ಹಾಕುತ್ತದೆ, ಕಚ್ಚಾ ಪ್ರಕ್ರಿಯಾ ದಾಖಲೆಗಳಲ್ಲ)
- ಫಾರ್ಮಾಕೋವಿಜಿಲೆನ್ಸ್ ಡೇಟಾಬೇಸ್‌ಗಳು (VAERS / EudraVigilance)
- ಮರುಪಡೆಯುವಿಕೆ ಸಮನ್ವಯ ವ್ಯವಸ್ಥೆಗಳು
