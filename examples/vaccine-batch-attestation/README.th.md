# ชุดเครื่องมือการรับรองล็อตวัคซีน

ตัวอย่าง myriam-kit แบบพร้อมใช้งานสำหรับผู้ผลิตวัคซีน หน่วยงานประกัน
คุณภาพ และหน่วยงานสาธารณสุขที่ต้องการเผยแพร่ตราประทับการปล่อยล็อตที่ยึด
โยงกับ Bitcoin และตรวจสอบการแก้ไขได้

## ชุดเครื่องมือนี้ป้องกันอะไร

**โหมดความล้มเหลวที่ตราประทับจับได้:**

- **การดัดแปลงผลการทดสอบการปล่อยล็อตย้อนหลังอย่างเงียบ ๆ** ไฟล์ PDF ทดสอบ
  การปล่อยล็อตทั้ง 8 ไฟล์ในตัวอย่างถูกแฮชไว้ การแฮชใหม่หลังการปล่อยล็อตจะ
  เปิดเผยการดัดแปลงใด ๆ
- **ข้อพิพาทเกี่ยวกับว่าข้อกำหนดใดที่ใช้ในขณะปล่อยล็อต** FACT 02 ที่ปิดผนึก
  มีรหัสประจำตัวล็อตอย่างเป็นทางการและแฮชของบันทึกการผลิต ข้อพิพาทใน
  อนาคตที่ว่า "ข้อกำหนดเป็น X เสมอมา" มีจุดยึดสาธารณะ
- **การลดขอบเขตการเรียกคืนอย่างเงียบ ๆ** คำมั่นด้านการเฝ้าระวังความปลอดภัย
  ทางเภสัชกรรมใน FACT 05 ถูกปิดผนึก การไม่ปฏิบัติตามสามารถสังเกตได้
  สาธารณะ
- **การกำหนดข้อกำหนดใหม่ย้อนหลังสำหรับล็อตที่ไม่ผ่านข้อกำหนด** หากผล
  การทดสอบการปล่อยล็อตเป็น PASS ที่ก้ำกึ่งแล้วถูกจัดประเภทใหม่เป็น FAIL
  หลังการปล่อยล็อต แฮชของรายงานที่ปิดผนึกจะพิสูจน์สิ่งที่ PDF ผลลัพธ์
  ต้นฉบับระบุไว้

**โหมดความล้มเหลวที่ไม่ครอบคลุม:**

- การฉ้อโกงในกระบวนการผลิตเอง (ตราประทับยึดผลการทดสอบ ไม่ใช่ความถูกต้อง
  สมบูรณ์ของการผลิต)
- เหตุการณ์ไม่พึงประสงค์ที่เป็นผลทางชีวภาพของล็อตที่ผ่านมาตรฐาน (ตรา
  ประทับไม่รับรองความปลอดภัยทางคลินิก เพียงแต่รับรองความถูกต้องสมบูรณ์ของ
  เอกสาร)
- ข้อพิพาทเกี่ยวกับว่าฉบับใดของข้อกำหนดทางกฎระเบียบเป็นทางการ

## เมื่อใดควรใช้ชุดเครื่องมือนี้

พิจารณาปิดผนึกการปล่อยล็อตหาก:

- คุณเป็นผู้ผลิตวัคซีนที่ต้องการแสดงความถูกต้องสมบูรณ์เชิงนิติวิทยาศาสตร์
  ของบันทึกการปล่อยล็อตเกินกว่าที่หน่วยงานกำกับดูแลกำหนด
- คุณเป็นทีม QA ที่นำแนวทาง "เผยแพร่สิ่งที่เรารับรอง" มาใช้
- คุณเป็นผู้ผลิตขนาดเล็ก/ระดับชาติในเขตอำนาจที่มีโครงสร้างพื้นฐานทาง
  กฎระเบียบที่อ่อนแอกว่า ต้องการยึดบันทึกล็อตไว้กับผู้ตรวจสอบระดับโลก
- คุณเป็นหน่วยงานสาธารณสุขหรือ NGO ที่ดำเนินโครงการผลิตวัคซีนเชิงไม่
  พาณิชย์ (เช่น สำหรับโรคที่ถูกละเลย)

## บรรทัดฐานการเสียชีวิตทางประวัติศาสตร์

- **การปนเปื้อนเฮปาริน (2008, จีน):** chondroitin sulfate ที่ถูก oversulfate
  ปลอมแปลงในล็อตเฮปาริน รายการการทดสอบการปล่อยล็อตที่ปิดผนึกจะทำให้การ
  ดัดแปลงบันทึกการทดสอบหลังการปล่อยล็อตอย่างเงียบ ๆ สามารถตรวจจับได้ทาง
  วิทยาการเข้ารหัส
- **Tylenol cyanide (1982):** ข้อพิพาทย้อนหลังเกี่ยวกับจุดกำเนิดของการ
  ดัดแปลง แฮชของบันทึกการผลิตที่ปิดผนึกยึดสถานะก่อนการจัดจำหน่าย
- **การเรียกคืนล็อตวัคซีนหลายกรณี** ที่คำถามว่า "ล็อตนี้เคยถูกทดสอบอย่าง
  เหมาะสมหรือไม่" กลายเป็นข้อพิพาทหลายปีต่อมา

ชุดเครื่องมือนี้ไม่ใช่การแก้ไขสำหรับกรณีใด ๆ ในความหมายที่เคร่งครัด — แต่
ในแต่ละกรณี การรับรองล็อตที่ปิดผนึกจะให้เส้นทางเวลาเชิงนิติวิทยาศาสตร์ที่
ชัดเจนกว่า

## วิธี fork ชุดเครื่องมือนี้สำหรับล็อตจริง

```bash
cp -r ~/Genesis/myriam-kit/examples/vaccine-batch-attestation ~/release-FBP-XXXX
cd ~/release-FBP-XXXX
$EDITOR facts/*.txt       # replace fictional content with real batch
bash build/seal.sh        # SEAL BEFORE distribution begins
python3 build/verify.py   # confirm
```

**เวลาที่สำคัญยิ่ง**: ปิดผนึกในขณะที่ QA ปล่อยล็อต ก่อนที่ล็อตจะถูก
แจกจ่าย ตราประทับหลังการแจกจ่ายมีค่าทางนิติวิทยาศาสตร์ที่อ่อนแอกว่า

## การบูรณาการกับโครงสร้างพื้นฐานทางเภสัชกรรมที่มีอยู่

- **การส่งเอกสาร eCTD (FDA / EMA):** URL การเผยแพร่ที่ปิดผนึกหรือ CID IPFS
  สามารถอ้างอิงได้ในส่วน Module 3 (คุณภาพ)
- **การพิจารณาคุณสมบัติเบื้องต้นของ WHO:** รายการที่ปิดผนึกแสดงห่วงโซ่การ
  ทดสอบก่อนการปล่อยล็อตให้กับเขตอำนาจที่ไม่ใช่ FDA/EMA
- **ข้อมูลย้อนกลับด้านการเฝ้าระวังความปลอดภัยทางเภสัชกรรม:** FACT 05 ผูกมัด
  ผู้ผลิตให้รายงานจำนวน AE รวมที่ปิดผนึกในเดือนที่ 6 และ 12
- **ระบบเรียกคืน (RxConnect, MedWatch):** หากจำเป็นต้องเรียกคืน ขอบเขตการ
  เรียกคืนเองจะถูกปิดผนึกภายใต้ MYRIAM ให้การยืนยันสาธารณะว่าล็อตที่
  แจกจ่ายทั้งหมดกำลังได้รับการจัดการ

## สิ่งที่ชุดเครื่องมือนี้ไม่ทดแทน

- การยื่นเอกสารทางกฎระเบียบของ FDA / EMA / WHO
- บันทึกตรวจสอบ GMP (ตราประทับยึดผลลัพธ์ ไม่ใช่บันทึกกระบวนการดิบ)
- ฐานข้อมูลการเฝ้าระวังความปลอดภัยทางเภสัชกรรม (VAERS / EudraVigilance)
- ระบบประสานงานการเรียกคืน
