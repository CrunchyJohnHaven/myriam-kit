# የክትባት ስብስብ Attestation ስብስብ

ለክትባት አምራቾች፣ ለጥራት ማረጋገጫ ባለሥልጣናት፣ እና ለሕዝብ ጤና ኤጀንሲዎች የ Bitcoin-anchored፣ የመጣረስ-ምልክት ያለው የስብስብ መለቀቅ ማኅተም ለማተም የተዘጋጀ turnkey myriam-kit ምሳሌ።

## ይህ ስብስብ ምን የሚከላከለው

**ማኅተሙ የሚይዛቸው የውድቀት ሁኔታዎች፦**

- **የመለቀቅ የፍተሻ ውጤቶችን ጸጥተኛ ድኅረ-ኋላ ማሻሻያ።** በምሳሌው ውስጥ ያሉ 8ቱም የመለቀቅ-ፍተሻ PDFs ሁሉም hashed ናቸው። ድኅረ-መለቀቅ እንደገና-hashing ማንኛውንም ማሻሻያ ያሳያል።
- **በመለቀቅ ጊዜ ምን ዓይነት ዝርዝሮች ተግባራዊ ስለነበሩ የተደረጉ ክርክሮች።** የታሸገው FACT 02 መደበኛውን የስብስብ መታወቂያ እና የማምረቻ-መዝገብ hash ይዟል። የወደፊት "ዝርዝሩ ሁልጊዜ X ነበር" ክርክሮች የሕዝብ ማረፊያ አላቸው።
- **ጸጥተኛ የመመለስ-ስፋት ማሳነስ።** በ FACT 05 ውስጥ ያሉ የ Pharmacovigilance ቃል ኪዳኖች ታሽገዋል፤ እነዚህን ያለመክበር በሕዝብ ሊታይ የሚችል ነው።
- **ለ out-of-spec ስብስብ ድኅረ-ኋላ እንደገና-spec ማድረግ።** የመለቀቅ-ፍተሻ ውጤት እንደ ድንበር-ላይ PASS ሆኖ ድኅረ-መለቀቅ እንደ FAIL እንደገና ቢከፋፈል፣ የታሸጉ የሪፖርት hashes የመጀመሪያው ውጤት PDF ምን እንዳለ ያረጋግጣሉ።

**ያልተሸፈኑ የውድቀት ሁኔታዎች፦**

- በማምረቻው ሂደት ራሱ ላይ ማጭበርበር (ማኅተሙ የፍተሻ ውጤቶችን ያስፈራል፣ የማምረቻ ታማኝነትን አይደለም)።
- አለበለዚያ-ያለፈ ስብስብ ባዮሎጂያዊ ውጤቶች የሆኑ የጎንዮሽ ክስተቶች (ማኅተሙ ክሊኒካዊ ደህንነትን አያረጋግጥም፣ የሰነድ ታማኝነትን ብቻ ነው)።
- ስለ ሕግ ዝርዝር የትኛው ስሪት ሥልጣን እንዳለው የተደረጉ ክርክሮች።

## ይህን ስብስብ መቼ መጠቀም

የስብስብ መለቀቅ ማተምን ያስቡበት ከሆነ፦

- ከተቆጣጣሪዎች ከሚፈልጉት በላይ የመለቀቅ መዛግብት የፎረንሲክ ታማኝነትን ለማሳየት የሚፈልጉ የክትባት አምራች ከሆኑ
- "የምናረጋግጠውን እናትማለን" ልምድ የሚቀበል QA ቡድን ከሆኑ
- በደካማ የሕግ መሠረተ ልማት ባለበት ግዛት ውስጥ ያሉ ትንሽ/ብሔራዊ-ደረጃ አምራች ከሆኑ፣ የስብስብ መዛግብትን ከዓለም አቀፍ verifier ጋር ለማስፈር የሚፈልጉ
- ለንግድ-ያልሆነ የክትባት ምርት ፕሮግራም (ለምሳሌ ለችላ-ለተባሉ በሽታዎች) የሚሠራ የሕዝብ-ጤና ኤጀንሲ ወይም NGO ከሆኑ

## ታሪካዊ የሞት ቀዳሚ ምሳሌ

- **የ Heparin ብክለት (2008፣ ቻይና):** በ heparin ስብስቦች ውስጥ የተጭበረበረ oversulfated chondroitin sulfate። የታሸገ የመለቀቅ-ፍተሻ ዝርዝር የፍተሻ መዝገብ ጸጥተኛ ድኅረ-መለቀቅ ማሻሻያ በ cryptographic መንገድ ሊታወቅ የሚችል እንዲሆን ያደርግ ነበር።
- **የ Tylenol cyanide (1982):** ስለ መጣረስ መነሻ ቦታ ድኅረ-ኋላ ክርክሮች። የታሸገ የማምረቻ-መዝገብ hash የቅድመ-ስርጭት ሁኔታን ያስፍራል።
- **የተለያዩ የክትባት ስብስብ መመለሶች** "ይህ ስብስብ መቼም በትክክል ተፈትኖ ነበር?" የሚለው ጥያቄ ከዓመታት በኋላ የክርክር ጉዳይ የሚሆንባቸው።

ስብስቡ በጥብቅ ስሜት ለማንኛቸውም እነዚህ መፍትሔ አይደለም — ግን በእያንዳንዱ ሁኔታ ውስጥ፣ የታሸጉ የስብስብ attestations ግልጽ የፎረንሲክ የጊዜ መስመር ይሰጡ ነበር።
