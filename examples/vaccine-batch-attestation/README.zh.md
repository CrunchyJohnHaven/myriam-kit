# 疫苗批次见证套件

*中文翻译 — 原始英文版本请参阅 [README.md](README.md)*

一个面向疫苗生产商、质量保证机构以及公共卫生机构的开箱即用 myriam-kit
示例,供其发布一份以 Bitcoin 锚定、可察觉篡改的批次放行密封件。

## 本套件可抵御的情形

**密封件可捕获的失效模式:**

- **对放行检测结果的无声追溯性修改。** 示例中全部 8 份放行检测 PDF 均
  被哈希。放行后重新哈希即可揭示任何修改。
- **关于放行时所适用规格的争议。** 已密封的 FACT 02 包含权威批次标识
  与生产记录哈希。未来"规格一直就是 X"之类的争议便有了公开锚点。
- **无声地缩小召回范围。** FACT 05 中的药物警戒(pharmacovigilance)
  承诺已被密封;未予履行的情况可被公开观察到。
- **对超规格批次进行追溯性重定规格。** 若某项放行检测结果实际为边缘
  PASS,却在放行后被重新归类为 FAIL,则已密封报告的哈希可证明原始结果
  PDF 的真实内容。

**未涵盖的失效模式:**

- 生产过程本身的舞弊(密封件锚定的是检测结果,而非生产过程的完整性)。
- 一个本应合格批次的生物学后果导致的不良事件(密封件不证明临床安全性,
  仅证明文档完整性)。
- 关于哪一版本的监管规格具有权威性的争议。

## 何时使用本套件

在以下情况下可考虑对一次批次放行进行密封:

- 您是一家疫苗生产商,希望在监管要求之外展示放行记录的取证完整性
- 您是一支 QA 团队,正采用"我们见证什么,就公开什么"的实践
- 您是一家位于监管基础设施较薄弱地区的小型/全国性生产商,希望将批次
  记录锚定至一个全球验证体系
- 您是一家运行非商业性疫苗生产计划(例如针对被忽视疾病)的公共卫生
  机构或 NGO

## 历史死亡先例

- **肝素污染事件(Heparin 2008,中国):** 肝素批次中混入伪劣的过硫酸
  软骨素。一份密封的放行检测清单将使检测记录在放行后的无声修改在密码
  学上可被检出。
- **泰诺氰化物事件(Tylenol 1982):** 关于篡改原点的追溯性争议。一份
  密封的生产记录哈希可锚定分发前的状态。
- **多起疫苗批次召回事件**,其中"该批次当初是否被妥善检测"这一问题在
  多年后变得有争议。

严格意义上,本套件并不能修复上述任何一种问题——但在每个案例中,密封
的批次见证都本可提供一条更清晰的取证时间线。

## 如何为真实批次派生本套件

```bash
cp -r ~/Genesis/myriam-kit/examples/vaccine-batch-attestation ~/release-FBP-XXXX
cd ~/release-FBP-XXXX
$EDITOR facts/*.txt       # replace fictional content with real batch
bash build/seal.sh        # SEAL BEFORE distribution begins
python3 build/verify.py   # confirm
```

**关键时机**:在 QA 放行的那一刻、批次开始分发**之前**完成密封。分发
之后再密封,在取证意义上较弱。

## 与现有制药基础设施的整合

- **eCTD 申报(FDA / EMA):** 已密封的发布 URL 或 IPFS CID 可在
  Module 3(质量)章节中引用
- **WHO Prequalification:** 已密封的清单可向 FDA/EMA 以外的辖区展示
  放行前的检测链条
- **药物警戒反馈:** FACT 05 承诺生产商以密封形式在 6 个月与 12 个月
  报告聚合的不良事件(AE)计数
- **召回系统(RxConnect、MedWatch):** 若需召回,召回范围本身也在
  MYRIAM 下被密封,从而向公众提供可验证的证明,表明所有已分发批号都正
  被处理

## 本套件**不**替代的内容

- FDA / EMA / WHO 监管申报
- GMP 审计轨迹(密封件锚定的是**结果**,而非原始过程记录)
- 药物警戒数据库(VAERS / EudraVigilance)
- 召回协调系统
