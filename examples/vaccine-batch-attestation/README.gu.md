# રસી બેચ સાક્ષ્યીકરણ કિટ

Bitcoin-લંગરિત, છેડછાડ-સ્પષ્ટ બેચ રિલીઝ સીલ પ્રકાશિત કરવા માંગતા રસી ઉત્પાદકો, ગુણવત્તા ખાતરી અધિકારીઓ અને જાહેર આરોગ્ય એજન્સીઓ માટે એક તૈયાર-ઉપયોગ myriam-kit ઉદાહરણ.

## આ કિટ શાનાથી રક્ષણ આપે છે

**સીલ પકડે છે તેવી નિષ્ફળતા પદ્ધતિઓ:**

- **રિલીઝ ટેસ્ટ પરિણામોમાં શાંત પૂર્વવ્યાપક ફેરફાર.** ઉદાહરણમાંની તમામ 8 રિલીઝ-ટેસ્ટિંગ PDFs hash કરવામાં આવી છે. રિલીઝ પછી ફરી-hashing કોઈપણ ફેરફાર જાહેર કરે છે.
- **રિલીઝ સમયે કયા સ્પષ્ટીકરણો લાગુ પડ્યા તે અંગે વિવાદો.** સીલ કરેલ FACT 02 માં કેનોનિકલ બેચ ઓળખ અને ઉત્પાદન-રેકોર્ડ hash છે. ભવિષ્યના "સ્પેક હંમેશા X હતી" વિવાદો માટે જાહેર લંગર છે.
- **શાંત રિકોલ-વ્યાપ ન્યૂનતમકરણ.** FACT 05 માં ફાર્માકોવિજિલન્સ પ્રતિબદ્ધતાઓ સીલ કરવામાં આવી છે; તેમનું પાલન ન કરવું જાહેરમાં અવલોકનક્ષમ છે.
- **સ્પેક-બહારના બેચ માટે પૂર્વવ્યાપક પુનઃ-સ્પેક.** જો રિલીઝ-ટેસ્ટ પરિણામ વાસ્તવમાં બોર્ડરલાઇન PASS હતું જે રિલીઝ પછી FAIL તરીકે પુનઃ-વર્ગીકૃત થાય, તો સીલ કરેલ રિપોર્ટ hash મૂળ પરિણામ PDF શું કહ્યું તે સાબિત કરે છે.

**આવરી લેવામાં ન આવેલી નિષ્ફળતા પદ્ધતિઓ:**

- ઉત્પાદન પ્રક્રિયામાં જ છેતરપિંડી (સીલ ટેસ્ટ પરિણામો લંગરે છે, ઉત્પાદન અખંડિતતા નહીં).
- અન્યથા-ઉત્તીર્ણ બેચના જૈવિક પરિણામો છે તેવી પ્રતિકૂળ ઘટનાઓ (સીલ ક્લિનિકલ સલામતી પ્રમાણિત કરતી નથી, માત્ર દસ્તાવેજ અખંડિતતા).
- નિયમનકારી સ્પષ્ટીકરણનું કયું વર્ઝન અધિકૃત છે તે અંગે વિવાદો.

## આ કિટનો ઉપયોગ ક્યારે કરવો

બેચ રિલીઝ સીલ કરવાનું વિચારો જો:

- તમે રસી ઉત્પાદક છો જે નિયમનકારોની માંગ કરતાં વધુ રિલીઝ રેકોર્ડની ફોરેન્સિક અખંડિતતા દર્શાવવા માંગો છો
- તમે "અમે જે સાક્ષી આપીએ છીએ તે પ્રકાશિત કરો" પ્રથા અપનાવતી QA ટીમ છો
- તમે નબળી નિયમનકારી માળખાગત સુવિધાવાળા અધિકારક્ષેત્રમાં નાના/રાષ્ટ્રીય-સ્કેલ ઉત્પાદક છો, વૈશ્વિક ચકાસણીકર્તા સામે બેચ રેકોર્ડ લંગરવા માંગો છો
- તમે ઉપેક્ષિત રોગો માટે બિન-વ્યવસાયિક રસી ઉત્પાદન કાર્યક્રમ ચલાવતી જાહેર-આરોગ્ય એજન્સી અથવા NGO છો

## ઐતિહાસિક મૃત્યુ પૂર્વઉદાહરણ

- **Heparin દૂષણ (2008, China):** heparin બેચોમાં નકલી oversulfated chondroitin sulfate. એક સીલ કરેલ રિલીઝ-ટેસ્ટ મેનિફેસ્ટ ટેસ્ટિંગ રેકોર્ડમાં રિલીઝ-પછીના શાંત ફેરફારને ક્રિપ્ટોગ્રાફિક રીતે શોધી શકાય તેવો બનાવ્યો હોત.
- **Tylenol cyanide (1982):** છેડછાડના મૂળ બિંદુ વિશે પૂર્વવ્યાપક વિવાદો. એક સીલ કરેલ ઉત્પાદન-રેકોર્ડ hash વિતરણ-પૂર્વ સ્થિતિને લંગરે છે.
- **વિવિધ રસી બેચ રિકોલ** જ્યાં "આ બેચ ક્યારેય યોગ્ય રીતે ટેસ્ટ થયું હતું કે કેમ" પ્રશ્ન વર્ષો પછી વિવાદાસ્પદ બને છે.

આ કિટ આમાંની કોઈપણ બાબત માટે કડક અર્થમાં ઉપાય નથી — પરંતુ દરેક કેસમાં, સીલ કરેલ બેચ સાક્ષ્યીકરણોએ સ્પષ્ટ ફોરેન્સિક સમયરેખા પૂરી પાડી હોત.

## વાસ્તવિક બેચ માટે આ કિટ કેવી રીતે fork કરવી

```bash
cp -r ~/Genesis/myriam-kit/examples/vaccine-batch-attestation ~/release-FBP-XXXX
cd ~/release-FBP-XXXX
$EDITOR facts/*.txt       # કાલ્પનિક સામગ્રીને વાસ્તવિક બેચથી બદલો
bash build/seal.sh        # વિતરણ શરૂ થાય તે પહેલાં SEAL કરો
python3 build/verify.py   # પુષ્ટિ કરો
```

**ગંભીર સમય**: બેચ વિતરિત થાય તે પહેલાં, QA રિલીઝની ક્ષણે સીલ કરો. વિતરણ પછીની સીલ ફોરેન્સિક રીતે નબળી છે.

## વર્તમાન ફાર્માસ્યુટિકલ માળખાગત સુવિધાઓ સાથે એકીકરણ

- **eCTD સબમિશન (FDA / EMA):** સીલ કરેલ પ્રકાશન URL અથવા IPFS CID Module 3 (ગુણવત્તા) વિભાગમાં સંદર્ભિત કરી શકાય
- **WHO Prequalification:** સીલ કરેલ મેનિફેસ્ટ FDA/EMA-સિવાયના અધિકારક્ષેત્રો માટે રિલીઝ-પૂર્વ ટેસ્ટિંગ સાંકળ દર્શાવે છે
- **ફાર્માકોવિજિલન્સ પ્રતિસાદ:** FACT 05 ઉત્પાદકને 6 અને 12 મહિને કુલ AE ગણતરીના સીલ કરેલ રિપોર્ટિંગ માટે પ્રતિબદ્ધ કરે છે
- **રિકોલ સિસ્ટમ્સ (RxConnect, MedWatch):** જો રિકોલની જરૂર હોય, તો રિકોલની વ્યાપ પોતે MYRIAM હેઠળ સીલ થાય છે, બધા વિતરિત લોટ સંબોધવામાં આવી રહ્યા છે તેની જાહેર ચકાસણી પૂરી પાડે છે

## આ કિટ શું બદલતી નથી

- FDA / EMA / WHO નિયમનકારી સબમિશન
- GMP ઓડિટ ટ્રેલ (સીલ પરિણામો લંગરે છે, કાચા પ્રક્રિયા રેકોર્ડ નહીં)
- ફાર્માકોવિજિલન્સ ડેટાબેઝ (VAERS / EudraVigilance)
- રિકોલ સંકલન સિસ્ટમો
