# വാക്സിൻ ബാച്ച് സാക്ഷ്യപ്പെടുത്തൽ കിറ്റ്

Bitcoin-നങ്കൂരമിട്ട, കൃത്രിമത്വം-തെളിയിക്കാവുന്ന ബാച്ച് റിലീസ് മുദ്ര പ്രസിദ്ധീകരിക്കാൻ ആഗ്രഹിക്കുന്ന വാക്സിൻ നിർമ്മാതാക്കൾ, ഗുണനിലവാര ഉറപ്പ് അധികാരികൾ, പൊതുജനാരോഗ്യ ഏജൻസികൾ എന്നിവർക്കായുള്ള ഒരു സജ്ജമായ myriam-kit ഉദാഹരണം.

## ഈ കിറ്റ് എന്തിനെതിരെ പ്രതിരോധിക്കുന്നു

**മുദ്ര പിടിക്കുന്ന പരാജയ രീതികൾ:**

- **റിലീസ് ടെസ്റ്റ് ഫലങ്ങളുടെ നിശ്ശബ്ദ പിന്നോട്ടുള്ള പരിഷ്കരണം.** ഉദാഹരണത്തിലെ എല്ലാ 8 റിലീസ്-ടെസ്റ്റിംഗ് PDFകളും hash ചെയ്തിരിക്കുന്നു. റിലീസിന് ശേഷം വീണ്ടും-hashing ചെയ്യുന്നത് ഏതെങ്കിലും പരിഷ്കരണം വെളിപ്പെടുത്തുന്നു.
- **റിലീസ് സമയത്ത് ഏത് സ്പെസിഫിക്കേഷനുകൾ ബാധകമായിരുന്നു എന്നതിനെക്കുറിച്ചുള്ള തർക്കങ്ങൾ.** മുദ്രയിട്ട FACT 02-ൽ കാനോനിക്കൽ ബാച്ച് തിരിച്ചറിയലും നിർമ്മാണ-റെക്കോർഡ് hash-ഉം അടങ്ങിയിരിക്കുന്നു. ഭാവിയിലെ "സ്പെസ് എപ്പോഴും X ആയിരുന്നു" തർക്കങ്ങൾക്ക് ഒരു പൊതു നങ്കൂരമുണ്ട്.
- **നിശ്ശബ്ദ തിരിച്ചുവിളിക്കൽ-വ്യാപ്തി മിനിമൈസേഷൻ.** FACT 05-ലെ ഫാർമക്കോവിജിലൻസ് പ്രതിബദ്ധതകൾ മുദ്രയിട്ടിരിക്കുന്നു; അവ പാലിക്കാത്തത് പൊതുവായി നിരീക്ഷിക്കാവുന്നതാണ്.
- **സ്പെസ്-ന് പുറത്തുള്ള ബാച്ചിനായി പിന്നോട്ടുള്ള പുനഃ-സ്പെസ്.** ഒരു റിലീസ്-ടെസ്റ്റ് ഫലം യഥാർത്ഥത്തിൽ അതിർത്തിയിലെ PASS ആയിരുന്നു അത് റിലീസിന് ശേഷം FAIL ആയി പുനഃവർഗ്ഗീകരിക്കപ്പെടുകയാണെങ്കിൽ, മുദ്രയിട്ട റിപ്പോർട്ട് hash-കൾ യഥാർത്ഥ ഫല PDF എന്താണ് പറഞ്ഞത് എന്ന് തെളിയിക്കുന്നു.

**ഉൾപ്പെടാത്ത പരാജയ രീതികൾ:**

- നിർമ്മാണ പ്രക്രിയയിൽ തന്നെ തട്ടിപ്പ് (മുദ്ര ടെസ്റ്റ് ഫലങ്ങളെ നങ്കൂരമിടുന്നു, നിർമ്മാണ സമഗ്രതയെയല്ല).
- അല്ലാത്തപക്ഷം-വിജയിച്ച ബാച്ചിന്റെ ജൈവിക പരിണാമങ്ങളായ പ്രതികൂല സംഭവങ്ങൾ (മുദ്ര ക്ലിനിക്കൽ സുരക്ഷ സാക്ഷ്യപ്പെടുത്തുന്നില്ല, പ്രമാണ സമഗ്രത മാത്രം).
- ഒരു റെഗുലേറ്ററി സ്പെസിഫിക്കേഷന്റെ ഏത് പതിപ്പാണ് ആധികാരികമെന്നതിനെക്കുറിച്ചുള്ള തർക്കങ്ങൾ.

## ഈ കിറ്റ് എപ്പോൾ ഉപയോഗിക്കണം

ഒരു ബാച്ച് റിലീസ് മുദ്രയിടുന്നത് പരിഗണിക്കുക:

- റെഗുലേറ്റർമാർ ആവശ്യപ്പെടുന്നതിലുമപ്പുറം റിലീസ് റെക്കോർഡുകളുടെ ഫോറൻസിക് സമഗ്രത പ്രദർശിപ്പിക്കാൻ ആഗ്രഹിക്കുന്ന വാക്സിൻ നിർമ്മാതാവ് ആണെങ്കിൽ
- "ഞങ്ങൾ സാക്ഷ്യപ്പെടുത്തുന്നത് പ്രസിദ്ധീകരിക്കുക" എന്ന സമ്പ്രദായം സ്വീകരിക്കുന്ന QA ടീം
- ദുർബലമായ റെഗുലേറ്ററി അടിസ്ഥാന സൗകര്യങ്ങളുള്ള ഒരു അധികാരപരിധിയിലെ ചെറുകിട/ദേശീയ-സ്കെയിൽ നിർമ്മാതാവ്, ഒരു ആഗോള പരിശോധകന് എതിരായി ബാച്ച് രേഖകൾ നങ്കൂരമിടാൻ ആഗ്രഹിക്കുന്നു
- അവഗണിക്കപ്പെട്ട രോഗങ്ങൾക്കായി വാണിജ്യേതര വാക്സിൻ ഉൽപാദന പരിപാടി നടത്തുന്ന പൊതുജനാരോഗ്യ ഏജൻസി അല്ലെങ്കിൽ NGO

## ചരിത്രപരമായ മരണ പൂർവ്വദൃഷ്ടാന്തം

- **Heparin മലിനീകരണം (2008, China):** heparin ബാച്ചുകളിലെ വ്യാജ oversulfated chondroitin sulfate. ഒരു മുദ്രയിട്ട റിലീസ്-ടെസ്റ്റ് മാനിഫെസ്റ്റ് ടെസ്റ്റിംഗ് റെക്കോർഡിന്റെ റിലീസിന് ശേഷമുള്ള നിശ്ശബ്ദ പരിഷ്കരണത്തെ ക്രിപ്റ്റോഗ്രാഫിക്കായി കണ്ടെത്താൻ കഴിയുമാക്കിയിരുന്നു.
- **Tylenol cyanide (1982):** കൃത്രിമത്വ ഉത്ഭവ സ്ഥാനത്തെക്കുറിച്ചുള്ള പിന്നോട്ടുള്ള തർക്കങ്ങൾ. ഒരു മുദ്രയിട്ട നിർമ്മാണ-റെക്കോർഡ് hash വിതരണത്തിന് മുമ്പുള്ള അവസ്ഥയെ നങ്കൂരമിടുന്നു.
- **വിവിധ വാക്സിൻ ബാച്ച് റീകോളുകൾ** അവിടെ "ഈ ബാച്ച് ശരിയായി പരിശോധിച്ചിരുന്നോ" എന്ന ചോദ്യം വർഷങ്ങൾക്ക് ശേഷം തർക്കിക്കപ്പെടുന്നു.

ഈ കിറ്റ് കർശനമായ അർത്ഥത്തിൽ ഇവയിൽ ഏതെങ്കിലുമൊന്നിന്റെ പരിഹാരമല്ല — എന്നാൽ ഓരോ കേസിലും, മുദ്രയിട്ട ബാച്ച് സാക്ഷ്യപ്പെടുത്തലുകൾ വ്യക്തമായ ഒരു ഫോറൻസിക് കാലരേഖ നൽകുമായിരുന്നു.

## യഥാർത്ഥ ബാച്ചിനായി ഈ കിറ്റ് fork ചെയ്യുന്നതെങ്ങനെ

```bash
cp -r ~/Genesis/myriam-kit/examples/vaccine-batch-attestation ~/release-FBP-XXXX
cd ~/release-FBP-XXXX
$EDITOR facts/*.txt       # ഫിക്ഷണൽ ഉള്ളടക്കത്തെ യഥാർത്ഥ ബാച്ച് ഉള്ളടക്കം കൊണ്ട് മാറ്റിസ്ഥാപിക്കുക
bash build/seal.sh        # വിതരണം ആരംഭിക്കുന്നതിന് മുമ്പ് SEAL ചെയ്യുക
python3 build/verify.py   # സ്ഥിരീകരിക്കുക
```

**നിർണ്ണായക സമയം**: ബാച്ച് വിതരണം ചെയ്യപ്പെടുന്നതിന് മുമ്പ്, QA റിലീസിന്റെ നിമിഷത്തിൽ മുദ്രയിടുക. വിതരണത്തിന് ശേഷമുള്ള ഒരു മുദ്ര ഫോറൻസിക്കായി ദുർബലമാണ്.

## നിലവിലുള്ള ഔഷധ അടിസ്ഥാന സൗകര്യങ്ങളുമായുള്ള സംയോജനം

- **eCTD സമർപ്പണങ്ങൾ (FDA / EMA):** മുദ്രയിട്ട പ്രസിദ്ധീകരണ URL അല്ലെങ്കിൽ IPFS CID Module 3 (ഗുണനിലവാരം) വിഭാഗത്തിൽ പരാമർശിക്കാം
- **WHO Prequalification:** മുദ്രയിട്ട മാനിഫെസ്റ്റ് FDA/EMA-ഇതര അധികാരപരിധികളിലേക്കുള്ള റിലീസിന് മുമ്പുള്ള ടെസ്റ്റിംഗ് ശൃംഖലയെ പ്രദർശിപ്പിക്കുന്നു
- **ഫാർമക്കോവിജിലൻസ് പ്രതികരണം:** FACT 05 നിർമ്മാതാവിനെ 6, 12 മാസങ്ങളിൽ മൊത്തം AE എണ്ണങ്ങളുടെ മുദ്രയിട്ട റിപ്പോർട്ടിംഗിലേക്ക് പ്രതിജ്ഞാബദ്ധനാക്കുന്നു
- **തിരിച്ചുവിളിക്കൽ സംവിധാനങ്ങൾ (RxConnect, MedWatch):** ഒരു റീകോൾ ആവശ്യമെങ്കിൽ, റീകോളിന്റെ വ്യാപ്തി തന്നെ MYRIAM-ന് കീഴിൽ മുദ്രയിടും, എല്ലാ വിതരണം ചെയ്ത ലോട്ടുകളും അഭിസംബോധന ചെയ്യപ്പെടുന്നുണ്ടെന്നതിന് പൊതു സ്ഥിരീകരണം നൽകുന്നു

## ഈ കിറ്റ് മാറ്റിസ്ഥാപിക്കാത്തത്

- FDA / EMA / WHO റെഗുലേറ്ററി സമർപ്പണങ്ങൾ
- GMP ഓഡിറ്റ് ട്രെയിലുകൾ (മുദ്ര ഫലങ്ങളെ നങ്കൂരമിടുന്നു, അസംസ്കൃത പ്രക്രിയ രേഖകളെയല്ല)
- ഫാർമക്കോവിജിലൻസ് ഡാറ്റാബേസുകൾ (VAERS / EudraVigilance)
- തിരിച്ചുവിളിക്കൽ സമന്വയ സംവിധാനങ്ങൾ
