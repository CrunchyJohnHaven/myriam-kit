  ជាមួយមូលដ្ឋានទិន្នន័យឯកជន។ Hashes កម្រិត Image (kit កម្រិត
  forensic-image) នៃ disk ពេញលេញអាចទទួលយកបាន; hashes កម្រិតឯកសារនៃ
  សម្ភារៈបែបនេះ **មិនអាចទទួលយកបាន**។
- **ការរកឃើញការត្រួតពិនិត្យ ការបកស្រាយ មតិ ឬការសន្និដ្ឋានអំពីអ្វី
  ដែល bytes មានន័យ។** ត្រាយុថ្កា BYTES ណាដែលត្រូវបានទទួលបាន និងពេល
  ណា។ ការបកស្រាយជារបស់របាយការណ៍ការត្រួតពិនិត្យ discoverable
  មិនមែននៅលើ ledger សាធារណៈដែលអាចបង្ហាញការកែប្រែទេ។
- **ការអះអាងនៃការទទួលយក ឬការសន្និដ្ឋានច្បាប់ផ្សេងទៀត** ("ភស្តុតាង
  នេះអាចទទួលយកបាន" "warrant ស្របច្បាប់" "ខ្សែសង្វាក់ការគ្រប់គ្រង
  ត្រឹមត្រូវក្រោម §X.Y")។ ការសន្និដ្ឋានច្បាប់គឺមុខងាររបស់តុលាការ។
- **ការបិទត្រាដោយនរណាម្នាក់ក្រៅពី principal ដែលមានសិទ្ធិរបស់ភ្នាក់ងារ
  ទទួលបាន។** អ្នកជំនាញការការពារអាចប្រើឧបករណ៍ដើម្បីបិទត្រាការផ្ទៀង
  ផ្ទាត់ឯករាជ្យ *ផ្ទាល់ខ្លួន* របស់ពួកគេនៃ manifest; ពួកគេមិនអាច
  បិទត្រាការទទួលបាន ORIGINAL ក្រោមឈ្មោះភ្នាក់ងារទទួលបានទេ។
- **ការផ្សារភ្ជាប់តួនាទី-ទៅ-បុគ្គលក្នុងត្រាសាធារណៈ។** ការផ្សារភ្ជាប់
  ត្រូវបានបិទត្រាក្នុងកំណត់ត្រា companion ដាច់ដោយឡែក (hash នៅ FACT
  05) ក្រោមការគ្រប់គ្រងការចូលរបស់ភ្នាក់ងារ។
- **ការពិតស៊ើបអង្កេតមុនការរឹបអូស ឬមុន warrant។** សេចក្តីថ្លែងការណ៍
  informant ការតាមដានមុន warrant ការបញ្ចូល parallel-construction —
  គ្មានណាមួយជាកម្មសិទ្ធិលើត្រាការទទួលបាន។ ត្រាមានវិសាលភាពចំពោះព្រឹត្ត
  ការណ៍ការទទួលបាន។

ការបដិសេធខ្លួនវាផ្ទាល់អាចត្រូវបានបិទត្រាជាការពិត ("ភ្នាក់ងារនេះស្នើ
ការបិទត្រានៃ X; ឧបករណ៍បានបដិសេធតាមច្បាប់បដិសេធ Y") បង្កើតកំណត់ត្រា
សាធារណៈនៃការប៉ុនប៉ងប្រើប្រាស់ខុស។

## អ្វីដែលឧបករណ៍នេះបិទត្រា

8 ការពិតនៅគ្រាននៃការទទួលបាន៖

1. **ការជូនដំណឹងឧទាហរណ៍** — ប្រកាសការបោះផ្សាយជាឧទាហរណ៍ (ការ
   បញ្ជាក់ពិតប្រាកដដកចេញ ឬជំនួសវា)។
2. **បរិបទនៃការទទួលបាន** — អត្តសញ្ញាណករណី ការដកស្រង់ warrant
   ភ្នាក់ងារទទួលបាន អាជ្ញាធរស្នើ កាលបរិច្ឆេទ/ពេលវេលានៃការរឹបអូស និង
   ការទទួលបាន ទីតាំងនៃការទទួលបាន (ទីតាំងនៃការរឹបអូសត្រូវកាត់ខ្លី
   ទៅជាកំណត់ត្រា companion ដែលបិទត្រាដាច់ដោយឡែកដើម្បីជៀសវាងការ
   កំណត់អត្តសញ្ញាណទីតាំងឡើងវិញ)។
3. **Manifest ភស្តុតាង** — ក្នុងមួយ forensic-image-item: SHA-256 hash
   ទំហំឯកសារ identifier ឧបករណ៍ប្រភពអនាមិក វិធីសាស្ត្រទទួលបាន
   ទម្រង់ forensic image។
4. **វិធីសាស្ត្រនៃការទទួលបាន** — ឈ្មោះឧបករណ៍នីតិវេទ្យា និងកំណែ
   ច្បាស់លាស់ (ជាមួយ hash binary ឧបករណ៍) ម៉ាក/ម៉ូដែល write-blocker និង
   firmware ឯកសារយោងផ្ទៀងផ្ទាត់ NIST CFTT នីតិវិធីផ្ទៀងផ្ទាត់ដែលប្រើ
   ជម្រើសទម្រង់ image។
5. **Log ខ្សែសង្វាក់ការគ្រប់គ្រង** — ខ្សែពេលក្នុងមួយព្រឹត្តិការណ៍ពី
   ការរឹបអូសដំបូងដល់ការបញ្ចប់ imaging អ្នកគ្រប់គ្រងកំណត់អត្តសញ្ញាណ
   ដោយតួនាទី (មិនមែនដោយឈ្មោះទេ) ជាមួយ hash នៃកំណត់ត្រា companion ដែល
   បិទត្រាដាច់ដោយឡែក (ការផ្សារភ្ជាប់តួនាទី-ទៅ-បុគ្គល log GPS ដឹក
   ជញ្ជូន log ចូលគ្រាប់ភស្តុតាង រូបថត tamper-seal ។ល។)។
6. **គុណវុឌ្ឍិអ្នកត្រួតពិនិត្យតាមតួនាទី** — វិញ្ញាបនបត្រដែលកាន់
   ស្ថានភាពអប់រំបន្ត ចំនួនបទពិសោធន៍មុនជាអ្នកជំនាញ-សាក្សី ការប៉ះពាល់
   Daubert ការបង្ហាញ conflict-of-interest — ទាំងអស់នៅកម្រិតតួនាទី
   មិនមែនផ្ទាល់ខ្លួនទេ។
7. **ការបង្ហាញ companion និងកំណត់ត្រាបដិសេធ** — ការស៊ើបអង្កេតផ្នែក
   ខាងគ្នា ការប្រកួត warrant ដែលនៅរង់ចាំ លំហាត់ភស្តុតាងដែលត្រូវបាន
   ទប់ស្កាត់ ពិធីការ privilege-filter / taint-team មានប្រសិទ្ធភាព
   ការប៉ុនប៉ងទទួលបានពីមុន ការពិនិត្យ bug-tracker អ្នកលក់; បញ្ជី
   ច្បាស់នៃគំរូដែលឧបករណ៍បានបដិសេធមិនបិទត្រា។
8. **Principal ចុះហត្ថលេខា** — ការកំណត់អត្តសញ្ញាណភ្នាក់ងារទទួលបាន +
   អ្នកចុះហត្ថលេខាកម្រិតតួនាទី (អ្នកត្រួតពិនិត្យនាំមុខ) + អ្នកសហ-
   ចុះហត្ថលេខាកម្រិតតួនាទី (អ្នកគ្រប់គ្រងឯកតា) + ព័ត៌មាន cryptographic
   key។

ការពិតនីមួយៗបញ្ចប់ដោយអនុសញ្ញា refusal-gate: *"ការពិតនេះអះអាងតែ X
ប៉ុណ្ណោះ។ វាមិនអះអាង Y ទេ"* (ដែល Y ជារបៀបនៃការបរាជ័យដែលជិតបំផុតពី
ផ្នែក "DOES NOT do" ខាងលើ)។

## អ្វីដែលឧបករណ៍នេះការពារ

- **ការផ្លាស់ប្តូរ forensic image ស្ងាត់ៗក្រោយការទទួលបាន។** ការ
  ផ្លាស់ប្តូរ byte-level ណាមួយចំពោះ image ណាមួយផលិត SHA-256 ខុសគ្នា
  ហើយ image ក្រោយការផ្លាស់ប្តូរនឹងមិនផ្ទៀងផ្ទាត់ប្រឆាំងនឹង manifest
  FACT 03 ដែលបានបិទត្រាទេ។
- **ការអះអាង "ឧបករណ៍ផ្សេងគ្នាបានប្រើ។"** FACT 04 កត់ត្រាកំណែឧបករណ៍
  ច្បាស់លាស់ជាមួយ hash binary ឧបករណ៍; ការអះអាងពេលក្រោយ "យើងពិតជា
  បានប្រើ v4.7.0 មិនមែន v4.7.1" អាចបដិសេធដោយសាធារណៈ។
- **ការអះអាងថា write-blocker មិនត្រូវបានចូលរួម ឬប្រើម៉ូដែលផ្សេងគ្នា។**
  FACT 04 បង្កក manifest write-blocker។
- **ការផ្លាស់ប្តូរ log custody។** ការផ្លាស់ប្តូរណាមួយចំពោះខ្សែពេល
  custody-event FACT 05 (បន្ថែម ដក ឬ re-timestamping ព្រឹត្តិការណ៍)
  ផ្លាស់ប្តូរ FACT 05 hash ហើយអាចរកឃើញដោយសាធារណៈ។
- **ការជំនួសព័ត៌មានបញ្ជាក់នៃអ្នកត្រួតពិនិត្យផ្សេងគ្នា។** FACT 06
  បង្កកប្រវត្តិរូបព័ត៌មានបញ្ជាក់កម្រិតតួនាទីនៅពេលទទួលបាន។
- **ការដកការបង្ហាញ companion ដោយជ្រើសរើស។** FACT 07 រាប់ប្រភេទនៃ
  ការបង្ហាញដែលរំពឹង; ប្រភេទទទេត្រូវតែបញ្ជាក់យ៉ាងច្បាស់ ("គ្មាននៅពេល
  ទទួលបាន") ដូច្នេះការនិយាយពេលក្រោយ "យើងមិនបានដឹងថាវាពាក់ព័ន្ធ" អាច
  បដិសេធដោយសាធារណៈ។
- **ការអះអាងពេលក្រោយថាអ្នកត្រួតពិនិត្យមាន conflict ដែលមិនត្រូវបាន
  បង្ហាញ។** ការបង្ហាញ conflict នៃ FACT 06 ត្រូវបានបង្កកនៅពេលទទួល
  បាន; ការប្រកួតពេលក្រោយអាចត្រូវបានសាកល្បងប្រឆាំងនឹងវា។

ឧបករណ៍ **មិន** ការពារប្រឆាំងនឹងអំពើពុករលួយក្នុងពេលជាមួយ — ក្រុមដែល
ដោយចេតនាបិទត្រា metadata ក្លែងបន្លំនៅពេលទទួលបានបង្កើតត្រា
