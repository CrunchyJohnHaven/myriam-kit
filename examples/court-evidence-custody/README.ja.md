# 裁判証拠 証拠保全(Chain-of-Custody)封緘キット

フォレンジック取得チーム — コンピュータフォレンジック調査官、モバイル端末抽出技術者、ボディカム証拠保管担当者、ラボ監督者 — のためのターンキー型 myriam-kit のサンプル。彼らは、デジタル証拠取得の瞬間に、Bitcoin にアンカリングされた改竄検出可能(tamper-evident)な封緘として、*どのバイトが取得されたか*と*いつ取得されたか*を公開し、証拠保全(chain-of-custody)に関する紛争に公的なアンカーを与えることを望みます。

---

## このキットが**しない**こと(まず読んでください)

封緘は、暗号学的タイムスタンプであり、それが捉えるのは、どのフォレンジックイメージが、いつ、どの機関により、どの令状の下で、どの方法論を通じて取得されたか、というものです。それは事案について、それ以上の問いには答えることができません。以下は**封緘の領域の外側**にあります:

1. **連邦証拠規則(またはその他の証拠規則)の下での証拠能力(admissibility)**。封緘はバイトとタイムスタンプを固定します。証拠能力 — FRE 901 認証、FRE 702 専門家証言 / Daubert、FRE 403 不当な偏見の衡量、または各州裁判所におけるそれらの相当物 — は法手続上の問いです。完全に封緘された取得であっても証拠能力なしと判断されることがありえます。封緘されていない取得であっても証拠能力ありとされることがありえます。封緘は完全性に関する付加的な証拠であって、裁判所による証拠能力判断の代替ではありません。

2. **押収が合法であったかどうか**。修正第 4 条への適合、令状の有効性、範囲遵守、排除法則の適用可能性 — これらはすべて封緘の上流にあります。封緘は、令状 FICT-SW-2026-0188 のもとで取得が行われたという事実を記録します。当該令状が適法に発付され、適法に執行されたかどうかは、裁判所が判断します。

3. **証拠が有罪寄りか、無罪寄りか、中立か**。封緘はハッシュを固定します。それらのハッシュが何を含むかについての解釈は、フォレンジック検査報告書に属し、それは独自の開示手続と証拠制度を持つ別個の文書です。封緘は「FACT N: 被疑者のラップトップには有罪を示す素材が含まれていた」のような封緘を拒絶します — 拒否規則 R4 を参照。

4. **証拠保全(chain-of-custody)が適用される州の証拠法のもとで手続上正しいかどうか**。封緘は、機関のリアルタイム保全イベントログ(FACT 05)を記録します。当該ログが特定の州の証拠保全要件を満たすかどうかは、証拠採用審理における裁判所の判断によります。

5. **フォレンジック方法論が信頼に足るものであったかどうか**。Daubert およびその系譜(および Frye が今なお適用される州レベルの相当物)は、方法論の信頼性を評価します。封緘はどのツールとどのバージョンが使われたかを記録します(FACT 04)。それらのツールが特定の裁判所の信頼性基準を満たすかどうかは、裁判所が判断します。過去の事案では、取得後にツールのバグが文書化されたこともあります。封緘の役割は、バージョン-およびバイナリ-ハッシュの記録を公的に反証可能(refutable)にすることであり、ツールそのものを保証することではありません。

6. **取得以前に証拠が改竄されていたかどうか**。封緘は、撮像の瞬間以降のバイトを固定します。被疑者、警察官、または第三者が撮像前にソースデバイスを変更した場合、封緘はそれを検出できません — 変更されたバイトを、原本と同じ完全性で封緘します。封緘の価値は、撮像のタイムスタンプから始まります。

7. **令状の範囲が遵守されたかどうか**。令状条項(warrant rider)は何が取得されてよいかを定義し、封緘は何が実際に取得されたかを記録します。取得が令状の範囲を超えていたかどうかは、これも法的な問いであり、独自の救済手段(該当する州の手続による証拠排除申立て)があります。

8. **捜査対象の私人を再識別すること**。封緘は、被疑者、被告人、被害者、目撃者、または建物の占有者を再識別しうるいかなる氏名、識別子、フィンガープリントの公表をも拒絶します。事件番号(case docket reference)が、案件の所在を特定する正当性を有する当事者にとっての適切なハンドルです。

**上記の正直な限界に関するセクションこそが、このキットの中核を担う部分です**。封緘=証拠能力と信じるフォレンジックチームは、これを盾として使うでしょう。キットははっきりとこう述べる必要があります: *これは取得時点に関する非否認(non-repudiation)プリミティブであり、適法な取得、方法論的妥当性、または裁判所が承認する証拠保全の証明ではありません。*

## このキットが封緘を**拒絶せねばならない**もの

このキットは、有効な機関署名権限を持つユーザーが提出したとしても、以下の事実パターンを拒絶するよう設定されています:

- **被疑者、被告人、被害者、目撃者、その他の私的な第三者の個人識別**。事件番号は、当事者性を有する者にとってのハンドルです。公開封緘が再識別の索引になってはなりません。
- **公開封緘におけるフォレンジック調査官その他機関職員の個人名**。役割は公開、人物-役割の対応付けは別途封緘され、裁判所手続のもとで開示可能です。
- **児童性的搾取コンテンツ(CSAM)、生体テンプレート、または脆弱者を再識別しうるあらゆるデータのファイル単位ハッシュ(またはフィンガープリント)— ハッシュであっても**。個別ファイルの SHA-256 は、永続的に検索可能な識別子です。公開台帳上にそれを公表することは、私的データベースとの相関を介して対象者を再被害化(re-victimize)するおそれがあります。ディスク全体のイメージ単位(forensic-image-level)ハッシュは許容されます。当該素材のファイル単位ハッシュは決して許容されません。
- **バイト列の意味についての検査所見、解釈、意見、結論**。封緘は、どのバイトが、いつ取得されたかを固定します。解釈は、開示可能な検査報告書に属するものであって、改竄検出可能な公開台帳に載せるものではありません。
- **証拠能力に関する主張その他の法的結論**(「この証拠は証拠能力がある」「令状は合法であった」「証拠保全は §X.Y の下で適正であった」)。法的結論は裁判所の機能です。
- **取得機関の権限ある主体以外の者による封緘**。防御側の専門家は、自らがマニフェストを独立に検証した結果について本キットで封緘してもよいですが、取得機関の名義で「元の」取得を封緘することはできません。
- **公開封緘の内部における役割-個人の対応付け**。当該対応付けは、機関のアクセス制御のもとで別の付随記録に封緘されます(FACT 05 でハッシュ化)。
- **押収以前または令状以前の捜査事実**。情報提供者の供述、令状以前の監視、平行構築(parallel construction)の入力 — これらはいずれも取得封緘の上に載せるべきではありません。封緘の範囲は取得イベントに限定されます。

拒絶それ自体が一つの事実として封緘されうるものであり(「本機関は X の封緘を要請し、キットは拒否規則 Y により拒絶した」)、これは誤用の試みの公的記録を生み出します。

## このキットが**封緘するもの**

取得の瞬間における 8 つの事実:

1. **例示通知** — 当該公表が例示であることを宣言する(実際の証言ではこれを除去または置換する)。
2. **取得コンテキスト** — 事件識別子、令状の引用、取得機関、請求権限、押収および取得の日時、取得場所(押収場所は、対象地点の再識別を避けるため、別途封緘される付随記録に伏字処理される)。
3. **証拠マニフェスト** — フォレンジックイメージ項目ごと: SHA-256 ハッシュ、ファイルサイズ、ソースデバイスの匿名化識別子、取得方法、フォレンジックイメージ形式。
4. **取得方法論** — フォレンジックツール名と正確なバージョン(ツールバイナリのハッシュ付き)、ライトブロッカーの製造元/モデルおよびファームウェア、NIST CFTT 検証参照、使用された検証手続、イメージ形式の選択。
5. **証拠保全ログ** — 初動押収から撮像終了までのイベント単位タイムライン、保全担当者を役割(氏名ではなく)で識別。役割-個人の対応付け、移送 GPS ログ、証拠保管庫アクセスログ、改竄シール写真など、別途封緘される付随記録のハッシュを伴う。
6. **役割別の検査官資格** — 保有する認証、継続教育の状況、過去の専門家証言経験の数、Daubert の経験、利益相反の開示 — すべて役割レベルであり個人レベルではない。
7. **付随開示および拒絶記録** — 並行調査、係属中の令状チャレンジ、証拠排除申立て、有効なプリビレッジ・フィルタ / テイント・チーム手続、過去の取得試行、ベンダーのバグトラッカーレビュー。本キットが封緘を拒絶したパターンの明示的リスト。
8. **署名主体** — 取得機関の識別 + 役割レベルの署名者(主任検査官) + 役割レベルの連署者(部隊監督者) + 暗号鍵情報。

各事実は、拒否ゲートの慣行で締めくくられます: *「本事実は X のみを主張する。Y は主張しない」*(ここで Y は、上記「しないこと」セクションから最も近い失敗モード)。

## このキットが防御するもの

- **フォレンジックイメージの取得後の沈黙裡な改竄**。いずれかのイメージに対するバイトレベルの変更は異なる SHA-256 を生み、変更後のイメージは封緘済み FACT 03 マニフェストに対して検証に失敗します。
- **「異なるツールが使われた」との主張**。FACT 04 はツールバイナリのハッシュ付きで正確なツールバージョンを記録します。後日の「実際には v4.7.1 ではなく v4.7.0 を使った」との主張は公的に反証可能となります。
- **ライトブロッカーが作動していなかった、または異なるモデルが使われたとの主張**。FACT 04 がライトブロッカーのマニフェストを凍結します。
- **保全ログの改変**。FACT 05 の保全イベントタイムラインに対する変更(イベントの追加、削除、または再タイムスタンプ化)は FACT 05 のハッシュを変化させ、公的に検出可能となります。
- **異なる検査官の資格への差し替え**。FACT 06 は取得時の役割レベル資格プロファイルを凍結します。
- **付随開示の選択的省略**。FACT 07 は期待される開示カテゴリを列挙します。空のカテゴリは明示的に証言される必要があり(「取得時点では該当なし」)、後日の「それが関連するとは思わなかった」が公的に反証可能となります。
- **検査官に開示されていなかった利益相反があったとする取得後の主張**。FACT 06 の利益相反開示は取得時に凍結されており、後の異議申し立てはそれに照らして検証可能です。

このキットは、同時並行的な汚損(contemporaneous corruption)に対しては**防御しません** — 取得時に意図的に虚偽のメタデータを封緘するチームは、虚偽メタデータの暗号学的に有効な封緘を生み出します。本プリミティブはその点について正直です: それは、機関の同時並行的主張に対する非否認プリミティブであり、それらの主張が正しいことの証明ではありません。

## このキットが助けになったであろう歴史的または準歴史的事案

このキットは、過去に、デジタル証拠の証拠保全が争点となった事案で、有用なアンカーを提供したであろうものです — *解決*ではありません。被告人(有罪判決を受けた者も含めて私人である)を名指しすることを避けるため、これらは個別の名前ではなく事案の類型で記述します:

- **2010 年代半ばの連邦フォレンジックツールバグ論争** — 広く使用されていたフォレンジック撮像ツールに、取得後、特定のソース媒体タイプに影響する境界事例のバグが発見された事案。ツールのバージョン-およびバイナリ-ハッシュ・データが凍結された封緘済み取得マニフェストがあれば、防御側と訴追側が正確にどのバージョンが使用されたかについて合意でき、紛争の一軸が排除されたはずです。

- **モバイル端末抽出出力に関する複数の州事案** — 抽出ツールのベンダーが後にバグ修正パッチをリリースし、「そのバグが本件に影響したか」が問題となった事案。封緘された方法論の事実があれば、当該機関が現時点でどのバージョンが使われたと記憶しているかではなく、凍結された版次情報に照らして問いを解くことができます。

- **ボディカム引き渡しに関する紛争** — BWC ベンダーのドッキングシステムが、後に伝送中の映像の改変を許していたとの主張がなされた事案。機関がボディカムアーカイブを受領した瞬間の封緘されたマニフェストは、機関がドッキングシステムから受け取ったバイトを固定し、機関側でのその後の編集が公的に検出可能となります。

- **コンピュータフォレンジック事案におけるハッシュ衝突論争** — 防御側の専門家が、(レガシーなワークフローで未だ使用される)MD5 では不十分であると主張した事案。MD5 と SHA-256 の両方を記録する封緘済み取得は、MD5 が論駁されても SHA-256 が残存することを可能にします。

- **取得から審理までの間における検査方法論の進化に関する紛争**。事案が取得から審理までの間に数年経過する場合、取得時に標準であった方法論が審理時には更新されていることがあります。封緘された記録は、後日の基準ではなく、取得の瞬間に通用していた基準に照らして方法論を裁判所が評価することを可能にします。

このキットは、上述のいずれの類型においても、有罪・無罪に関する根底にある紛争を解決しなかったでしょう。それが解決したであろうのは、特定の証拠サブ紛争 — 機関が主張する時点に、機関が主張する方法で、バイト、ツール、手続がそうであったか — のみです。

## このキットを使用するとき

- 貴機関には安定したフォレンジック SOP が存在し、既存の紙と署名による証拠保全の上に、Bitcoin にアンカリングされた公的証言を改竄検出可能な層として追加したい場合。
- 本件においてフォレンジック証拠への防御側の重大な異議申し立てが見込まれる場合(高い争点性、新規方法論、争いのあるツールバージョン等)。
- PII や脆弱者を再識別しうるデータを露出することなく、列挙された事実セットを公表できる場合(キットの拒否規則がこれを強制します。事実が拒否規則の破棄を要するなら、キットを使わないでください)。
- 取得機関の総務部および検察庁が公表計画をレビューした場合。事件識別子を含む成果物の公表は、進行中の捜査に影響を及ぼします。総務部のレビューは任意ではありません。
- 貴管轄の防御側ディスカバリー規則が、機関が役割-個人の対応付け記録(FACT 05 でハッシュ化)を別途保持することと矛盾しない場合。貴管轄が検査官氏名の公的開示を要求するなら、それに合わせてキットを改修する必要があります。

## このキットを使用**しない**とき

- **機関の既存の証拠保全文書の代替として使用しないこと**。本キットは*付加的*であって代替ではありません。機関の CMS、保全ログ、ディスカバリー提供義務は変わりません。
- **取得が CSAM、生体テンプレート、その他の脆弱者を再識別しうるデータをファイル単位で含む場合は使用しないこと**。ディスク全体のイメージ単位ハッシュは許容されますが、当該素材のファイル単位ハッシュは決して許容されません。たとえ拒否規則の例外措置のもとであっても。R3 を破らずに事実セットを構築できないなら、キットを使わないでください。
- **令状が合法であった、押収が適正であった、または証拠が証拠能力を持つと主張するために使用しないこと** — キットはそれをすることができず、拒否規則 R5 がそのような主張をすべて拒絶します。
- **取得が完了する前に封緘するよう圧力を受けて使用しないこと**。部分的取得の封緘は状態を誤って提示します。
- **検査官の個人名を公表するために使用しないこと** — 管轄が特にそれを要求し、検査官個別に同意している場合を除き。デフォルトは役割レベル公表です。
- **明示的な裁判所の許可なしに、封緘された事案または法廷封緘下の事案で使用しないこと**。事件識別子を Bitcoin 上に公表することは不可逆です。事案が後で封緘される場合、公的封緘は撤回できません。本キットは公開記録の取得のためのものです。
- **正当性の演劇(legitimacy theater)として使用しないこと**。瑕疵ある取得の封緘は、瑕疵に対する防御ではなく、瑕疵の永続的な公的記録となります。プリミティブは両面に切れます。それがその誠実さです。

## 実際の取得のためにこのキットをフォークする方法

```bash
cp -r ~/Genesis/myriam-kit/examples/court-evidence-custody ~/case-FICT-CR-XXXX
cd ~/case-FICT-CR-XXXX
$EDITOR facts/*.txt    # 架空の内容を実物で置換
                       # 全体に拒否規則 R1-R8 を適用
bash build/seal.sh     # 撮像完了時または直後に封緘
python3 build/verify.py
```

**タイミングが決定的**: 撮像終了時または直後に封緘してください(例示タイムラインの FACT 05 イベント E17)。撮像中の封緘は部分取得の記録を生みます。はるかに後の封緘は、その間機関がイメージを変更しえた間隙を生みます。正しい窓は、証拠セット内の最後の項目についてイメージ検証が完了した直後です。

**レビューが決定的**: 封緘前に、主任ではない第二のフォレンジック検査官および機関の総務を通して公表内容を検証してください。一度封緘されれば、公表は永続的です。

## 既存の証拠保全インフラとの統合

- **機関の事件管理システム(CMS)**。CMS は機関の主要な記録システムであり続けます。封緘は CMS を置き換えるものではなく、CMS が記録するバイトと方法論にコミットすることで、事後の CMS 編集が公的に検出可能となります。
- **連邦証拠規則 / Daubert / 州の証拠法**。封緘は、改竄検出可能な完全性を提供することで、FRE 901 認証記録に寄与します。それのみで FRE 702 / Daubert を満たすものではありません。方法論の信頼性は依然として証拠採用審理で試されます。
- **防御側ディスカバリー(FRCP 16、Brady、Giglio、各州相当物)**。FACT 05 でハッシュ化された付随記録(役割-個人の対応付け、保全ログ、改竄シール写真等)は通常のディスカバリーの対象として残ります。封緘は、それらの記録が取得の瞬間に何を主張していたかを固定します。これにより、後日の「当時記録は異なっていた」との主張は公的に反証可能となります。
- **NIST コンピュータフォレンジックツールテスト(CFTT)**。本キットは FACT 04 / FACT 05 で CFTT 検証ログを参照します。参照はハッシュであってログ自体ではないため、CFTT 手続は影響を受けません。
- **裁判所選任の中立者および防御側専門家**。いずれも、ディスカバリーで受領するフォレンジックイメージを FACT 03 マニフェストと再ハッシュ比較できます。これは機関の協力とは独立に行えます。これが防御側にとっての本キットの主要な付加価値です。
- **上訴審の審査**。数年後、上訴審のフォレンジックレビュアーは、機関の進化したかもしれない現在の記憶ではなく、封緘された FACT 04 記録に対して当初の取得方法論を検証できます。
