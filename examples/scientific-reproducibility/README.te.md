# శాస్త్రీయ పునరుత్పాదకత కిట్ (Scientific Reproducibility Kit)

*తెలుగు అనువాదం — అసలు ఆంగ్ల వెర్షన్ కోసం, చూడండి [README.md](README.md)*

పరిశోధన బృందాలు, జర్నల్‌లు, మరియు పునరుత్పాదకత-కేంద్రిత సంస్థల కోసం ఒక సిద్ధంగా ఉన్న myriam-kit ఉదాహరణ, ఒక పేపర్ లేదా preprint తో పాటు Bitcoin-యాంకర్డ్, చెడగొట్టడం-స్పష్టమైన పునరుత్పాదకత మానిఫెస్ట్‌ను ప్రచురించాలనుకునే వారికి.

## ఈ కిట్ ఉత్పత్తి చేసేది

ఒక అధ్యయనం కోసం, సీలు చేయబడిన ప్రచురణ కలిగి ఉంటుంది:

1. అధ్యయనం గుర్తింపు (పేపర్ శీర్షిక, రచయితలు, సంస్థ, ఆర్థిక సహాయకులు, COI)
2. **ముడి డేటా ఆర్కైవ్ SHA-256** — నియమబద్ధ డేటాసెట్
3. **విశ్లేషణ పైప్‌లైన్ హ్యాష్** — script + Docker image digest + pinned dependencies
4. **ఆశించిన అవుట్‌పుట్ బండిల్ హ్యాష్** — విశ్వాసపాత్రమైన పునరుత్పాదక ఉత్పత్తి చేయవలసినది
5. పునరుత్పాదన ప్రోటోకాల్ + ఫలితం-తర్వాత ప్రచురణ నిబద్ధతలు
6. సంతక principal(s)

ఈ సీలు **పూర్వ-విశ్లేషణ** దశ కోసం ఉద్దేశించబడింది: డేటా స్తంభింపజేయబడింది, పైప్‌లైన్ ఖరారు చేయబడింది, ఆశించిన అవుట్‌పుట్ లెక్కించబడింది, పేపర్ రాయడం ప్రయోజనం కోసం ఏ ఫలితాలు చూడబడే ముందు అన్నీ.

## ఈ కిట్ ఏది రక్షిస్తుంది

**సీలు పట్టుకునే వైఫల్య విధానాలు:**

- **p-hacking**: ఫలితాలు చూసిన తర్వాత విశ్లేషణలో post-hoc మార్పులు, సీలు చేయబడిన FACT 04 మరియు ప్రచురించబడిన పైప్‌లైన్ మధ్య diff ద్వారా గుర్తించదగినవి.
- **HARKing** (Hypothesizing After Results are Known): FACT 02 లో పూర్వ-నమోదిత ఊహ byte-stable.
- **పేపర్-తర్వాత డేటా తారుమారు**: FACT 03 కి వ్యతిరేకంగా డేటా ఆర్కైవ్‌ను తిరిగి-హ్యాష్ చేయడం ఏ మార్పునైనా వెల్లడిస్తుంది.
- **పైప్‌లైన్ డ్రిఫ్ట్**: FACT 04 కి వ్యతిరేకంగా విశ్లేషణ పైప్‌లైన్‌ను తిరిగి-హ్యాష్ చేయడం నిశ్శబ్ద మార్పులను పట్టుకుంటుంది.
- **రచయితృత్వ గేమింగ్**: ప్రచురణ తర్వాత నిశ్శబ్దంగా రచయితలను జోడించడం లేదా తీసివేయడం సీలు చేయబడిన FACT 02 కి వ్యతిరేకంగా గుర్తించదగినదిగా మారుతుంది.

**కవర్ చేయని వైఫల్య విధానాలు:**

- ముడి-డేటా సేకరణలో మోసం (సీలు సీలింగ్ సమయంలో డేటాను యాంకర్ చేస్తుంది, సేకరణ సమయంలో కాదు)
- జీవశాస్త్రీయ / అనుభావిక పునరుత్పాదక-రహితం (సీలు గణన పునరుత్పాదకతను మాత్రమే యాంకర్ చేస్తుంది)
- నమూనా-పరిమాణం లేదా జనాభా-ప్రామాణికత సమస్యలు (ఇవి అధ్యయన-రూపకల్పన ప్రశ్నలు)
- అంతర్లీన దావా యొక్క నిజం (సీలు ఏది దావా చేయబడిందో దాన్ని యాంకర్ చేస్తుంది, ప్రపంచం దావాకు సరిపోతుందా అని కాదు)

## ఈ కిట్‌ను ఎప్పుడు ఉపయోగించాలి

కింది సందర్భాలలో సీలు చేయడాన్ని పరిగణించండి:

- మీరు ఒక అధిక-ప్రాముఖ్యత గల విశ్లేషణ (క్లినికల్ ట్రయల్, AI సిస్టమ్ మూల్యాంకనం, ప్రతిరూపణ అధ్యయనం) నడుపుతున్న పరిశోధన బృందం మరియు OSF / GitHub టైమ్-స్టాంప్‌ల కంటే మించి ఫోరెన్సిక్ సమగ్రత కోరుకుంటున్నారు
- మీరు అసమాన ఖ్యాతి-రక్షణ కోరుకునే ప్రారంభ-కెరీర్ పరిశోధకుడు: సీలు చేయబడిన పూర్వ-విశ్లేషణ అంటే భవిష్యత్తులో "వారు p-hacked" విమర్శకు పబ్లిక్ falsifier ఉంది
- మీరు పేపర్‌ల ఉపసమితి కోసం cryptographic-గ్రేడ్ పూర్వ-నమోదును ప్రోత్సహించాలనుకునే జర్నల్ ఎడిటర్ లేదా ప్రతిరూపణ-ప్రాజెక్ట్ నాయకుడు
- మీరు post-hoc tweaks అర్థవంతమైన రంగ పర్యవసానాలు కలిగి ఉన్న విశ్లేషణపై పని చేస్తున్నారు (psych, behavioral econ, medical research)

## చారిత్రక మరణాలు / హాని పూర్వ ఉదాహరణ

ఈ కిట్ యొక్క మరణాలు-సంబంధం పరోక్షం:

- **స్టెమ్ సెల్ / క్యాన్సర్ పరిశోధన పునరుత్పాదక-రహితం:** అనేక పేపర్‌లు తప్పుగా దిశానిర్దేశం చేయబడిన క్లినికల్ ట్రయల్‌ల ద్వారా రోగి హానికి దారితీశాయి. అసలు పైప్‌లైన్ యొక్క సీలు చేయబడిన పూర్వ-నమోదు పోలికను యాంకర్ చేసి ఉండేది.
- **Tamiflu యొక్క ప్రభావకారిత వివాదం:** ముడి-డేటా యాక్సెస్ వివాదాస్పదమైనందున Cochrane review సంవత్సరాలు పట్టింది. ప్రచురణ సమయంలో సీలు చేయబడిన డేటా-హ్యాష్‌లు డేటా-యాక్సెస్ వివాదాలను తక్షణ పరిష్కరణ చేసేవి.
- **సాధారణ పునరుత్పాదక-రహిత సంక్షోభం:** పునరుత్పాదక-రహిత బయోమెడికల్ పరిశోధన యొక్క రంగ-స్థాయి మరణాల ఖర్చు ముఖ్యమైనది (Begley & Ellis 2012 >50% preclinical findings పునరుత్పాదక-రహితం అని అంచనా వేసింది). Cryptographic యాంకరింగ్ ఒక నిర్మాణాత్మక లివర్.

## ఈ కిట్‌ను ఎలా fork చేయాలి

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# CRITICAL: do this BEFORE seeing any results
shasum -a 256 raw_data_v0.tar.gz       # compute the canonical data hash
shasum -a 256 analysis.py              # hash your pipeline
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # populate with real values
bash build/seal.sh                     # seal BEFORE running the analysis
python3 build/verify.py                # confirm

# Now run the analysis. Hash the output and confirm matches FACT 05.
# If it doesn't match, the pipeline is non-deterministic; fix and re-seal.
```

## ప్రస్తుత పండిత మౌలిక సదుపాయాలతో సమైక్యత

- **arXiv / bioRxiv / medRxiv**: సారాంశం లేదా అనుబంధ సామగ్రిలో MYRIAM సీలు URL లేదా IPFS CID చేర్చండి
- **OSF**: సాంప్రదాయ పూర్వ-నమోదు ఫారంతో పాటు OSF ప్రాజెక్ట్‌కు సీలు ఆర్టిఫాక్ట్‌ను జతచేయండి
- **AsPredicted**: పూర్వ-నమోదు పత్రంలో MYRIAM సీలు URL చేర్చండి
- **GitHub**: విశ్లేషణ పైప్‌లైన్ యొక్క commit hash అనేది FACT 04 యాంకర్ చేసేది; GitHub మూలాన్ని అందిస్తుంది, MYRIAM బైట్‌లను యాంకర్ చేస్తుంది
- **Zenodo / Figshare / OpenAIRE**: ముడి డేటాను డిపాజిట్ చేయండి, ఆపై డిపాజిట్ యొక్క hash ను సీలు చేయండి; డిపాజిట్ మెటాడేటా కంటే సీలు నిశ్శబ్దంగా భర్తీ చేయడానికి ఫోరెన్సిక్‌గా కఠినమైనది
- **జర్నల్ అనుబంధ సామగ్రి**: సమర్పణ సమయంలో అనుబంధ ఫైల్‌గా MYRIAM సీలు ఆర్టిఫాక్ట్‌ను చేర్చండి

## రచయితృత్వ క్రమం మరియు క్రెడిట్ గురించి ఒక గమనిక

సీలు సీలింగ్ సమయంలో AUTHOR LIST ను యాంకర్ చేస్తుంది. ప్రచురణ-తర్వాత తలెత్తే రచయితృత్వ వివాదాలను సీలు చేయబడిన జాబితాకు వ్యతిరేకంగా మూల్యాంకనం చేయవచ్చు. ఇది ఒక చిన్న జవాబుదారీతన లివర్ — నిశ్శబ్దంగా జోడించబడిన లేదా తీసివేయబడిన ECRs కోసం ముఖ్యమైనది.
