# Набор для научной воспроизводимости

Готовый пример myriam-kit для исследовательских групп, журналов и
учреждений, ориентированных на воспроизводимость, желающих публиковать
заякоренный в Bitcoin манифест воспроизводимости со свойством
обнаружения вмешательства (tamper-evident) — вместе со статьёй или
препринтом.

## Что создаёт этот набор

Запечатанная публикация, содержащая для одного исследования:

1. Идентификацию исследования (название статьи, авторы, учреждение,
   спонсоры, конфликт интересов)
2. **SHA-256 архива сырых данных** — канонический датасет
3. **Хэш аналитического конвейера** — скрипт + дайджест Docker-образа
   + зафиксированные (pinned) зависимости
4. **Хэш пакета ожидаемого вывода** — то, что должен получить
   добросовестный воспроизводитель
5. Протокол воспроизведения + обязательства по публикации
   результатов
6. Подписывающий принципал (или принципалы)

Печать предназначена для стадии **до анализа**: данные зафиксированы,
конвейер финализирован, ожидаемый вывод вычислен — ВСЁ это до того,
как будут увидены какие-либо результаты с целью написания статьи.

## От чего защищает этот набор

**Сценарии отказа, выявляемые печатью:**

- **p-hacking**: пост-фактум изменения анализа после ознакомления с
  результатами обнаруживаются через diff между запечатанным FACT 04
  и опубликованным конвейером.
- **HARKing** (Hypothesizing After Results are Known):
  предзарегистрированная гипотеза в FACT 02 побайтово стабильна.
- **Манипуляция данными после статьи**: повторное хэширование архива
  данных относительно FACT 03 выявляет любые модификации.
- **Дрейф конвейера**: повторное хэширование аналитического конвейера
  относительно FACT 04 ловит скрытые изменения.
- **Манипуляции с авторством**: скрытое добавление или удаление
  авторов после публикации становится обнаружимым относительно
  запечатанного FACT 02.

**Сценарии отказа, НЕ покрываемые набором:**

- Мошенничество при сборе сырых данных (печать фиксирует данные в
  момент опечатывания, а не сбора)
- Биологическая / эмпирическая невоспроизводимость (печать фиксирует
  только вычислительную воспроизводимость)
- Проблемы с размером выборки или валидностью популяции (это
  вопросы дизайна исследования)
- Истинность исходного утверждения (печать фиксирует то, что было
  заявлено, а не то, соответствует ли мир этому заявлению)

## Когда использовать этот набор

Рассмотрите возможность опечатывания, если:

- Вы исследовательская группа, выполняющая анализ с высокими
  ставками (клиническое испытание, оценка ИИ-системы, репликационное
  исследование), и хотите получить судебно-значимую целостность за
  пределами временных меток OSF / GitHub
- Вы исследователь на ранней стадии карьеры, желающий асимметричной
  репутационной защиты: запечатанный предварительный анализ означает,
  что будущая критика «они занимались p-hacking» будет иметь
  публичный фальсификатор
- Вы редактор журнала или руководитель проекта по репликации,
  желающий поощрить криптографически надёжную предрегистрацию для
  части статей
- Вы работаете над анализом, в котором пост-фактум корректировки
  имеют значимые последствия для области (психология, поведенческая
  экономика, медицинские исследования)

## Исторический прецедент смертности / вреда

Связь этого набора со смертностью является косвенной:

- **Невоспроизводимость в исследованиях стволовых клеток / рака:**
  ряд статей привёл к вреду для пациентов через неверно направленные
  клинические испытания. Запечатанная предрегистрация исходного
  конвейера заякорила бы сравнение.
- **Споры об эффективности Tamiflu:** Cochrane-обзор занял годы
  отчасти из-за того, что доступ к сырым данным оспаривался.
  Запечатанные хэши данных на момент публикации сделали бы споры о
  доступе к данным немедленно разрешимыми.
- **Общий кризис невоспроизводимости:** уровень смертности на уровне
  области из-за невоспроизводимых биомедицинских исследований
  значителен (Begley & Ellis 2012 оценили, что более 50%
  доклинических находок невоспроизводимы). Криптографическое
  заякоривание — один из структурных рычагов.

## Как форкнуть этот набор

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# КРИТИЧНО: сделайте это ДО ознакомления с какими-либо результатами
shasum -a 256 raw_data_v0.tar.gz       # вычислить канонический хэш данных
shasum -a 256 analysis.py              # хэшировать ваш конвейер
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # заполнить реальными значениями
bash build/seal.sh                     # опечатывайте ДО запуска анализа
python3 build/verify.py                # подтвердите

# Теперь запустите анализ. Хэшируйте вывод и подтвердите соответствие FACT 05.
# Если соответствия нет, конвейер недетерминирован; исправьте и опечатайте заново.
```

## Интеграция с существующей научной инфраструктурой

- **arXiv / bioRxiv / medRxiv**: включите URL печати MYRIAM или CID
  в IPFS в аннотацию или дополнительные материалы
- **OSF**: приложите артефакт печати к проекту OSF наряду с
  обычной формой предрегистрации
- **AsPredicted**: включите URL печати MYRIAM в документ
  предрегистрации
- **GitHub**: хэш коммита аналитического конвейера — это то, что
  FACT 04 заякоривает; GitHub предоставляет исходный код, а
  MYRIAM заякоривает байты
- **Zenodo / Figshare / OpenAIRE**: задепозитьте сырые данные,
  затем опечатайте хэш депозита; печать судебно труднее скрытно
  заменить, чем метаданные депозита
- **Дополнительные материалы журнала**: включите артефакт печати
  MYRIAM как дополнительный файл при подаче

## Что этот набор НЕ заменяет

- Одобрение IRB / этического комитета
- Рецензирование
- Репликационные исследования (вычислительная воспроизводимость ≠
  эмпирическая воспроизводимость)
- Соглашения о доступе к чувствительным данным

## Замечание о порядке авторов и о соавторстве

Печать заякоривает СПИСОК АВТОРОВ на момент опечатывания. Споры об
авторстве, возникающие после публикации, можно оценивать
относительно запечатанного списка. Это небольшой рычаг
подотчётности — важный для исследователей на ранней стадии карьеры
(ECR), которых добавляют или удаляют скрытно.
