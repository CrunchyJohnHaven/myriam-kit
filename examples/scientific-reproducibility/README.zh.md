# 科学可复现性套件

*中文翻译 — 英文原文见 [README.md](README.md)*

一个开箱即用的 myriam-kit 范例,面向希望随论文或预印本一同发布**经 Bitcoin 锚定、可察觉篡改的可复现性清单**的研究团队、期刊与可复现性导向的机构。

## 本套件产出什么

针对一项研究的一份密封发布,内容包含:

1. 研究身份信息(论文标题、作者、所属机构、资助方、利益冲突)
2. **原始数据归档的 SHA-256** —— 规范数据集
3. **分析流水线哈希** —— 脚本 + Docker 镜像摘要 + 固定版本的依赖
4. **预期输出包哈希** —— 一位忠实复现者应当产出的内容
5. 复现协议与结果公布后的发布承诺
6. 签署主体

该密封意在锚定**预分析阶段**:数据已冻结、流水线已定型、预期输出已计算完成,以上一切**在**为撰写论文目的而看到任何结果之前完成。

## 本套件防御什么

**密封能捕捉到的失败模式:**

- **p-hacking**:看到结果之后才对分析做的事后修改,可通过密封的 FACT 04 与公开流水线之间的 diff 检测出来。
- **HARKing**(Hypothesizing After Results are Known,结果已知后才提出假设):FACT 02 中的预注册假设是字节稳定的。
- **论文发表后的数据操纵**:对数据归档重新哈希并比对 FACT 03,可揭示任何改动。
- **流水线漂移**:对分析流水线重新哈希并比对 FACT 04,可捕捉到悄然变更。
- **作者名单做手脚**:发表后悄悄添加或移除作者,可对照已密封的 FACT 02 被检测出来。

**未覆盖的失败模式:**

- 原始数据采集环节的造假(密封锚定的是密封时刻的数据,而非采集时刻)
- 生物学 / 经验层面的不可复现性(密封只锚定**计算**层面的可复现性)
- 样本量或总体效度问题(这些属于研究设计问题)
- 底层主张的真伪(密封锚定的是**所宣称的内容**,而非世界是否与该主张相符)

## 何时使用本套件

如果您符合以下情形,可考虑密封:

- 您所在的研究团队正在进行高风险分析(临床试验、AI 系统评测、复制研究),希望获得超越 OSF / GitHub 时间戳的法证级完整性
- 您是早期职业研究者,希望获得不对称的声誉保护:经密封的预分析意味着未来"他们做过 p-hacking"之类的批评有了一个公开的可证伪物
- 您是期刊编辑或复制项目负责人,希望对某一子集论文鼓励**密码学级**的预注册
- 您正在做的分析中,事后微调具有重大的领域后果(心理学、行为经济学、医学研究)

## 历史死亡 / 危害先例

本套件与死亡相关性是间接的:

- **干细胞 / 癌症研究不可复现:** 多篇论文已通过误导性的临床试验造成患者伤害。对原始流水线做经密封的预注册,本可为比对锚定基准。
- **达菲(Tamiflu)疗效之争:** Cochrane 综述耗时多年,部分原因在于原始数据访问权存在争议。在发表时密封数据哈希,本可使数据访问争议立即可裁决。
- **总体不可复现性危机:** 不可复现生物医学研究在领域层面的死亡代价相当可观(Begley & Ellis 2012 估计逾 50% 的临床前发现不可复现)。密码学锚定是其中一项结构性杠杆。

## 如何派生本套件

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# 关键:务必在看到任何结果**之前**完成以下步骤
shasum -a 256 raw_data_v0.tar.gz       # 计算规范数据哈希
shasum -a 256 analysis.py              # 对您的流水线做哈希
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # 填入真实值
bash build/seal.sh                     # 在运行分析**之前**完成密封
python3 build/verify.py                # 确认

# 现在再运行分析。对输出做哈希,并确认其与 FACT 05 匹配。
# 若不匹配,则说明流水线是非确定性的;修复后重新密封。
```

## 与现有学术基础设施的集成

- **arXiv / bioRxiv / medRxiv**:在摘要或补充材料中给出 MYRIAM 密封的 URL 或 IPFS CID
- **OSF**:将密封产物作为附件与常规预注册表单一同挂在 OSF 项目上
- **AsPredicted**:在预注册文档中给出 MYRIAM 密封的 URL
- **GitHub**:分析流水线的 commit 哈希正是 FACT 04 所锚定的对象;GitHub 提供源码,MYRIAM 锚定字节
- **Zenodo / Figshare / OpenAIRE**:存入原始数据,然后密封存入项的哈希;相比存入项的元数据,密封在法证意义上更难被悄然替换
- **期刊补充材料**:投稿时将 MYRIAM 密封产物作为补充文件一同提交

## 本套件**不能**替代什么

- IRB / 伦理委员会审批
- 同行评审
- 复制研究(计算层面的可复现性 ≠ 经验层面的可复现性)
- 敏感数据的数据访问协议

## 关于作者排序与署名的说明

密封锚定的是**密封时刻的作者名单**。发表后产生的署名争议可对照已密封的名单进行评估。这是一个小而有用的问责杠杆 —— 对那些可能被悄悄加入或移出名单的早期职业研究者尤为重要。
