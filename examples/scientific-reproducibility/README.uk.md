# Набір для наукової відтворюваності

Готовий до використання приклад myriam-kit для дослідницьких груп,
журналів та орієнтованих на відтворюваність установ, які бажають
опублікувати поряд зі статтею чи препринтом маніфест відтворюваності,
заякорений у Bitcoin та стійкий до підробок.

## Що цей набір створює

Запечатану публікацію, що містить для одного дослідження:

1. Ідентифікацію дослідження (заголовок статті, автори, установа,
   фінансувачі, COI)
2. **SHA-256 архіву необроблених даних** — канонічний набір даних
3. **Хеш конвеєра аналізу** — скрипт + дайджест Docker-образу +
   зафіксовані залежності
4. **Хеш очікуваного пакета виводу** — те, що сумлінний відтворювач
   повинен отримати
5. Протокол відтворення + зобов'язання щодо публікації після
   отримання результатів
6. Підписант(и)-принципал(и)

Печатка призначена для стадії **до-аналізу**: дані заморожено,
конвеєр фіналізовано, очікуваний вивід обчислено, УСЕ перш ніж буде
переглянуто будь-який результат з метою написання статті.

## Від чого цей набір захищає

**Режими відмови, які печатка вловлює:**

- **p-hacking**: апостеріорні зміни в аналізі після перегляду
  результатів виявляються через diff між запечатаним FACT 04 та
  опублікованим конвеєром.
- **HARKing** (Hypothesizing After Results are Known — формулювання
  гіпотез після того, як відомі результати): передреєстрована
  гіпотеза у FACT 02 байт-стабільна.
- **Маніпуляція даними після статті**: повторне хешування архіву
  даних проти FACT 03 виявляє будь-яку модифікацію.
- **Дрейф конвеєра**: повторне хешування конвеєра аналізу проти
  FACT 04 вловлює тихі зміни.
- **Маніпуляції авторством**: тихе додавання або видалення авторів
  після публікації стає виявним проти запечатаного FACT 02.

**Режими відмови, які НЕ покриваються:**

- Шахрайство при зборі необроблених даних (печатка заякорює дані в
  момент запечатування, а не збору)
- Біологічна / емпірична невідтворюваність (печатка заякорює лише
  обчислювальну відтворюваність)
- Питання розміру вибірки чи валідності популяції (це питання
  дизайну дослідження)
- Істинність основоположного твердження (печатка заякорює те, що
  було заявлено, а не те, чи відповідає світ цьому твердженню)

## Коли використовувати цей набір

Розгляньте можливість запечатування, якщо:

- Ви — дослідницька група, що проводить аналіз високої важливості
  (клінічне випробування, оцінювання системи ШІ, дослідження
  реплікації), і хочете криміналістичної цілісності понад часові
  мітки OSF / GitHub
- Ви — дослідник на початку кар'єри, який хоче асиметричного
  репутаційного захисту: запечатаний до-аналіз означає, що майбутня
  критика «вони робили p-hacking» матиме публічний фальсифікатор
- Ви — редактор журналу або керівник проекту з реплікації, який
  хоче заохочувати передреєстрацію криптографічного рівня для
  підмножини статей
- Ви працюєте над аналізом, де апостеріорні налаштування мають
  значущі наслідки для галузі (психологія, поведінкова економіка,
  медичні дослідження)

## Історичний прецедент смертності / шкоди

Стосунок цього набору до смертності непрямий:

- **Невідтворюваність у дослідженнях стовбурових клітин / раку:**
  кілька статей призвели до шкоди пацієнтам через хибно скеровані
  клінічні випробування. Запечатана передреєстрація оригінального
  конвеєра заякорила б порівняння.
- **Суперечка щодо ефективності Tamiflu:** огляд Cochrane тривав
  роки частково через те, що доступ до необроблених даних
  оспорювався. Запечатані хеші даних на момент публікації зробили б
  суперечки про доступ до даних негайно розв'язуваними.
- **Загальна криза невідтворюваності:** галузевий рівень смертності
  від невідтворюваних біомедичних досліджень значний (Begley &
  Ellis 2012 оцінили, що >50% доклінічних знахідок невідтворювані).
  Криптографічне якорення є одним зі структурних важелів.

## Як форкнути цей набір

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# КРИТИЧНО: зробіть це ПЕРЕД переглядом будь-яких результатів
shasum -a 256 raw_data_v0.tar.gz       # обчисліть канонічний хеш даних
shasum -a 256 analysis.py              # хешуйте свій конвеєр
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # заповніть реальними значеннями
bash build/seal.sh                     # запечатайте ПЕРЕД запуском аналізу
python3 build/verify.py                # підтвердьте

# Тепер запустіть аналіз. Хешуйте вивід і підтвердьте відповідність FACT 05.
# Якщо не відповідає, конвеєр недетермінований; виправте та запечатайте знову.
```

## Інтеграція з існуючою науковою інфраструктурою

- **arXiv / bioRxiv / medRxiv**: включіть URL печатки MYRIAM або
  IPFS CID в анотацію чи додаткові матеріали
- **OSF**: прикріпіть артефакт печатки до проекту OSF поруч зі
  звичайною формою передреєстрації
- **AsPredicted**: включіть URL печатки MYRIAM у документ
  передреєстрації
- **GitHub**: хеш коміту конвеєра аналізу — це те, що заякорює
  FACT 04; GitHub надає вихідний код, MYRIAM заякорює байти
- **Zenodo / Figshare / OpenAIRE**: депонуйте необроблені дані, потім
  запечатайте хеш депозиту; печатку криміналістично складніше тихо
  замінити, ніж метадані депозиту
- **Додаткові матеріали журналів**: включіть артефакт печатки MYRIAM
  як додатковий файл при поданні

## Чого цей набір НЕ замінює

- Затвердження IRB / комітету з етики
- Рецензування
- Дослідження реплікації (обчислювальна відтворюваність ≠ емпірична
  відтворюваність)
- Угоди про доступ до даних для чутливих даних

## Примітка про порядок авторства та заслугу

Печатка заякорює СПИСОК АВТОРІВ на момент запечатування. Суперечки
про авторство, що виникають після публікації, можуть бути оцінені
проти запечатаного списку. Це невеликий важіль підзвітності —
важливий для дослідників на початку кар'єри, яких тихо додають або
видаляють.
