# 과학적 재현성 키트

논문 또는 프리프린트와 함께 Bitcoin에 앵커링된 변조 탐지 가능
(tamper-evident) 재현성 매니페스트를 공표하고자 하는 연구 그룹,
저널, 그리고 재현성 중심의 기관을 위한 턴키 myriam-kit 예시입니다.

## 이 키트가 생성하는 것

한 연구에 대해 다음을 포함하는 봉인된 공표:

1. 연구 식별(논문 제목, 저자, 소속 기관, 연구비 출처, 이해 상충)
2. **원시 데이터 아카이브 SHA-256** — 정규 데이터셋
3. **분석 파이프라인 해시** — 스크립트 + Docker 이미지 다이제스트 +
   고정된 의존성
4. **예상 출력 번들 해시** — 충실한 재현자가 생성해야 하는 것
5. 재현 프로토콜 + 결과 이후 공표 공약
6. 서명 주체(principal)

이 봉인은 **분석 전** 단계를 위한 것입니다: 데이터를 고정하고,
파이프라인을 확정하며, 예상 출력을 산출한 후, 논문 작성 목적으로
어떤 결과도 보기 전에 봉인합니다.

## 이 키트가 방어하는 대상

**봉인이 포착하는 실패 모드:**

- **p-hacking**: 결과를 본 이후의 사후적 분석 변경은 봉인된
  FACT 04와 공표된 파이프라인 간의 diff를 통해 탐지 가능합니다.
- **HARKing**(Hypothesizing After Results are Known): FACT 02의
  사전 등록된 가설은 바이트 단위로 안정합니다.
- **논문 이후 데이터 조작**: 데이터 아카이브를 FACT 03에 대해 재
  해싱하면 어떤 변경도 드러납니다.
- **파이프라인 드리프트**: 분석 파이프라인을 FACT 04에 대해 재
  해싱하면 은밀한 변경이 포착됩니다.
- **저자 명단 조작**: 공표 이후 저자를 은밀히 추가하거나 제거하는
  행위는 봉인된 FACT 02에 대해 탐지 가능해집니다.

**커버되지 않는 실패 모드:**

- 원시 데이터 수집의 부정(봉인은 봉인 시점의 데이터를 앵커링하며,
  수집 시점은 아닙니다)
- 생물학적/경험적 비재현성(봉인은 오직 계산적 재현성만을 앵커링합니다)
- 표본 크기 또는 모집단 타당성 문제(이는 연구 설계의 문제입니다)
- 기저 주장의 진실성(봉인은 무엇이 주장되었는지를 앵커링할 뿐,
  세계가 그 주장과 일치하는지를 앵커링하지 않습니다)

## 이 키트를 사용해야 할 때

다음에 해당한다면 봉인을 고려하십시오:

- 고위험 분석(임상시험, AI 시스템 평가, 복제 연구)을 수행하는 연구
  그룹이며 OSF / GitHub 타임스탬프를 넘어선 법의학적 완전성을
  원하는 경우
- 비대칭적인 평판 보호를 원하는 초기 경력 연구자: 봉인된 사전
  분석은 향후 "그들은 p-hacking을 했다"는 비판에 대한 공개적인
  반증 수단이 됩니다
- 일부 논문에 대해 암호학적 등급의 사전 등록을 장려하고자 하는
  저널 편집자 또는 복제 프로젝트 책임자
- 사후 조정이 해당 분야에 실질적인 결과를 미치는 분석(심리학,
  행동경제학, 의학 연구)에 종사하는 경우

## 역사적 사망/피해 전례

이 키트의 사망률 관련성은 간접적입니다:

- **줄기세포 / 암 연구의 비재현성:** 여러 논문이 잘못된 방향의 임상
  시험을 통해 환자 피해로 이어졌습니다. 원본 파이프라인의 봉인된
  사전 등록이 있었다면 비교의 앵커가 되었을 것입니다.
- **타미플루 효능 논란:** Cochrane 리뷰가 수년이 걸린 이유 중 일부는
  원시 데이터 접근이 쟁점이 되었기 때문입니다. 출판 시점의 봉인된
  데이터 해시가 있었다면 데이터 접근 분쟁이 즉시 해결 가능했을
  것입니다.
- **일반적인 비재현성 위기:** 재현 불가능한 생의학 연구의 분야
  수준 사망 비용은 상당합니다(Begley & Ellis 2012는 전임상 발견의
  50% 이상이 재현 불가능하다고 추정). 암호학적 앵커링은 하나의
  구조적 지렛대입니다.

## 이 키트를 포크하는 방법

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# 중요: 어떤 결과도 보기 전에 이를 수행할 것
shasum -a 256 raw_data_v0.tar.gz       # 정규 데이터 해시 계산
shasum -a 256 analysis.py              # 파이프라인 해시
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # 실제 값으로 채우기
bash build/seal.sh                     # 분석 실행 전 봉인
python3 build/verify.py                # 확인

# 이제 분석을 실행합니다. 출력을 해시하여 FACT 05와 일치함을 확인하십시오.
# 일치하지 않으면 파이프라인이 비결정적입니다; 수정 후 재봉인.
```

## 기존 학술 인프라와의 통합

- **arXiv / bioRxiv / medRxiv**: MYRIAM 봉인 URL 또는 IPFS CID를
  초록 또는 보충 자료에 포함
- **OSF**: 기존 사전 등록 양식과 함께 봉인 아티팩트를 OSF
  프로젝트에 첨부
- **AsPredicted**: 사전 등록 문서에 MYRIAM 봉인 URL 포함
- **GitHub**: 분석 파이프라인의 커밋 해시가 곧 FACT 04가 앵커링하는
  대상입니다; GitHub는 소스를 제공하고, MYRIAM은 바이트를
  앵커링합니다
- **Zenodo / Figshare / OpenAIRE**: 원시 데이터를 디포짓한 후, 그
  디포짓의 해시를 봉인하십시오; 봉인은 디포짓 메타데이터보다
  법의학적으로 은밀하게 교체하기가 더 어렵습니다
- **저널 보충 자료**: 제출 시점에 MYRIAM 봉인 아티팩트를 보충
  파일로 포함

## 이 키트가 대체하지 않는 것

- IRB / 윤리위원회 승인
- 동료 심사
- 복제 연구(계산적 재현성 ≠ 경험적 재현성)
- 민감 데이터에 대한 데이터 접근 합의

## 저자 순서 및 크레딧에 관한 주의

봉인은 봉인 시점의 저자 명단을 앵커링합니다. 공표 이후 발생하는
저자 자격 관련 분쟁은 봉인된 명단에 대해 평가될 수 있습니다. 이는
작은 책임 지렛대로서, 은밀히 추가되거나 제거되는 초기 경력
연구자(ECR)에게 중요합니다.
