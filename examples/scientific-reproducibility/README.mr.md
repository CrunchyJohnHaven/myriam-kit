# वैज्ञानिक पुनरुत्पादकता किट

संशोधन गट, जर्नल्स, आणि पुनरुत्पादकता-केंद्रित संस्थांसाठी एक
टर्नकी myriam-kit उदाहरण, ज्यांना एखाद्या पेपर किंवा प्रीप्रिंटसह
Bitcoin-अँकर केलेली, टॅम्पर-एव्हिडंट पुनरुत्पादकता मॅनिफेस्ट
प्रकाशित करायची आहे.

## ही किट काय तयार करते

एका अभ्यासासाठी, सीलबंद प्रकाशनात समाविष्ट आहे:

1. अभ्यास ओळख (पेपर शीर्षक, लेखक, संस्था, अनुदानदाते, COI)
2. **कच्चा डेटा अर्काइव्ह SHA-256** — कॅनोनिकल डेटासेट
3. **विश्लेषण पाइपलाइन हॅश** — स्क्रिप्ट + Docker इमेज डायजेस्ट
   + पिन केलेल्या अवलंबित्व
4. **अपेक्षित आउटपुट बंडल हॅश** — विश्वासू पुनरुत्पादकाने काय
   तयार करावे
5. पुनरुत्पादन प्रोटोकॉल + परिणामानंतरच्या प्रकाशन वचनबद्धता
6. स्वाक्षरी करणारे प्रिन्सिपल

सील **पूर्व-विश्लेषण** टप्प्यासाठी हेतू आहे: डेटा गोठवलेला,
पाइपलाइन अंतिम केलेली, अपेक्षित आउटपुट मोजलेले, सर्व काही
पेपर लिहिण्याच्या उद्देशाने कोणतेही परिणाम पाहण्यापूर्वी.

## ही किट कशापासून संरक्षण करते

**सील पकडणारे अपयश मोड:**

- **p-hacking**: परिणाम पाहिल्यानंतर विश्लेषणात केलेले पोस्ट-हॉक
  बदल सीलबंद FACT 04 आणि प्रकाशित पाइपलाइन यांच्यातील डिफद्वारे
  शोधले जाऊ शकतात.
- **HARKing** (परिणाम कळल्यानंतर गृहीतक): FACT 02 मधील पूर्व-नोंदणी
  केलेले गृहीतक बाइट-स्थिर आहे.
- **पेपर-नंतरचे डेटा हाताळणे**: FACT 03 विरुद्ध डेटा अर्काइव्ह
  पुन्हा हॅश केल्याने कोणतेही बदल उघड होतात.
- **पाइपलाइन ड्रिफ्ट**: FACT 04 विरुद्ध विश्लेषण पाइपलाइन पुन्हा
  हॅश केल्याने मूक बदल पकडले जातात.
- **लेखकत्व गेमिंग**: प्रकाशनानंतर मूकपणे लेखक जोडणे किंवा काढून
  टाकणे सीलबंद FACT 02 विरुद्ध शोधले जाऊ शकते.

**अंतर्भूत नसलेले अपयश मोड:**

- कच्च्या-डेटा संकलनातील फसवणूक (सील डेटाला सीलिंगच्या बिंदूवर
  अँकर करते, संकलनात नाही)
- जैविक / प्रयोगात्मक नॉन-पुनरुत्पादकता (सील फक्त संगणकीय
  पुनरुत्पादकता अँकर करते)
- नमुना-आकार किंवा लोकसंख्या-वैधता समस्या (हे अभ्यास-रचना प्रश्न
  आहेत)
- अंतर्निहित दाव्याचे सत्य (सील दावा केलेले अँकर करते, जग दाव्याशी
  जुळते की नाही हे नाही)

## ही किट कधी वापरावी

सीलिंगचा विचार करा जर:

- तुम्ही उच्च-स्टेक्स विश्लेषण चालवणारा संशोधन गट असाल (क्लिनिकल
  ट्रायल, AI सिस्टम मूल्यमापन, प्रतिकृती अभ्यास) आणि OSF /
  GitHub टाइम-स्टॅम्पच्या पलीकडे फॉरेन्सिक अखंडता हवी असेल
- तुम्ही असममित प्रतिष्ठा-संरक्षण इच्छिणारे प्रारंभिक-कारकीर्द
  संशोधक असाल: सीलबंद पूर्व-विश्लेषणाचा अर्थ भविष्यातील "त्यांनी
  p-hack केले" टीकेला सार्वजनिक फॉल्सिफायर आहे
- तुम्ही पेपरच्या उपसंचासाठी क्रिप्टोग्राफिक-दर्जाची पूर्व-नोंदणी
  प्रोत्साहित करू इच्छिणारे जर्नल संपादक किंवा प्रतिकृती-प्रकल्प
  नेते असाल
- तुम्ही अशा विश्लेषणावर काम करत असाल जिथे पोस्ट-हॉक बदलांचे
  अर्थपूर्ण क्षेत्रीय परिणाम आहेत (मानसशास्त्र, वर्तणूक
  अर्थशास्त्र, वैद्यकीय संशोधन)

## ऐतिहासिक मृत्युदर / हानी पूर्वनिदर्शन

या किटची मृत्यु-संबंधितता अप्रत्यक्ष आहे:

- **स्टेम सेल / कर्करोग संशोधन नॉन-पुनरुत्पादकता:** अनेक पेपर्सने
  चुकीच्या-दिशा देणाऱ्या क्लिनिकल ट्रायल्सद्वारे रुग्ण हानीस
  नेले आहे. मूळ पाइपलाइनची सीलबंद पूर्व-नोंदणी तुलनेला अँकर
  केली असती.
- **Tamiflu परिणामकारकता वाद:** कच्चा-डेटा प्रवेश विवादित
  असल्यामुळे Cochrane पुनरावलोकनास वर्षे लागली. प्रकाशनाच्या
  वेळी सीलबंद डेटा-हॅशने डेटा-प्रवेश वाद लगेच सोडवण्यायोग्य
  केले असते.
- **सामान्य नॉन-पुनरुत्पादकता संकट:** नॉन-पुनरुत्पादक बायोमेडिकल
  संशोधनाची क्षेत्र-स्तरीय मृत्यु किंमत लक्षणीय आहे (Begley &
  Ellis 2012 ने >50% प्रीक्लिनिकल निष्कर्ष नॉन-पुनरुत्पादक
  असल्याचा अंदाज वर्तवला). क्रिप्टोग्राफिक अँकरिंग एक संरचनात्मक
  लीव्हर आहे.

## ही किट कशी फोर्क करावी

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# महत्त्वपूर्ण: कोणतेही परिणाम पाहण्यापूर्वी हे करा
shasum -a 256 raw_data_v0.tar.gz       # कॅनोनिकल डेटा हॅश मोजा
shasum -a 256 analysis.py              # तुमची पाइपलाइन हॅश करा
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # वास्तविक मूल्यांसह भरा
bash build/seal.sh                     # विश्लेषण चालवण्यापूर्वी सील करा
python3 build/verify.py                # पुष्टी करा

# आता विश्लेषण चालवा. आउटपुट हॅश करा आणि FACT 05 शी जुळते याची पुष्टी करा.
# जुळत नसल्यास, पाइपलाइन नॉन-निर्धारक आहे; निराकरण करा आणि पुन्हा सील करा.
```

## विद्यमान विद्वत्तापूर्ण पायाभूत सुविधांशी एकत्रीकरण

- **arXiv / bioRxiv / medRxiv**: अमूर्त किंवा पूरक सामग्रीमध्ये
  MYRIAM सील URL किंवा IPFS CID समाविष्ट करा
- **OSF**: पारंपारिक पूर्व-नोंदणी फॉर्मसह OSF प्रकल्पाला सील
  आर्टिफॅक्ट जोडा
- **AsPredicted**: पूर्व-नोंदणी दस्तऐवजात MYRIAM सील URL समाविष्ट करा
- **GitHub**: विश्लेषण पाइपलाइनचा कमिट हॅश हाच FACT 04 अँकर करतो;
  GitHub स्त्रोत प्रदान करते, MYRIAM बाइट्स अँकर करते
- **Zenodo / Figshare / OpenAIRE**: कच्चा डेटा जमा करा, नंतर
  जमाचा हॅश सील करा; सील जमा मेटाडेटापेक्षा मूकपणे बदलण्यासाठी
  फॉरेन्सिकदृष्ट्या अधिक कठीण आहे
- **जर्नल पूरक सामग्री**: सादरीकरणाच्या वेळी पूरक फाइल म्हणून
  MYRIAM सील आर्टिफॅक्ट समाविष्ट करा

## ही किट काय बदलत नाही

- IRB / नैतिकता-समिती मान्यता
- पीअर रिव्ह्यू
- प्रतिकृती अभ्यास (संगणकीय पुनरुत्पादकता ≠ प्रयोगात्मक
  पुनरुत्पादकता)
- संवेदनशील डेटासाठी डेटा-प्रवेश करार

## लेखक क्रम आणि श्रेय यावर एक टिप

सील सीलिंगच्या वेळी लेखक यादी अँकर करते. प्रकाशनानंतर उद्भवणारे
लेखकत्व वाद सीलबंद यादीच्या विरुद्ध मूल्यांकन केले जाऊ शकतात.
हे एक लहान उत्तरदायित्व लीव्हर आहे — मूकपणे जोडले किंवा काढून
टाकले जाणाऱ्या ECR-साठी महत्वाचे.
