# مجموعة أدوات قابلية إعادة الإنتاج العلمية

مثال myriam-kit جاهز للاستخدام لمجموعات البحث والمجلات والمؤسسات
المركزة على قابلية إعادة الإنتاج التي تريد نشر بيان قابلية إعادة الإنتاج
المثبت على Bitcoin والذي يكشف عن التلاعب، جنبًا إلى جنب مع ورقة بحثية أو
نسخة أولية.

## ما الذي تنتجه هذه المجموعة

منشور مختوم يحتوي، لدراسة واحدة، على:

1. تعريف الدراسة (عنوان الورقة، المؤلفون، المؤسسة، الممولون، COI)
2. **SHA-256 لأرشيف البيانات الخام** — مجموعة البيانات القانونية
3. **هاش خط أنابيب التحليل** — السكريبت + ملخص صورة Docker +
   التبعيات المثبتة
4. **هاش حزمة المخرجات المتوقعة** — ما يجب أن ينتجه المُعيد الأمين
5. بروتوكول إعادة الإنتاج + التزامات النشر بعد النتائج
6. الأصيل (الأصلاء) الموقعون

السيل مخصص لمرحلة **ما قبل التحليل**: البيانات مجمدة، وخط الأنابيب
نهائي، والمخرجات المتوقعة محسوبة، كل ذلك قبل رؤية أي نتائج لغرض كتابة
الورقة.

## ما الذي تدافع عنه هذه المجموعة

**أنماط الفشل التي يلتقطها السيل:**

- **p-hacking**: التغييرات اللاحقة على التحليل بعد رؤية النتائج قابلة
  للاكتشاف عبر diff بين FACT 04 المختوم وخط الأنابيب المنشور.
- **HARKing** (Hypothesizing After Results are Known): الفرضية المسجلة
  مسبقًا في FACT 02 ثابتة بايتيًا.
- **التلاعب بالبيانات بعد الورقة**: إعادة هاش أرشيف البيانات مقابل FACT 03
  تكشف عن أي تعديل.
- **انجراف خط الأنابيب**: إعادة هاش خط أنابيب التحليل مقابل FACT 04 تلتقط
  التغييرات الصامتة.
- **التلاعب بالتأليف**: إضافة المؤلفين أو إزالتهم بصمت بعد النشر تصبح
  قابلة للاكتشاف مقابل FACT 02 المختوم.

**أنماط الفشل غير المغطاة:**

- الاحتيال في جمع البيانات الخام (السيل يثبت البيانات في نقطة الختم،
  وليس في الجمع)
- عدم قابلية إعادة الإنتاج البيولوجية / التجريبية (السيل يثبت فقط قابلية
  إعادة الإنتاج الحاسوبية)
- مشاكل حجم العينة أو صحة المجتمع (هذه أسئلة تتعلق بتصميم الدراسة)
- حقيقة الادعاء الأساسي (السيل يثبت ما تم الادعاء به، وليس ما إذا كان
  العالم يطابق الادعاء)

## متى تستخدم هذه المجموعة

ضع في اعتبارك الختم إذا:

- كنت مجموعة بحثية تجري تحليلًا عالي المخاطر (تجربة سريرية، تقييم نظام
  AI، دراسة تكرار) وتريد سلامة جنائية تتجاوز ختم الوقت لـ OSF / GitHub
- كنت باحثًا في مرحلة مبكرة من حياته المهنية يريد حماية سمعة غير
  متماثلة: الختم قبل التحليل يعني أن النقد المستقبلي بـ "لقد قاموا
  بـ p-hack" له مزور عام
- كنت محرر مجلة أو قائد مشروع تكرار يريد تشجيع التسجيل المسبق على
  مستوى التشفير لمجموعة فرعية من الأوراق
- كنت تعمل على تحليل حيث التعديلات اللاحقة لها عواقب ميدانية ذات معنى
  (علم النفس، الاقتصاد السلوكي، البحث الطبي)

## السابقة التاريخية للوفيات / الأذى

أهمية هذه المجموعة من حيث الوفيات غير مباشرة:

- **عدم قابلية إعادة إنتاج أبحاث الخلايا الجذعية / السرطان:** أدت عدة
  أوراق إلى أذى للمرضى عبر التجارب السريرية المضللة. كان التسجيل المسبق
  المختوم لخط الأنابيب الأصلي سيثبت المقارنة.
- **جدل فعالية Tamiflu:** استغرقت مراجعة Cochrane سنوات جزئيًا لأن
  الوصول إلى البيانات الخام كان موضع نزاع. كانت هاشات البيانات المختومة
  عند النشر ستجعل نزاعات الوصول إلى البيانات قابلة للحل فورًا.
- **أزمة عدم قابلية إعادة الإنتاج العامة:** تكلفة الوفيات على مستوى
  الميدان للبحث الطبي الحيوي غير القابل لإعادة الإنتاج كبيرة (قدر
  Begley & Ellis 2012 أن >50% من نتائج ما قبل السريرية غير قابلة لإعادة
  الإنتاج). التثبيت التشفيري هو رافعة هيكلية واحدة.

## كيفية fork هذه المجموعة

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# CRITICAL: do this BEFORE seeing any results
shasum -a 256 raw_data_v0.tar.gz       # compute the canonical data hash
shasum -a 256 analysis.py              # hash your pipeline
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # populate with real values
bash build/seal.sh                     # seal BEFORE running the analysis
python3 build/verify.py                # confirm

# Now run the analysis. Hash the output and confirm matches FACT 05.
# If it doesn't match, the pipeline is non-deterministic; fix and re-seal.
```

## التكامل مع البنية التحتية الأكاديمية الحالية

- **arXiv / bioRxiv / medRxiv**: قم بتضمين URL ختم MYRIAM أو IPFS CID في
  الملخص أو المواد التكميلية
- **OSF**: قم بإرفاق قطعة الختم بمشروع OSF جنبًا إلى جنب مع نموذج التسجيل
  المسبق التقليدي
- **AsPredicted**: قم بتضمين URL ختم MYRIAM في وثيقة التسجيل المسبق
- **GitHub**: هاش commit لخط أنابيب التحليل هو ما يثبته FACT 04؛ GitHub
  يوفر المصدر، MYRIAM يثبت البايتات
- **Zenodo / Figshare / OpenAIRE**: قم بإيداع البيانات الخام، ثم اختم
  هاش الإيداع؛ من الأصعب جنائيًا استبدال الختم بصمت من بيانات تعريف
  الإيداع
- **المواد التكميلية للمجلة**: قم بتضمين قطعة ختم MYRIAM كملف تكميلي
  عند التقديم

## ما الذي لا تستبدله هذه المجموعة

- موافقة IRB / لجنة الأخلاقيات
- مراجعة الأقران
- دراسات التكرار (قابلية إعادة الإنتاج الحاسوبية ≠ قابلية إعادة الإنتاج
  التجريبية)
- اتفاقيات الوصول إلى البيانات للبيانات الحساسة

## ملاحظة حول ترتيب المؤلفين والاعتماد

السيل يثبت AUTHOR LIST في وقت الختم. يمكن تقييم نزاعات التأليف التي
تنشأ بعد النشر مقابل القائمة المختومة. هذه رافعة مساءلة صغيرة — مهمة
للباحثين في مرحلة مبكرة من حياتهم المهنية الذين تتم إضافتهم أو إزالتهم
بصمت.
