# 科学的再現性キット

論文またはプレプリントと並べて、Bitcoin に錨を下ろした改ざん明示
(tamper-evident)の再現性マニフェストを公開したい研究グループ、ジャーナル、
および再現性を重視する機関のための、ターンキー形式の myriam-kit の例です。

## このキットが生み出すもの

1 つの研究について、以下を含む封印された公開物:

1. 研究の同定情報(論文タイトル、著者、所属機関、資金提供者、利益相反)
2. **生データアーカイブの SHA-256** — 正規データセット
3. **解析パイプラインのハッシュ** — スクリプト + Docker イメージ
   ダイジェスト + ピン留めされた依存関係
4. **期待される出力バンドルのハッシュ** — 忠実な再現者が生成すべき
   もの
5. 再現プロトコル + 結果後の公開公約
6. 署名プリンシパル

この封印は **解析前** 段階を意図しています: データを固定し、パイプラインを
最終化し、期待される出力を計算し、これらすべてを、論文を書く目的で
結果が見られる前に行います。

## このキットが防御するもの

**封印が捕捉する失敗モード:**

- **p-hacking**: 結果を見た後の事後的な解析変更は、封印された
  FACT 04 と公開されたパイプラインの差分により検出可能。
- **HARKing**(Hypothesizing After Results are Known): FACT 02 の
  事前登録された仮説はバイト単位で安定。
- **論文発表後のデータ操作**: データアーカイブを FACT 03 に対して
  再ハッシュすると、いかなる変更も明らかになる。
- **パイプライン・ドリフト**: 解析パイプラインを FACT 04 に対して
  再ハッシュすることで、暗黙の変更を捕捉。
- **著者リスト操作**: 公開後に著者を密かに追加・削除することは、
  封印された FACT 02 に照らして検出可能となる。

**カバーされない失敗モード:**

- 生データ収集における不正(封印は封印時点でのデータを錨で固定する
  もので、収集時点ではない)
- 生物学的/経験的な非再現性(封印は計算的再現性のみを錨で固定する)
- 標本サイズや母集団妥当性の問題(これらは研究設計の問題)
- 根底にある主張の真実性(封印は何が主張されたかを錨で固定するもので
  あって、世界がその主張と一致するかどうかではない)

## このキットを使うべきとき

以下に該当する場合、封印を検討してください:

- 高ステークスの解析(臨床試験、AI システム評価、再現研究)を実行する
  研究グループであり、OSF / GitHub のタイムスタンプを超える法医学的
  完全性を求めている
- 非対称的な評判保護を求める若手研究者: 封印された事前解析は、
  将来「彼らは p-hacking をした」という批判に対する公開された
  反証手段となる
- ジャーナル編集者または再現プロジェクトのリーダーであり、論文の
  一部について暗号学グレードの事前登録を奨励したい
- 事後的な手直しが領域に意味ある結果をもたらす解析(心理学、行動経済学、
  医学研究)に取り組んでいる

## 歴史的な死亡/危害の前例

このキットの死亡率関連性は間接的です:

- **幹細胞 / がん研究の非再現性:** 複数の論文が誤った方向へ向かわせる
  臨床試験を通じて患者への危害を引き起こした。元のパイプラインの
  封印された事前登録があれば、比較の錨となっていただろう。
- **タミフルの有効性論争:** Cochrane レビューに何年もかかった理由の
  一部は、生データへのアクセスが争われたことにある。出版時の封印
  されたデータハッシュがあれば、データアクセス論争はただちに
  解決可能となっていただろう。
- **一般的な非再現性危機:** 再現不能な生物医学研究の分野レベルでの
  死亡コストは大きい(Begley & Ellis 2012 は前臨床所見の 50% 超が
  再現不能と推定)。暗号学的な錨付けはひとつの構造的レバーである。

## このキットのフォーク方法

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# 重要: いかなる結果も見る前に実行すること
shasum -a 256 raw_data_v0.tar.gz       # 正規データのハッシュを計算
shasum -a 256 analysis.py              # パイプラインをハッシュ化
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # 実際の値を入力
bash build/seal.sh                     # 解析実行前に封印
python3 build/verify.py                # 確認

# ここで解析を実行。出力をハッシュ化し FACT 05 と一致することを確認。
# 一致しない場合、パイプラインは非決定的; 修正して再封印。
```

## 既存の学術インフラとの統合

- **arXiv / bioRxiv / medRxiv**: MYRIAM 封印 URL または IPFS CID を
  抄録または補足資料に含める
- **OSF**: 従来の事前登録フォームと並べて、封印アーティファクトを
  OSF プロジェクトに添付
- **AsPredicted**: 事前登録文書に MYRIAM 封印 URL を含める
- **GitHub**: 解析パイプラインのコミットハッシュこそが FACT 04 が
  錨で固定するもの; GitHub はソースを提供し、MYRIAM はバイトを
  錨で固定する
- **Zenodo / Figshare / OpenAIRE**: 生データをデポジットし、その後
  デポジットのハッシュを封印する; 封印はデポジットのメタデータ
  よりも法医学的に密かな置き換えが困難
- **ジャーナルの補足資料**: 提出時に MYRIAM 封印アーティファクトを
  補足ファイルとして含める

## このキットの代わりにはならないもの

- IRB / 倫理委員会の承認
- ピアレビュー
- 再現研究(計算的再現性 ≠ 経験的再現性)
- 機微データのデータアクセス契約

## 著者順序とクレジットに関する注記

封印は封印時点での著者リストを錨で固定します。公開後に生じる著者
資格に関する論争は、封印されたリストに照らして評価できます。これは
小さな説明責任のレバーであり、密かに追加または削除されることがある
若手研究者(ECR)にとって重要です。
