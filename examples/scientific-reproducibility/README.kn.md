# ವೈಜ್ಞಾನಿಕ ಪುನರುತ್ಪಾದನೀಯತೆ ಕಿಟ್

ಸಂಶೋಧನಾ ಗುಂಪುಗಳು, ನಿಯತಕಾಲಿಕಗಳು, ಮತ್ತು ಪುನರುತ್ಪಾದನೀಯತೆ-ಕೇಂದ್ರಿತ
ಸಂಸ್ಥೆಗಳಿಗಾಗಿ ಟರ್ನ್‌ಕೀ myriam-kit ಉದಾಹರಣೆ, ಒಂದು ಪತ್ರಿಕೆ ಅಥವಾ
ಪೂರ್ವಮುದ್ರಣದ ಜೊತೆಗೆ Bitcoin-ಆಂಕರ್ ಆಗಿರುವ, ಟ್ಯಾಂಪರ್-ಎವಿಡೆಂಟ್
ಪುನರುತ್ಪಾದನೀಯತೆ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಅನ್ನು ಪ್ರಕಟಿಸಲು ಬಯಸುತ್ತಾರೆ.

## ಈ ಕಿಟ್ ಏನನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ

ಒಂದು ಅಧ್ಯಯನಕ್ಕಾಗಿ ಸೀಲ್ ಮಾಡಿದ ಪ್ರಕಟಣೆ, ಒಳಗೊಂಡಿರುತ್ತದೆ:

1. ಅಧ್ಯಯನದ ಗುರುತಿಸುವಿಕೆ (ಪತ್ರಿಕೆ ಶೀರ್ಷಿಕೆ, ಲೇಖಕರು, ಸಂಸ್ಥೆ,
   ಧನಸಹಾಯಕರು, COI)
2. **ಕಚ್ಚಾ ಡೇಟಾ ಆರ್ಕೈವ್ SHA-256** — ಕ್ಯಾನೋನಿಕಲ್ ಡೇಟಾಸೆಟ್
3. **ವಿಶ್ಲೇಷಣಾ ಪೈಪ್‌ಲೈನ್ ಹ್ಯಾಶ್** — ಸ್ಕ್ರಿಪ್ಟ್ + Docker ಇಮೇಜ್
   ಡೈಜೆಸ್ಟ್ + ಪಿನ್ ಮಾಡಿದ ಅವಲಂಬನೆಗಳು
4. **ನಿರೀಕ್ಷಿತ ಔಟ್‌ಪುಟ್ ಬಂಡಲ್ ಹ್ಯಾಶ್** — ನಿಷ್ಠಾವಂತ ಪುನರುತ್ಪಾದಕನು
   ಉತ್ಪಾದಿಸಬೇಕಾದದ್ದು
5. ಪುನರುತ್ಪಾದನಾ ಪ್ರೋಟೋಕಾಲ್ + ಫಲಿತಾಂಶ-ನಂತರದ ಪ್ರಕಟಣೆ ಬದ್ಧತೆಗಳು
6. ಸಹಿ ಮಾಡುವ ಪ್ರಿನ್ಸಿಪಾಲ್(ಗಳು)

ಸೀಲ್ **ಪೂರ್ವ-ವಿಶ್ಲೇಷಣಾ** ಹಂತಕ್ಕಾಗಿ ಉದ್ದೇಶಿಸಲಾಗಿದೆ: ಡೇಟಾ
ಫ್ರೀಜ್ ಮಾಡಲಾಗಿದೆ, ಪೈಪ್‌ಲೈನ್ ಅಂತಿಮಗೊಳಿಸಲಾಗಿದೆ, ನಿರೀಕ್ಷಿತ
ಔಟ್‌ಪುಟ್ ಲೆಕ್ಕ ಹಾಕಲಾಗಿದೆ, ಎಲ್ಲವೂ ಪತ್ರಿಕೆಯನ್ನು ಬರೆಯುವ ಉದ್ದೇಶಕ್ಕಾಗಿ
ಯಾವುದೇ ಫಲಿತಾಂಶಗಳನ್ನು ನೋಡುವ ಮೊದಲು.

## ಈ ಕಿಟ್ ಯಾವುದರ ವಿರುದ್ಧ ರಕ್ಷಿಸುತ್ತದೆ

**ಸೀಲ್ ಹಿಡಿಯುವ ವೈಫಲ್ಯ ಮೋಡ್‌ಗಳು:**

- **p-hacking**: ಫಲಿತಾಂಶಗಳನ್ನು ನೋಡಿದ ನಂತರ ವಿಶ್ಲೇಷಣೆಗೆ
  ಪೋಸ್ಟ್-ಹಾಕ್ ಬದಲಾವಣೆಗಳನ್ನು ಸೀಲ್ ಮಾಡಿದ FACT 04 ಮತ್ತು ಪ್ರಕಟಿತ
  ಪೈಪ್‌ಲೈನ್ ನಡುವಿನ ಡಿಫ್ ಮೂಲಕ ಪತ್ತೆಹಚ್ಚಬಹುದು.
- **HARKing** (ಫಲಿತಾಂಶಗಳು ತಿಳಿದ ನಂತರ ಊಹಿಸುವಿಕೆ): FACT 02 ರಲ್ಲಿನ
  ಪೂರ್ವ-ನೋಂದಾಯಿಸಿದ ಊಹೆ ಬೈಟ್-ಸ್ಥಿರವಾಗಿದೆ.
- **ಪತ್ರಿಕೆ-ನಂತರದ ಡೇಟಾ ಮ್ಯಾನಿಪ್ಯುಲೇಶನ್**: FACT 03 ರ ವಿರುದ್ಧ
  ಡೇಟಾ ಆರ್ಕೈವ್ ಅನ್ನು ಮರು-ಹ್ಯಾಶ್ ಮಾಡುವುದು ಯಾವುದೇ ಮಾರ್ಪಾಡನ್ನು
  ಬಹಿರಂಗಪಡಿಸುತ್ತದೆ.
- **ಪೈಪ್‌ಲೈನ್ ಡ್ರಿಫ್ಟ್**: FACT 04 ರ ವಿರುದ್ಧ ವಿಶ್ಲೇಷಣಾ ಪೈಪ್‌ಲೈನ್
  ಅನ್ನು ಮರು-ಹ್ಯಾಶ್ ಮಾಡುವುದು ಮೌನ ಬದಲಾವಣೆಗಳನ್ನು ಹಿಡಿಯುತ್ತದೆ.
- **ಲೇಖಕತ್ವ ಗೇಮಿಂಗ್**: ಪ್ರಕಟಣೆಯ ನಂತರ ಲೇಖಕರನ್ನು ಮೌನವಾಗಿ
  ಸೇರಿಸುವುದು ಅಥವಾ ತೆಗೆದುಹಾಕುವುದು ಸೀಲ್ ಮಾಡಿದ FACT 02 ರ ವಿರುದ್ಧ
  ಪತ್ತೆಹಚ್ಚಬಹುದು.

**ಒಳಗೊಳ್ಳದ ವೈಫಲ್ಯ ಮೋಡ್‌ಗಳು:**

- ಕಚ್ಚಾ-ಡೇಟಾ ಸಂಗ್ರಹಣೆಯಲ್ಲಿ ವಂಚನೆ (ಸೀಲ್ ಡೇಟಾವನ್ನು ಸೀಲಿಂಗ್
  ಬಿಂದುವಿನಲ್ಲಿ ಆಂಕರ್ ಮಾಡುತ್ತದೆ, ಸಂಗ್ರಹಣೆಯಲ್ಲಿ ಅಲ್ಲ)
- ಜೈವಿಕ / ಅನುಭವಜನ್ಯ ಪುನರುತ್ಪಾದನಾ-ರಹಿತತೆ (ಸೀಲ್ ಕೇವಲ
  ಕಂಪ್ಯೂಟೇಶನಲ್ ಪುನರುತ್ಪಾದನೀಯತೆಯನ್ನು ಆಂಕರ್ ಮಾಡುತ್ತದೆ)
- ಮಾದರಿ-ಗಾತ್ರ ಅಥವಾ ಜನಸಂಖ್ಯೆ-ಮಾನ್ಯತೆ ಸಮಸ್ಯೆಗಳು (ಇವು ಅಧ್ಯಯನ-ವಿನ್ಯಾಸ
  ಪ್ರಶ್ನೆಗಳು)
- ಆಧಾರವಾಗಿರುವ ಹಕ್ಕಿನ ಸತ್ಯ (ಸೀಲ್ ಹಕ್ಕು ಮಾಡಲಾದದ್ದನ್ನು ಆಂಕರ್
  ಮಾಡುತ್ತದೆ, ಜಗತ್ತು ಹಕ್ಕಿಗೆ ಹೊಂದಿಕೆಯಾಗುತ್ತದೆಯೇ ಎಂಬುದನ್ನು ಅಲ್ಲ)

## ಈ ಕಿಟ್ ಅನ್ನು ಯಾವಾಗ ಬಳಸಬೇಕು

ಸೀಲಿಂಗ್ ಅನ್ನು ಪರಿಗಣಿಸಿ:

- ನೀವು ಹೆಚ್ಚಿನ-ಪಾಲಿನ ವಿಶ್ಲೇಷಣೆಯನ್ನು ನಡೆಸುತ್ತಿರುವ ಸಂಶೋಧನಾ
  ಗುಂಪಾಗಿದ್ದರೆ (ಕ್ಲಿನಿಕಲ್ ಟ್ರಯಲ್, AI ಸಿಸ್ಟಂ ಮೌಲ್ಯಮಾಪನ, ಪ್ರತಿಕೃತಿ
  ಅಧ್ಯಯನ) ಮತ್ತು OSF / GitHub ಸಮಯ-ಮುದ್ರೆಗಳಿಗಿಂತ ಆಚೆ ಫೋರೆನ್ಸಿಕ್
  ಸಮಗ್ರತೆಯನ್ನು ಬಯಸಿದರೆ
- ನೀವು ಆರಂಭಿಕ-ವೃತ್ತಿ ಸಂಶೋಧಕರಾಗಿದ್ದರೆ ಮತ್ತು ಅಸಮ್ಮಿತ ಖ್ಯಾತಿ-ರಕ್ಷಣೆಯನ್ನು
  ಬಯಸಿದರೆ: ಸೀಲ್ ಮಾಡಿದ ಪೂರ್ವ-ವಿಶ್ಲೇಷಣೆ ಎಂದರೆ ಭವಿಷ್ಯದ "ಅವರು
  p-hack ಮಾಡಿದರು" ಎಂಬ ಟೀಕೆಗೆ ಸಾರ್ವಜನಿಕ ಫಾಲ್ಸಿಫೈಯರ್ ಇರುತ್ತದೆ
- ನೀವು ನಿಯತಕಾಲಿಕ ಸಂಪಾದಕ ಅಥವಾ ಪ್ರತಿಕೃತಿ-ಯೋಜನೆಯ ನಾಯಕರಾಗಿದ್ದರೆ
  ಮತ್ತು ಪತ್ರಿಕೆಗಳ ಉಪಸಮುಚ್ಚಯಕ್ಕೆ ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್-ಗ್ರೇಡ್
  ಪೂರ್ವ-ನೋಂದಣಿಯನ್ನು ಪ್ರೋತ್ಸಾಹಿಸಲು ಬಯಸಿದರೆ
- ನೀವು ಪೋಸ್ಟ್-ಹಾಕ್ ಬದಲಾವಣೆಗಳು ಅರ್ಥಪೂರ್ಣ ಕ್ಷೇತ್ರ ಪರಿಣಾಮಗಳನ್ನು
  ಹೊಂದಿರುವ ವಿಶ್ಲೇಷಣೆಯ ಮೇಲೆ ಕೆಲಸ ಮಾಡುತ್ತಿದ್ದರೆ (ಮನೋವಿಜ್ಞಾನ,
  ವರ್ತನೆಯ ಅರ್ಥಶಾಸ್ತ್ರ, ವೈದ್ಯಕೀಯ ಸಂಶೋಧನೆ)

## ಐತಿಹಾಸಿಕ ಮರಣ / ಹಾನಿ ಪೂರ್ವನಿದರ್ಶನ

ಈ ಕಿಟ್‌ನ ಮರಣ-ಸಂಬಂಧಿತತೆ ಪರೋಕ್ಷವಾಗಿದೆ:

- **ಸ್ಟೆಮ್ ಸೆಲ್ / ಕ್ಯಾನ್ಸರ್ ಸಂಶೋಧನೆ ಪುನರುತ್ಪಾದನಾ-ರಹಿತತೆ:**
  ಹಲವಾರು ಪತ್ರಿಕೆಗಳು ತಪ್ಪು-ನಿರ್ದೇಶಿತ ಕ್ಲಿನಿಕಲ್ ಟ್ರಯಲ್‌ಗಳ ಮೂಲಕ
  ರೋಗಿಗಳ ಹಾನಿಗೆ ಕಾರಣವಾಗಿವೆ. ಮೂಲ ಪೈಪ್‌ಲೈನ್‌ನ ಸೀಲ್ ಮಾಡಿದ
  ಪೂರ್ವ-ನೋಂದಣಿಯು ಹೋಲಿಕೆಯನ್ನು ಆಂಕರ್ ಮಾಡುತ್ತಿತ್ತು.
- **Tamiflu ಪರಿಣಾಮಕಾರಿತ್ವ ವಿವಾದ:** ಕಚ್ಚಾ-ಡೇಟಾ ಪ್ರವೇಶವು
  ವಿವಾದಾಸ್ಪದವಾಗಿದ್ದರಿಂದ Cochrane ವಿಮರ್ಶೆಗೆ ವರ್ಷಗಳು ಬೇಕಾದವು.
  ಪ್ರಕಟಣೆಯ ಸಮಯದಲ್ಲಿ ಸೀಲ್ ಮಾಡಿದ ಡೇಟಾ-ಹ್ಯಾಶ್‌ಗಳು ಡೇಟಾ-ಪ್ರವೇಶದ
  ವಿವಾದಗಳನ್ನು ತಕ್ಷಣವೇ ಪರಿಹರಿಸಬಲ್ಲವಾಗಿಸುತ್ತಿತ್ತು.
- **ಸಾಮಾನ್ಯ ಪುನರುತ್ಪಾದನಾ-ರಹಿತತೆ ಸಂಕಟ:** ಪುನರುತ್ಪಾದನಾ-ರಹಿತ
  ಬಯೋಮೆಡಿಕಲ್ ಸಂಶೋಧನೆಯ ಕ್ಷೇತ್ರ-ಮಟ್ಟದ ಮರಣ ವೆಚ್ಚ ಗಮನಾರ್ಹವಾಗಿದೆ
  (Begley & Ellis 2012 >50% ಪ್ರೀಕ್ಲಿನಿಕಲ್ ಸಂಶೋಧನೆಗಳು
  ಪುನರುತ್ಪಾದನಾ-ರಹಿತ ಎಂದು ಅಂದಾಜಿಸಿದರು). ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್
  ಆಂಕರಿಂಗ್ ಒಂದು ರಚನಾತ್ಮಕ ಲಿವರ್ ಆಗಿದೆ.

## ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ವಿದ್ವತ್ ಮೂಲಸೌಕರ್ಯದೊಂದಿಗೆ ಏಕೀಕರಣ

- **arXiv / bioRxiv / medRxiv**: ಸಾರಾಂಶ ಅಥವಾ ಪೂರಕ ಸಾಮಗ್ರಿಗಳಲ್ಲಿ
  MYRIAM ಸೀಲ್ URL ಅಥವಾ IPFS CID ಸೇರಿಸಿ
- **OSF**: ಸಾಂಪ್ರದಾಯಿಕ ಪೂರ್ವ-ನೋಂದಣಿ ಫಾರ್ಮ್ ಜೊತೆಗೆ OSF
  ಯೋಜನೆಗೆ ಸೀಲ್ ಆರ್ಟಿಫ್ಯಾಕ್ಟ್ ಅನ್ನು ಲಗತ್ತಿಸಿ
- **AsPredicted**: ಪೂರ್ವ-ನೋಂದಣಿ ದಾಖಲೆಯಲ್ಲಿ MYRIAM ಸೀಲ್ URL ಸೇರಿಸಿ
- **GitHub**: ವಿಶ್ಲೇಷಣಾ ಪೈಪ್‌ಲೈನ್‌ನ ಕಮಿಟ್ ಹ್ಯಾಶ್ ಎಂಬುದು FACT 04
  ಆಂಕರ್ ಮಾಡುತ್ತದೆ; GitHub ಮೂಲವನ್ನು ಒದಗಿಸುತ್ತದೆ, MYRIAM ಬೈಟ್‌ಗಳನ್ನು
  ಆಂಕರ್ ಮಾಡುತ್ತದೆ
- **Zenodo / Figshare / OpenAIRE**: ಕಚ್ಚಾ ಡೇಟಾವನ್ನು ಠೇವಣಿ ಇಡಿ,
  ನಂತರ ಠೇವಣಿಯ ಹ್ಯಾಶ್ ಅನ್ನು ಸೀಲ್ ಮಾಡಿ; ಸೀಲ್ ಠೇವಣಿ ಮೆಟಾಡೇಟಾಗಿಂತ
  ಮೌನವಾಗಿ ಬದಲಾಯಿಸಲು ಫೋರೆನ್ಸಿಕ್ ಆಗಿ ಕಷ್ಟಕರವಾಗಿದೆ
- **ನಿಯತಕಾಲಿಕ ಪೂರಕ ಸಾಮಗ್ರಿಗಳು**: ಸಲ್ಲಿಕೆಯ ಸಮಯದಲ್ಲಿ ಪೂರಕ ಫೈಲ್‌ನಂತೆ
  MYRIAM ಸೀಲ್ ಆರ್ಟಿಫ್ಯಾಕ್ಟ್ ಅನ್ನು ಸೇರಿಸಿ
