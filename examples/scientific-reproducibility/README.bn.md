# বৈজ্ঞানিক পুনরুৎপাদনযোগ্যতা কিট

গবেষণা গোষ্ঠী, জার্নাল, এবং পুনরুৎপাদনযোগ্যতা-কেন্দ্রিক প্রতিষ্ঠানের
জন্য একটি টার্নকি myriam-kit উদাহরণ, যারা একটি গবেষণাপত্র বা প্রিপ্রিন্টের
সাথে Bitcoin-নোঙর করা, কারচুপি-স্পষ্ট পুনরুৎপাদনযোগ্যতা মেনিফেস্ট প্রকাশ
করতে চান।

## এই কিট কী উৎপাদন করে

একটি গবেষণার জন্য একটি সিলযুক্ত প্রকাশনা যেখানে রয়েছে:

1. গবেষণা সনাক্তকরণ (গবেষণাপত্রের শিরোনাম, লেখক, প্রতিষ্ঠান, অর্থদাতা, COI)
2. **কাঁচা ডেটা সংরক্ষণাগারের SHA-256** — প্রামাণিক ডেটাসেট
3. **বিশ্লেষণ পাইপলাইন হ্যাশ** — স্ক্রিপ্ট + Docker ইমেজ ডাইজেস্ট +
   পিন করা নির্ভরতা
4. **প্রত্যাশিত আউটপুট বান্ডেল হ্যাশ** — একজন বিশ্বস্ত পুনরুৎপাদনকারীর
   যা উৎপাদন করা উচিত
5. পুনরুৎপাদন প্রোটোকল + ফলাফল-পরবর্তী প্রকাশনার প্রতিশ্রুতি
6. স্বাক্ষরকারী মুখ্য কর্তা

সিলটি **পূর্ব-বিশ্লেষণ** পর্যায়ের জন্য উদ্দিষ্ট: ডেটা হিমায়িত,
পাইপলাইন চূড়ান্ত, প্রত্যাশিত আউটপুট গণনা করা, গবেষণাপত্র লেখার উদ্দেশ্যে
কোনও ফলাফল দেখার আগে এই সবকিছু।

## এই কিট কী থেকে রক্ষা করে

**ব্যর্থতার যে ধরনগুলি সিলটি ধরে:**

- **p-hacking**: ফলাফল দেখার পরে বিশ্লেষণে post-hoc পরিবর্তন সিলযুক্ত
  FACT 04 এবং প্রকাশিত পাইপলাইনের মধ্যে diff-এর মাধ্যমে শনাক্তযোগ্য।
- **HARKing** (Hypothesizing After Results are Known): FACT 02-এ পূর্ব-
  নিবন্ধিত হাইপোথিসিস বাইট-স্থিতিশীল।
- **গবেষণাপত্র-পরবর্তী ডেটা কারচুপি**: FACT 03-এর বিরুদ্ধে ডেটা
  সংরক্ষণাগার পুনঃহ্যাশ করা যেকোনো পরিবর্তন প্রকাশ করে।
- **পাইপলাইন ড্রিফট**: FACT 04-এর বিরুদ্ধে বিশ্লেষণ পাইপলাইন পুনঃহ্যাশ
  করা নীরব পরিবর্তন ধরে।
- **লেখকত্ব কারচুপি**: প্রকাশনার পরে নীরবে লেখক যোগ করা বা সরানো
  সিলযুক্ত FACT 02-এর বিরুদ্ধে শনাক্তযোগ্য হয়ে ওঠে।

**যে ব্যর্থতার ধরন অন্তর্ভুক্ত নয়:**

- কাঁচা-ডেটা সংগ্রহে জালিয়াতি (সিল সিলিং-এর সময়ে ডেটাকে নোঙর করে,
  সংগ্রহে নয়)
- জৈবিক / অভিজ্ঞতাজনিত অ-পুনরুৎপাদনযোগ্যতা (সিল শুধুমাত্র
  কম্পিউটেশনাল পুনরুৎপাদনযোগ্যতা নোঙর করে)
- নমুনা-আকার বা জনসংখ্যা-বৈধতা সমস্যা (এগুলি গবেষণা-নকশা প্রশ্ন)
- অন্তর্নিহিত দাবির সত্য (সিল নোঙর করে কী দাবি করা হয়েছিল, পৃথিবী
  দাবির সাথে মেলে কিনা তা নয়)

## কখন এই কিট ব্যবহার করবেন

সিল করার কথা বিবেচনা করুন যদি:

- আপনি একটি গবেষণা গোষ্ঠী যা একটি উচ্চ-ঝুঁকির বিশ্লেষণ (ক্লিনিক্যাল
  ট্রায়াল, AI সিস্টেম মূল্যায়ন, প্রতিলিপি গবেষণা) চালাচ্ছেন এবং OSF /
  GitHub সময়-মুদ্রাঙ্কনের বাইরে ফরেনসিক অখণ্ডতা চান
- আপনি একজন প্রাথমিক-ক্যারিয়ার গবেষক যিনি অসামঞ্জস্যপূর্ণ
  খ্যাতিগত সুরক্ষা চান: সিলযুক্ত পূর্ব-বিশ্লেষণ মানে ভবিষ্যতের
  "তারা p-hack করেছিল" সমালোচনার একটি পাবলিক ফলসিফায়ার আছে
- আপনি একজন জার্নাল সম্পাদক বা প্রতিলিপি-প্রকল্পের প্রধান যিনি
  গবেষণাপত্রের একটি উপসেটের জন্য ক্রিপ্টোগ্রাফিক-গ্রেড পূর্ব-নিবন্ধন
  উৎসাহিত করতে চান
- আপনি এমন একটি বিশ্লেষণে কাজ করছেন যেখানে post-hoc টুইকগুলির অর্থপূর্ণ
  ক্ষেত্রের পরিণতি রয়েছে (মনোবিজ্ঞান, আচরণগত অর্থনীতি, চিকিৎসা গবেষণা)

## ঐতিহাসিক মৃত্যুহার / ক্ষতির পূর্বনজির

এই কিটের মৃত্যু-প্রাসঙ্গিকতা পরোক্ষ:

- **স্টেম সেল / ক্যান্সার গবেষণা অ-পুনরুৎপাদনযোগ্যতা:** বেশ কয়েকটি
  গবেষণাপত্র ভুল-নির্দেশিত ক্লিনিক্যাল ট্রায়ালের মাধ্যমে রোগীর ক্ষতির
  দিকে পরিচালিত করেছে। মূল পাইপলাইনের সিলযুক্ত পূর্ব-নিবন্ধন তুলনাকে
  নোঙর করত।
- **Tamiflu-এর কার্যকারিতা বিতর্ক:** Cochrane পর্যালোচনায় আংশিকভাবে
  বছর লেগেছিল কারণ কাঁচা-ডেটা অ্যাক্সেস বিতর্কিত ছিল। প্রকাশনায়
  সিলযুক্ত ডেটা-হ্যাশ ডেটা-অ্যাক্সেস বিরোধগুলিকে তাৎক্ষণিকভাবে
  সমাধানযোগ্য করে তুলত।
- **সাধারণ অ-পুনরুৎপাদনযোগ্যতা সংকট:** অ-পুনরুৎপাদনযোগ্য বায়োমেডিকেল
  গবেষণার ক্ষেত্র-স্তরের মৃত্যুর খরচ উল্লেখযোগ্য (Begley & Ellis 2012
  অনুমান করেছিল যে preclinical ফলাফলের >50% অ-পুনরুৎপাদনযোগ্য)।
  ক্রিপ্টোগ্রাফিক নোঙর একটি কাঠামোগত লিভার।

## এই কিট কীভাবে fork করবেন

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# CRITICAL: do this BEFORE seeing any results
shasum -a 256 raw_data_v0.tar.gz       # compute the canonical data hash
shasum -a 256 analysis.py              # hash your pipeline
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # populate with real values
bash build/seal.sh                     # seal BEFORE running the analysis
python3 build/verify.py                # confirm

# Now run the analysis. Hash the output and confirm matches FACT 05.
# If it doesn't match, the pipeline is non-deterministic; fix and re-seal.
```

## বিদ্যমান পাণ্ডিত্যপূর্ণ অবকাঠামোর সাথে একীকরণ

- **arXiv / bioRxiv / medRxiv**: সারাংশ বা সম্পূরক উপাদানে MYRIAM সিল URL
  বা IPFS CID অন্তর্ভুক্ত করুন
- **OSF**: প্রচলিত পূর্ব-নিবন্ধন ফর্মের পাশাপাশি OSF প্রকল্পে সিল
  আর্টিফ্যাক্ট সংযুক্ত করুন
- **AsPredicted**: পূর্ব-নিবন্ধন নথিতে MYRIAM সিল URL অন্তর্ভুক্ত করুন
- **GitHub**: বিশ্লেষণ পাইপলাইনের commit হ্যাশ যা FACT 04 নোঙর করে;
  GitHub উৎস প্রদান করে, MYRIAM বাইটগুলি নোঙর করে
- **Zenodo / Figshare / OpenAIRE**: কাঁচা ডেটা জমা দিন, তারপর জমার হ্যাশ
  সিল করুন; সিলটি জমা মেটাডেটার চেয়ে নীরবে প্রতিস্থাপন করা ফরেনসিকভাবে
  কঠিন
- **জার্নাল সম্পূরক উপাদান**: জমা দেওয়ার সময় একটি সম্পূরক ফাইল হিসাবে
  MYRIAM সিল আর্টিফ্যাক্ট অন্তর্ভুক্ত করুন
