# அறிவியல் மறு-உருவாக்க கிட் (Scientific Reproducibility Kit)

*தமிழ் மொழிபெயர்ப்பு — அசல் ஆங்கிலப் பதிப்பிற்கு, பார்க்கவும் [README.md](README.md)*

ஒரு கட்டுரை அல்லது preprint உடன், Bitcoin-நங்கூரப்படுத்தப்பட்ட, சேதம்-வெளிப்படையான மறு-உருவாக்க manifest ஐ வெளியிட விரும்பும் ஆராய்ச்சி குழுக்கள், இதழ்கள், மற்றும் மறு-உருவாக்கம்-கவனம் கொண்ட நிறுவனங்களுக்கான உடனடி பயன்பாட்டுக்கு தயாரான myriam-kit எடுத்துக்காட்டு.

## இந்த கிட் உருவாக்குவது

ஒரு ஆய்வுக்கான, முத்திரையிட்ட வெளியீடு பின்வருபவற்றைக் கொண்டுள்ளது:

1. ஆய்வு அடையாளம் (கட்டுரை தலைப்பு, ஆசிரியர்கள், நிறுவனம், நிதியளிப்பவர்கள், COI)
2. **மூல தரவு காப்பகம் SHA-256** — நியமக் தரவுத்தொகுப்பு
3. **பகுப்பாய்வு pipeline hash** — script + Docker image digest + pinned dependencies
4. **எதிர்பார்க்கப்பட்ட output bundle hash** — ஒரு உண்மையான மறு-உருவாக்குபவர் உற்பத்தி செய்ய வேண்டியது
5. மறு-உருவாக்க நெறிமுறை + முடிவு-பிற்கால வெளியீட்டு உறுதிமொழிகள்
6. கையெழுத்திடும் principal(s)

முத்திரை **முன்-பகுப்பாய்வு** நிலைக்காக நோக்கப்பட்டுள்ளது: தரவு உறையவைக்கப்பட்டது, pipeline இறுதி செய்யப்பட்டது, எதிர்பார்க்கப்பட்ட output கணக்கிடப்பட்டது, கட்டுரை எழுதுவதற்காக எந்த முடிவுகளும் காணப்படுவதற்கு முன் அனைத்தும்.

## இந்த கிட் எதற்கு எதிராக பாதுகாக்கிறது

**முத்திரை பிடிக்கும் தோல்வி முறைகள்:**

- **p-hacking**: முடிவுகளைப் பார்த்த பிறகு பகுப்பாய்வுக்கு post-hoc மாற்றங்கள், முத்திரையிட்ட FACT 04 மற்றும் வெளியிடப்பட்ட pipeline க்கு இடையே உள்ள diff வழியாக கண்டறியக்கூடியவை.
- **HARKing** (Hypothesizing After Results are Known): FACT 02 இல் முன்-பதிவு செய்யப்பட்ட கருதுகோள் byte-stable.
- **கட்டுரை-பிற்கால தரவு கையாளுதல்**: FACT 03 க்கு எதிராக தரவு காப்பகத்தை மீண்டும்-hashing எந்த மாற்றத்தையும் வெளிப்படுத்துகிறது.
- **Pipeline drift**: FACT 04 க்கு எதிராக பகுப்பாய்வு pipeline ஐ மீண்டும்-hashing அமைதியான மாற்றங்களை பிடிக்கிறது.
- **ஆசிரியர் gaming**: வெளியீட்டிற்குப் பிறகு அமைதியாக ஆசிரியர்களை சேர்த்தல் அல்லது நீக்கம் முத்திரையிட்ட FACT 02 க்கு எதிராக கண்டறியக்கூடியதாகிறது.

**மூடப்படாத தோல்வி முறைகள்:**

- மூல-தரவு சேகரிப்பில் மோசடி (முத்திரை, முத்திரையிடும் புள்ளியில் தரவை நங்கூரமிடுகிறது, சேகரிப்பு புள்ளியில் அல்ல)
- உயிரியல் / அனுபவ மறு-உருவாக்கம்-இன்மை (முத்திரை கணினி மறு-உருவாக்கத்தை மட்டுமே நங்கூரமிடுகிறது)
- மாதிரி-அளவு அல்லது மக்கள்தொகை-செல்லுபடியாகுதல் சிக்கல்கள் (இவை ஆய்வு-வடிவமைப்பு கேள்விகள்)
- அடிப்படை கூற்றின் உண்மை (முத்திரை, என்ன கூறப்பட்டது என்பதை நங்கூரமிடுகிறது, உலகம் கூற்றுடன் பொருந்துகிறதா என்பதை அல்ல)

## இந்த கிட்டை எப்போது பயன்படுத்த வேண்டும்

பின்வரும் சூழ்நிலையில் முத்திரையிடுவதைக் கருத்தில் கொள்ளுங்கள்:

- நீங்கள் ஒரு உயர்-ரிஸ்க் பகுப்பாய்வை (clinical trial, AI அமைப்பு மதிப்பீடு, replication study) நடத்தும் ஒரு ஆராய்ச்சி குழு மற்றும் OSF / GitHub time-stamps ஐ விட அப்பால் forensic நேர்மையை விரும்புகிறீர்கள்
- நீங்கள் ஒரு early-career ஆராய்ச்சியாளர், அசமசீர் கண்ணியம்-பாதுகாப்பை விரும்புகிறீர்கள்: முத்திரையிட்ட முன்-பகுப்பாய்வு என்றால் எதிர்காலத்தில் "அவர்கள் p-hacked" விமர்சனத்திற்கு ஒரு பொது falsifier உள்ளது
- நீங்கள் ஒரு இதழ் ஆசிரியர் அல்லது replication-project தலைவர், கட்டுரைகளின் ஒரு துணைதொகுதிக்கு cryptographic-தர முன்-பதிவை ஊக்குவிக்க விரும்புகிறீர்கள்
- நீங்கள் post-hoc tweaks அர்த்தமுள்ள துறை விளைவுகளைக் கொண்டுள்ள ஒரு பகுப்பாய்வில் வேலை செய்கிறீர்கள் (psych, behavioral econ, medical research)

## வரலாற்று இறப்பு / பாதிப்பு முன்னுதாரணம்

இந்த கிட்டின் இறப்பு-தொடர்பு மறைமுகமானது:

- **Stem cell / cancer ஆராய்ச்சி மறு-உருவாக்கம்-இன்மை:** பல கட்டுரைகள் தவறான வழிநடத்தப்பட்ட clinical trials வழியாக நோயாளி பாதிப்புக்கு வழிவகுத்துள்ளன. அசல் pipeline இன் முத்திரையிட்ட முன்-பதிவு ஒப்பீட்டை நங்கூரமிட்டிருக்கும்.
- **Tamiflu இன் செயல்திறன் சர்ச்சை:** மூல-தரவு அணுகல் சர்ச்சைக்குரியதாக இருந்ததால் Cochrane review பல வருடங்கள் ஆனது. வெளியீட்டில் முத்திரையிட்ட தரவு-hashes தரவு-அணுகல் சர்ச்சைகளை உடனடியாக தீர்க்கக்கூடியதாக ஆக்கியிருக்கும்.
- **பொது மறு-உருவாக்கம்-இன்மை நெருக்கடி:** மறு-உருவாக்கம்-இல்லாத biomedical ஆராய்ச்சியின் துறை-நிலை இறப்பு செலவு குறிப்பிடத்தக்கது (Begley & Ellis 2012 >50% preclinical கண்டுபிடிப்புகள் மறு-உருவாக்கம்-இல்லாதவை என்று மதிப்பிட்டது). Cryptographic நங்கூரமிடுதல் ஒரு கட்டமைப்பு லீவர்.

## இந்த கிட்டை எப்படி fork செய்வது

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# CRITICAL: do this BEFORE seeing any results
shasum -a 256 raw_data_v0.tar.gz       # compute the canonical data hash
shasum -a 256 analysis.py              # hash your pipeline
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # populate with real values
bash build/seal.sh                     # seal BEFORE running the analysis
python3 build/verify.py                # confirm

# Now run the analysis. Hash the output and confirm matches FACT 05.
# If it doesn't match, the pipeline is non-deterministic; fix and re-seal.
```

## ஏற்கனவே உள்ள ஆராய்ச்சி கட்டமைப்புடன் ஒருங்கிணைப்பு

- **arXiv / bioRxiv / medRxiv**: சுருக்கம் அல்லது துணை பொருட்களில் MYRIAM முத்திரை URL அல்லது IPFS CID ஐ சேர்க்கவும்
- **OSF**: வழக்கமான முன்-பதிவு படிவத்துடன் OSF திட்டத்திற்கு முத்திரை artifact ஐ இணைக்கவும்
- **AsPredicted**: முன்-பதிவு ஆவணத்தில் MYRIAM முத்திரை URL ஐ சேர்க்கவும்
- **GitHub**: பகுப்பாய்வு pipeline இன் commit hash தான் FACT 04 நங்கூரமிடுவது; GitHub மூலத்தை வழங்குகிறது, MYRIAM bytes ஐ நங்கூரமிடுகிறது
- **Zenodo / Figshare / OpenAIRE**: மூல தரவை வைப்புத்தொகையாக வைக்கவும், பின்னர் வைப்புத்தொகையின் hash ஐ முத்திரையிடவும்; வைப்புத்தொகை மெட்டாடேட்டாவை விட முத்திரை, அமைதியாக மாற்றுவதற்கு forensically கடினமானது
- **இதழ் துணை பொருட்கள்**: சமர்ப்பணத்தில் ஒரு துணை கோப்பாக MYRIAM முத்திரை artifact ஐ சேர்க்கவும்

## இந்த கிட் மாற்றாதவை

- IRB / நெறிமுறை-குழு ஒப்புதல்
- Peer review
- Replication ஆய்வுகள் (கணினி மறு-உருவாக்கம் ≠ அனுபவ மறு-உருவாக்கம்)
- உணர்திறன் தரவுகளுக்கான தரவு-அணுகல் ஒப்பந்தங்கள்

## ஆசிரியர் வரிசை மற்றும் கடன் பற்றிய ஒரு குறிப்பு

முத்திரை, முத்திரையிடும் நேரத்தில் AUTHOR LIST ஐ நங்கூரமிடுகிறது. வெளியீட்டிற்குப் பிறகு எழும் ஆசிரியர் சர்ச்சைகள் முத்திரையிட்ட பட்டியலுக்கு எதிராக மதிப்பிடப்படலாம். இது ஒரு சிறிய பொறுப்புக்கூறும் லீவர் — அமைதியாக சேர்க்கப்பட்ட அல்லது நீக்கப்பட்ட ECRs க்கு முக்கியமானது.
