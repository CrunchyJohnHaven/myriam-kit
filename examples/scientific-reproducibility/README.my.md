# သိပ္ပံပညာ ပြန်လည်ထုတ်လုပ်နိုင်မှု ကိရိယာ

*မြန်မာဘာသာပြန် — မူရင်းအင်္ဂလိပ်ဗားရှင်းအတွက် [README.md](README.md) ကြည့်ပါ*

သုတေသနအဖွဲ့များ၊ ဂျာနယ်များနှင့် ပြန်လည်ထုတ်လုပ်နိုင်မှုကို
အာရုံစိုက်သော အဖွဲ့အစည်းများအတွက် Bitcoin ဖြင့်အခိုင်အမာချည်နှောင်ထားသော၊
ပြောင်းလဲခြင်းကို သိသာစေသော ပြန်လည်ထုတ်လုပ်နိုင်မှု manifest ကို စာတမ်း
သို့မဟုတ် ပရီပရင့်နှင့်အတူ ထုတ်ဝေလိုသူများအတွက် တန်းပင်ပြင်ဆင်ထားသော
myriam-kit ဥပမာတစ်ခုဖြစ်သည်။

## ဤကိရိယာက ထုတ်လုပ်ပေးသည့်အရာ

လေ့လာမှုတစ်ခုအတွက် ပါဝင်သော တံဆိပ်ခတ်ထားသော ထုတ်ဝေမှု-

1. လေ့လာမှု အထောက်အထား (စာတမ်းခေါင်းစဉ်၊ စာရေးသူများ၊ အဖွဲ့အစည်း၊
   ရန်ပုံငွေထောက်ပံ့သူများ၊ COI)
2. **Raw data archive SHA-256** — တရားဝင် dataset
3. **Analysis pipeline hash** — script + Docker image digest +
   pinned dependencies
4. **Expected output bundle hash** — တိကျမှန်ကန်စွာ ပြန်လည်ထုတ်လုပ်သူတစ်ဦးက
   ထုတ်လုပ်ရမည့်အရာ
5. ပြန်လည်ထုတ်လုပ်ခြင်း protocol + ရလဒ်ပြီးနောက် ထုတ်ဝေမှု ကတိကဝတ်များ
6. လက်မှတ်ထိုးသူ principal(s)

တံဆိပ်သည် **pre-analysis** အဆင့်အတွက် ရည်ရွယ်သည်- data ကို ထိန်းသိမ်းပြီး၊
pipeline ပြီးစီးပြီး၊ မျှော်လင့်ထားသော output ကို တွက်ချက်ပြီး၊ စာတမ်းရေးသား
ရန် ရည်ရွယ်ချက်အတွက် မည်သည့်ရလဒ်မှ မမြင်ရသေးမီ ဤအဆင့်အားလုံးကို
ဆောင်ရွက်ရမည်။

## ဤကိရိယာ၏ ကာကွယ်ပေးနိုင်သည့်အရာ

**တံဆိပ်က ဖမ်းမိနိုင်သော ပျက်ကွက်မှု ပုံစံများ-**

- **p-hacking**: ရလဒ်များကို မြင်ပြီးနောက် analysis ကို ပြောင်းလဲမှုများသည်
  တံဆိပ်ခတ်ထားသော FACT 04 နှင့် ထုတ်ဝေထားသော pipeline အကြား diff မှ
  တစ်ဆင့် တွေ့ရှိနိုင်သည်။
- **HARKing** (ရလဒ်များကို သိရှိပြီးနောက် Hypothesizing လုပ်ခြင်း): FACT 02
  ထဲတွင် ကြိုတင်မှတ်ပုံတင်ထားသော သီအိုရီသည် byte-stable ဖြစ်သည်။
- **စာတမ်းအပြီး data ပြောင်းလဲမှု**: data archive ကို FACT 03 နှင့်
  ပြန်လည် hashing ပြုလုပ်ခြင်းသည် မည်သည့်ပြုပြင်ပြောင်းလဲမှုကိုမဆို
  ဖော်ပြသည်။
- **Pipeline drift**: analysis pipeline ကို FACT 04 နှင့် ပြန်လည် hashing
  ပြုလုပ်ခြင်းသည် တိတ်ဆိတ်စွာ ပြောင်းလဲမှုများကို ဖမ်းမိသည်။
- **Authorship gaming**: ထုတ်ဝေပြီးနောက် စာရေးသူများကို တိတ်ဆိတ်စွာ ထည့်သွင်း
  သို့မဟုတ် ဖယ်ရှားခြင်းသည် တံဆိပ်ခတ်ထားသော FACT 02 နှင့်နှိုင်းယှဉ်လျှင်
  တွေ့ရှိနိုင်လာသည်။

**မဖုံးအုပ်နိုင်သော ပျက်ကွက်မှု ပုံစံများ-**

- ကုန်ကြမ်း data စုဆောင်းခြင်းတွင် လိမ်လည်မှု (တံဆိပ်သည် data ကို တံဆိပ်
  ခတ်သည့်အချိန်တွင်သာ ချည်နှောင်သည်၊ စုဆောင်းချိန်တွင် မဟုတ်)
- ဇီဝဗေဒ / လေ့လာတွေ့ရှိချက် ပြန်လည်ထုတ်လုပ်နိုင်စွမ်းမရှိမှု (တံဆိပ်သည်
  computational ပြန်လည်ထုတ်လုပ်နိုင်စွမ်းကိုသာ ချည်နှောင်သည်)
- နမူနာအရွယ်အစား သို့မဟုတ် လူဦးရေ-တရားဝင်မှု ပြဿနာများ (ဤအရာများသည်
  လေ့လာမှု-ဒီဇိုင်း မေးခွန်းများဖြစ်သည်)
- အခြေခံ ပြောကြားချက်၏ အမှန်တရား (တံဆိပ်သည် ပြောကြားခဲ့သည့်အရာကို
  ချည်နှောင်သည်၊ ကမ္ဘာက ပြောကြားချက်နှင့် ကိုက်ညီသည် မညီသည် မဟုတ်)

## ဤကိရိယာကို ဘယ်အချိန်အသုံးပြုရမည်

အောက်ပါအခြေအနေများတွင် တံဆိပ်ခတ်ရန် စဉ်းစားပါ-

- သင်သည် ရလဒ်များမြင့်မားသော analysis (clinical trial, AI system
  evaluation, replication study) ကို လုပ်ဆောင်နေသော သုတေသန အဖွဲ့တစ်ဖွဲ့
  ဖြစ်ပြီး OSF / GitHub time-stamps ထက် forensic ဆိုင်ရာ ပိုမို
  တိကျမှု လိုချင်လျှင်
- သင်သည် အစောပိုင်း-အသက်မွေးဝမ်းကျောင်း သုတေသီတစ်ဦးဖြစ်ပြီး မညီမျှသော
  ဂုဏ်သတင်း ကာကွယ်မှု လိုချင်လျှင်- pre-analysis ကို တံဆိပ်ခတ်ထားခြင်းသည်
  အနာဂတ်တွင် "သူတို့က p-hack လုပ်ခဲ့သည်" ဆိုသော ဝေဖန်မှုကို လူသိရှင်ကြား
  ငြင်းပယ်နိုင်စေသည်
- သင်သည် စာတမ်းတချို့အတွက် cryptographic-grade pre-registration ကို
  အားပေးလိုသော ဂျာနယ်အယ်ဒီတာ သို့မဟုတ် replication-project ခေါင်းဆောင်
  တစ်ဦးဖြစ်လျှင်
- post-hoc tweaks များက သိသာထင်ရှားသော နယ်ပယ်ဆိုင်ရာ ဆက်ပါးအကျိုးဆက်များ
  ရှိသော analysis တစ်ခုကို လုပ်ဆောင်နေလျှင် (စိတ်ပညာ၊ behavioral econ၊
  ဆေးဘက်ဆိုင်ရာ သုတေသန)

## သမိုင်းဆိုင်ရာ သေဆုံးမှု / အန္တရာယ်ဖြစ်စေသော ဥပမာများ

ဤကိရိယာ၏ သေဆုံးမှု-ဆက်စပ်မှုသည် တိုက်ရိုက်မဟုတ်ပါ-

- **Stem cell / cancer ပြန်လည်ထုတ်လုပ်နိုင်စွမ်းမရှိမှု-** စာတမ်းအချို့သည်
  မှားယွင်းသော clinical trials များမှတဆင့် လူနာများကို ထိခိုက်စေခဲ့သည်။
  မူရင်း pipeline ၏ တံဆိပ်ခတ်ထားသော pre-registration သည် နှိုင်းယှဉ်မှုကို
  ချည်နှောင်ပေးနိုင်ခဲ့မည်။
- **Tamiflu ၏ ထိရောက်မှု ငြင်းခုံမှု-** Cochrane review သည် ကုန်ကြမ်း
  data ရရှိနိုင်မှုကို ငြင်းခုံခြင်းကြောင့် တစ်စိတ်တစ်ပိုင်း နှစ်ပေါင်းများစွာ
  ကြာခဲ့သည်။ ထုတ်ဝေချိန်တွင် တံဆိပ်ခတ်ထားသော data-hashes များက data-
  ရရှိနိုင်မှု အငြင်းပွားမှုများကို ချက်ချင်း ဖြေရှင်းနိုင်စေခဲ့မည်။
- **ယေဘုယျ ပြန်လည်ထုတ်လုပ်နိုင်စွမ်းမရှိမှု အကျပ်အတည်း-** ပြန်လည်ထုတ်လုပ်၍
  မရသော biomedical သုတေသန၏ နယ်ပယ်-အဆင့် သေဆုံးမှု ကုန်ကျစရိတ်သည် သိသာ
  ထင်ရှားသည် (Begley & Ellis 2012 က preclinical findings ၏ >50% ကို
  ပြန်လည်ထုတ်လုပ်၍မရဟု ခန့်မှန်းခဲ့သည်)။ Cryptographic anchoring သည်
  ဖွဲ့စည်းပုံ ဆိုင်ရာ အညွှန်းတစ်ခုဖြစ်သည်။

## ဤကိရိယာကို fork လုပ်နည်း

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# အရေးကြီး- မည်သည့်ရလဒ်ကိုမှ မမြင်ရသေးမီ ဤအဆင့်ကို လုပ်ဆောင်ပါ
shasum -a 256 raw_data_v0.tar.gz       # canonical data hash ကို တွက်ချက်ပါ
shasum -a 256 analysis.py              # သင်၏ pipeline ကို hash လုပ်ပါ
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # အမှန်တကယ်ဖြစ်သော တန်ဖိုးများဖြင့် ဖြည့်ပါ
bash build/seal.sh                     # analysis ကို မစတင်မီ တံဆိပ်ခတ်ပါ
python3 build/verify.py                # အတည်ပြုပါ

# ယခု analysis ကို လုပ်ဆောင်ပါ။ output ကို hash လုပ်ပြီး FACT 05 နှင့်
# တိုက်ဆိုင်ကြောင်း အတည်ပြုပါ။ မတိုက်ဆိုင်ပါက pipeline သည် non-deterministic
# ဖြစ်နေသည်; ပြုပြင်ပြီး ပြန်တံဆိပ်ခတ်ပါ။
```

## ရှိပြီးသား ပညာရှင်ဆိုင်ရာ အခြေခံအဆောက်အအုံများနှင့် ပေါင်းစပ်ခြင်း

- **arXiv / bioRxiv / medRxiv**: MYRIAM seal URL သို့မဟုတ် IPFS CID ကို
  abstract သို့မဟုတ် supplementary materials တွင် ထည့်သွင်းပါ
- **OSF**: seal artifact ကို ပုံမှန် pre-registration ဖောင်နှင့်အတူ OSF
  project သို့ ပူးတွဲတင်ပါ
- **AsPredicted**: pre-registration စာရွက်စာတမ်းတွင် MYRIAM seal URL ကို
  ထည့်သွင်းပါ
- **GitHub**: analysis pipeline ၏ commit hash သည် FACT 04 ချည်နှောင်ထား
  သည့်အရာဖြစ်သည်; GitHub က source ကိုပေးပြီး MYRIAM က bytes ကို ချည်နှောင်သည်
- **Zenodo / Figshare / OpenAIRE**: ကုန်ကြမ်း data ကို သိမ်းပြီး၊ deposit
  ၏ hash ကို တံဆိပ်ခတ်ပါ; deposit metadata ထက် တံဆိပ်ကို တိတ်ဆိတ်စွာ
  အစားထိုးရန် forensically ပိုခက်သည်
- **Journal supplementary materials**: တင်သွင်းချိန်တွင် MYRIAM seal
  artifact ကို supplementary file အဖြစ် ထည့်သွင်းပါ

## ဤကိရိယာက အစားထိုးမပေးနိုင်သည့်အရာ

- IRB / ethics-committee အတည်ပြုချက်
- Peer review
- Replication studies (computational reproducibility ≠ empirical
  reproducibility)
- ထိခိုက်လွယ်သော data အတွက် Data-access agreements

## စာရေးသူ အစီအစဉ်နှင့် ဂုဏ်ပြုခြင်းနှင့်ပတ်သက်သော မှတ်ချက်

တံဆိပ်သည် တံဆိပ်ခတ်ချိန်တွင် AUTHOR LIST ကို ချည်နှောင်သည်။ ထုတ်ဝေပြီးနောက်
ပေါ်ပေါက်လာသော authorship နှင့်ပတ်သက်သော အငြင်းပွားမှုများကို တံဆိပ်ခတ်ထား
သော စာရင်းနှင့်နှိုင်းယှဉ်၍ အကဲဖြတ်နိုင်သည်။ ဤသည်မှာ သေးငယ်သော တာဝန်ခံမှု
ညွှန်ပြချက်တစ်ခုဖြစ်သည် — တိတ်ဆိတ်စွာ ထည့်သွင်း သို့မဟုတ် ဖယ်ရှားခြင်း
ခံရသော ECRs များအတွက် အရေးကြီးသည်။
