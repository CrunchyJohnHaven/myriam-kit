# વૈજ્ઞાનિક પુનઃઉત્પાદનક્ષમતા કિટ

સંશોધન જૂથો, જર્નલ્સ, અને પુનઃઉત્પાદનક્ષમતા-કેન્દ્રિત સંસ્થાઓ માટે
ટર્નકી myriam-kit ઉદાહરણ જે પેપર અથવા પ્રીપ્રિન્ટની સાથે
Bitcoin-એન્કરિત, ટેમ્પર-એવિડન્ટ પુનઃઉત્પાદનક્ષમતા મેનિફેસ્ટ
પ્રકાશિત કરવા માંગે છે.

## આ કિટ શું ઉત્પન્ન કરે છે

એક અભ્યાસ માટે, સીલ કરેલું પ્રકાશન જેમાં શામેલ છે:

1. અભ્યાસ ઓળખ (પેપર શીર્ષક, લેખકો, સંસ્થા, ભંડોળ આપનારા, COI)
2. **કાચા ડેટા આર્કાઇવ SHA-256** — કેનોનિકલ ડેટાસેટ
3. **વિશ્લેષણ પાઇપલાઇન હેશ** — સ્ક્રિપ્ટ + Docker ઇમેજ ડાયજેસ્ટ +
   પિન કરેલી નિર્ભરતાઓ
4. **અપેક્ષિત આઉટપુટ બંડલ હેશ** — જે વિશ્વાસપાત્ર પુનઃઉત્પાદકે
   ઉત્પન્ન કરવું જોઈએ
5. પુનઃઉત્પાદન પ્રોટોકોલ + પોસ્ટ-પરિણામ પ્રકાશન પ્રતિબદ્ધતાઓ
6. હસ્તાક્ષર કરનાર પ્રિન્સિપાલ(ઓ)

સીલ **પૂર્વ-વિશ્લેષણ** તબક્કા માટે ઉદ્દિષ્ટ છે: ડેટા ફ્રીઝ કરેલો,
પાઇપલાઇન અંતિમ રૂપ આપેલી, અપેક્ષિત આઉટપુટ ગણતરી કરેલી, બધું જ
પેપર લખવાના હેતુ માટે કોઈ પરિણામો જોવાય તે પહેલાં.

## આ કિટ શેની સામે રક્ષણ આપે છે

**સીલ જે નિષ્ફળતા મોડ પકડે છે:**

- **p-hacking**: પરિણામો જોયા પછી વિશ્લેષણમાં પોસ્ટ-હોક ફેરફારો
  સીલ કરેલા FACT 04 અને પ્રકાશિત પાઇપલાઇન વચ્ચેના ડિફ દ્વારા
  શોધી શકાય છે.
- **HARKing** (પરિણામો જાણ્યા પછી પૂર્વધારણા બનાવવી): FACT 02 માં
  પૂર્વ-નોંધાયેલ પૂર્વધારણા બાઇટ-સ્થિર છે.
- **પેપર પછી ડેટા મેનિપ્યુલેશન**: FACT 03 સામે ડેટા આર્કાઇવને
  ફરીથી હેશ કરવાથી કોઈપણ ફેરફાર પ્રગટ થાય છે.
- **પાઇપલાઇન ડ્રિફ્ટ**: FACT 04 સામે વિશ્લેષણ પાઇપલાઇનને ફરીથી
  હેશ કરવાથી શાંત ફેરફારો પકડાય છે.
- **લેખકત્વ ગેમિંગ**: પ્રકાશન પછી શાંતપણે લેખકો ઉમેરવા અથવા દૂર
  કરવા સીલ કરેલા FACT 02 સામે શોધી શકાય છે.

**નિષ્ફળતા મોડ જે આવરી લેવામાં આવ્યા નથી:**

- કાચા-ડેટા સંગ્રહમાં છેતરપિંડી (સીલ ડેટાને સીલિંગના બિંદુએ એન્કર
  કરે છે, સંગ્રહના નહીં)
- જૈવિક / પ્રયોગાત્મક બિન-પુનઃઉત્પાદનક્ષમતા (સીલ ફક્ત કોમ્પ્યુટેશનલ
  પુનઃઉત્પાદનક્ષમતાને એન્કર કરે છે)
- નમૂના-કદ અથવા વસ્તી-માન્યતા મુદ્દાઓ (આ અભ્યાસ-ડિઝાઇન પ્રશ્નો છે)
- અંતર્ગત દાવાનું સત્ય (સીલ જે દાવો કરવામાં આવ્યો તે એન્કર કરે છે,
  વિશ્વ દાવા સાથે મેળ ખાય છે કે નહીં તે નહીં)

## આ કિટ ક્યારે વાપરવી

સીલ કરવાનું ધ્યાનમાં લો જો:

- તમે ઉચ્ચ-દાવ વિશ્લેષણ ચલાવતું સંશોધન જૂથ છો (ક્લિનિકલ ટ્રાયલ,
  AI સિસ્ટમ મૂલ્યાંકન, પ્રતિકૃતિ અભ્યાસ) અને OSF / GitHub સમય-મુદ્રાઓથી
  આગળ ફોરેન્સિક અખંડિતતા ઇચ્છો છો
- તમે પ્રારંભિક-કારકિર્દી સંશોધક છો જે અસમમિત પ્રતિષ્ઠા-સંરક્ષણ
  ઇચ્છો છો: સીલ કરેલા પૂર્વ-વિશ્લેષણનો અર્થ છે ભવિષ્યની "તેમણે
  p-hack કર્યું" ટીકાનો જાહેર ફોલ્સિફાયર છે
- તમે જર્નલ સંપાદક અથવા પ્રતિકૃતિ-પ્રોજેક્ટ લીડ છો જે પેપરના
  સબસેટ માટે ક્રિપ્ટોગ્રાફિક-ગ્રેડ પૂર્વ-નોંધણીને પ્રોત્સાહિત
  કરવા માંગે છે
- તમે એવા વિશ્લેષણ પર કામ કરી રહ્યા છો જ્યાં પોસ્ટ-હોક ટ્વીક્સના
  અર્થપૂર્ણ ક્ષેત્રીય પરિણામો છે (મનોવિજ્ઞાન, વર્તણૂક અર્થશાસ્ત્ર,
  તબીબી સંશોધન)

## ઐતિહાસિક મૃત્યુ દર / નુકસાનનો પૂર્વગ્રહ

આ કિટની મૃત્યુ-સંબંધિતતા પરોક્ષ છે:

- **સ્ટેમ સેલ / કેન્સર સંશોધન બિન-પુનઃઉત્પાદનક્ષમતા:** કેટલાંક
  પેપરો ખોટી-દિશા આપેલા ક્લિનિકલ ટ્રાયલ્સ દ્વારા દર્દીને નુકસાન
  તરફ દોરી ગયા છે. મૂળ પાઇપલાઇનની સીલ કરેલી પૂર્વ-નોંધણીએ
  સરખામણીને એન્કર કરી હોત.
- **Tamiflu અસરકારકતા વિવાદ:** Cochrane સમીક્ષામાં વર્ષો લાગ્યા
  કારણ કે કાચા-ડેટા ઍક્સેસ વિવાદિત હતી. પ્રકાશન સમયે સીલ કરેલા
  ડેટા-હેશે ડેટા-ઍક્સેસ વિવાદોને તરત જ ઉકેલવા યોગ્ય બનાવ્યા હોત.
- **સામાન્ય બિન-પુનઃઉત્પાદનક્ષમતા સંકટ:** બિન-પુનઃઉત્પાદન યોગ્ય
  બાયોમેડિકલ સંશોધનની ક્ષેત્ર-સ્તરની મૃત્યુ દર નોંધપાત્ર છે
  (Begley & Ellis 2012 એ >50% પ્રીક્લિનિકલ તારણો
  બિન-પુનઃઉત્પાદનક્ષમ હોવાનો અંદાજ લગાવ્યો). ક્રિપ્ટોગ્રાફિક
  એન્કરિંગ એક માળખાકીય લિવર છે.

## આ કિટને ફોર્ક કેવી રીતે કરવી

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# મહત્વપૂર્ણ: કોઈપણ પરિણામો જોતા પહેલાં આ કરો
shasum -a 256 raw_data_v0.tar.gz       # કેનોનિકલ ડેટા હેશ ગણતરી કરો
shasum -a 256 analysis.py              # તમારી પાઇપલાઇન હેશ કરો
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # વાસ્તવિક મૂલ્યો સાથે ભરો
bash build/seal.sh                     # વિશ્લેષણ ચલાવતા પહેલાં સીલ કરો
python3 build/verify.py                # પુષ્ટિ કરો

# હવે વિશ્લેષણ ચલાવો. આઉટપુટ હેશ કરો અને FACT 05 સાથે મેળ ખાય તે પુષ્ટિ કરો.
# જો મેળ ન ખાય, તો પાઇપલાઇન બિન-નિર્ધારક છે; ઠીક કરો અને ફરીથી સીલ કરો.
```

## વર્તમાન વિદ્વતાપૂર્ણ ઇન્ફ્રાસ્ટ્રક્ચર સાથે એકીકરણ

- **arXiv / bioRxiv / medRxiv**: એબ્સ્ટ્રેક્ટ અથવા પૂરક સામગ્રીમાં
  MYRIAM સીલ URL અથવા IPFS CID શામેલ કરો
- **OSF**: પરંપરાગત પૂર્વ-નોંધણી ફોર્મ સાથે OSF પ્રોજેક્ટ સાથે
  સીલ આર્ટિફેક્ટ જોડો
- **AsPredicted**: પૂર્વ-નોંધણી દસ્તાવેજમાં MYRIAM સીલ URL શામેલ કરો
- **GitHub**: વિશ્લેષણ પાઇપલાઇનનો કમિટ હેશ એ છે જે FACT 04
  એન્કર કરે છે; GitHub સ્રોત પ્રદાન કરે છે, MYRIAM બાઇટ્સ એન્કર
  કરે છે
- **Zenodo / Figshare / OpenAIRE**: કાચા ડેટા જમા કરો, પછી જમાના
  હેશને સીલ કરો; સીલને જમા મેટાડેટા કરતાં શાંતપણે બદલવાનું ફોરેન્સિક
  રીતે વધુ મુશ્કેલ છે
- **જર્નલ પૂરક સામગ્રી**: સબમિશન સમયે પૂરક ફાઇલ તરીકે MYRIAM સીલ
  આર્ટિફેક્ટ શામેલ કરો

## આ કિટ શું બદલતી નથી

- IRB / એથિક્સ-કમિટી મંજૂરી
- પીઅર સમીક્ષા
- પ્રતિકૃતિ અભ્યાસ (કોમ્પ્યુટેશનલ પુનઃઉત્પાદનક્ષમતા ≠ પ્રયોગાત્મક
  પુનઃઉત્પાદનક્ષમતા)
- સંવેદનશીલ ડેટા માટે ડેટા-ઍક્સેસ કરારો

## લેખકના ક્રમ અને ક્રેડિટ પર નોંધ

સીલ સીલિંગ સમયે લેખકોની યાદી એન્કર કરે છે. પ્રકાશન પછી ઉદ્ભવતા
લેખકત્વ વિશેના વિવાદો સીલ કરેલી યાદી સામે મૂલ્યાંકન કરી શકાય છે.
આ એક નાનો જવાબદારી લિવર છે — ECRs માટે મહત્વપૂર્ણ જે શાંતપણે
ઉમેરાય છે અથવા દૂર કરાય છે.
