How to Verify

  1. Download all {{N}} fact files from facts/.
  2. Recompute each file's SHA-256 and confirm it matches the per-fact hash shown above.
  3. Compute the Merkle root (algorithm in SPEC §3), or run verify.py.
  4. Verify the OpenTimestamps receipt against Bitcoin: ots verify merkle_root.txt.ots (with both merkle_root.txt and merkle_root.txt.ots present in the same directory).
  5. Independent IPFS check: ipfs add -r --cid-version=1 -Q site-dir/ and confirm the resulting CID matches {{IPFS_CID}}.

What this protocol promises (and doesn't)

The protocol delivers detectability of tampering, not impossibility of tampering. It delivers content-addressed mirroring at the IPFS layer, but not guaranteed long-term pinning durability without additional pin redundancy. It does not assert that any sealed fact is true; it asserts only that the fact was published in its current bytewise form at the sealed time.

See the project's threat model for the full list of what the protocol does and does not defend against.
