# Scientific Reproducibility Kit

*คำแปลภาษาไทย — สำหรับเวอร์ชันต้นฉบับภาษาอังกฤษ ดู [README.md](README.md)*

ตัวอย่าง myriam-kit แบบสำเร็จรูปสำหรับกลุ่มวิจัย วารสาร และสถาบันที่
เน้นการทำซ้ำ ที่ต้องการเผยแพร่ manifest การทำซ้ำที่ตรึงไว้กับ Bitcoin
และตรวจพบการแก้ไขได้ ควบคู่กับ paper หรือ preprint

## สิ่งที่ชุดเครื่องมือนี้ผลิต

การเผยแพร่ที่ผนึกซึ่งบรรจุ สำหรับการศึกษาหนึ่งเรื่อง:

1. การระบุการศึกษา (ชื่อ paper ผู้เขียน สถาบัน ผู้ให้ทุน COI)
2. **SHA-256 ของไฟล์เก็บถาวรข้อมูลดิบ** — ชุดข้อมูล canonical
3. **แฮชของ pipeline การวิเคราะห์** — script + Docker image digest +
   pinned dependencies
4. **แฮชของ output bundle ที่คาดหวัง** — สิ่งที่ผู้ทำซ้ำที่ซื่อสัตย์
   ควรผลิต
5. โปรโตคอลการทำซ้ำ + commitment การเผยแพร่หลังผลลัพธ์
6. ผู้ลงนาม

ตราผนึกมีเจตนาสำหรับขั้นตอน **pre-analysis**: ข้อมูลแช่แข็ง pipeline
เสร็จสิ้น output ที่คาดหวังคำนวณแล้ว ทั้งหมดก่อนที่จะเห็นผลลัพธ์ใด ๆ
เพื่อจุดประสงค์ในการเขียน paper

## สิ่งที่ชุดเครื่องมือนี้ป้องกัน

**โหมดความล้มเหลวที่ตราผนึกจับ:**

- **p-hacking**: การเปลี่ยนแปลงหลังเหตุการณ์ต่อการวิเคราะห์หลังจากเห็น
  ผลลัพธ์สามารถตรวจพบได้ผ่าน diff ระหว่าง FACT 04 ที่ผนึกและ pipeline
  ที่เผยแพร่
- **HARKing** (Hypothesizing After Results are Known): สมมติฐานที่
  ลงทะเบียนล่วงหน้าใน FACT 02 มีความเสถียรของไบต์
- **การแก้ไขข้อมูลหลัง paper**: การ re-hash ไฟล์เก็บถาวรข้อมูลเทียบกับ
  FACT 03 เปิดเผยการแก้ไขใด ๆ
- **Pipeline drift**: การ re-hash pipeline การวิเคราะห์เทียบกับ FACT 04
  จับการเปลี่ยนแปลงแบบเงียบ ๆ
- **Authorship gaming**: การเพิ่มหรือลบผู้เขียนเงียบ ๆ หลังการเผยแพร่
  สามารถตรวจพบได้เทียบกับ FACT 02 ที่ผนึก

**โหมดความล้มเหลวที่ไม่ครอบคลุม:**

- การฉ้อโกงในการรวบรวมข้อมูลดิบ (ตราผนึกตรึงข้อมูล ณ จุดผนึก ไม่ใช่
  การรวบรวม)
- การไม่ทำซ้ำได้ทางชีวภาพ / เชิงประจักษ์ (ตราผนึกตรึงการทำซ้ำเชิง
  คำนวณเท่านั้น)
- ปัญหาขนาดตัวอย่างหรือความถูกต้องของประชากร (สิ่งเหล่านี้เป็นคำถาม
  ของการออกแบบการศึกษา)
- ความจริงของคำกล่าวอ้างพื้นฐาน (ตราผนึกตรึงสิ่งที่ถูกกล่าวอ้าง ไม่ใช่
  ว่าโลกตรงกับคำกล่าวอ้างหรือไม่)

## เมื่อใดควรใช้ชุดเครื่องมือนี้

พิจารณาผนึกหาก:

- คุณเป็นกลุ่มวิจัยที่ดำเนินการวิเคราะห์เดิมพันสูง (การทดลองทางคลินิก
  การประเมินระบบ AI การศึกษาทำซ้ำ) และต้องการความถูกต้องสมบูรณ์ทาง
  forensic เกินกว่าการประทับเวลา OSF / GitHub
- คุณเป็นนักวิจัยอาชีพต้นที่ต้องการการป้องกันชื่อเสียงแบบไม่สมมาตร:
  pre-analysis ที่ผนึกหมายความว่าการวิจารณ์ในอนาคต "พวกเขา p-hacked"
  มีตัวพิสูจน์เท็จสาธารณะ
- คุณเป็นบรรณาธิการวารสารหรือหัวหน้าโครงการทำซ้ำที่ต้องการสนับสนุนการ
  ลงทะเบียนล่วงหน้าระดับเชิงการเข้ารหัสสำหรับ paper ส่วนหนึ่ง
- คุณกำลังทำการวิเคราะห์ที่การปรับแต่งหลังเหตุการณ์มีผลกระทบที่มี
  ความหมายในสาขา (จิตวิทยา เศรษฐศาสตร์เชิงพฤติกรรม การวิจัยทางการแพทย์)

## บรรทัดฐานการเสียชีวิต / อันตรายทางประวัติศาสตร์

ความเกี่ยวข้องกับการเสียชีวิตของชุดเครื่องมือนี้เป็นทางอ้อม:

- **การไม่ทำซ้ำได้ของการวิจัยสเต็มเซลล์ / มะเร็ง:** paper หลายฉบับนำ
  ไปสู่อันตรายต่อผู้ป่วยผ่านการทดลองทางคลินิกที่ถูกชี้นำผิด การ
  ลงทะเบียนล่วงหน้าที่ผนึกของ pipeline ต้นฉบับจะตรึงการเปรียบเทียบ
- **ข้อโต้แย้งประสิทธิผลของ Tamiflu:** การทบทวน Cochrane ใช้เวลา
  หลายปีส่วนหนึ่งเพราะการเข้าถึงข้อมูลดิบถูกถกเถียง แฮชข้อมูลที่
  ผนึก ณ การเผยแพร่จะทำให้ข้อพิพาทการเข้าถึงข้อมูลแก้ไขได้ทันที
- **วิกฤตการไม่ทำซ้ำได้ทั่วไป:** ต้นทุนการเสียชีวิตในระดับสาขาของ
  การวิจัยทางชีวการแพทย์ที่ไม่สามารถทำซ้ำได้นั้นมีนัยสำคัญ (Begley &
  Ellis 2012 ประมาณ >50% ของผลการค้นพบเชิงคลินิกพรีคลินิกไม่สามารถ
  ทำซ้ำได้) การตรึงเชิงการเข้ารหัสคือ leverage โครงสร้างหนึ่ง

## วิธี fork ชุดเครื่องมือนี้

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# CRITICAL: do this BEFORE seeing any results
shasum -a 256 raw_data_v0.tar.gz       # compute the canonical data hash
shasum -a 256 analysis.py              # hash your pipeline
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # populate with real values
bash build/seal.sh                     # seal BEFORE running the analysis
python3 build/verify.py                # confirm

# Now run the analysis. Hash the output and confirm matches FACT 05.
# If it doesn't match, the pipeline is non-deterministic; fix and re-seal.
```

## การบูรณาการกับโครงสร้างพื้นฐานทางวิชาการที่มีอยู่

- **arXiv / bioRxiv / medRxiv**: รวม URL ตราผนึก MYRIAM หรือ CID IPFS
  ในบทคัดย่อหรือวัสดุเสริม
- **OSF**: แนบสิ่งประดิษฐ์ตราผนึกไปยังโครงการ OSF ควบคู่กับฟอร์มการ
  ลงทะเบียนล่วงหน้าแบบดั้งเดิม
- **AsPredicted**: รวม URL ตราผนึก MYRIAM ในเอกสารการลงทะเบียนล่วงหน้า
- **GitHub**: commit hash ของ pipeline การวิเคราะห์คือสิ่งที่ FACT 04
  ตรึง GitHub ให้ source MYRIAM ตรึงไบต์
- **Zenodo / Figshare / OpenAIRE**: ฝากข้อมูลดิบ จากนั้นผนึกแฮชของ
  การฝาก ตราผนึกเป็นเรื่องยากกว่าทาง forensic ที่จะแทนแบบเงียบ ๆ
  เทียบกับ metadata ของการฝาก
- **วัสดุเสริมของวารสาร**: รวมสิ่งประดิษฐ์ตราผนึก MYRIAM เป็นไฟล์
  เสริม ณ การยื่นเรื่อง

## สิ่งที่ชุดเครื่องมือนี้ "ไม่" ทดแทน

- การอนุมัติ IRB / คณะกรรมการจริยธรรม
- การทบทวนของเพื่อนร่วมงาน
- การศึกษาทำซ้ำ (การทำซ้ำเชิงคำนวณ ≠ การทำซ้ำเชิงประจักษ์)
- ข้อตกลงการเข้าถึงข้อมูลสำหรับข้อมูลที่ละเอียดอ่อน

## หมายเหตุเกี่ยวกับการจัดลำดับผู้เขียนและเครดิต

ตราผนึกตรึง AUTHOR LIST ณ เวลาผนึก ข้อพิพาทเรื่องการเป็นผู้เขียนที่
เกิดขึ้นหลังการเผยแพร่สามารถถูกประเมินเทียบกับรายการที่ผนึก นี่คือ
leverage ความรับผิดชอบเล็ก ๆ — สำคัญสำหรับ ECR ที่ถูกเพิ่มหรือลบเงียบ ๆ
