# ശാസ്ത്രീയ പുനരുൽപ്പാദനക്ഷമത കിറ്റ്

ഗവേഷണ ഗ്രൂപ്പുകൾ, ജേണലുകൾ, പുനരുൽപ്പാദനക്ഷമത കേന്ദ്രീകൃത
സ്ഥാപനങ്ങൾ എന്നിവയ്ക്കായി, ഒരു പേപ്പറിനോ പ്രീപ്രിന്റിനോ ഒപ്പം
Bitcoin-അങ്കർ ചെയ്ത, ടാമ്പർ-എവിഡന്റ് പുനരുൽപ്പാദനക്ഷമത
മാനിഫെസ്റ്റ് പ്രസിദ്ധീകരിക്കാൻ ആഗ്രഹിക്കുന്ന ഒരു ടേൺകീ
myriam-kit ഉദാഹരണം.

## ഈ കിറ്റ് എന്താണ് ഉൽപ്പാദിപ്പിക്കുന്നത്

ഒരു പഠനത്തിനായി, സീൽ ചെയ്ത പ്രസിദ്ധീകരണത്തിൽ ഉൾപ്പെടുന്നു:

1. പഠന തിരിച്ചറിയൽ (പേപ്പർ ശീർഷകം, രചയിതാക്കൾ, സ്ഥാപനം,
   ധനസഹായം നൽകുന്നവർ, COI)
2. **അസംസ്കൃത ഡാറ്റ ആർക്കൈവ് SHA-256** — കാനോനിക്കൽ ഡാറ്റാസെറ്റ്
3. **വിശകലന പൈപ്പ്‌ലൈൻ ഹാഷ്** — സ്ക്രിപ്റ്റ് + Docker ഇമേജ്
   ഡൈജസ്റ്റ് + പിൻ ചെയ്ത ഡിപ്പൻഡൻസികൾ
4. **പ്രതീക്ഷിത ഔട്ട്പുട്ട് ബണ്ടിൽ ഹാഷ്** — വിശ്വസ്തനായ
   പുനരുൽപ്പാദകൻ ഉൽപ്പാദിപ്പിക്കേണ്ടത്
5. പുനരുൽപ്പാദന പ്രോട്ടോക്കോൾ + ഫലത്തിന് ശേഷമുള്ള പ്രസിദ്ധീകരണ
   പ്രതിബദ്ധതകൾ
6. ഒപ്പിടുന്ന പ്രിൻസിപ്പൽ(കൾ)

സീൽ **പ്രീ-വിശകലനം** ഘട്ടത്തിനായി ഉദ്ദേശിച്ചുള്ളതാണ്: ഡാറ്റ
ഫ്രീസ് ചെയ്തു, പൈപ്പ്‌ലൈൻ അന്തിമമാക്കി, പ്രതീക്ഷിത ഔട്ട്പുട്ട്
കണക്കാക്കി, എല്ലാം പേപ്പർ എഴുതുന്നതിന് വേണ്ടി ഏതെങ്കിലും
ഫലങ്ങൾ കാണുന്നതിന് മുമ്പ്.

## ഈ കിറ്റ് എന്തിനെതിരെ പ്രതിരോധിക്കുന്നു

**സീൽ പിടിക്കുന്ന പരാജയ മോഡുകൾ:**

- **p-hacking**: ഫലങ്ങൾ കണ്ടതിന് ശേഷം വിശകലനത്തിലെ പോസ്റ്റ്-ഹോക്
  മാറ്റങ്ങൾ സീൽ ചെയ്ത FACT 04 ഉം പ്രസിദ്ധീകരിച്ച പൈപ്പ്‌ലൈനും
  തമ്മിലുള്ള ഡിഫ് വഴി കണ്ടെത്താനാകും.
- **HARKing** (ഫലങ്ങൾ അറിഞ്ഞശേഷം സിദ്ധാന്തമുണ്ടാക്കൽ): FACT 02-ലെ
  മുൻ-രജിസ്റ്റർ ചെയ്ത സിദ്ധാന്തം ബൈറ്റ്-സ്ഥിരമാണ്.
- **പേപ്പറിന് ശേഷമുള്ള ഡാറ്റ കൃത്രിമം**: FACT 03-നെതിരെ ഡാറ്റ
  ആർക്കൈവ് വീണ്ടും ഹാഷ് ചെയ്യുന്നത് ഏതെങ്കിലും പരിഷ്കരണം
  വെളിപ്പെടുത്തുന്നു.
- **പൈപ്പ്‌ലൈൻ ഡ്രിഫ്റ്റ്**: FACT 04-നെതിരെ വിശകലന പൈപ്പ്‌ലൈൻ
  വീണ്ടും ഹാഷ് ചെയ്യുന്നത് നിശ്ശബ്ദ മാറ്റങ്ങൾ പിടിക്കുന്നു.
- **രചയിതാവ് ഗെയിമിംഗ്**: പ്രസിദ്ധീകരണത്തിന് ശേഷം നിശ്ശബ്ദമായി
  രചയിതാക്കളെ ചേർക്കുകയോ നീക്കം ചെയ്യുകയോ ചെയ്യുന്നത് സീൽ ചെയ്ത
  FACT 02-നെതിരെ കണ്ടെത്താനാകും.

**ഉൾപ്പെടാത്ത പരാജയ മോഡുകൾ:**

- അസംസ്കൃത-ഡാറ്റ ശേഖരണത്തിലെ വഞ്ചന (സീൽ ഡാറ്റയെ സീലിംഗ്
  ഘട്ടത്തിൽ അങ്കർ ചെയ്യുന്നു, ശേഖരണത്തിലല്ല)
- ജൈവിക / അനുഭവജന്യ പുനരുൽപ്പാദന-രാഹിത്യം (സീൽ കമ്പ്യൂട്ടേഷണൽ
  പുനരുൽപ്പാദനക്ഷമത മാത്രമേ അങ്കർ ചെയ്യുന്നുള്ളൂ)
- സാമ്പിൾ വലുപ്പം അല്ലെങ്കിൽ ജനസംഖ്യാ-സാധുത പ്രശ്നങ്ങൾ (ഇവ
  പഠന-രൂപകൽപ്പന ചോദ്യങ്ങളാണ്)
- അടിസ്ഥാന അവകാശവാദത്തിന്റെ സത്യം (സീൽ അവകാശപ്പെട്ടത് അങ്കർ
  ചെയ്യുന്നു, ലോകം അവകാശവാദവുമായി പൊരുത്തപ്പെടുന്നുണ്ടോ എന്നല്ല)

## ഈ കിറ്റ് എപ്പോൾ ഉപയോഗിക്കണം

സീലിംഗ് പരിഗണിക്കുക:

- നിങ്ങൾ ഉയർന്ന-സ്റ്റേക്ക് വിശകലനം നടത്തുന്ന ഒരു ഗവേഷണ ഗ്രൂപ്പാണെങ്കിൽ
  (ക്ലിനിക്കൽ ട്രയൽ, AI സിസ്റ്റം മൂല്യനിർണ്ണയം, റെപ്ലിക്കേഷൻ പഠനം)
  കൂടാതെ OSF / GitHub ടൈം-സ്റ്റാമ്പുകൾക്കപ്പുറം ഫോറൻസിക് സമഗ്രത
  ആഗ്രഹിക്കുന്നുണ്ടെങ്കിൽ
- നിങ്ങൾ അസമമിത പ്രശസ്തി-സംരക്ഷണം ആഗ്രഹിക്കുന്ന ഒരു ആദ്യകാല-ജോലിജീവിത
  ഗവേഷകനാണെങ്കിൽ: സീൽ ചെയ്ത പ്രീ-വിശകലനം എന്നാൽ ഭാവിയിലെ "അവർ
  p-hack ചെയ്തു" എന്ന വിമർശനത്തിന് ഒരു പൊതു ഫാൾസിഫയർ ഉണ്ടെന്നാണ്
- നിങ്ങൾ പേപ്പറുകളുടെ ഒരു ഉപവിഭാഗത്തിന് ക്രിപ്റ്റോഗ്രാഫിക്-ഗ്രേഡ്
  മുൻ-രജിസ്ട്രേഷൻ പ്രോത്സാഹിപ്പിക്കാൻ ആഗ്രഹിക്കുന്ന ഒരു ജേണൽ
  എഡിറ്റർ അല്ലെങ്കിൽ റെപ്ലിക്കേഷൻ-പ്രോജക്റ്റ് നേതാവാണെങ്കിൽ
- പോസ്റ്റ്-ഹോക് ട്വീക്കുകൾക്ക് അർത്ഥവത്തായ ഫീൽഡ് അനന്തരഫലങ്ങളുള്ള
  ഒരു വിശകലനത്തിൽ നിങ്ങൾ ജോലി ചെയ്യുകയാണെങ്കിൽ (മനഃശാസ്ത്രം,
  സ്വഭാവ സാമ്പത്തികശാസ്ത്രം, വൈദ്യശാസ്ത്ര ഗവേഷണം)

## ചരിത്രപരമായ മരണനിരക്ക് / ദോഷ പൂർവ്വാധികാരം

ഈ കിറ്റിന്റെ മരണ-പ്രസക്തി പരോക്ഷമാണ്:

- **സ്റ്റെം സെൽ / ക്യാൻസർ ഗവേഷണ പുനരുൽപ്പാദന-രാഹിത്യം:** പല
  പേപ്പറുകളും തെറ്റായ ദിശയിലുള്ള ക്ലിനിക്കൽ ട്രയലുകൾ വഴി രോഗി
  ദോഷത്തിലേക്ക് നയിച്ചിട്ടുണ്ട്. യഥാർത്ഥ പൈപ്പ്‌ലൈനിന്റെ സീൽ
  ചെയ്ത മുൻ-രജിസ്ട്രേഷൻ താരതമ്യത്തെ അങ്കർ ചെയ്യുമായിരുന്നു.
- **Tamiflu ഫലപ്രാപ്തി വിവാദം:** അസംസ്കൃത-ഡാറ്റ ആക്സസ്സ്
  തർക്കത്തിലായതിനാൽ Cochrane അവലോകനത്തിന് വർഷങ്ങൾ വേണ്ടി വന്നു.
  പ്രസിദ്ധീകരണ സമയത്ത് സീൽ ചെയ്ത ഡാറ്റ-ഹാഷുകൾ ഡാറ്റ-ആക്സസ്
  തർക്കങ്ങൾ ഉടനടി പരിഹരിക്കാവുന്നതാക്കുമായിരുന്നു.
- **പൊതുവായ പുനരുൽപ്പാദന-രാഹിത്യ പ്രതിസന്ധി:** പുനരുൽപ്പാദന-രഹിത
  ബയോമെഡിക്കൽ ഗവേഷണത്തിന്റെ ഫീൽഡ്-തല മരണ ചെലവ് ഗണ്യമാണ് (Begley
  & Ellis 2012 >50% പ്രീക്ലിനിക്കൽ കണ്ടെത്തലുകൾ പുനരുൽപ്പാദന-രഹിതമെന്ന്
  കണക്കാക്കി). ക്രിപ്റ്റോഗ്രാഫിക് ആങ്കറിംഗ് ഒരു ഘടനാപരമായ
  ലിവറാണ്.

## ഈ കിറ്റ് എങ്ങനെ ഫോർക്ക് ചെയ്യാം

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# നിർണ്ണായകം: ഏതെങ്കിലും ഫലങ്ങൾ കാണുന്നതിന് മുമ്പ് ഇത് ചെയ്യുക
shasum -a 256 raw_data_v0.tar.gz       # കാനോനിക്കൽ ഡാറ്റ ഹാഷ് കണക്കാക്കുക
shasum -a 256 analysis.py              # നിങ്ങളുടെ പൈപ്പ്‌ലൈൻ ഹാഷ് ചെയ്യുക
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # യഥാർത്ഥ മൂല്യങ്ങൾ ഉപയോഗിച്ച് പൂരിപ്പിക്കുക
bash build/seal.sh                     # വിശകലനം പ്രവർത്തിപ്പിക്കുന്നതിന് മുമ്പ് സീൽ ചെയ്യുക
python3 build/verify.py                # സ്ഥിരീകരിക്കുക

# ഇപ്പോൾ വിശകലനം പ്രവർത്തിപ്പിക്കുക. ഔട്ട്പുട്ട് ഹാഷ് ചെയ്ത് FACT 05-മായി പൊരുത്തപ്പെടുന്നുവെന്ന് സ്ഥിരീകരിക്കുക.
# പൊരുത്തപ്പെടുന്നില്ലെങ്കിൽ, പൈപ്പ്‌ലൈൻ നോൺ-ഡിറ്റർമിനിസ്റ്റിക് ആണ്; പരിഹരിച്ച് വീണ്ടും സീൽ ചെയ്യുക.
```

## നിലവിലുള്ള പണ്ഡിത അടിസ്ഥാന സൗകര്യങ്ങളുമായുള്ള സംയോജനം

- **arXiv / bioRxiv / medRxiv**: അമൂർത്തത്തിലോ അനുബന്ധ സാമഗ്രികളിലോ
  MYRIAM സീൽ URL അല്ലെങ്കിൽ IPFS CID ഉൾപ്പെടുത്തുക
- **OSF**: പരമ്പരാഗത മുൻ-രജിസ്ട്രേഷൻ ഫോമിനൊപ്പം OSF പ്രോജക്റ്റിലേക്ക്
  സീൽ ആർട്ടിഫാക്റ്റ് ചേർക്കുക
- **AsPredicted**: മുൻ-രജിസ്ട്രേഷൻ രേഖയിൽ MYRIAM സീൽ URL ഉൾപ്പെടുത്തുക
- **GitHub**: വിശകലന പൈപ്പ്‌ലൈനിന്റെ കമ്മിറ്റ് ഹാഷാണ് FACT 04
  അങ്കർ ചെയ്യുന്നത്; GitHub ഉറവിടം നൽകുന്നു, MYRIAM ബൈറ്റുകൾ
  അങ്കർ ചെയ്യുന്നു
- **Zenodo / Figshare / OpenAIRE**: അസംസ്കൃത ഡാറ്റ ഡെപ്പോസിറ്റ്
  ചെയ്യുക, എന്നിട്ട് ഡെപ്പോസിറ്റിന്റെ ഹാഷ് സീൽ ചെയ്യുക;
  ഡെപ്പോസിറ്റ് മെറ്റാഡാറ്റയേക്കാൾ സീൽ നിശ്ശബ്ദമായി
  മാറ്റിസ്ഥാപിക്കാൻ ഫോറൻസിക്കായി കൂടുതൽ ബുദ്ധിമുട്ടാണ്

## രചയിതാവ് ഓർഡറിംഗും ക്രെഡിറ്റും സംബന്ധിച്ച ഒരു കുറിപ്പ്

സീൽ സീലിംഗ് സമയത്ത് രചയിതാക്കളുടെ പട്ടിക അങ്കർ ചെയ്യുന്നു.
പ്രസിദ്ധീകരണത്തിന് ശേഷം ഉയരുന്ന രചയിതാവ് സംബന്ധിച്ച തർക്കങ്ങൾ
സീൽ ചെയ്ത പട്ടികക്കെതിരെ വിലയിരുത്താനാകും. ഇത് ഒരു ചെറിയ
ഉത്തരവാദിത്ത ലിവറാണ് — നിശ്ശബ്ദമായി ചേർക്കപ്പെടുകയോ നീക്കം
ചെയ്യപ്പെടുകയോ ചെയ്യുന്ന ECR-കൾക്ക് പ്രധാനമാണ്.
