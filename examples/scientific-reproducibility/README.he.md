# ערכת שחזוריות מדעית

דוגמת myriam-kit מן המוכן לקבוצות מחקר, כתבי עת ומוסדות
המתמקדים בשחזוריות, המעוניינים לפרסם מניפסט שחזוריות מעוגן
ב-Bitcoin וחושף-זיוף לצד מאמר או טרום-הדפסה.

## מה הערכה הזו מפיקה

פרסום חתום המכיל, עבור מחקר אחד:

1. זיהוי המחקר (כותרת המאמר, מחברים, מוסד, מממנים, ניגוד עניינים)
2. **SHA-256 של ארכיון נתונים גולמיים** — מערך הנתונים הקנוני
3. **תקציר (hash) של צינור הניתוח** — סקריפט + digest של תמונת Docker +
   תלויות מקובעות
4. **תקציר חבילת הפלט הצפוי** — מה ששחזרן נאמן אמור להפיק
5. פרוטוקול שחזור + התחייבויות לפרסום לאחר התוצאות
6. מפרסם או מפרסמים חותמים

החתימה מיועדת לשלב **טרום-הניתוח**: הנתונים מוקפאים, הצינור
מסופי, הפלט הצפוי מחושב, הכול לפני שמישהו רואה תוצאות לצורך
כתיבת המאמר.

## מפני מה הערכה הזו מגינה

**אופני כשל שהחתימה תופסת:**

- **p-hacking**: שינויים בדיעבד בניתוח לאחר ראיית התוצאות הם
  בני-זיהוי דרך diff בין FACT 04 החתום לבין הצינור המפורסם.
- **HARKing** (ניסוח השערה לאחר שהתוצאות ידועות — Hypothesizing
  After Results are Known): ההשערה הרשומה-מראש ב-FACT 02 יציבה
  ברמת בית (byte).
- **מניפולציה של נתונים לאחר פרסום המאמר**: תקצור-מחדש של ארכיון
  הנתונים מול FACT 03 חושף כל שינוי.
- **סחיפה של הצינור**: תקצור-מחדש של צינור הניתוח מול FACT 04
  תופס שינויים שקטים.
- **משחק במחברים**: הוספה או הסרה שקטה של מחברים לאחר הפרסום
  הופכת לבת-זיהוי מול FACT 02 החתום.

**אופני כשל שאינם מכוסים:**

- הונאה באיסוף נתונים גולמיים (החתימה מעגנת את הנתונים בנקודת
  החתימה, לא בנקודת האיסוף)
- אי-שחזוריות ביולוגית / אמפירית (החתימה מעגנת רק שחזוריות
  חישובית)
- בעיות גודל מדגם או תוקף אוכלוסייה (אלו שאלות של עיצוב המחקר)
- אמיתות הטענה הבסיסית (החתימה מעגנת את מה שנטען, לא את
  השאלה האם העולם תואם את הטענה)

## מתי להשתמש בערכה זו

שקלו לחתום אם:

- אתם קבוצת מחקר המבצעת ניתוח עתיר-סיכון (ניסוי קליני,
  הערכת מערכת בינה מלאכותית, מחקר שחזור) ורוצים שלמות
  ברמה פורנזית מעבר לחותמות-זמן של OSF / GitHub
- אתם חוקרים בתחילת דרכם המבקשים הגנה מוניטין א-סימטרית:
  טרום-ניתוח חתום פירושו שכל ביקורת עתידית של "הם עשו
  p-hacking" יש לה מפריך פומבי
- אתם עורכי כתב עת או מובילי פרויקט שחזור המבקשים לעודד
  רישום מקדים ברמה קריפטוגרפית לתת-קבוצה של מאמרים
- אתם עובדים על ניתוח שבו שינויים בדיעבד נושאים השלכות
  משמעותיות לשדה (פסיכולוגיה, כלכלה התנהגותית, מחקר רפואי)

## תקדים היסטורי של תמותה / נזק

הרלוונטיות של ערכה זו לתמותה היא עקיפה:

- **אי-שחזוריות במחקר תאי גזע / סרטן:** כמה מאמרים הובילו
  לפגיעה בחולים דרך ניסויים קליניים שכּוּונו מוטעית. רישום
  מקדים חתום של הצינור המקורי היה מעגן את ההשוואה.
- **המחלוקת על יעילות Tamiflu:** סקירת Cochrane נמשכה שנים
  בין השאר משום שהגישה לנתונים הגולמיים הייתה שנויה במחלוקת.
  תקצירי נתונים חתומים בעת הפרסום היו הופכים מחלוקות גישה
  לנתונים לפתירות באופן מיידי.
- **משבר אי-שחזוריות כללי:** עלות התמותה ברמת השדה ממחקר
  ביו-רפואי לא-בר-שחזור משמעותית (Begley & Ellis 2012 העריכו
  ש->50% מהממצאים הפרה-קליניים אינם בני-שחזור). עגינה
  קריפטוגרפית היא מנוף מבני אחד.

## כיצד לפצל (fork) את הערכה הזו

```bash
cp -r ~/Genesis/myriam-kit/examples/scientific-reproducibility ~/my-paper-seal
cd ~/my-paper-seal

# קריטי: עשו זאת לפני ראיית תוצאות כלשהן
shasum -a 256 raw_data_v0.tar.gz       # חשבו את התקציר הקנוני של הנתונים
shasum -a 256 analysis.py              # תקצרו את הצינור שלכם
docker buildx imagetools inspect <image> --format '{{.Manifest.Digest}}'

$EDITOR facts/*.txt                    # מלאו בערכים אמיתיים
bash build/seal.sh                     # חתמו לפני הרצת הניתוח
python3 build/verify.py                # אישור

# כעת הריצו את הניתוח. תקצרו את הפלט וודאו שהוא תואם ל-FACT 05.
# אם אינו תואם, הצינור אינו דטרמיניסטי; תקנו וחתמו מחדש.
```

## אינטגרציה עם תשתית מחקרית קיימת

- **arXiv / bioRxiv / medRxiv**: כללו את כתובת ה-URL של חתימת
  MYRIAM או את ה-CID של IPFS בתקציר או בחומרים המשלימים
- **OSF**: צרפו את ארטיפקט החתימה לפרויקט OSF לצד טופס הרישום
  המקדים המקובל
- **AsPredicted**: כללו את כתובת ה-URL של חתימת MYRIAM במסמך
  הרישום המקדים
- **GitHub**: ה-commit hash של צינור הניתוח הוא מה ש-FACT 04
  מעגן; GitHub מספק את המקור, MYRIAM מעגן את הבתים
- **Zenodo / Figshare / OpenAIRE**: הפקידו נתונים גולמיים, ואז
  חתמו על ה-hash של ההפקדה; קשה יותר פורנזית להחליף את
  החתימה בשקט מאשר את מטא-נתוני ההפקדה
- **חומרים משלימים של כתב עת**: צרפו את ארטיפקט חתימת MYRIAM
  כקובץ משלים בעת ההגשה

## מה הערכה הזו אינה מחליפה

- אישור IRB / ועדת אתיקה
- ביקורת עמיתים (peer review)
- מחקרי שחזור (שחזוריות חישובית ≠ שחזוריות אמפירית)
- הסכמי גישה לנתונים עבור נתונים רגישים

## הערה על סדר המחברים והקרדיט

החתימה מעגנת את רשימת המחברים בעת החתימה. מחלוקות לגבי
המחברות שמתעוררות לאחר הפרסום ניתנות להערכה מול הרשימה
החתומה. זהו מנוף אחריותיות קטן — חשוב עבור חוקרים בתחילת
דרכם שמתווספים או מוסרים בשקט.
