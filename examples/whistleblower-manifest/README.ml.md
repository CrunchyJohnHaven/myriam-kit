# വിസിൽബ്ലോവർ മാനിഫെസ്റ്റ് സീൽ കിറ്റ്

*മലയാളം പരിഭാഷ — യഥാർത്ഥ ഇംഗ്ലീഷ് പതിപ്പിനായി, കാണുക [README.md](README.md)*

**വിസിൽബ്ലോവർക്കും പൊതുജനങ്ങൾക്കും ഇടയിലുള്ള ഇടനിലക്കാർ തെളിവുകൾ കുഴിച്ചുമൂടുന്ന, എഡിറ്റ് ചെയ്യുന്ന, അല്ലെങ്കിൽ തർക്കിക്കുന്ന** പ്രത്യേക പരാജയ രീതി നേരിടുന്ന വിസിൽബ്ലോവർമാർക്കായുള്ള ഒരു ടേൺകീ myriam-kit ഉദാഹരണം. ഇത് പ്രതിരോധിക്കുന്ന കാനോനിക്കൽ ചരിത്രപരമായ കേസുകൾ:

- ബോയിംഗ് 737 MAX (ആന്തരിക സുരക്ഷാ റിപ്പോർട്ടുകൾ കുറച്ചുകാണുകയോ/കുഴിച്ചുമൂടുകയോ ചെയ്തു)
- Theranos (ലാബ്-ടെസ്റ്റ് ആശങ്കകൾ വർഷങ്ങളോളം അടിച്ചമർത്തി)
- FDA ഇൻസ്പെക്ടർ റിപ്പോർട്ടുകൾ സമർപ്പിച്ചതിന് ശേഷം എഡിറ്റ് ചെയ്തു
- Frances Haugen / Facebook (മുദ്രവെച്ച മാനിഫെസ്റ്റ് രേഖകൾ ആധികാരികമാണെന്നും അവകാശപ്പെട്ട സമയത്ത് നിലവിലുണ്ടായിരുന്നുവെന്നും തെളിയിക്കുമായിരുന്നു)

പ്രോട്ടോക്കോൾ പദപ്രയോഗത്തിൽ ഈ കിറ്റ് എന്തിനെതിരെ പ്രതിരോധിക്കുന്നു:

- [FACT 05](facts/05_threat_model.txt) ലെ T1–T3: രേഖാ ഉള്ളടക്കങ്ങളുടെ പൊതു തർക്കം, അവകാശപ്പെട്ട സമയത്ത് രേഖകൾ മുദ്രവെച്ചിട്ടില്ലെന്ന അവകാശവാദങ്ങൾ, പിന്നോട്ടുള്ള കെട്ടിച്ചമയ്ക്കലിന്റെ അവകാശവാദങ്ങൾ.
- T9: മൂന്നാം കക്ഷികൾ പത്രപ്രവർത്തകർക്ക് മാറ്റം വരുത്തിയ പകർപ്പുകൾ കൈമാറുന്നു.

ഈ കിറ്റ് **പ്രതിരോധിക്കാത്തത്**:

- വിസിൽബ്ലോവർക്കെതിരായ വ്യക്തിപരമായ പ്രതികാരം (T7).
- വിസിൽബ്ലോവർക്ക് പിൻവലിക്കാനുള്ള സമ്മർദ്ദം (T4, FACT 07 വഴി ഭാഗികമായി ലഘൂകരിച്ചു, ഇത് നിശ്ശബ്ദ പിൻവലിക്കൽ പൊതുവായി കണ്ടെത്താനാകുന്നതാക്കുന്നു).
- വെളിപ്പെടുത്തലിനായി തന്നെ വിസിൽബ്ലോവർക്കെതിരായ നിയമപരമായ നടപടി (T5).
- വിസിൽബ്ലോവറുടെ സ്വഭാവത്തെ അപകീർത്തിപ്പെടുത്തൽ (T8).
- ഇടനിലക്കാർ കഥ നിശ്ശബ്ദമായി ഉപേക്ഷിക്കൽ (T10).

ഈ കിറ്റ് ഉപയോഗിക്കാൻ ആലോചിക്കുന്ന വിസിൽബ്ലോവർ ആണെങ്കിൽ, **ആദ്യം ഈ README മുഴുവൻ വായിക്കുക**. പ്രോട്ടോക്കോൾ ഒരു ഉപകരണമാണ്. പ്രോട്ടോക്കോൾ ഒരു നിയമോപദേശകൻ, പത്രപ്രവർത്തകൻ, റെഗുലേറ്റർ, അല്ലെങ്കിൽ ശ്രദ്ധാപൂർവമായ പ്രവർത്തന സുരക്ഷയ്ക്ക് പകരമല്ല.

## ഈ കിറ്റ് എന്ത് ഉത്പാദിപ്പിക്കുന്നു

ക്രിപ്റ്റോഗ്രാഫിക്കലി മുദ്രവെച്ച തെളിവിന്റെ *മാനിഫെസ്റ്റ്* (തെളിവ് തന്നെയല്ല) ഇത് തെളിയിക്കുന്നു:

1. ഓരോ തെളിവ് ഫയലിന്റെയും നിർദ്ദിഷ്ട ബൈറ്റുകൾ മുദ്രവെച്ച നിമിഷത്തിൽ നിലവിലുണ്ടായിരുന്നു.
2. മുദ്രവെക്കൽ ഒരു നിർദ്ദിഷ്ട സമയത്തോ അതിന് മുമ്പോ സംഭവിച്ചു (Bitcoin OTS സാക്ഷ്യപ്പെടുത്തലിലൂടെ).
3. ഉത്ഭവം, ആരോപണം, ഭീഷണി മാതൃക, സ്വീകർത്താവിന്റെ പ്രോട്ടോക്കോൾ, നിശ്ശബ്ദ-പിൻവലിക്കൽ-ഇല്ലാത്ത പ്രതിജ്ഞാബദ്ധത എന്നിവയെല്ലാം ഒരുമിച്ച് പ്രഖ്യാപിച്ചു.

നിങ്ങൾ അവ ഒരു സ്വീകർത്താവിന് അയക്കുന്നതുവരെ അടിസ്ഥാന രേഖകൾ നിങ്ങളുടെ കൈവശം തന്നെ ഇരിക്കും. നിങ്ങൾ മുദ്രവെച്ച അതേ ബൈറ്റുകൾ തന്നെ ലഭിച്ചുവെന്ന് സ്വീകർത്താക്കൾ പരിശോധിക്കുന്നു.

## ഈ കിറ്റ് എപ്പോൾ ഉപയോഗിക്കണം

ഒരു പത്രപ്രവർത്തകന്, റെഗുലേറ്റർക്ക്, അല്ലെങ്കിൽ അഭിഭാഷകന് തെളിവുകൾ കൈമാറുന്നതിന് **മുമ്പ്** ഒരു മാനിഫെസ്റ്റ് മുദ്രവെക്കാൻ നിങ്ങൾ ആലോചിക്കണം, ഇനിപ്പറയുന്നവയെല്ലാം ശരിയാണെങ്കിൽ:

- ബൈറ്റ്വൈസ് ആധികാരികമായി സംരക്ഷിക്കാൻ നിങ്ങൾ ആഗ്രഹിക്കുന്ന ഡിജിറ്റൽ രൂപത്തിലുള്ള (PDFs, ഇമെയിലുകൾ, ചിത്രങ്ങൾ മുതലായവ) തെളിവുകൾ നിങ്ങൾക്കുണ്ട്.
- ഇടനിലക്കാർ തെളിവുകൾ മാറ്റാം, തർക്കിക്കാം, അല്ലെങ്കിൽ കുഴിച്ചുമൂടാം എന്ന് നിങ്ങൾ പ്രതീക്ഷിക്കുന്നു.
- കൈമാറുന്നതുവരെ അടിസ്ഥാന രേഖകൾ നിങ്ങളുടെ കൈവശം സൂക്ഷിക്കാൻ കഴിയും.
- രേഖകളുടെ ഉള്ളടക്കം വെളിപ്പെടുത്താതെ പൊതുവായി മാനിഫെസ്റ്റ് പ്രസിദ്ധീകരിക്കാൻ കഴിയും (ഹാഷുകൾ മാത്രമേ പൊതുവായുള്ളൂ).
- വിസിൽബ്ലോവർ എന്ന നിലയിൽ പൊതുവായി തിരിച്ചറിയപ്പെടാനുള്ള അപകടസാധ്യതകൾ നിങ്ങൾ തൂക്കിനോക്കിയിട്ടുണ്ട് (സിഗ്നിംഗ്-പ്രിൻസിപ്പൽ ഫീൽഡ്). അജ്ഞാതമാണെങ്കിൽ, എല്ലാ ബന്ധപ്പെട്ട പ്രസിദ്ധീകരണങ്ങളിലും സ്ഥിരമായി ഒരു ഓമനപ്പേര് ഉപയോഗിക്കുക.
- നിങ്ങളുടെ അധികാരപരിധിയിലെ വിസിൽബ്ലോവർ-സംരക്ഷണ നിയമത്തെക്കുറിച്ച് നിങ്ങൾ ഒരു അഭിഭാഷകനെ സമീപിച്ചിട്ടുണ്ട്.

## ഈ കിറ്റ് എപ്പോൾ ഉപയോഗിക്കരുത്

ഇനിപ്പറയുന്ന സന്ദർഭങ്ങളിൽ ഈ കിറ്റ് ഉപയോഗിക്കരുത്:

- രേഖകളിൽ തന്നെ മൂന്നാം കക്ഷികളെക്കുറിച്ചുള്ള (രോഗികൾ, ഉപഭോക്താക്കൾ, ജീവനക്കാർ) വ്യക്തിഗത വിവരങ്ങൾ അടങ്ങിയിട്ടുണ്ടെങ്കിൽ, അത് പൊതുവായി ഹാഷ് ചെയ്യാൻ പാടില്ല. ഒറിജിനലിന് പകരം എഡിറ്റ് ചെയ്ത പതിപ്പ് ഹാഷ് ചെയ്യുക.
- മാനിഫെസ്റ്റ് പ്രസിദ്ധീകരിക്കുന്ന പ്രവൃത്തി തന്നെ വിസിൽബ്ലോവർ സമ്മതിച്ചിട്ടില്ലാത്ത രീതിയിൽ അവരെ തിരിച്ചറിയും.
- നിങ്ങൾ ഇതുവരെ ഒരു അഭിഭാഷകനെ സമീപിച്ചിട്ടില്ല. പ്രോട്ടോക്കോൾ ഒരു ഉപകരണമാണ്; വിസിൽബ്ലോവർ-സംരക്ഷണ നിയമം അധികാരപരിധി-നിർദ്ദിഷ്ടമാണ്, അത് പ്രോട്ടോക്കോളിന് പുറത്താണ്.
- ഏതെങ്കിലും ഒറ്റ ആരോപണം പിന്നീട് പിൻവലിക്കാൻ നിങ്ങൾ ആഗ്രഹിച്ചേക്കാം. സീൽ സ്ഥിരമാണ്; സത്യസന്ധമായി പുതുക്കാനുള്ള ഏക മാർഗം പുതിയ മുദ്രവെച്ച തിരുത്തലുകളാണ്.
- നിങ്ങൾ നിർബന്ധത്തിലോ സമയ സമ്മർദ്ദത്തിലോ ഇത് ചെയ്യുന്നു. പ്രോട്ടോക്കോൾ ബലപ്രയോഗത്തെ വർദ്ധിപ്പിക്കുന്നു; വിസമ്മത ഗേറ്റ് (myriam-kit `SPEC §8`) പ്രസാധകർക്ക് ബാധകമാകുന്നതുപോലെ വിസിൽബ്ലോവർമാർക്കും ബാധകമാണ്.

## യഥാർത്ഥ തെളിവ് ഫയലുകൾക്കായി ഹാഷുകൾ കണക്കാക്കൽ

SHA-256 ഉപയോഗിച്ച് ഒരു രേഖ ഫയൽ ഹാഷ് ചെയ്യാൻ:

```bash
shasum -a 256 my_document.pdf
```

ആദ്യത്തെ 64 ഹെക്സ് അക്ഷരങ്ങൾ SHA-256 ആണ്. അത് മാനിഫെസ്റ്റിൽ ഇടുക.

നിങ്ങൾക്ക് ഒരേസമയം പല ഫയലുകൾ ഹാഷ് ചെയ്യാം:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

ഇത് `facts/03_evidence_manifest.txt` നുള്ള ഡാറ്റ നൽകുന്നു.

## പ്രവർത്തന സുരക്ഷാ കുറിപ്പുകൾ

MYRIAM പ്രോട്ടോക്കോൾ പ്രവർത്തന സുരക്ഷ നൽകുന്നില്ല. അത് *തെളിവ് സ്ഥിരത* നൽകുന്നു. നിങ്ങൾക്ക് ഇപ്പോഴും ആവശ്യമാണ്:

- നിങ്ങളുടെ അഭിഭാഷകന് സുരക്ഷിത ആശയവിനിമയ ചാനൽ ([Signal](https://signal.org/), നേരിട്ട്, അല്ലെങ്കിൽ നിങ്ങളുടെ അഭിഭാഷകന്റെ സ്ഥാപിത സുരക്ഷിത ചാനൽ)
- അടിസ്ഥാന രേഖകൾക്ക് സുരക്ഷിത സ്റ്റോറേജ് ലൊക്കേഷൻ (വിശ്രമത്തിൽ എൻക്രിപ്റ്റ് ചെയ്തത്, നിങ്ങളുടെ തൊഴിലുടമയ്ക്ക് ആക്സസ് ചെയ്യാൻ കഴിയുന്ന ക്ലൗഡ് സേവനങ്ങളിൽ അല്ല)
- സീലിംഗിനായി ശുദ്ധമായ ഉപകരണം (തൊഴിലുടമ അക്കൗണ്ടുകളുമായി ബന്ധമില്ലാത്ത ഒരു വ്യക്തിഗത ലാപ്ടോപ്പ്; മികച്ചത് ഈ ആവശ്യത്തിനു മാത്രം ഉപയോഗിച്ച ഒന്ന്)
- പ്രസിദ്ധീകരണത്തിനായി ശുദ്ധമായ നെറ്റ്‌വർക്ക് (നിങ്ങളുടെ തൊഴിലുടമയുടെ നെറ്റ്‌വർക്കല്ല, നിങ്ങളുടെ തൊഴിലുടമയ്ക്ക് നിങ്ങളുടെ ISP-യെ സബ്പോണ ചെയ്യാൻ കഴിയുമെങ്കിൽ നിങ്ങളുടെ വീട്ടിലെ IP-യും അല്ല)

വിസിൽബ്ലോവർമാർക്കായുള്ള പ്രവർത്തന സുരക്ഷയെക്കുറിച്ച് ശുപാർശ ചെയ്യപ്പെടുന്ന വായന:

- [The Intercept's SecureDrop documentation](https://securedrop.org/)
- [Freedom of the Press Foundation's resources](https://freedom.press/)
- [EFF's surveillance self-defense guide](https://ssd.eff.org/)

സീലിംഗ് പ്രക്രിയ ആരംഭിക്കുന്നതിന് മുമ്പ് അവയെ സമീപിക്കുക. പ്രോട്ടോക്കോൾ ആവാഹിക്കുന്നതിന് മുമ്പ് എടുത്ത മോശം ഓപ്സെക് തിരഞ്ഞെടുപ്പുകൾക്കെതിരെ പ്രോട്ടോക്കോൾ സംരക്ഷിക്കുന്നില്ല.

## സ്വീകർത്താക്കൾ എന്ത് അറിയണം

മുദ്രവെച്ച മാനിഫെസ്റ്റിനൊപ്പം സ്വീകർത്താക്കൾക്ക് ഈ README അയക്കുക. മാനിഫെസ്റ്റിൽ തന്നെ സ്വീകർത്താവിന്റെ പ്രോട്ടോക്കോൾ (FACT 06) അടങ്ങിയിട്ടുണ്ട്, എന്നാൽ README സ്വീകർത്താവിന്റെ പങ്ക് വിശദമാക്കുന്നു:

- **പത്രപ്രവർത്തകർ**: ആദ്യം മാനിഫെസ്റ്റിന്റെ സ്വന്തം സമഗ്രത പരിശോധിക്കുക. തുടർന്ന് ഒരു പ്രത്യേക സുരക്ഷിത ചാനലിലൂടെ രേഖകൾ അഭ്യർത്ഥിക്കുക. പ്രസിദ്ധീകരണത്തിന് മുമ്പ് ഓരോ രേഖയുടെയും SHA-256 മാനിഫെസ്റ്റിനോട് പൊരുത്തപ്പെടുന്നുവെന്ന് പരിശോധിക്കുക. വിസിൽബ്ലോവറുടെ സമ്മതമില്ലാതെ രേഖകൾ പ്രസിദ്ധീകരിക്കരുത്; മുദ്രവെച്ച മാനിഫെസ്റ്റിന്റെ അസ്തിത്വവും നിങ്ങളുടെ പരിശോധനാ നിഗമനവും പ്രസിദ്ധീകരിക്കുക.
- **റെഗുലേറ്റർമാർ**: അതേ പരിശോധനാ പ്രക്രിയ. ഔപചാരിക കസ്റ്റഡി-ചെയിൻ ശേഖരണത്തിന് പുറത്ത് ലഭ്യമായ ഏറ്റവും ശക്തമായ തെളിവ് രൂപമാണ് മുദ്രവെച്ച മാനിഫെസ്റ്റും പൊരുത്തപ്പെടുന്ന രേഖകളും.
- **അഭിഭാഷകർ**: മുദ്രവെച്ച മാനിഫെസ്റ്റ് അവകാശപ്പെട്ട സമയത്ത് രേഖകളുടെ അസ്തിത്വത്തിന് ഫോറൻസിക് ആങ്കറിംഗ് നൽകുന്നു. ഇത് ഏതെങ്കിലും പ്രത്യേക അധികാരപരിധിയുടെ തെളിവ് നിയമങ്ങളെ തൃപ്തിപ്പെടുത്തുന്നുണ്ടോ എന്നത് നിങ്ങൾ നിർണ്ണയിക്കേണ്ടതാണ്.

## ഈ കിറ്റ് എന്തല്ല

- നിയമോപദേശത്തിന് പകരമല്ല.
- ആരെങ്കിലും നിങ്ങളുടെ വെളിപ്പെടുത്തലിന്മേൽ പ്രവർത്തിക്കുമെന്നതിന്റെ ഉറപ്പല്ല.
- പ്രതികാരത്തിനെതിരായ സംരക്ഷണമല്ല.
- രേഖകളുടെ ഉള്ളടക്കങ്ങളുടെ ആധികാരികതയല്ല (മുദ്രവെച്ച സമയത്തെ ബൈറ്റുകളുടേത് മാത്രം).
- ആരുമായും അജ്ഞാതമായി ആശയവിനിമയം നടത്താനുള്ള മാർഗമല്ല — സീൽ പൊതുവാണ്; നിങ്ങളുടെ സിഗ്നിംഗ്-പ്രിൻസിപ്പൽ പേര് യഥാർത്ഥമാണെങ്കിൽ, നിങ്ങൾ തിരിച്ചറിയപ്പെടും.

## സമാപനം

മുദ്രവെച്ച മാനിഫെസ്റ്റ് ഒരു ചെറിയ കാര്യമാണ്. ഇത് ഒരു പ്രത്യേക സമയത്ത് ഒരു പ്രത്യേക രൂപത്തിൽ വലിയ ബൈറ്റ് സ്ട്രീമുകളുടെ ഒരു സെറ്റ് നിലവിലുണ്ടായിരുന്നു എന്ന് തെളിയിക്കുന്ന ഒരു ബൈറ്റ് സ്ട്രീമാണ്. ആ ചെറുപ്പമാണ് അതിന്റെ ശക്തി: അത് ഒരു കാര്യം മാത്രം ചെയ്യുന്നു, Bitcoin ബ്ലോക്ക്‌ചെയിന്റെ ജീവിതകാലത്തിലുടനീളം ആ ഒരു കാര്യം ചെയ്യുന്നു.

ഒരു വിസിൽബ്ലോവർ ആകാൻ ആലോചിക്കുന്നതുകൊണ്ടാണ് നിങ്ങൾ ഇത് വായിക്കുന്നതെങ്കിൽ, ദയവായി: ആദ്യം ഒരു അഭിഭാഷകനോട് സംസാരിക്കുക. മുകളിലുള്ള പ്രവർത്തന സുരക്ഷാ ഗൈഡുകൾ ഉപയോഗിക്കുക. തുടർന്ന്, ആ എല്ലാ മേലോട്ടുള്ള പരിഗണനകളും ക്രമത്തിലാണെങ്കിൽ മാത്രം, ഈ കിറ്റിനെ നിങ്ങളുടെ വെളിപ്പെടുത്തലിന്റെ ക്രിപ്റ്റോഗ്രാഫിക് നട്ടെല്ലായി പരിഗണിക്കുക. ക്രിപ്റ്റോ എളുപ്പമുള്ള ഭാഗമാണ്. ക്രിപ്റ്റോയ്ക്ക് മേലെ എല്ലാം യഥാർത്ഥ അപകടം വസിക്കുന്നിടത്താണ്.
