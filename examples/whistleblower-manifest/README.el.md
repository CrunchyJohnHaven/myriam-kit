# Κιτ Σφράγισης Δηλώσεως Καταγγέλλοντος

Ένα παράδειγμα myriam-kit έτοιμο προς χρήση για καταγγέλλοντες
(whistleblowers) που αντιμετωπίζουν τη συγκεκριμένη μορφή αποτυχίας
όπου **τα στοιχεία θάβονται, τροποποιούνται ή αμφισβητούνται από
ενδιάμεσους μεταξύ του καταγγέλλοντος και του κοινού**. Οι κανονικές
ιστορικές περιπτώσεις έναντι των οποίων αυτό αμύνεται:

- Boeing 737 MAX (οι εσωτερικές αναφορές ασφαλείας
  υποτιμήθηκαν/θάφτηκαν)
- Theranos (οι ανησυχίες περί εργαστηριακών εξετάσεων καταπιέστηκαν
  για χρόνια)
- Αναφορές επιθεωρητών του FDA επεξεργάστηκαν μετά την υποβολή
- Frances Haugen / Facebook (μία σφραγισμένη δήλωση θα είχε
  αποδείξει ότι τα έγγραφα ήταν αυθεντικά και υπήρχαν τη
  διεκδικούμενη στιγμή)

Έναντι τίνος αμύνεται αυτό το κιτ (με όρους πρωτοκόλλου):

- T1–T3 στο [FACT 05](facts/05_threat_model.txt): δημόσια
  αμφισβήτηση του περιεχομένου των εγγράφων, ισχυρισμοί ότι τα
  έγγραφα δεν ήταν σφραγισμένα τη διεκδικούμενη στιγμή, ισχυρισμοί
  αναδρομικής κατασκευής.
- T9: τρίτοι που παραδίδουν στους δημοσιογράφους αλλοιωμένα
  αντίγραφα.

Έναντι τίνος **δεν** αμύνεται αυτό το κιτ:

- Προσωπικά αντίποινα κατά του καταγγέλλοντος (T7).
- Πίεση στον καταγγέλλοντα να ανακαλέσει (T4, εν μέρει μετριάζεται
  από το FACT 07 που καθιστά την σιωπηρή ανάκληση δημοσίως
  ανιχνεύσιμη).
- Νομικές ενέργειες κατά του καταγγέλλοντος για την ίδια την
  αποκάλυψη (T5).
- Δυσφήμηση του χαρακτήρα του καταγγέλλοντος (T8).
- Ενδιάμεσοι που σιωπηρά εγκαταλείπουν την ιστορία (T10).

Εάν είστε καταγγέλλων που εξετάζετε τη χρήση αυτού του κιτ,
**διαβάστε ολόκληρο αυτό το README πρώτα**. Το πρωτόκολλο είναι
ένα εργαλείο. Το πρωτόκολλο δεν αντικαθιστά νομικό σύμβουλο,
δημοσιογράφο, ρυθμιστική αρχή ή προσεκτική επιχειρησιακή ασφάλεια.

## Τι παράγει αυτό το κιτ

Μία κρυπτογραφικά σφραγισμένη *δήλωση* στοιχείων (όχι τα ίδια τα
στοιχεία) που αποδεικνύει:

1. Τα συγκεκριμένα bytes κάθε αρχείου στοιχείων υπήρχαν τη στιγμή
   της σφράγισης.
2. Η σφράγιση συνέβη κατά ή πριν από μία συγκεκριμένη στιγμή (μέσω
   βεβαίωσης Bitcoin OTS).
3. Η προέλευση, ο ισχυρισμός, το μοντέλο απειλών, το πρωτόκολλο
   παραλήπτη και η δέσμευση μη-σιωπηρής-ανάκλησης δηλώθηκαν όλα
   μαζί.

Τα υποκείμενα έγγραφα παραμένουν στην κατοχή σας μέχρι να τα
στείλετε σε έναν παραλήπτη. Οι παραλήπτες επαληθεύουν ότι έλαβαν
τα ίδια bytes που σφραγίσατε.

## Πότε να χρησιμοποιήσετε αυτό το κιτ

Πρέπει να εξετάσετε τη σφράγιση μιας δήλωσης **πριν** μεταδώσετε
στοιχεία σε δημοσιογράφο, ρυθμιστική αρχή ή δικηγόρο, εάν όλα τα
ακόλουθα ισχύουν:

- Έχετε στοιχεία σε ψηφιακή μορφή (PDF, e-mail, εικόνες κ.λπ.) που
  θέλετε να διατηρηθούν ως αυθεντικά κατά byte.
- Αναμένετε ότι οι ενδιάμεσοι μπορεί να αλλοιώσουν, να
  αμφισβητήσουν ή να θάψουν τα στοιχεία.
- Μπορείτε να κρατήσετε τα υποκείμενα έγγραφα στην κατοχή σας μέχρι
  τη μετάδοση.
- Μπορείτε να δημοσιεύσετε τη δήλωση δημοσίως **χωρίς** να
  αποκαλύψετε το περιεχόμενο των εγγράφων (μόνο τα hash είναι
  δημόσια).
- Έχετε σταθμίσει τους κινδύνους να γίνετε δημοσίως αναγνωρίσιμοι
  ως καταγγέλλοντες (το πεδίο signing-principal). Εάν είστε
  ανώνυμοι, χρησιμοποιήστε ένα ψευδώνυμο σταθερά σε όλες τις
  σχετικές δημοσιεύσεις.
- Έχετε συμβουλευτεί δικηγόρο σχετικά με τον νόμο προστασίας
  καταγγελλόντων στη δικαιοδοσία σας.

## Πότε ΔΕΝ πρέπει να χρησιμοποιήσετε αυτό το κιτ

ΜΗΝ χρησιμοποιείτε αυτό το κιτ εάν:

- Τα ίδια τα έγγραφα περιέχουν προσωπικές πληροφορίες σχετικά με
  τρίτους (ασθενείς, πελάτες, εργαζομένους) που δεν πρέπει να
  γίνουν δημοσίως κατακερματίσιμα (hash-able). Κατακερματίστε
  μία αποσαφηνισμένη έκδοση, όχι το πρωτότυπο.
- Η ίδια η πράξη δημοσίευσης της δήλωσης θα αναγνώριζε τον
  καταγγέλλοντα με τρόπο για τον οποίο δεν έχει δώσει
  συγκατάθεση.
- Δεν έχετε συμβουλευτεί ακόμα δικηγόρο. Το πρωτόκολλο είναι ένα
  εργαλείο· ο νόμος προστασίας καταγγελλόντων είναι ειδικός κατά
  δικαιοδοσία και ζει εκτός του πρωτοκόλλου.
- Μπορεί να θέλετε αργότερα να ανακαλέσετε οποιονδήποτε
  μεμονωμένο ισχυρισμό. Η σφραγίδα είναι μόνιμη· νέες σφραγισμένες
  διορθώσεις είναι ο μόνος ειλικρινής τρόπος αναθεώρησης.
- Το κάνετε αυτό υπό εξαναγκασμό ή χρονική πίεση. Το πρωτόκολλο
  ενισχύει τον εξαναγκασμό· η πύλη άρνησης (myriam-kit `SPEC §8`)
  ισχύει για τους καταγγέλλοντες όπως και για τους εκδότες.

## Πώς να κάνετε fork αυτό το κιτ για μία πραγματική αποκάλυψη

```bash
# 1. Αντιγράψτε αυτό το παράδειγμα σε έναν νέο κατάλογο
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. Αφαιρέστε το φανταστικό περιεχόμενο· αντικαταστήστε με τα πραγματικά σας γεγονότα.
# Κάθε αρχείο κάτω από το facts/ είναι ένα γεγονός. Διαβάστε καθένα και αντικαταστήστε το.
$EDITOR facts/01_example_notice.txt   # αλλάξτε σε μία πραγματική δήλωση εξωφύλλου
$EDITOR facts/02_provenance.txt       # πραγματική προέλευση των στοιχείων σας
$EDITOR facts/03_evidence_manifest.txt # κατακερματίστε τα πραγματικά σας έγγραφα και απαριθμήστε τα
$EDITOR facts/04_allegation_summary.txt # ο πραγματικός σας ισχυρισμός
$EDITOR facts/05_threat_model.txt     # οι πραγματικές αναμενόμενες αποκρίσεις σας
$EDITOR facts/06_recipient_protocol.txt # προσαρμόστε για τον παραλήπτη σας
$EDITOR facts/07_no_retraction_commitment.txt # η δήλωση μη-ανάκλησης σας
$EDITOR facts/08_signing_principal.txt # η πραγματική ταυτότητα υπογραφής σας (ή ψευδώνυμο)

# 3. Ενημερώστε το site/index.html ώστε να αντικατοπτρίζει τα πραγματικά γεγονότα.
# Το script seal.sh ΔΕΝ συμπληρώνει αυτόματα το HTML· η χειροκίνητη
# επεξεργασία είναι το βήμα ανθρώπινης αναθεώρησης.
$EDITOR site/index.html

# 4. Σφραγίστε.
bash build/seal.sh
# Έξοδοι: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. Επαληθεύστε τοπικά.
python3 build/verify.py

# 6. Δημοσιεύστε τη δήλωση (ΟΧΙ τα έγγραφα) σε οποιοδήποτε από τα:
#    - Έναν στατικό host που ελέγχετε (Cloudflare Pages, GitHub Pages κ.λπ.)
#    - Το δίκτυο IPFS στο εκτυπωμένο CID
#    - Και τα δύο (συνιστάται)

# 7. Ξεχωριστά, μεταδώστε τα υποκείμενα έγγραφα στον παραλήπτη
#    μέσω ενός ασφαλούς καναλιού της επιλογής σας (Signal, SecureDrop κ.λπ.).
#    Ο παραλήπτης επαληθεύει τα hash των αρχείων έναντι της δήλωσής σας.
```

## Υπολογισμός hash για τα πραγματικά αρχεία στοιχείων

Για να κατακερματίσετε ένα αρχείο εγγράφου με SHA-256:

```bash
shasum -a 256 my_document.pdf
```

Οι πρώτοι 64 δεκαεξαδικοί χαρακτήρες είναι το SHA-256. Βάλτε το
αυτό στη δήλωση.

Μπορείτε να κατακερματίσετε πολλά αρχεία ταυτόχρονα:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

Αυτό σας δίνει τα δεδομένα για το `facts/03_evidence_manifest.txt`.

## Σημειώσεις επιχειρησιακής ασφάλειας

Το πρωτόκολλο MYRIAM ΔΕΝ παρέχει επιχειρησιακή ασφάλεια. Παρέχει
*ανθεκτικότητα τεκμηρίωσης*. Εξακολουθείτε να χρειάζεστε:

- Ένα ασφαλές κανάλι επικοινωνίας με τον δικηγόρο σας
  ([Signal](https://signal.org/), δια ζώσης, ή το καθιερωμένο
  ασφαλές κανάλι του δικηγόρου σας)
- Μία ασφαλή τοποθεσία αποθήκευσης για τα υποκείμενα έγγραφα
  (κρυπτογραφημένα σε ηρεμία, όχι σε υπηρεσίες cloud που μπορεί να
  έχει πρόσβαση ο εργοδότης σας)
- Μία καθαρή συσκευή για τη σφράγιση (έναν προσωπικό φορητό
  υπολογιστή που δεν συνδέεται με λογαριασμούς εργοδότη· κατά
  προτίμηση έναν που χρησιμοποιείται μόνο για αυτόν τον σκοπό)
- Ένα καθαρό δίκτυο για τη δημοσίευση (όχι το δίκτυο του εργοδότη
  σας, όχι το οικιακό σας IP εάν ο εργοδότης σας ενδέχεται να
  κλητεύσει τον ISP σας)

Συνιστώμενη ανάγνωση σχετικά με την επιχειρησιακή ασφάλεια για
καταγγέλλοντες:

- [Τεκμηρίωση SecureDrop του The Intercept](https://securedrop.org/)
- [Πόροι του Freedom of the Press Foundation](https://freedom.press/)
- [Οδηγός αυτο-άμυνας επιτήρησης του EFF](https://ssd.eff.org/)

Συμβουλευτείτε τους ΠΡΙΝ ξεκινήσετε τη διαδικασία σφράγισης. Το
πρωτόκολλο δεν προστατεύει από κακές επιλογές opsec που έγιναν
πριν επικληθεί το πρωτόκολλο.

## Τι πρέπει να γνωρίζουν οι παραλήπτες

Στείλτε στους παραλήπτες αυτό το README μαζί με τη σφραγισμένη
δήλωση. Η ίδια η δήλωση περιέχει το πρωτόκολλο παραλήπτη (FACT 06),
αλλά το README επεκτείνει τον ρόλο του παραλήπτη:

- **Δημοσιογράφοι**: επαληθεύστε πρώτα την ίδια την ακεραιότητα της
  δήλωσης. Στη συνέχεια ζητήστε τα έγγραφα μέσω ξεχωριστού
  ασφαλούς καναλιού. Επαληθεύστε ότι το SHA-256 κάθε εγγράφου
  ταιριάζει με τη δήλωση πριν τη δημοσίευση. ΜΗΝ δημοσιεύετε τα
  έγγραφα χωρίς τη συγκατάθεση του καταγγέλλοντος· δημοσιεύστε
  την ύπαρξη της σφραγισμένης δήλωσης και το συμπέρασμα
  επαλήθευσής σας.
- **Ρυθμιστικές αρχές**: ίδια διαδικασία επαλήθευσης. Η σφραγισμένη
  δήλωση συν τα έγγραφα που ταιριάζουν είναι η ισχυρότερη
  διαθέσιμη μορφή τεκμηρίωσης εκτός της επίσημης συλλογής
  αλυσίδας επιμέλειας.
- **Δικηγόροι**: η σφραγισμένη δήλωση παρέχει εγκληματολογική
  αγκύρωση για την ύπαρξη των εγγράφων τη διεκδικούμενη στιγμή.
  Εάν ικανοποιεί τους κανόνες απόδειξης οποιασδήποτε συγκεκριμένης
  δικαιοδοσίας, αυτό είναι για να το προσδιορίσετε.

## Τι ΔΕΝ είναι αυτό το κιτ

- Δεν είναι υποκατάστατο νομικού συμβούλου.
- Δεν είναι εγγύηση ότι κάποιος θα ενεργήσει για την αποκάλυψή
  σας.
- Δεν είναι προστασία έναντι αντιποίνων.
- Δεν είναι αυθεντικοποίηση του περιεχομένου των εγγράφων (μόνο
  των bytes-τη-στιγμή-της-σφράγισης).
- Δεν είναι τρόπος ανώνυμης επικοινωνίας με κανέναν — η σφραγίδα
  είναι δημόσια· εάν το όνομα signing-principal σας είναι
  πραγματικό, αναγνωρίζεστε.

## Κλείσιμο

Μία σφραγισμένη δήλωση είναι ένα μικρό πράγμα. Είναι ένα ρεύμα
bytes που αποδεικνύει ότι ένα μεγαλύτερο σύνολο ρευμάτων bytes
υπήρχε σε συγκεκριμένη μορφή σε συγκεκριμένη στιγμή. Αυτή η
μικρότητα είναι η δύναμή της: κάνει μόνο ένα πράγμα, και κάνει
αυτό το ένα πράγμα σε όλη τη διάρκεια ζωής του blockchain του
Bitcoin.

Εάν διαβάζετε αυτό επειδή σκέφτεστε να γίνετε καταγγέλλοντες,
παρακαλώ: μιλήστε πρώτα σε έναν δικηγόρο. Χρησιμοποιήστε τους
οδηγούς επιχειρησιακής ασφάλειας παραπάνω. Στη συνέχεια, εάν και
μόνο εάν όλες αυτές οι εκ των άνω εκτιμήσεις είναι σε τάξη,
εξετάστε αυτό το κιτ ως την κρυπτογραφική σπονδυλική στήλη της
αποκάλυψής σας. Η κρυπτογραφία είναι το εύκολο μέρος. Όλα όσα
βρίσκονται πάνω από την κρυπτογραφία είναι εκεί όπου ζει ο
πραγματικός κίνδυνος.
