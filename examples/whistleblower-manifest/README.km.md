# ឧបករណ៍បោះត្រាបញ្ជីភស្តុតាងរបស់អ្នកប្តឹងបង្ហើរ (Whistleblower Manifest Seal Kit)

ឧទាហរណ៍ myriam-kit រួចជាស្រេច សម្រាប់អ្នកប្តឹងបង្ហើរប្រឈមនឹងករណីបរាជ័យជាក់លាក់ដែល **ភស្តុតាងត្រូវបានកប់ កែប្រែ ឬប្រកែកដោយអន្តរការី រវាងអ្នកប្តឹងបង្ហើរ និងសាធារណជន**។ ករណីប្រវត្តិសាស្ត្រដែលឧបករណ៍នេះការពារ៖

- Boeing 737 MAX (របាយការណ៍សុវត្ថិភាពផ្ទៃក្នុងត្រូវបានបន្ថយតម្លៃ/កប់)
- Theranos (កង្វល់នៃការធ្វើតេស្តមន្ទីរពិសោធន៍ត្រូវបានគេបង្ក្រាបអស់រយៈពេលជាច្រើនឆ្នាំ)
- របាយការណ៍អ្នកត្រួតពិនិត្យ FDA ត្រូវបានកែប្រែបន្ទាប់ពីដាក់បញ្ជូន
- Frances Haugen / Facebook (បញ្ជីភស្តុតាងដែលបោះត្រា នឹងបានបង្ហាញថា ឯកសារពិតប្រាកដ និងមាននៅពេលដែលបានអះអាង)

អ្វីដែលឧបករណ៍នេះការពារ (តាមលក្ខខណ្ឌពិធីការ)៖

- T1–T3 នៅ [FACT 05](facts/05_threat_model.txt)៖ ការប្រកែកជាសាធារណៈអំពីខ្លឹមសារឯកសារ ការអះអាងថាឯកសារមិនត្រូវបានបោះត្រានៅពេលអះអាង ការអះអាងពីការផ្តើតក្រោយ។
- T9៖ ភាគីទីបីផ្ដល់សារព័ត៌មានជូននឹងច្បាប់ចម្លងដែលបានកែប្រែ។

អ្វីដែលឧបករណ៍នេះ **មិន** ការពារ៖

- ការសងសឹកផ្ទាល់ខ្លួនលើអ្នកប្តឹងបង្ហើរ (T7)។
- សម្ពាធលើអ្នកប្តឹងបង្ហើរឲ្យដកវិញ (T4 ត្រូវបានកាត់បន្ថយផ្នែកខ្លះដោយ FACT 07 ដែលធ្វើឲ្យការដកវិញស្ងាត់ៗអាចរកឃើញជាសាធារណៈ)។
- សកម្មភាពច្បាប់ប្រឆាំងនឹងអ្នកប្តឹងបង្ហើរសម្រាប់ការបង្ហាញខ្លួនឯង (T5)។
- ការធ្វើឲ្យឧបករណ៍កិត្តិសព្ទរបស់អ្នកប្តឹងបង្ហើរអាប់ឱន (T8)។
- អន្តរការីបោះបង់ចោលរឿងស្ងាត់ៗ (T10)។

បើអ្នកគឺជាអ្នកប្តឹងបង្ហើរដែលកំពុងពិចារណាប្រើឧបករណ៍នេះ **សូមអាន README ទាំងមូលនេះជាមុនសិន**។ ពិធីការគឺជាឧបករណ៍។ ពិធីការមិនមែនជាការជំនួសសម្រាប់មេធាវី អ្នកសារព័ត៌មាន អាជ្ញាធរ ឬសុវត្ថិភាពប្រតិបត្តិការប្រុងប្រយ័ត្នឡើយ។

## អ្វីដែលឧបករណ៍នេះផលិត

*បញ្ជី* ភស្តុតាងដែលបោះត្រាសម្ងាត់ (មិនមែនភស្តុតាងខ្លួនឯងទេ) ដែលបង្ហាញថា៖

1. បៃជាក់លាក់នៃឯកសារភស្តុតាងនីមួយៗ មាននៅពេលនៃការបោះត្រា។
2. ការបោះត្រាបានកើតឡើងនៅ ឬមុនពេលជាក់លាក់ (តាមរយៈសក្ខីភាព Bitcoin OTS)។
3. ប្រភពកំណើត ការចោទប្រកាន់ គំរូការគំរាមកំហែង ពិធីការអ្នកទទួល និងការប្តេជ្ញាគ្មាន-ការដកវិញ-ស្ងាត់ ត្រូវបានប្រកាសជាមួយគ្នាទាំងអស់។

ឯកសារមូលដ្ឋាននៅជាមួយអ្នកនៅឡើយ រហូតដល់អ្នកបញ្ជូនពួកវាទៅអ្នកទទួល។ អ្នកទទួលផ្ទៀងផ្ទាត់ថា ពួកគេបានទទួលបៃដូចគ្នាដែលអ្នកបោះត្រា។

## ពេលណាគួរប្រើឧបករណ៍នេះ

អ្នកគួរពិចារណាបោះត្រាបញ្ជី **មុនពេល** បញ្ជូនភស្តុតាងទៅអ្នកសារព័ត៌មាន អាជ្ញាធរ ឬមេធាវី ប្រសិនបើទាំងអស់ខាងក្រោមនេះពិត៖

- អ្នកមានភស្តុតាងជារូបភាពឌីជីថល (PDFs អ៊ីមែល រូបភាព ។ល។) ដែលអ្នកចង់រក្សាទុកជាបៃពិតប្រាកដ។
- អ្នករំពឹងថា អន្តរការីអាចកែប្រែ ប្រកែក ឬកប់ភស្តុតាង។
- អ្នកអាចរក្សាឯកសារមូលដ្ឋាននៅជាមួយអ្នកនៅឡើយ រហូតដល់ការបញ្ជូន។
- អ្នកអាចផ្សព្វផ្សាយបញ្ជីជាសាធារណៈ **ដោយមិន** បង្ហាញខ្លឹមសារនៃឯកសារ (មានតែហាស់ប៉ុណ្ណោះដែលជាសាធារណៈ)។
- អ្នកបានវាយតម្លៃហានិភ័យនៃការអាចសម្គាល់ខ្លួនជាសាធារណៈថាជាអ្នកប្តឹងបង្ហើរ (វាល signing-principal)។ បើអនាមិក សូមប្រើនាមប៉ាករបង់ឲ្យបានស្ថិតស្ថេរលើការផ្សព្វផ្សាយដែលពាក់ព័ន្ធទាំងអស់។
- អ្នកបានពិគ្រោះជាមួយមេធាវីអំពីច្បាប់ការការពារអ្នកប្តឹងបង្ហើរក្នុងដែនយុត្តាធិការរបស់អ្នក។

## ពេលណាមិនត្រូវប្រើឧបករណ៍នេះ

កុំប្រើឧបករណ៍នេះប្រសិនបើ៖

- ឯកសារខ្លួនវាមានព័ត៌មានផ្ទាល់ខ្លួនអំពីភាគីទីបី (អ្នកជំងឺ អតិថិជន បុគ្គលិក) ដែលមិនគួរធ្វើឲ្យអាចហាស់ជាសាធារណៈ។ ហាស់កំណែដែលបានកាត់ចេញ មិនមែនកំណែដើមឡើយ។
- សកម្មភាពនៃការផ្សព្វផ្សាយបញ្ជី ខ្លួនវានឹងសម្គាល់អ្នកប្តឹងបង្ហើរ តាមរបៀបដែលគេមិនបានយល់ព្រម។
- អ្នកមិនទាន់បានពិគ្រោះមេធាវី។ ពិធីការគឺជាឧបករណ៍ ច្បាប់ការការពារអ្នកប្តឹងបង្ហើរគឺជាក់លាក់តាមដែនយុត្តាធិការ ហើយរស់នៅខាងក្រៅពិធីការ។
- អ្នកអាចចង់ដកការចោទប្រកាន់ណាមួយក្រោយ។ ត្រាគឺអចិន្ត្រៃយ៍ ការកែតម្រូវដែលបោះត្រាថ្មីគឺជារបៀបស្មោះត្រង់តែមួយគត់ដើម្បីកែប្រែ។
- អ្នកកំពុងធ្វើនេះក្រោមការបង្ខំ ឬសម្ពាធពេលវេលា។ ពិធីការពង្រីកការបង្ខំ ច្រកបដិសេធ (myriam-kit `SPEC §8`) ត្រូវបានអនុវត្តចំពោះអ្នកប្តឹងបង្ហើរផងដែរ ដូចជាអ្នកផ្សព្វផ្សាយ។

## របៀប fork ឧបករណ៍នេះសម្រាប់ការបង្ហាញព័ត៌មានពិត

```bash
# 1. Copy this example into a new directory
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. Remove the fictional content; replace with your real facts.
# Each file under facts/ is one fact. Read each and replace.
$EDITOR facts/01_example_notice.txt   # change to a real cover statement
$EDITOR facts/02_provenance.txt       # real provenance of your evidence
$EDITOR facts/03_evidence_manifest.txt # hash your real documents and list them
$EDITOR facts/04_allegation_summary.txt # your real allegation
$EDITOR facts/05_threat_model.txt     # your real expected responses
$EDITOR facts/06_recipient_protocol.txt # adjust for your recipient
$EDITOR facts/07_no_retraction_commitment.txt # your no-retraction declaration
$EDITOR facts/08_signing_principal.txt # your real signing identity (or pseudonym)

# 3. Update site/index.html to reflect the real facts.
# The seal.sh script does NOT auto-populate the HTML; the manual edit
# is the human-review step.
$EDITOR site/index.html

# 4. Seal.
bash build/seal.sh
# Outputs: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. Verify locally.
python3 build/verify.py

# 6. Publish the manifest (NOT the documents) to any of:
#    - A static host you control (Cloudflare Pages, GitHub Pages, etc.)
#    - The IPFS network at the printed CID
#    - Both (recommended)

# 7. Separately, transmit the underlying documents to the recipient
#    via a secure channel of your choice (Signal, SecureDrop, etc.).
#    The recipient verifies file hashes against your manifest.
```

## ការគណនាហាស់សម្រាប់ឯកសារភស្តុតាងពិត

ដើម្បីហាស់ឯកសារដោយ SHA-256៖

```bash
shasum -a 256 my_document.pdf
```

តួអក្សរ hex ដំបូង 64 តួ គឺ SHA-256។ ដាក់វាក្នុងបញ្ជី។

អ្នកអាចហាស់ឯកសារច្រើនក្នុងពេលតែមួយ៖

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

នោះផ្តល់ឲ្យអ្នកនូវទិន្នន័យសម្រាប់ `facts/03_evidence_manifest.txt`។

- បណ្តាញទំនាក់ទំនងសុវត្ថិភាពទៅមេធាវីរបស់អ្នក ([Signal](https://signal.org/) ទល់មុខគ្នា ឬបណ្តាញសុវត្ថិភាពដែលបានបង្កើតរបស់មេធាវីរបស់អ្នក)

## អ្វីដែលអ្នកទទួលគួរដឹង

ផ្ញើ README នេះទៅអ្នកទទួលជាមួយបញ្ជីដែលបោះត្រា។ បញ្ជីខ្លួនវាមានពិធីការអ្នកទទួល (FACT 06) ប៉ុន្តែ README ពង្រីកតួនាទីអ្នកទទួល៖

- **អ្នកសារព័ត៌មាន**៖ ផ្ទៀងផ្ទាត់សុចរិតភាពបញ្ជីខ្លួនវាជាមុនសិន។ បន្ទាប់មក សុំឯកសារតាមរយៈបណ្តាញសុវត្ថិភាពដាច់ដោយឡែក។ ផ្ទៀងផ្ទាត់ SHA-256 របស់ឯកសារនីមួយៗ ឲ្យត្រូវនឹងបញ្ជី មុនការផ្សព្វផ្សាយ។ កុំផ្សព្វផ្សាយឯកសារដោយគ្មានការយល់ព្រមរបស់អ្នកប្តឹងបង្ហើរឡើយ ផ្សព្វផ្សាយការមាននៃបញ្ជីដែលបោះត្រា និងសេចក្តីសន្និដ្ឋាននៃការផ្ទៀងផ្ទាត់របស់អ្នក។
- **អាជ្ញាធរ**៖ ដំណើរការផ្ទៀងផ្ទាត់ដូចគ្នា។ បញ្ជីដែលបោះត្រាបូកជាមួយឯកសារដែលត្រូវគ្នា គឺជាទម្រង់ភស្តុតាងខ្លាំងបំផុតដែលមាន នៅខាងក្រៅការប្រមូលខ្សែសង្វាក់-ការគ្រប់គ្រងជាផ្លូវការ។
- **មេធាវី**៖ បញ្ជីដែលបោះត្រាផ្តល់នូវការយោងផ្នែកវេជ្ជសាស្ត្រនីតិសាស្ត្រ សម្រាប់ការមាននៃឯកសារនៅពេលដែលបានអះអាង។ ថាតើវាបំពេញច្បាប់ភស្តុតាងរបស់ដែនយុត្តាធិការជាក់លាក់ណាមួយឬអត់ គឺសម្រាប់អ្នកសម្រេច។

## អ្វីដែលឧបករណ៍នេះ មិនមែន

## ការបិទ

បញ្ជីដែលបោះត្រាគឺជារបស់តូចមួយ។ វាគឺជាស្ទ្រីមបៃមួយ ដែលបង្ហាញថា សំណុំធំជាងនៃស្ទ្រីមបៃមាននៅទម្រង់ជាក់លាក់នៅពេលជាក់លាក់។ ភាពតូចនោះគឺជាកម្លាំងរបស់វា៖ វាធ្វើតែរឿងមួយ ហើយវាធ្វើរឿងតែមួយនោះ ឆ្លងកាត់អាយុកាលនៃខ្សែបណ្តាញ Bitcoin។
