# விசில்ப்ளோயர் மேனிஃபெஸ்ட் சீல் கிட்

விசில்ப்ளோயர்களுக்காக (காட்டிக்கொடுப்பவர்களுக்காக) உள்ள ஒரு தயார்-நிலை myriam-kit உதாரணம், இது குறிப்பிட்ட தோல்வி முறையை எதிர்கொள்கிறது — **விசில்ப்ளோயருக்கும் பொதுமக்களுக்கும் இடையே உள்ள இடைத்தரகர்களால் சாட்சியம் புதைக்கப்படுதல், திருத்தப்படுதல், அல்லது மறுக்கப்படுதல்**. இந்த கிட் தற்காத்துக்கொள்ளும் நியதி வரலாற்று வழக்குகள்:

- Boeing 737 MAX (உள் பாதுகாப்பு அறிக்கைகள் புறக்கணிக்கப்பட்டன/புதைக்கப்பட்டன)
- Theranos (ஆய்வக-சோதனை கவலைகள் பல ஆண்டுகளாக அடக்கப்பட்டன)
- FDA ஆய்வாளர் அறிக்கைகள் சமர்ப்பிக்கப்பட்ட பிறகு திருத்தப்பட்டன
- Frances Haugen / Facebook (ஒரு சீல் செய்யப்பட்ட மேனிஃபெஸ்ட் ஆவணங்கள் உண்மையானவை என்றும், கூறப்பட்ட நேரத்தில் இருந்தன என்றும் நிரூபித்திருக்கும்)

நெறிமுறை மொழியில் இந்த கிட் எதிலிருந்து தற்காத்துக்கொள்கிறது:

- [FACT 05](facts/05_threat_model.txt) இல் உள்ள T1–T3: ஆவண உள்ளடக்கம் குறித்த பொது சர்ச்சை, ஆவணங்கள் கூறப்பட்ட நேரத்தில் சீல் செய்யப்படவில்லை என்ற வாதங்கள், பின்னதியிட்ட புனையல் என்ற வாதங்கள்.
- T9: மூன்றாம் தரப்பினர் பத்திரிகையாளர்களுக்கு மாற்றப்பட்ட பிரதிகளை வழங்குதல்.

இந்த கிட் எதிலிருந்து தற்காத்துக்கொள்ள **இல்லை**:

- விசில்ப்ளோயருக்கு எதிரான தனிப்பட்ட பழிவாங்கல் (T7).
- மீளப்பெறுவதற்கான விசில்ப்ளோயர் மீதான அழுத்தம் (T4, FACT 07 இனால் ஓரளவு குறைக்கப்படுகிறது, இது அமைதியான மீளப்பெறுதலை பொதுவாக கண்டறியக்கூடியதாக ஆக்குகிறது).
- வெளிப்படுத்தல் செயலுக்காகவே விசில்ப்ளோயருக்கு எதிரான சட்ட நடவடிக்கை (T5).
- விசில்ப்ளோயரின் பண்பை இழிவுபடுத்துதல் (T8).
- இடைத்தரகர்கள் கதையை அமைதியாக கைவிடுதல் (T10).

நீங்கள் ஒரு விசில்ப்ளோயராக இந்த கிட்டை பயன்படுத்த எண்ணினால், **முதலில் இந்த முழு README-ஐ படியுங்கள்**. நெறிமுறை ஒரு கருவி. நெறிமுறை ஒரு வழக்கறிஞருக்கு, பத்திரிகையாளருக்கு, ஒழுங்குமுறையாளருக்கு, அல்லது கவனமான செயல்பாட்டு பாதுகாப்பிற்கு (operational security) மாற்றாக அமையாது.

## இந்த கிட் என்ன உருவாக்குகிறது

சாட்சியத்தின் (சாட்சியம் தானல்ல) ஒரு கிரிப்டோகிராஃபிக் முறையில் சீல் செய்யப்பட்ட *மேனிஃபெஸ்ட்*, இது நிரூபிக்கிறது:

1. ஒவ்வொரு சாட்சிய கோப்பின் குறிப்பிட்ட பைட்கள் சீல் செய்த தருணத்தில் இருந்தன.
2. சீலிங் ஒரு குறிப்பிட்ட நேரத்தில் அல்லது அதற்கு முன் நிகழ்ந்தது (Bitcoin OTS சான்றுபடுத்தலின் வழியாக).
3. மூலம் (provenance), குற்றச்சாட்டு, அச்சுறுத்தல் மாதிரி, பெறுநர் நெறிமுறை, மற்றும் அமைதியான-மீளப்பெறாமை உறுதிமொழி — அனைத்தும் ஒருசேர அறிவிக்கப்பட்டன.

அடிப்படை ஆவணங்கள் நீங்கள் ஒரு பெறுநருக்கு அனுப்பும் வரை உங்கள் வசத்தில் இருக்கும். பெறுநர்கள் தங்களுக்கு கிடைத்த பைட்கள் நீங்கள் சீல் செய்தவையே என்பதை சரிபார்க்கிறார்கள்.

## இந்த கிட்டை எப்போது பயன்படுத்த வேண்டும்

ஒரு பத்திரிகையாளர், ஒழுங்குமுறையாளர், அல்லது வழக்கறிஞருக்கு சாட்சியத்தை அனுப்புவதற்கு **முன்** மேனிஃபெஸ்ட்டை சீல் செய்வதை பரிசீலிக்க வேண்டும், பின்வரும் அனைத்தும் உண்மையாக இருந்தால்:

- உங்களிடம் டிஜிட்டல் வடிவில் சாட்சியம் (PDF, மின்னஞ்சல்கள், படங்கள், போன்றவை) உள்ளது, மற்றும் அதை பைட்-வாரியாக உண்மையானதாக பாதுகாக்க விரும்புகிறீர்கள்.
- இடைத்தரகர்கள் சாட்சியத்தை மாற்றலாம், சர்ச்சைக்குரியதாக்கலாம், அல்லது புதைக்கலாம் என்று எதிர்பார்க்கிறீர்கள்.
- அடிப்படை ஆவணங்களை அனுப்புதல் வரை உங்கள் வசத்தில் வைத்திருக்க முடியும்.
- ஆவணங்களின் உள்ளடக்கத்தை வெளிப்படுத்தாமல் மேனிஃபெஸ்ட்டை பொதுவாக **வெளியிட முடியும்** (ஹேஷ்கள் மட்டுமே பொதுவானவை).
- ஒரு விசில்ப்ளோயராக பொதுவாக அடையாளம் காணப்படும் அபாயங்களை நீங்கள் எடைபோட்டிருக்கிறீர்கள் (signing-principal புலம்). அநாமதேயமாக இருந்தால், அனைத்து தொடர்புடைய வெளியீடுகளிலும் ஒரு புனைப்பெயரை (pseudonym) தொடர்ந்து பயன்படுத்துங்கள்.
- உங்கள் அதிகார வரம்பில் உள்ள விசில்ப்ளோயர்-பாதுகாப்பு சட்டம் குறித்து ஒரு வழக்கறிஞரை ஆலோசித்துள்ளீர்கள்.

## இந்த கிட்டை எப்போது பயன்படுத்த **வேண்டாம்**

இந்த கிட்டை பயன்படுத்த வேண்டாம் என்றால்:

- ஆவணங்களே மூன்றாம் தரப்பினர் பற்றிய தனிப்பட்ட தகவல்களை (நோயாளிகள், வாடிக்கையாளர்கள், ஊழியர்கள்) கொண்டுள்ளன, அவை பொது-ஹேஷ்-தகுந்ததாக ஆக்கப்படக் கூடாது. திருத்தப்பட்ட பதிப்பை ஹேஷ் செய்யுங்கள், அசலை அல்ல.
- மேனிஃபெஸ்ட்டை வெளியிடும் செயலே விசில்ப்ளோயரை அவர் ஒப்புக்கொள்ளாத வழியில் அடையாளம் காட்டும்.
- நீங்கள் இன்னும் ஒரு வழக்கறிஞரை ஆலோசிக்கவில்லை. நெறிமுறை ஒரு கருவி; விசில்ப்ளோயர்-பாதுகாப்பு சட்டம் அதிகார-வரம்பு-குறிப்பிட்டது, மற்றும் நெறிமுறைக்கு வெளியே வாழ்கிறது.
- நீங்கள் பின்னர் ஏதேனும் ஒரு குற்றச்சாட்டை மீளப்பெற விரும்பலாம். சீல் நிரந்தரமானது; புதிய சீல் செய்யப்பட்ட திருத்தங்கள் மட்டுமே திருத்துவதற்கான ஒரே நேர்மையான வழி.
- நீங்கள் இதை வற்புறுத்தலின் கீழ் அல்லது நேர அழுத்தத்தின் கீழ் செய்கிறீர்கள். நெறிமுறை வற்புறுத்தலை பெருக்குகிறது; மறுப்பு வாயில் (myriam-kit `SPEC §8`) வெளியீட்டாளர்களுக்கு பொருந்துவதைப் போலவே விசில்ப்ளோயர்களுக்கும் பொருந்தும்.

## உண்மையான வெளிப்படுத்தலுக்காக இந்த கிட்டை எவ்வாறு ஃபோர்க் செய்வது

```bash
# 1. இந்த உதாரணத்தை ஒரு புதிய கோப்பகத்திற்கு நகலெடுக்கவும்
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. கற்பனை உள்ளடக்கத்தை அகற்றவும்; உங்கள் உண்மையான உண்மைகளுடன் மாற்றவும்.
# facts/ கீழ் உள்ள ஒவ்வொரு கோப்பும் ஒரு உண்மை. ஒவ்வொன்றையும் படித்து மாற்றவும்.
$EDITOR facts/01_example_notice.txt   # உண்மையான கவர் அறிக்கையாக மாற்றவும்
$EDITOR facts/02_provenance.txt       # உங்கள் சாட்சியத்தின் உண்மையான மூலம்
$EDITOR facts/03_evidence_manifest.txt # உங்கள் உண்மையான ஆவணங்களை ஹேஷ் செய்து பட்டியலிடுங்கள்
$EDITOR facts/04_allegation_summary.txt # உங்கள் உண்மையான குற்றச்சாட்டு
$EDITOR facts/05_threat_model.txt     # உங்கள் உண்மையான எதிர்பார்க்கப்படும் பதில்கள்
$EDITOR facts/06_recipient_protocol.txt # உங்கள் பெறுநருக்கு சரிசெய்யுங்கள்
$EDITOR facts/07_no_retraction_commitment.txt # உங்கள் மீளப்பெறாத பிரகடனம்
$EDITOR facts/08_signing_principal.txt # உங்கள் உண்மையான கையெழுத்து அடையாளம் (அல்லது புனைப்பெயர்)

# 3. உண்மையான உண்மைகளை பிரதிபலிக்க site/index.html-ஐ புதுப்பிக்கவும்.
# seal.sh ஸ்கிரிப்ட் HTML-ஐ தானாக நிரப்பாது; கையேடு திருத்தம்
# மனித-மதிப்பாய்வு படி.
$EDITOR site/index.html

# 4. சீல் செய்யுங்கள்.
bash build/seal.sh
# வெளியீடுகள்: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. உள்ளூரில் சரிபார்க்கவும்.
python3 build/verify.py

# 6. மேனிஃபெஸ்ட்டை (ஆவணங்களை அல்ல) பின்வருவனவற்றில் ஏதேனும் ஒன்றில் வெளியிடுங்கள்:
#    - நீங்கள் கட்டுப்படுத்தும் ஒரு நிலையான ஹோஸ்ட் (Cloudflare Pages, GitHub Pages, போன்றவை.)
#    - அச்சிடப்பட்ட CID இல் IPFS பிணையம்
#    - இரண்டுமே (பரிந்துரைக்கப்படுகிறது)

# 7. தனியாக, அடிப்படை ஆவணங்களை பெறுநருக்கு
#    உங்கள் தேர்வின் பாதுகாப்பான சேனலின் (Signal, SecureDrop, போன்றவை.) வழியாக அனுப்புங்கள்.
#    பெறுநர் கோப்பு ஹேஷ்களை உங்கள் மேனிஃபெஸ்ட்டுக்கு எதிராக சரிபார்க்கிறார்.
```

## உண்மையான சாட்சிய கோப்புகளுக்கான ஹேஷ்களை கணக்கிடுதல்

ஒரு ஆவண கோப்பை SHA-256 உடன் ஹேஷ் செய்ய:

```bash
shasum -a 256 my_document.pdf
```

முதல் 64 ஹெக்ஸ் எழுத்துக்கள் SHA-256. அதை மேனிஃபெஸ்ட்டில் வைக்கவும்.

ஒரே நேரத்தில் பல கோப்புகளை ஹேஷ் செய்ய முடியும்:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

## செயல்பாட்டு பாதுகாப்பு குறிப்புகள்

MYRIAM நெறிமுறை செயல்பாட்டு பாதுகாப்பை வழங்கவில்லை. அது *சாட்சிய நிலைத்தன்மையை* வழங்குகிறது. உங்களுக்கு இன்னும் தேவை:

- உங்கள் வழக்கறிஞருக்கு பாதுகாப்பான தகவல்தொடர்பு சேனல்
  ([Signal](https://signal.org/), நேரில், அல்லது உங்கள் வழக்கறிஞரின் நிறுவப்பட்ட பாதுகாப்பான சேனல்)
- அடிப்படை ஆவணங்களுக்கான பாதுகாப்பான சேமிப்பு இடம் (ஓய்வில் என்க்ரிப்ட் செய்யப்பட்டது, உங்கள் முதலாளி அணுக முடியாத கிளவுட் சேவைகளில் அல்ல)
- சீலிங்கிற்கான சுத்தமான சாதனம் (முதலாளி கணக்குகளுடன் தொடர்பற்ற தனிப்பட்ட லேப்டாப்; சிறந்தபட்சம் இந்த நோக்கத்திற்காக மட்டுமே பயன்படுத்தப்படும் ஒன்று)
- வெளியீட்டிற்கான சுத்தமான பிணையம் (உங்கள் முதலாளியின் பிணையம் அல்ல, உங்கள் முதலாளி உங்கள் ISP-ஐ சப்போனா செய்ய முடிந்தால் உங்கள் வீட்டின் IP அல்ல)

விசில்ப்ளோயர்களுக்கான செயல்பாட்டு பாதுகாப்பு குறித்த பரிந்துரைக்கப்பட்ட வாசிப்பு:

- [The Intercept's SecureDrop documentation](https://securedrop.org/)
- [Freedom of the Press Foundation's resources](https://freedom.press/)
- [EFF's surveillance self-defense guide](https://ssd.eff.org/)

சீலிங் செயல்முறையைத் தொடங்குவதற்கு முன் அவற்றை ஆலோசிக்கவும். நெறிமுறை அழைக்கப்படுவதற்கு முன் செய்யப்பட்ட மோசமான ஓப்செக் (opsec) தேர்வுகளுக்கு எதிராக நெறிமுறை பாதுகாக்காது.

## பெறுநர்கள் என்ன அறிய வேண்டும்

பெறுநர்களுக்கு இந்த README-ஐ சீல் செய்யப்பட்ட மேனிஃபெஸ்ட்டுடன் அனுப்புங்கள். மேனிஃபெஸ்ட்டே பெறுநர் நெறிமுறையை (FACT 06) கொண்டுள்ளது, ஆனால் README பெறுநர் பாத்திரத்தை விரிவாக்குகிறது:

- **பத்திரிகையாளர்கள்**: முதலில் மேனிஃபெஸ்ட்டின் சொந்த ஒருமைப்பாட்டை சரிபார்க்கவும். பிறகு ஒரு தனி பாதுகாப்பான சேனல் வழியாக ஆவணங்களை கோருங்கள். வெளியிடுவதற்கு முன் ஒவ்வொரு ஆவணத்தின் SHA-256 மேனிஃபெஸ்ட்டுடன் பொருந்துகிறதா என்பதை சரிபார்க்கவும். விசில்ப்ளோயரின் ஒப்புதல் இல்லாமல் ஆவணங்களை வெளியிட வேண்டாம்; சீல் செய்யப்பட்ட மேனிஃபெஸ்ட்டின் இருப்பையும் உங்கள் சரிபார்ப்பு முடிவையும் வெளியிடுங்கள்.
- **ஒழுங்குமுறையாளர்கள்**: அதே சரிபார்ப்பு செயல்முறை. சீல் செய்யப்பட்ட மேனிஃபெஸ்ட் பொருந்தும் ஆவணங்களுடன், முறையான பாதுகாப்பு-சங்கிலி சேகரிப்புக்கு வெளியே கிடைக்கும் வலுவான சாட்சிய வடிவம்.
- **வழக்கறிஞர்கள்**: சீல் செய்யப்பட்ட மேனிஃபெஸ்ட், ஆவணங்கள் கூறப்பட்ட நேரத்தில் இருந்ததற்கான தடயவியல் நங்கூரத்தை வழங்குகிறது. ஒரு குறிப்பிட்ட அதிகார வரம்பின் சாட்சிய-விதிகளை இது நிறைவேற்றுகிறதா என்பதை நீங்கள் தீர்மானிக்க வேண்டும்.

## இந்த கிட் என்ன அல்ல

- சட்ட ஆலோசனைக்கு மாற்றாக அல்ல.
- உங்கள் வெளிப்படுத்தலின் மீது யாரேனும் நடவடிக்கை எடுப்பார்கள் என்ற உத்தரவாதம் அல்ல.
- பழிவாங்கலுக்கு எதிரான பாதுகாப்பு அல்ல.
- ஆவணங்களின் உள்ளடக்கங்களின் அங்கீகாரம் அல்ல (சீலிங் நேரத்தில் உள்ள அவற்றின் பைட்கள் மட்டுமே).
- யாருடனும் அநாமதேயமாக தொடர்பு கொள்வதற்கான வழி அல்ல — சீல் பொதுவானது; உங்கள் signing-principal பெயர் உண்மையானதாக இருந்தால், நீங்கள் அடையாளம் காணப்படுகிறீர்கள்.

## முடிவு

ஒரு சீல் செய்யப்பட்ட மேனிஃபெஸ்ட் ஒரு சிறிய விஷயம். இது ஒரு பெரிய பைட் ஓடைகளின் தொகுப்பு ஒரு குறிப்பிட்ட வடிவத்தில் ஒரு குறிப்பிட்ட நேரத்தில் இருந்ததை நிரூபிக்கும் ஒரு பைட் ஓடை. அந்த சிறுமை அதன் வலிமை: அது ஒரே ஒரு விஷயத்தை செய்கிறது, மற்றும் அது அந்த ஒரு விஷயத்தை Bitcoin blockchain-இன் வாழ்நாள் முழுவதும் செய்கிறது.

ஒரு விசில்ப்ளோயராக ஆவதை பரிசீலிப்பதால் இதை படிக்கிறீர்கள் என்றால், தயவுசெய்து: முதலில் ஒரு வழக்கறிஞருடன் பேசுங்கள். மேலே உள்ள செயல்பாட்டு பாதுகாப்பு வழிகாட்டிகளைப் பயன்படுத்துங்கள். பிறகு, மற்றும் அந்த அனைத்து மேலதிக பரிசீலனைகளும் ஒழுங்கில் இருந்தால் மட்டுமே, இந்த கிட்டை உங்கள் வெளிப்படுத்தலின் கிரிப்டோகிராஃபிக் முதுகெலும்பாக கருதுங்கள். கிரிப்டோ எளிய பகுதி. கிரிப்டோவிற்கு மேலே உள்ள அனைத்தும் உண்மையான ஆபத்து வாழும் இடம்.
