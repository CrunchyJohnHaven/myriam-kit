# વ્હિસલબ્લોઅર મેનિફેસ્ટ સીલ કિટ

વ્હિસલબ્લોઅરો (ગુપ્ત માહિતી ઉઘાડ પાડનારાઓ) માટે એક તૈયાર-ઉપયોગ myriam-kit ઉદાહરણ, જે ચોક્કસ નિષ્ફળતા સ્થિતિનો સામનો કરે છે — **વ્હિસલબ્લોઅર અને જનતા વચ્ચેના મધ્યસ્થીઓ દ્વારા પુરાવા દાટવા, સંપાદિત કરવા, અથવા વિવાદાસ્પદ બનાવવા**. આ કિટ જે પ્રામાણિક ઐતિહાસિક કેસોથી રક્ષણ આપે છે:

- Boeing 737 MAX (આંતરિક સુરક્ષા અહેવાલોને અવગણવામાં/દાટવામાં આવ્યા)
- Theranos (પ્રયોગશાળા-પરીક્ષણની ચિંતાઓ વર્ષો સુધી દબાવવામાં આવી)
- FDA નિરીક્ષક અહેવાલો રજૂ કર્યા પછી સંપાદિત કરવામાં આવ્યા
- Frances Haugen / Facebook (સીલ કરેલ મેનિફેસ્ટ સાબિત કરી દેત કે દસ્તાવેજો અધિકૃત હતા અને દાવા કરેલા સમયે અસ્તિત્વમાં હતા)

પ્રોટોકોલની ભાષામાં આ કિટ શાનાથી રક્ષણ આપે છે:

- [FACT 05](facts/05_threat_model.txt) માં T1–T3: દસ્તાવેજની સામગ્રીનો જાહેર વિવાદ, દસ્તાવેજો દાવા કરેલા સમયે સીલ કરવામાં આવ્યા ન હતા તેવા દાવાઓ, પાછલી તારીખે બનાવટના દાવાઓ.
- T9: ત્રીજા પક્ષો પત્રકારોને બદલાયેલી નકલો સોંપે છે.

આ કિટ શાનાથી રક્ષણ **નથી** આપતી:

- વ્હિસલબ્લોઅર સામેનો વ્યક્તિગત બદલો (T7).
- પાછું ખેંચવા માટે વ્હિસલબ્લોઅર પર દબાણ (T4, FACT 07 દ્વારા આંશિક રીતે ઓછું કરવામાં આવે છે જે મૌન પાછું ખેંચવાને જાહેરમાં શોધી શકાય તેવું બનાવે છે).
- જાહેરાતના કાર્ય માટે જ વ્હિસલબ્લોઅર સામેની કાનૂની કાર્યવાહી (T5).
- વ્હિસલબ્લોઅરના ચારિત્ર્યને બદનામ કરવું (T8).
- મધ્યસ્થીઓ વાર્તાને મૌનપૂર્વક છોડી દે છે (T10).

જો તમે વ્હિસલબ્લોઅર છો અને આ કિટનો ઉપયોગ કરવાનું વિચારી રહ્યા છો, તો **પહેલા આ આખું README વાંચો**. પ્રોટોકોલ એક સાધન છે. પ્રોટોકોલ વકીલ, પત્રકાર, નિયમનકાર, અથવા સાવધાનીપૂર્વકની કાર્યકારી સુરક્ષા (operational security)નો વિકલ્પ નથી.

## આ કિટ શું ઉત્પન્ન કરે છે

પુરાવાનો (પુરાવો પોતે નહીં) એક ક્રિપ્ટોગ્રાફિક રીતે સીલ કરેલ *મેનિફેસ્ટ* જે સાબિત કરે છે:

1. દરેક પુરાવા ફાઇલના ચોક્કસ બાઇટ્સ સીલિંગની ક્ષણે અસ્તિત્વમાં હતા.
2. સીલિંગ ચોક્કસ સમયે અથવા તે પહેલાં થયું (Bitcoin OTS પ્રમાણીકરણ દ્વારા).
3. ઉદ્ગમ (provenance), આરોપ, ધમકી મોડેલ, પ્રાપ્તકર્તા પ્રોટોકોલ, અને મૌન-પાછું-ન-લેવાની પ્રતિબદ્ધતા — બધાં એક સાથે જાહેર કરવામાં આવ્યા.

મૂળભૂત દસ્તાવેજો જ્યાં સુધી તમે તેમને પ્રાપ્તકર્તાને ન મોકલો ત્યાં સુધી તમારા કબજામાં રહે છે. પ્રાપ્તકર્તાઓ ચકાસે છે કે તેમને મળેલ બાઇટ્સ તમે સીલ કરેલા સમાન છે.

## આ કિટનો ઉપયોગ ક્યારે કરવો

પત્રકાર, નિયમનકાર, અથવા વકીલને પુરાવા મોકલતા **પહેલા** મેનિફેસ્ટ સીલ કરવાનું વિચારવું જોઈએ, જો નીચેના બધા સાચા હોય:

- તમારી પાસે ડિજિટલ સ્વરૂપમાં પુરાવા (PDFs, ઈમેઈલ, છબીઓ, વગેરે) છે જેને તમે બાઇટ-વાર અધિકૃત તરીકે સાચવવા માંગો છો.
- તમે અપેક્ષા રાખો છો કે મધ્યસ્થીઓ પુરાવાને બદલી, વિવાદ, અથવા દાટી શકે છે.
- તમે મૂળભૂત દસ્તાવેજોને પ્રસારણ સુધી તમારા કબજામાં રાખી શકો છો.
- તમે દસ્તાવેજોની સામગ્રી જાહેર કર્યા **વગર** મેનિફેસ્ટને જાહેરમાં પ્રકાશિત કરી શકો છો (ફક્ત હેશ જાહેર છે).
- તમે વ્હિસલબ્લોઅર તરીકે જાહેરમાં ઓળખવાના જોખમોને તોલ્યા છે (signing-principal ક્ષેત્ર). જો અનામી હો, તો બધાં સંબંધિત પ્રકાશનોમાં એક છદ્મ નામ (pseudonym)નો સતત ઉપયોગ કરો.
- તમે તમારા અધિકારક્ષેત્રમાં વ્હિસલબ્લોઅર-સંરક્ષણ કાયદા વિશે વકીલની સલાહ લીધી છે.

## આ કિટનો ઉપયોગ ક્યારે **ન** કરવો

આ કિટનો ઉપયોગ ન કરો જો:

- દસ્તાવેજો પોતે ત્રીજા પક્ષો (દર્દીઓ, ગ્રાહકો, કર્મચારીઓ) વિશે વ્યક્તિગત માહિતી ધરાવે છે જે જાહેર-હેશ-યોગ્ય બનાવવી જોઈએ નહીં. સંપાદિત સંસ્કરણને હેશ કરો, મૂળને નહીં.
- મેનિફેસ્ટ પ્રકાશિત કરવાનું કાર્ય જ વ્હિસલબ્લોઅરને એવી રીતે ઓળખશે જે તેમણે સંમતિ આપી નથી.
- તમે હજુ સુધી વકીલની સલાહ લીધી નથી. પ્રોટોકોલ એક સાધન છે; વ્હિસલબ્લોઅર-સંરક્ષણ કાયદો અધિકારક્ષેત્ર-વિશિષ્ટ છે, અને પ્રોટોકોલની બહાર રહે છે.
- તમે પછી કોઈ એક આરોપ પાછો લેવા માંગતા હો. સીલ કાયમી છે; નવા સીલ કરેલા સુધારા સંશોધિત કરવાનો એકમાત્ર પ્રામાણિક માર્ગ છે.
- તમે આ બળજબરી હેઠળ અથવા સમયના દબાણ હેઠળ કરી રહ્યા છો. પ્રોટોકોલ બળજબરીને વધારે છે; નકારનું દ્વાર (myriam-kit `SPEC §8`) પ્રકાશકોને જેમ વ્હિસલબ્લોઅરોને પણ લાગુ પડે છે.

## વાસ્તવિક જાહેરાત માટે આ કિટને કેવી રીતે ફોર્ક કરવી

```bash
# 1. આ ઉદાહરણને નવી ડિરેક્ટરીમાં કૉપિ કરો
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. કાલ્પનિક સામગ્રીને દૂર કરો; તમારી વાસ્તવિક હકીકતો સાથે બદલો.
# facts/ હેઠળની દરેક ફાઇલ એક હકીકત છે. દરેક વાંચો અને બદલો.
$EDITOR facts/01_example_notice.txt   # વાસ્તવિક કવર સ્ટેટમેન્ટમાં બદલો
$EDITOR facts/02_provenance.txt       # તમારા પુરાવાનું વાસ્તવિક ઉદ્ગમ
$EDITOR facts/03_evidence_manifest.txt # તમારા વાસ્તવિક દસ્તાવેજો હેશ કરો અને સૂચિબદ્ધ કરો
$EDITOR facts/04_allegation_summary.txt # તમારો વાસ્તવિક આરોપ
$EDITOR facts/05_threat_model.txt     # તમારી અપેક્ષિત વાસ્તવિક પ્રતિક્રિયાઓ
$EDITOR facts/06_recipient_protocol.txt # તમારા પ્રાપ્તકર્તા માટે સમાયોજિત કરો
$EDITOR facts/07_no_retraction_commitment.txt # તમારી પાછું ન લેવાની ઘોષણા
$EDITOR facts/08_signing_principal.txt # તમારી વાસ્તવિક હસ્તાક્ષર ઓળખ (અથવા છદ્મ નામ)

# 3. વાસ્તવિક હકીકતોને પ્રતિબિંબિત કરવા માટે site/index.html અપડેટ કરો.
# seal.sh સ્ક્રિપ્ટ HTMLને આપોઆપ ભરતી નથી; મેન્યુઅલ સંપાદન
# માનવ-સમીક્ષા પગલું છે.
$EDITOR site/index.html

bash build/seal.sh
# આઉટપુટ: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. સ્થાનિક રીતે ચકાસો.
python3 build/verify.py

# 6. મેનિફેસ્ટ (દસ્તાવેજો નહીં) આમાંથી કોઈપણમાં પ્રકાશિત કરો:
#    - તમે નિયંત્રિત કરો છો તે સ્થિર હોસ્ટ (Cloudflare Pages, GitHub Pages, વગેરે.)
#    - છાપેલા CID પર IPFS નેટવર્ક
#    - બંને (ભલામણ કરેલ)

# 7. અલગથી, મૂળભૂત દસ્તાવેજો પ્રાપ્તકર્તાને
#    તમારી પસંદગીની સુરક્ષિત ચેનલ (Signal, SecureDrop, વગેરે.) દ્વારા પ્રસારિત કરો.
#    પ્રાપ્તકર્તા તમારા મેનિફેસ્ટ સામે ફાઇલ હેશ ચકાસે છે.
```

## વાસ્તવિક પુરાવા ફાઇલો માટે હેશની ગણતરી

દસ્તાવેજ ફાઇલને SHA-256 સાથે હેશ કરવા માટે:

```bash
shasum -a 256 my_document.pdf
```

પ્રથમ 64 હેક્સ અક્ષરો SHA-256 છે. તેને મેનિફેસ્ટમાં મૂકો.

તમે એક સાથે ઘણી ફાઇલો હેશ કરી શકો છો:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

તે `facts/03_evidence_manifest.txt` માટે ડેટા આપે છે.

## કાર્યકારી સુરક્ષા નોંધો

MYRIAM પ્રોટોકોલ કાર્યકારી સુરક્ષા પ્રદાન કરતું નથી. તે *પુરાવાની ટકાઉપણું* પ્રદાન કરે છે. તમને હજુ પણ જરૂર છે:

- તમારા વકીલ માટે સુરક્ષિત સંચાર ચેનલ
  ([Signal](https://signal.org/), રૂબરૂ, અથવા તમારા વકીલની સ્થાપિત સુરક્ષિત ચેનલ)
- મૂળભૂત દસ્તાવેજો માટે સુરક્ષિત સંગ્રહ સ્થાન (આરામમાં એન્ક્રિપ્ટેડ, તમારા એમ્પ્લોયરની ઍક્સેસ હોય તેવી ક્લાઉડ સેવાઓ પર નહીં)
- સીલિંગ માટે સ્વચ્છ ઉપકરણ (એમ્પ્લોયર એકાઉન્ટ્સ સાથે સંકળાયેલ ન હોય તેવું વ્યક્તિગત લેપટોપ; પ્રાધાન્ય ફક્ત આ હેતુ માટે વાપરવામાં આવેલ)
- પ્રકાશન માટે સ્વચ્છ નેટવર્ક (તમારા એમ્પ્લોયરનું નેટવર્ક નહીં, જો તમારા એમ્પ્લોયર તમારા ISPને સબ્પોના કરી શકે તો તમારા ઘરનો IP નહીં)

વ્હિસલબ્લોઅરો માટે કાર્યકારી સુરક્ષા પર ભલામણ કરેલ વાંચન:

- [The Intercept's SecureDrop documentation](https://securedrop.org/)
- [Freedom of the Press Foundation's resources](https://freedom.press/)
- [EFF's surveillance self-defense guide](https://ssd.eff.org/)

સીલિંગ પ્રક્રિયા શરૂ કરતા પહેલા તેમની સલાહ લો. પ્રોટોકોલ બોલાવાય તે પહેલા કરેલા ખરાબ ઓપસેક (opsec) પસંદગીઓ સામે પ્રોટોકોલ રક્ષણ આપતું નથી.

## પ્રાપ્તકર્તાઓએ શું જાણવું જોઈએ

પ્રાપ્તકર્તાઓને આ README સીલ કરેલા મેનિફેસ્ટ સાથે મોકલો. મેનિફેસ્ટ પોતે પ્રાપ્તકર્તા પ્રોટોકોલ (FACT 06) ધરાવે છે, પરંતુ README પ્રાપ્તકર્તાની ભૂમિકાને વિસ્તૃત કરે છે:

- **પત્રકારો**: પ્રથમ મેનિફેસ્ટની પોતાની અખંડિતતા ચકાસો. પછી અલગ સુરક્ષિત ચેનલ દ્વારા દસ્તાવેજોની વિનંતી કરો. પ્રકાશન પહેલાં દરેક દસ્તાવેજનું SHA-256 મેનિફેસ્ટ સાથે મેળ ખાય છે કે કેમ તે ચકાસો. વ્હિસલબ્લોઅરની સંમતિ વિના દસ્તાવેજો પ્રકાશિત કરશો નહીં; સીલ કરેલા મેનિફેસ્ટના અસ્તિત્વ અને તમારી ચકાસણીના નિષ્કર્ષને પ્રકાશિત કરો.
- **નિયમનકારો**: સમાન ચકાસણી પ્રક્રિયા. સીલ કરેલા મેનિફેસ્ટ સાથે મેળ ખાતા દસ્તાવેજો ઔપચારિક ચેઇન-ઓફ-કસ્ટડી સંગ્રહ બહાર ઉપલબ્ધ સૌથી મજબૂત પુરાવાનું સ્વરૂપ છે.
- **વકીલો**: સીલ કરેલ મેનિફેસ્ટ દસ્તાવેજો દાવા કરેલા સમયે અસ્તિત્વમાં હોવા માટે ફોરેન્સિક એન્કરિંગ પ્રદાન કરે છે. તે કોઈ ચોક્કસ અધિકારક્ષેત્રના પુરાવા-નિયમોને સંતોષે છે કે કેમ તે નક્કી કરવાનું તમારું છે.

## આ કિટ શું નથી

- કાનૂની સલાહનો વિકલ્પ નથી.
- તમારી જાહેરાત પર કોઈ પગલાં લેશે તેની ગેરંટી નથી.
- બદલા સામેનું રક્ષણ નથી.
- દસ્તાવેજોની સામગ્રીનું પ્રમાણીકરણ નથી (ફક્ત સીલિંગ-સમય પરના તેમના બાઇટ્સ).
- કોઈ સાથે અનામી રીતે વાતચીત કરવાનો માર્ગ નથી — સીલ જાહેર છે; જો તમારું signing-principal નામ વાસ્તવિક હોય, તો તમે ઓળખાઈ ગયા છો.

## સમાપન

સીલ કરેલ મેનિફેસ્ટ એક નાની વસ્તુ છે. તે એક બાઇટ સ્ટ્રીમ છે જે સાબિત કરે છે કે મોટા બાઇટ સ્ટ્રીમ્સનો સમૂહ ચોક્કસ સ્વરૂપમાં ચોક્કસ સમયે અસ્તિત્વમાં હતો. તે નાનાપણું તેની તાકાત છે: તે ફક્ત એક જ કામ કરે છે, અને તે એક કામ Bitcoin blockchainના આયુષ્ય દરમ્યાન કરે છે.

જો તમે વ્હિસલબ્લોઅર બનવાનું વિચારી રહ્યા હોવાથી આ વાંચી રહ્યા છો, તો કૃપા કરીને: પ્રથમ વકીલ સાથે વાત કરો. ઉપરની કાર્યકારી સુરક્ષા માર્ગદર્શિકાઓનો ઉપયોગ કરો. પછી, અને ફક્ત જો તે બધી અપસ્ટ્રીમ વિચારણાઓ ક્રમમાં હોય, તો જ આ કિટને તમારી જાહેરાતના ક્રિપ્ટોગ્રાફિક કરોડરજ્જુ તરીકે વિચારો. ક્રિપ્ટો સરળ ભાગ છે. ક્રિપ્ટોની ઉપરનું બધું જ વાસ્તવિક જોખમ રહે છે તે જગ્યા છે.
