# Whistleblower Manifest Seal Kit

ชุดเครื่องมือตัวอย่าง myriam-kit ที่พร้อมใช้งานสำหรับผู้แจ้งเบาะแสที่
เผชิญกับโหมดความล้มเหลวเฉพาะที่ **หลักฐานถูกฝัง แก้ไข หรือโต้แย้งโดย
ตัวกลางระหว่างผู้แจ้งเบาะแสและสาธารณะ** กรณีในประวัติศาสตร์ที่
เป็นแบบฉบับซึ่งสิ่งนี้ป้องกัน:

- Boeing 737 MAX (รายงานความปลอดภัยภายในที่ถูกลดความสำคัญ/ฝังไว้)
- Theranos (ข้อกังวลเกี่ยวกับการทดสอบในห้องปฏิบัติการถูกระงับเป็นเวลาหลายปี)
- รายงานผู้ตรวจของ FDA ถูกแก้ไขหลังการส่ง
- Frances Haugen / Facebook (manifest ที่ผนึกไว้แล้วจะพิสูจน์ได้ว่า
  เอกสารเป็นของแท้และมีอยู่ในเวลาที่ระบุไว้)

สิ่งที่ชุดเครื่องมือนี้ป้องกัน (ในเชิงโพรโทคอล):

- T1–T3 ใน [FACT 05](facts/05_threat_model.txt): การโต้แย้งสาธารณะ
  เกี่ยวกับเนื้อหาเอกสาร การอ้างว่าเอกสารไม่ได้ถูกผนึกในเวลาที่ระบุ
  การอ้างว่าเป็นการประดิษฐ์ย้อนหลัง
- T9: บุคคลที่สามมอบสำเนาที่ถูกแก้ไขให้กับนักข่าว

สิ่งที่ชุดเครื่องมือนี้ **ไม่** ป้องกัน:

- การตอบโต้ส่วนบุคคลต่อผู้แจ้งเบาะแส (T7)
- ความกดดันต่อผู้แจ้งเบาะแสให้ถอนคำกล่าว (T4, บรรเทาบางส่วน
  ผ่าน FACT 07 ที่ทำให้การถอนแบบเงียบ ๆ ตรวจพบได้โดยสาธารณะ)
- การดำเนินคดีทางกฎหมายต่อผู้แจ้งเบาะแสในการเปิดเผยข้อมูลเอง (T5)
- การทำให้คุณลักษณะของผู้แจ้งเบาะแสเสื่อมเสีย (T8)
- ตัวกลางที่ปล่อยเรื่องราวอย่างเงียบ ๆ (T10)

หากคุณเป็นผู้แจ้งเบาะแสที่กำลังพิจารณาใช้ชุดเครื่องมือนี้ **โปรดอ่าน
README ทั้งหมดนี้ก่อน** โพรโทคอลเป็นเครื่องมือ โพรโทคอลไม่ใช่
สิ่งทดแทนที่ปรึกษากฎหมาย นักข่าว ผู้ควบคุม หรือความปลอดภัยในการ
ปฏิบัติงานที่รอบคอบ

## สิ่งที่ชุดเครื่องมือนี้ผลิต

*manifest* ที่ผนึกด้วยการเข้ารหัสของหลักฐาน (ไม่ใช่ตัวหลักฐานเอง) ที่
พิสูจน์ว่า:

1. ไบต์ที่เฉพาะเจาะจงของแต่ละไฟล์หลักฐานมีอยู่ในเวลาที่ทำการผนึก
2. การผนึกเกิดขึ้นในหรือก่อนเวลาที่กำหนด (ผ่านการรับรอง Bitcoin OTS)
3. แหล่งที่มา ข้อกล่าวหา แบบจำลองภัยคุกคาม โพรโทคอลผู้รับ และ
   ข้อผูกพันไม่ถอนแบบเงียบ ๆ ทั้งหมดถูกประกาศพร้อมกัน

เอกสารต้นทางยังคงอยู่ในความครอบครองของคุณจนกว่าคุณจะส่งให้กับผู้รับ
ผู้รับจะตรวจสอบว่าได้รับไบต์เดียวกับที่คุณผนึกไว้

## เมื่อใดควรใช้ชุดเครื่องมือนี้

คุณควรพิจารณาการผนึก manifest **ก่อน** ส่งหลักฐานให้กับนักข่าว
ผู้ควบคุม หรือทนายความ ถ้าเงื่อนไขทั้งหมดต่อไปนี้เป็นจริง:

- คุณมีหลักฐานในรูปแบบดิจิทัล (PDF, อีเมล, ภาพ ฯลฯ) ที่
  คุณต้องการเก็บรักษาให้คงไบต์อย่างของแท้
- คุณคาดว่าตัวกลางอาจแก้ไข โต้แย้ง หรือฝังหลักฐานไว้
- คุณสามารถเก็บเอกสารต้นทางไว้ในความครอบครองของคุณจนถึง
  การส่ง
- คุณสามารถเผยแพร่ manifest ต่อสาธารณะ **โดยไม่** เปิดเผย
  เนื้อหาของเอกสาร (เฉพาะแฮชเท่านั้นที่เป็นสาธารณะ)
- คุณได้ชั่งน้ำหนักความเสี่ยงในการกลายเป็นที่ระบุได้ในที่
  สาธารณะในฐานะผู้แจ้งเบาะแส (ฟิลด์ signing principal) หากนิรนาม
  ให้ใช้นามแฝงอย่างสม่ำเสมอในการเผยแพร่ที่เกี่ยวข้องทั้งหมด
- คุณได้ปรึกษาทนายความเกี่ยวกับกฎหมายคุ้มครองผู้แจ้งเบาะแสในเขต
  อำนาจศาลของคุณแล้ว

## เมื่อใดที่ไม่ควรใช้ชุดเครื่องมือนี้

อย่าใช้ชุดเครื่องมือนี้หาก:

- เอกสารเองมีข้อมูลส่วนบุคคลเกี่ยวกับบุคคลที่สาม (ผู้ป่วย
  ลูกค้า พนักงาน) ที่ไม่ควรถูกทำให้แฮชได้ในที่สาธารณะ
  ให้แฮชเวอร์ชันที่ถูกตัดทอน ไม่ใช่ต้นฉบับ
- การเผยแพร่ manifest เองจะระบุตัวผู้แจ้งเบาะแสในลักษณะ
  ที่พวกเขาไม่ได้ยินยอม
- คุณยังไม่ได้ปรึกษาทนายความ โพรโทคอลเป็นเครื่องมือ;
  กฎหมายคุ้มครองผู้แจ้งเบาะแสขึ้นอยู่กับเขตอำนาจศาลและอยู่
  นอกโพรโทคอล
- คุณอาจต้องการถอนข้อกล่าวหาใด ๆ ในภายหลัง การผนึกเป็น
  ถาวร; การแก้ไขที่ผนึกใหม่เป็นวิธีเดียวที่ซื่อสัตย์ในการแก้ไข
- คุณกำลังทำสิ่งนี้ภายใต้การข่มขู่หรือความกดดันด้านเวลา โพรโทคอล
  ขยายการบังคับขู่เข็ญ; การปฏิเสธ (myriam-kit `SPEC §8`) ใช้ได้
  กับผู้แจ้งเบาะแสเช่นเดียวกับผู้เผยแพร่

## วิธี fork ชุดเครื่องมือนี้สำหรับการเปิดเผยข้อมูลจริง

```bash
# 1. คัดลอกตัวอย่างนี้ไปยังไดเรกทอรีใหม่
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. ลบเนื้อหาที่สมมติ; แทนที่ด้วยข้อเท็จจริงจริงของคุณ
# แต่ละไฟล์ภายใต้ facts/ คือหนึ่งข้อเท็จจริง อ่านแต่ละไฟล์และแทนที่
$EDITOR facts/01_example_notice.txt   # เปลี่ยนเป็นคำแถลงแบบจริง
$EDITOR facts/02_provenance.txt       # แหล่งที่มาจริงของหลักฐานของคุณ
$EDITOR facts/03_evidence_manifest.txt # แฮชเอกสารจริงของคุณและจัดทำรายการ
$EDITOR facts/04_allegation_summary.txt # ข้อกล่าวหาจริงของคุณ
$EDITOR facts/05_threat_model.txt     # การตอบสนองที่คุณคาดหวังจริง
$EDITOR facts/06_recipient_protocol.txt # ปรับสำหรับผู้รับของคุณ
$EDITOR facts/07_no_retraction_commitment.txt # คำประกาศไม่ถอนคำของคุณ
$EDITOR facts/08_signing_principal.txt # อัตลักษณ์การลงนามจริงของคุณ (หรือนามแฝง)

# 3. อัปเดต site/index.html เพื่อสะท้อนข้อเท็จจริงจริง
# สคริปต์ seal.sh ไม่ได้กรอก HTML อัตโนมัติ; การแก้ไขด้วยตนเอง
# คือขั้นตอนการตรวจสอบของมนุษย์
$EDITOR site/index.html

# 4. ผนึก
bash build/seal.sh
# ผลผลิต: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. ตรวจสอบในเครื่อง
python3 build/verify.py

# 6. เผยแพร่ manifest (ไม่ใช่เอกสาร) ไปยังหนึ่งใน:
#    - host แบบสแตติกที่คุณควบคุม (Cloudflare Pages, GitHub Pages ฯลฯ)
#    - เครือข่าย IPFS ที่ CID ที่พิมพ์ออกมา
#    - ทั้งคู่ (แนะนำ)

# 7. ส่งเอกสารต้นทางให้กับผู้รับแยกต่างหาก
#    ผ่านช่องทางที่ปลอดภัยที่คุณเลือก (Signal, SecureDrop ฯลฯ)
#    ผู้รับจะตรวจสอบแฮชไฟล์เทียบกับ manifest ของคุณ
```

## การคำนวณแฮชสำหรับไฟล์หลักฐานจริง

ในการแฮชไฟล์เอกสารด้วย SHA-256:

```bash
shasum -a 256 my_document.pdf
```

อักขระฐานสิบหก 64 ตัวแรกคือ SHA-256 ใส่ลงใน manifest

คุณสามารถแฮชหลายไฟล์พร้อมกันได้:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

สิ่งนั้นให้ข้อมูลสำหรับ `facts/03_evidence_manifest.txt`

## หมายเหตุเกี่ยวกับความปลอดภัยในการปฏิบัติงาน

โพรโทคอล MYRIAM ไม่ได้ให้ความปลอดภัยในการปฏิบัติงาน มันให้
*ความคงทนของหลักฐาน* คุณยังคงต้องการ:

- ช่องทางการสื่อสารที่ปลอดภัยกับทนายความของคุณ
  ([Signal](https://signal.org/) แบบเผชิญหน้า หรือช่องทางที่
  ปลอดภัยที่กำหนดไว้ของทนายความของคุณ)
- ที่จัดเก็บที่ปลอดภัยสำหรับเอกสารต้นทาง (เข้ารหัสในขณะที่
  พักไว้ ไม่อยู่ในบริการคลาวด์ที่นายจ้างของคุณสามารถเข้าถึงได้)
- อุปกรณ์ที่สะอาดสำหรับการผนึก (แล็ปท็อปส่วนตัวที่ไม่เชื่อมโยง
  กับบัญชีของนายจ้าง; ควรเป็นเครื่องที่ใช้เพื่อจุดประสงค์นี้
  เท่านั้น)
- เครือข่ายที่สะอาดสำหรับการเผยแพร่ (ไม่ใช่เครือข่ายของนายจ้าง
  ไม่ใช่ IP บ้านของคุณหากนายจ้างของคุณอาจออกหมายเรียก ISP ของคุณ)

แนะนำให้อ่านเกี่ยวกับความปลอดภัยในการปฏิบัติงานสำหรับผู้แจ้งเบาะแส:

- [เอกสาร SecureDrop ของ The Intercept](https://securedrop.org/)
- [แหล่งทรัพยากรของ Freedom of the Press Foundation](https://freedom.press/)
- [คู่มือป้องกันตัวเองจากการเฝ้าระวังของ EFF](https://ssd.eff.org/)

ปรึกษาเอกสารเหล่านี้ก่อนที่คุณจะเริ่มกระบวนการผนึก โพรโทคอลไม่
ป้องกันทางเลือก opsec ที่ไม่ดีที่ทำขึ้นก่อนการเรียกใช้โพรโทคอล

## สิ่งที่ผู้รับควรรู้

ส่ง README นี้ไปยังผู้รับพร้อมกับ manifest ที่ผนึกแล้ว ตัว manifest
มีโพรโทคอลผู้รับ (FACT 06) แต่ README ขยายบทบาทของผู้รับ:

- **นักข่าว**: ตรวจสอบความสมบูรณ์ของ manifest เองก่อน จากนั้น
  ขอเอกสารผ่านช่องทางที่ปลอดภัยแยกต่างหาก ตรวจสอบว่า SHA-256
  ของแต่ละเอกสารตรงกับ manifest ก่อนการเผยแพร่ อย่าเผยแพร่
  เอกสารโดยไม่ได้รับความยินยอมจากผู้แจ้งเบาะแส; เผยแพร่
  การมีอยู่ของ manifest ที่ผนึกแล้วและข้อสรุปการตรวจสอบของคุณ
- **ผู้ควบคุม**: กระบวนการตรวจสอบเดียวกัน manifest ที่ผนึกบวกกับ
  เอกสารที่ตรงกันเป็นรูปแบบหลักฐานที่แข็งแกร่งที่สุดที่มีอยู่
  นอกเหนือจากการเก็บรวบรวมแบบ chain-of-custody ที่เป็นทางการ
- **ทนายความ**: manifest ที่ผนึกแล้วให้การยึดเหนี่ยวเชิงนิติวิทยา
  ศาสตร์สำหรับการมีอยู่ของเอกสารในเวลาที่ระบุไว้ ว่าจะตอบสนอง
  กฎเกณฑ์หลักฐานของเขตอำนาจศาลใดเขตหนึ่งหรือไม่นั้นขึ้นอยู่กับ
  คุณที่จะพิจารณา

## สิ่งที่ชุดเครื่องมือนี้ไม่ใช่

- ไม่ใช่สิ่งทดแทนที่ปรึกษากฎหมาย
- ไม่ใช่การรับประกันว่าใครจะดำเนินการเกี่ยวกับการเปิดเผยข้อมูลของคุณ
- ไม่ใช่การป้องกันการตอบโต้
- ไม่ใช่การรับรองความถูกต้องของเนื้อหาเอกสาร (เพียงแต่ของ
  ไบต์ในเวลาที่ทำการผนึกเท่านั้น)
- ไม่ใช่วิธีในการสื่อสารแบบนิรนามกับใครก็ตาม — การผนึกเป็น
  สาธารณะ; ถ้าชื่อ signing principal ของคุณเป็นชื่อจริง คุณก็
  ถูกระบุตัวแล้ว

## บทปิดท้าย

manifest ที่ผนึกแล้วเป็นสิ่งเล็ก ๆ มันเป็นกระแสไบต์หนึ่งที่พิสูจน์
ว่ากระแสไบต์ชุดที่ใหญ่กว่ามีอยู่ในรูปแบบเฉพาะในเวลาเฉพาะ
ความเล็กนั้นคือความแข็งแกร่ง: มันทำเพียงสิ่งเดียว และมันทำสิ่งเดียว
นั้นตลอดอายุการใช้งานของบล็อกเชน Bitcoin

หากคุณกำลังอ่านสิ่งนี้เพราะคุณกำลังพิจารณาที่จะกลายเป็นผู้แจ้ง
เบาะแส โปรด: ปรึกษาทนายความก่อน ใช้คู่มือความปลอดภัยในการ
ปฏิบัติงานข้างต้น จากนั้น ก็ต่อเมื่อข้อพิจารณาต้นน้ำทั้งหมดเหล่านั้น
