# व्हिसलब्लोअर मॅनिफेस्ट सील किट

विशिष्ट अपयश मोडचा सामना करणाऱ्या व्हिसलब्लोअर्ससाठी एक टर्नकी myriam-kit उदाहरण जिथे **व्हिसलब्लोअर आणि सार्वजनिक यांच्या दरम्यान असलेल्या मध्यस्थांद्वारे पुरावा दबविला जातो, संपादित केला जातो किंवा वादात आणला जातो**. हे ज्या प्रामाणिक ऐतिहासिक प्रकरणांविरुद्ध संरक्षण करते:

- Boeing 737 MAX (अंतर्गत सुरक्षा अहवाल कमी लेखले / दफन केले)
- Theranos (प्रयोगशाळा-चाचणी चिंता वर्षानुवर्षे दडपल्या)
- FDA निरीक्षक अहवाल सबमिशननंतर संपादित केले
- Frances Haugen / Facebook (सीलबंद मॅनिफेस्टने सिद्ध केले असते की दस्तऐवज प्रामाणिक होते आणि निर्धारित वेळी अस्तित्वात होते)

हे किट कशापासून संरक्षण देते (प्रोटोकॉल अटींमध्ये):

- T1–T3 [FACT 05](facts/05_threat_model.txt) मध्ये: दस्तऐवज सामग्रीचा सार्वजनिक वाद, दस्तऐवज निर्धारित वेळी सीलबंद नव्हते असे दावे, पूर्वलक्षी निर्मितीचे दावे.
- T9: तृतीय पक्ष पत्रकारांना बदललेल्या प्रती सोपवित आहेत.

हे किट **यापासून संरक्षण देत नाही**:

- व्हिसलब्लोअरविरुद्ध वैयक्तिक प्रतिकार (T7).
- व्हिसलब्लोअरवर मागे घेण्याचा दबाव (T4, FACT 07 द्वारे अंशतः कमी केले जे शांत मागे घेणे सार्वजनिकपणे शोधण्यायोग्य बनवते).
- स्वतःच्या प्रकटीकरणासाठी व्हिसलब्लोअरविरुद्ध कायदेशीर कारवाई (T5).
- व्हिसलब्लोअरचे चारित्र्य बदनाम करणे (T8).
- मध्यस्थ शांतपणे कथा सोडून देणे (T10).

जर तुम्ही व्हिसलब्लोअर असाल आणि हे किट वापरण्याचा विचार करत असाल, तर **प्रथम हे संपूर्ण README वाचा**. प्रोटोकॉल एक साधन आहे. प्रोटोकॉल कायदेशीर सल्लागार, पत्रकार, नियामक किंवा काळजीपूर्वक ऑपरेशनल सिक्युरिटीचा पर्याय नाही.

## हे किट काय तयार करते

पुराव्याचे क्रिप्टोग्राफिकरित्या सीलबंद *मॅनिफेस्ट* (पुरावा स्वतः नाही) जे सिद्ध करते:

1. प्रत्येक पुरावा फाइलचे विशिष्ट बाइट्स सीलिंगच्या वेळी अस्तित्वात होते.
2. Bitcoin OTS साक्ष्याद्वारे सीलिंग विशिष्ट वेळी किंवा आधी झाले.
3. प्रोव्हनन्स, आरोप, धमकीचे मॉडेल, प्राप्तकर्ता प्रोटोकॉल, आणि शांत-मागे-घेणे-नाही प्रतिज्ञा सर्व एकत्र घोषित केल्या गेल्या.

अंतर्निहित दस्तऐवज तुम्ही प्राप्तकर्त्याला पाठवेपर्यंत तुमच्या ताब्यात राहतात. प्राप्तकर्ते सत्यापित करतात की त्यांना तुम्ही सीलबंद केलेले समान बाइट्स मिळाले.

## हे किट कधी वापरावे

तुम्ही पत्रकार, नियामक किंवा वकीलाला पुरावा प्रसारित करण्यापूर्वी मॅनिफेस्ट सील करण्याचा विचार करावा, जर खालील सर्व सत्य असेल:

- तुमच्याकडे डिजिटल स्वरूपात पुरावा आहे (PDFs, ईमेल, प्रतिमा इ.) जो तुम्हाला बाइट-वार प्रामाणिक म्हणून जतन करायचा आहे.
- तुम्ही अपेक्षा करता की मध्यस्थ पुरावा बदलू शकतात, वादात आणू शकतात किंवा दफन करू शकतात.
- तुम्ही प्रसारणापर्यंत अंतर्निहित दस्तऐवज तुमच्या ताब्यात ठेवू शकता.
- तुम्ही दस्तऐवजांची सामग्री प्रकट **केल्याशिवाय** मॅनिफेस्ट सार्वजनिकपणे प्रकाशित करू शकता (फक्त हॅश सार्वजनिक आहेत).
- तुम्ही सार्वजनिकपणे व्हिसलब्लोअर म्हणून ओळखले जाण्याच्या जोखमींचे मूल्यांकन केले आहे (स्वाक्षरी-प्रिन्सिपल फील्ड). जर अनामिक असेल, तर सर्व संबंधित प्रकाशनांमध्ये सातत्याने एक उपनाव वापरा.
- तुम्ही तुमच्या अधिकारक्षेत्रातील व्हिसलब्लोअर-संरक्षण कायद्याबद्दल वकीलाशी सल्ला घेतला आहे.

## हे किट कधी वापरू नये

जर असे असेल तर हे किट वापरू नका:

- दस्तऐवजांमध्ये स्वतःमध्ये तृतीय पक्षांची (रुग्ण, ग्राहक, कर्मचारी) वैयक्तिक माहिती असेल जी सार्वजनिक-हॅश करण्यायोग्य बनवू नये. मूळ ऐवजी संशोधित आवृत्तीचा हॅश घ्या.
- मॅनिफेस्ट प्रकाशित करण्याचा कृत्य स्वतः व्हिसलब्लोअरला त्यांनी संमती न दिलेल्या मार्गाने ओळखेल.
- तुम्ही अद्याप वकीलाशी सल्ला घेतलेला नाही. प्रोटोकॉल एक साधन आहे; व्हिसलब्लोअर-संरक्षण कायदा अधिकारक्षेत्र-विशिष्ट आहे आणि प्रोटोकॉलच्या बाहेर राहतो.
- तुम्हाला नंतर कोणताही एकल आरोप मागे घ्यायचा असेल. सील स्थायी आहे; नवीन सीलबंद सुधारणा सुधारित करण्याचा एकमेव प्रामाणिक मार्ग आहे.
- तुम्ही दबावाखाली किंवा वेळेच्या दबावाखाली हे करत आहात. प्रोटोकॉल बळजबरीला वाढवते; नकार गेट (myriam-kit `SPEC §8`) प्रकाशकांप्रमाणेच व्हिसलब्लोअर्सनाही लागू आहे.

## वास्तविक प्रकटीकरणासाठी हे किट कसे फोर्क करावे

```bash
# 1. Copy this example into a new directory
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. Remove the fictional content; replace with your real facts.
# Each file under facts/ is one fact. Read each and replace.
$EDITOR facts/01_example_notice.txt   # change to a real cover statement
$EDITOR facts/02_provenance.txt       # real provenance of your evidence
$EDITOR facts/03_evidence_manifest.txt # hash your real documents and list them
$EDITOR facts/04_allegation_summary.txt # your real allegation
$EDITOR facts/05_threat_model.txt     # your real expected responses
$EDITOR facts/06_recipient_protocol.txt # adjust for your recipient
$EDITOR facts/07_no_retraction_commitment.txt # your no-retraction declaration
$EDITOR facts/08_signing_principal.txt # your real signing identity (or pseudonym)

# 3. Update site/index.html to reflect the real facts.
# The seal.sh script does NOT auto-populate the HTML; the manual edit
# is the human-review step.
$EDITOR site/index.html

# 4. Seal.
bash build/seal.sh
# Outputs: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. Verify locally.
python3 build/verify.py

# 6. Publish the manifest (NOT the documents) to any of:
#    - A static host you control (Cloudflare Pages, GitHub Pages, etc.)
#    - The IPFS network at the printed CID
#    - Both (recommended)

# 7. Separately, transmit the underlying documents to the recipient
#    via a secure channel of your choice (Signal, SecureDrop, etc.).
#    The recipient verifies file hashes against your manifest.
```

## वास्तविक पुरावा फायलींसाठी हॅश गणना करणे

SHA-256 सह दस्तऐवज फाइल हॅश करण्यासाठी:

```bash
shasum -a 256 my_document.pdf
```

पहिले 64 हेक्स वर्ण SHA-256 आहेत. ते मॅनिफेस्टमध्ये टाका.

तुम्ही एकाच वेळी अनेक फायली हॅश करू शकता:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

ते तुम्हाला `facts/03_evidence_manifest.txt` साठी डेटा देते.

## ऑपरेशनल सिक्युरिटी टिपा

MYRIAM प्रोटोकॉल ऑपरेशनल सिक्युरिटी प्रदान करत नाही. ते *पुराव्याची टिकाऊपणा* प्रदान करते. तुम्हाला अद्याप आवश्यक आहे:

- तुमच्या वकीलाला एक सुरक्षित संप्रेषण चॅनेल ([Signal](https://signal.org/), समोरासमोर, किंवा तुमच्या वकीलाचे स्थापित सुरक्षित चॅनेल)
- अंतर्निहित दस्तऐवजांसाठी सुरक्षित संग्रह स्थान (विश्रांतीत एन्क्रिप्ट केलेले, तुमच्या नियोक्त्याला प्रवेश असलेल्या क्लाउड सेवांवर नाही)
- सीलिंगसाठी स्वच्छ डिव्हाइस (नियोक्ता खात्यांशी संबंधित नसलेला वैयक्तिक लॅपटॉप; शक्यतो केवळ या उद्देशासाठी वापरला जाणारा)
- प्रकाशनासाठी स्वच्छ नेटवर्क (तुमच्या नियोक्त्याचे नेटवर्क नाही, तुमच्या नियोक्त्याने तुमच्या ISP ला समन्स केले असेल तर तुमचा गृह IP नाही)

व्हिसलब्लोअर्ससाठी ऑपरेशनल सिक्युरिटीवर शिफारस केलेले वाचन:

- [The Intercept's SecureDrop documentation](https://securedrop.org/)
- [Freedom of the Press Foundation's resources](https://freedom.press/)
- [EFF's surveillance self-defense guide](https://ssd.eff.org/)

सीलिंग प्रक्रिया सुरू करण्यापूर्वी त्यांचा सल्ला घ्या. प्रोटोकॉल वाईट ऑप्सेक निवडींपासून संरक्षण करत नाही जे प्रोटोकॉल आवाहन करण्यापूर्वी केले गेले होते.

## प्राप्तकर्त्यांना काय माहित असावे

सीलबंद मॅनिफेस्टसह प्राप्तकर्त्यांना हे README पाठवा. मॅनिफेस्टमध्ये स्वतः प्राप्तकर्ता प्रोटोकॉल (FACT 06) आहे, परंतु README प्राप्तकर्त्याची भूमिका विस्तारित करते:

- **पत्रकार**: प्रथम मॅनिफेस्टची स्वतःची अखंडता सत्यापित करा. नंतर वेगळ्या सुरक्षित चॅनेलद्वारे दस्तऐवजांची विनंती करा. प्रकाशनापूर्वी प्रत्येक दस्तऐवजाचा SHA-256 मॅनिफेस्टशी जुळतो की नाही ते सत्यापित करा. व्हिसलब्लोअरच्या संमतीशिवाय दस्तऐवज प्रकाशित करू नका; सीलबंद मॅनिफेस्टचे अस्तित्व आणि तुमचे सत्यापन निष्कर्ष प्रकाशित करा.
- **नियामक**: समान सत्यापन प्रक्रिया. सीलबंद मॅनिफेस्ट अधिक जुळणारे दस्तऐवज औपचारिक चेन-ऑफ-कस्टडी संग्रहाबाहेर उपलब्ध सर्वात मजबूत पुराव्याचे स्वरूप आहे.
- **वकील**: सीलबंद मॅनिफेस्ट निर्धारित वेळी दस्तऐवजांच्या अस्तित्वासाठी फॉरेन्सिक अँकरिंग प्रदान करते. ते कोणत्याही विशिष्ट अधिकारक्षेत्राच्या पुरावा-नियमांना पूर्ण करते की नाही हे तुम्ही ठरवायचे आहे.

## हे किट काय नाही

- कायदेशीर सल्लागाराचा पर्याय नाही.
- कोणीतरी तुमच्या प्रकटीकरणावर कारवाई करेल याची हमी नाही.
- प्रतिकाराविरुद्ध संरक्षण नाही.
- दस्तऐवजांच्या सामग्रीचे प्रमाणीकरण नाही (केवळ त्यांच्या बाइट्स-टाइम-ऑफ-सीलिंगचे).
- कोणाशीही अनामिकरित्या संवाद साधण्याचा मार्ग नाही — सील सार्वजनिक आहे; जर तुमच्या स्वाक्षरी-प्रिन्सिपल नाव वास्तविक असेल, तर तुम्ही ओळखले जाता.

## समाप्ती

सीलबंद मॅनिफेस्ट एक लहान गोष्ट आहे. ही एक बाइट प्रवाह आहे जो एका विशिष्ट स्वरूपात विशिष्ट वेळी अस्तित्वात असलेल्या बाइट प्रवाहांच्या मोठ्या संचाला सिद्ध करते. ती लहानता तिची शक्ती आहे: ती फक्त एक गोष्ट करते, आणि ती Bitcoin blockchain च्या आयुष्यभर ती एक गोष्ट करते.

जर तुम्ही हे वाचत असाल कारण तुम्ही व्हिसलब्लोअर बनण्याचा विचार करत आहात, कृपया: प्रथम वकीलाशी बोला. वरील ऑपरेशनल सिक्युरिटी मार्गदर्शक वापरा. नंतर, आणि केवळ जर त्या सर्व अप्स्ट्रीम विचार क्रमाने असतील, तरच तुमच्या प्रकटीकरणाचा क्रिप्टोग्राफिक पाठीचा कणा म्हणून हे किट विचारात घ्या. क्रिप्टो हा सोपा भाग आहे. क्रिप्टोच्या वर असलेले सर्व काही जिथे खरा धोका राहतो.
