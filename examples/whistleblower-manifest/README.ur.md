# وِسل بلوور مینیفیسٹ سیل کِٹ

ان وِسل بلوورز کے لیے ایک ٹرن کی myriam-kit مثال جو اس مخصوص ناکامی
کی صورتحال کا سامنا کرتے ہیں جہاں **ثبوت کو وسل بلوور اور عوام کے
درمیان واقع ثالثین دفن کر دیتے ہیں، ترمیم کر دیتے ہیں یا متنازعہ
بنا دیتے ہیں**۔ وہ کلاسیکی تاریخی مقدمات جن کے خلاف یہ کِٹ دفاع کرتی ہے:

- Boeing 737 MAX (اندرونی حفاظتی رپورٹیں کم اہم سمجھی گئیں/دفن کی گئیں)
- Theranos (لیب ٹیسٹ کے خدشات کئی سال تک دبائے گئے)
- FDA معائنہ کاروں کی رپورٹیں جمع کرانے کے بعد ترمیم کی گئیں
- Frances Haugen / Facebook (ایک مہر شدہ مینیفیسٹ ثابت کرتا کہ
  دستاویزات اصلی تھیں اور دعویٰ کیے گئے وقت موجود تھیں)

یہ کِٹ کس چیز سے دفاع کرتی ہے (پروٹوکول کی زبان میں):

- [FACT 05](facts/05_threat_model.txt) میں T1–T3: دستاویز کے
  مندرجات کا عوامی تنازعہ، یہ دعوے کہ دستاویزات دعویٰ کیے گئے وقت
  مہر شدہ نہیں تھیں، گزشتہ تاریخ سے جعل سازی کے دعوے۔
- T9: تیسرے فریق صحافیوں کو تبدیل شدہ نقول دیتے ہیں۔

یہ کِٹ کس چیز سے دفاع **نہیں** کرتی:

- وسل بلوور کے خلاف ذاتی انتقام (T7)۔
- وسل بلوور پر دستبردار ہونے کا دباؤ (T4، جسے FACT 07 جزوی طور پر
  کم کرتا ہے کیونکہ وہ خاموش دستبرداری کو عوامی طور پر قابلِ شناخت
  بناتا ہے)۔
- خود انکشاف پر وسل بلوور کے خلاف قانونی کارروائی (T5)۔
- وسل بلوور کے کردار کو بدنام کرنا (T8)۔
- ثالثین کا خاموشی سے کہانی کو دبا دینا (T10)۔

اگر آپ ایک وسل بلوور ہیں جو اس کِٹ کے استعمال پر غور کر رہے ہیں،
**پہلے یہ پورا README پڑھیں**۔ پروٹوکول ایک اوزار ہے۔ پروٹوکول
قانونی مشیر، صحافی، ریگولیٹر یا محتاط عملیاتی سلامتی کا متبادل
نہیں ہے۔

## یہ کِٹ کیا پیدا کرتی ہے

ثبوت کا ایک کرپٹوگرافک طور پر مہر شدہ *مینیفیسٹ* (نہ کہ خود ثبوت)
جو ثابت کرتا ہے:

1. ہر ثبوت کی فائل کے مخصوص بائٹس مہر لگانے کے لمحے موجود تھے۔
2. مہر لگانا کسی مخصوص وقت میں یا اس سے پہلے ہوا (Bitcoin OTS
   تصدیق کے ذریعے)۔
3. اصل، الزام، تہدید کا ماڈل، وصول کنندہ کا پروٹوکول، اور خاموش
   دستبرداری نہ کرنے کا عہد — یہ سب ایک ساتھ اعلان کیے گئے تھے۔

بنیادی دستاویزات اس وقت تک آپ کی تحویل میں رہتی ہیں جب تک آپ انہیں
وصول کنندہ کو نہ بھیجیں۔ وصول کنندگان تصدیق کرتے ہیں کہ انہیں وہی
بائٹس موصول ہوئے جو آپ نے مہر کیے تھے۔

## یہ کِٹ کب استعمال کریں

آپ کو صحافی، ریگولیٹر یا وکیل کو ثبوت منتقل کرنے سے **پہلے** ایک
مینیفیسٹ مہر کرنے پر غور کرنا چاہیے، اگر مندرجہ ذیل سب سچ ہوں:

- آپ کے پاس ڈیجیٹل شکل میں ثبوت ہیں (PDFs، ای میلز، تصاویر وغیرہ)
  جنہیں آپ بائٹ بائٹ مستند طور پر محفوظ رکھنا چاہتے ہیں۔
- آپ کو توقع ہے کہ ثالثین ثبوت کو تبدیل، متنازعہ یا دفن کر سکتے ہیں۔
- آپ منتقلی تک بنیادی دستاویزات اپنی تحویل میں رکھ سکتے ہیں۔
- آپ مینیفیسٹ کو دستاویزات کے مندرجات ظاہر کیے بغیر (صرف ہیشز
  عوامی ہیں) عوامی طور پر شائع کر سکتے ہیں۔
- آپ نے ایک وسل بلوور کے طور پر عوامی طور پر قابلِ شناخت بننے کے
  خطرات کا وزن کیا ہے (signing-principal فیلڈ)۔ اگر گمنام ہیں، تو
  تمام متعلقہ اشاعتوں میں مستقل طور پر ایک تخلص استعمال کریں۔
- آپ نے اپنے دائرہ اختیار میں وسل بلوور تحفظ کے قانون کے بارے میں
  ایک وکیل سے مشورہ کیا ہے۔

## یہ کِٹ کب استعمال نہ کریں

اس کِٹ کا استعمال نہ کریں اگر:

- دستاویزات خود تیسرے فریق (مریضوں، صارفین، ملازمین) کے بارے میں
  ذاتی معلومات پر مشتمل ہیں جو عوامی طور پر ہیش نہیں کی جانی چاہیے۔
  اصل کے بجائے ایک ترمیم شدہ ورژن کو ہیش کریں۔
- مینیفیسٹ کی اشاعت بذاتِ خود وسل بلوور کی شناخت اس طرح کرے گی جس
  پر انہوں نے رضامندی نہیں دی۔
- آپ نے ابھی تک کسی وکیل سے مشورہ نہیں کیا ہے۔ پروٹوکول ایک اوزار
  ہے؛ وسل بلوور تحفظ کا قانون دائرہ اختیار کے لحاظ سے مخصوص ہے اور
  پروٹوکول سے باہر رہتا ہے۔
- آپ بعد میں کسی ایک الزام سے دستبردار ہونا چاہیں۔ مہر مستقل ہے؛
  نظرثانی کا واحد دیانتدار طریقہ نئی مہر شدہ تصحیحات ہیں۔
- آپ یہ مجبوری یا وقتی دباؤ کے تحت کر رہے ہیں۔ پروٹوکول جبر کو
  بڑھاتا ہے؛ انکار کا دروازہ (myriam-kit `SPEC §8`) جس طرح ناشرین
  پر لاگو ہوتا ہے اسی طرح وسل بلوورز پر بھی۔

## حقیقی انکشاف کے لیے اس کِٹ کو فورک کیسے کریں

```bash
# 1. اس مثال کو ایک نئی ڈائریکٹری میں کاپی کریں
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. افسانوی مواد کو ہٹا دیں؛ اپنے حقیقی حقائق سے بدلیں۔
# facts/ کے تحت ہر فائل ایک حقیقت ہے۔ ہر ایک کو پڑھیں اور تبدیل کریں۔
$EDITOR facts/01_example_notice.txt   # ایک حقیقی سرورق بیان میں تبدیل کریں
$EDITOR facts/02_provenance.txt       # آپ کے ثبوت کی حقیقی اصل
$EDITOR facts/03_evidence_manifest.txt # اپنی حقیقی دستاویزات کو ہیش کریں اور درج کریں
$EDITOR facts/04_allegation_summary.txt # آپ کا حقیقی الزام
$EDITOR facts/05_threat_model.txt     # آپ کے حقیقی متوقع ردِعمل
$EDITOR facts/06_recipient_protocol.txt # اپنے وصول کنندہ کے لیے ایڈجسٹ کریں
$EDITOR facts/07_no_retraction_commitment.txt # دستبردار نہ ہونے کا آپ کا اعلان
$EDITOR facts/08_signing_principal.txt # آپ کی حقیقی دستخطی شناخت (یا تخلص)

# 3. site/index.html کو حقیقی حقائق کی عکاسی کے لیے اپ ڈیٹ کریں۔
# seal.sh اسکرپٹ HTML کو خودکار طور پر آباد نہیں کرتا؛ دستی ترمیم
# انسانی جائزے کا قدم ہے۔
$EDITOR site/index.html

# 4. مہر لگائیں۔
bash build/seal.sh
# آؤٹ پٹ: merkle_root.txt، merkle_root.txt.ots، IPFS CID

# 5. مقامی طور پر تصدیق کریں۔
python3 build/verify.py

# 6. مینیفیسٹ شائع کریں (دستاویزات نہیں) ان میں سے کسی پر:
#    - آپ کی کنٹرول کردہ ایک سٹیٹک ہوسٹ (Cloudflare Pages، GitHub Pages وغیرہ)
#    - چھپی ہوئی CID پر IPFS نیٹ ورک
#    - دونوں (تجویز کردہ)

# 7. علیحدہ طور پر، بنیادی دستاویزات کو اپنی پسند کے ایک محفوظ چینل
#    (Signal، SecureDrop، وغیرہ) کے ذریعے وصول کنندہ کو منتقل کریں۔
#    وصول کنندہ آپ کے مینیفیسٹ کے خلاف فائل ہیشز کی تصدیق کرتا ہے۔
```

## حقیقی ثبوت کی فائلوں کے لیے ہیشز کا حساب لگانا

SHA-256 کے ساتھ ایک دستاویز فائل کو ہیش کرنے کے لیے:

```bash
shasum -a 256 my_document.pdf
```

پہلے 64 ہیکس کریکٹرز SHA-256 ہیں۔ اسے مینیفیسٹ میں ڈالیں۔

آپ ایک ساتھ کئی فائلوں کو ہیش کر سکتے ہیں:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

اس سے آپ کو `facts/03_evidence_manifest.txt` کے لیے ڈیٹا ملتا ہے۔

## آپریشنل سیکیورٹی نوٹس

MYRIAM پروٹوکول آپریشنل سلامتی فراہم نہیں کرتا۔ یہ *ثبوتی پائیداری*
فراہم کرتا ہے۔ آپ کو اب بھی ضرورت ہے:

- اپنے وکیل سے ایک محفوظ مواصلاتی چینل
  ([Signal](https://signal.org/)، بالمشافہ، یا آپ کے وکیل کا
  قائم شدہ محفوظ چینل)
- بنیادی دستاویزات کے لیے ایک محفوظ ذخیرہ کرنے کا مقام (آرام پر
  انکرپٹڈ، آپ کے آجر کے قابلِ رسائی کلاؤڈ سروسز پر نہیں)
- مہر لگانے کے لیے ایک صاف ڈیوائس (آجر کے اکاؤنٹس سے غیر منسلک
  ایک ذاتی لیپ ٹاپ؛ ترجیحاً ایک جو صرف اس مقصد کے لیے استعمال ہوا ہو)
- اشاعت کے لیے ایک صاف نیٹ ورک (نہ آپ کے آجر کا نیٹ ورک، نہ آپ کا
  گھریلو IP اگر آپ کا آجر آپ کے ISP کو طلبی نوٹس بھیج سکتا ہے)

وسل بلوورز کے لیے آپریشنل سیکیورٹی پر تجویز کردہ مطالعہ:

- [The Intercept's SecureDrop documentation](https://securedrop.org/)
- [Freedom of the Press Foundation's resources](https://freedom.press/)
- [EFF's surveillance self-defense guide](https://ssd.eff.org/)

مہر لگانے کا عمل شروع کرنے سے پہلے ان سے مشورہ کریں۔ پروٹوکول ان
خراب آپسیک انتخابات سے تحفظ نہیں دیتا جو پروٹوکول کو کال کرنے سے
پہلے کیے جاتے ہیں۔

## وصول کنندگان کو کیا جاننا چاہیے

مہر شدہ مینیفیسٹ کے ساتھ وصول کنندگان کو یہ README بھیجیں۔ مینیفیسٹ
خود وصول کنندہ کے پروٹوکول (FACT 06) پر مشتمل ہے، لیکن README
وصول کنندہ کے کردار کو وسعت دیتا ہے:

- **صحافی**: پہلے مینیفیسٹ کی اپنی سالمیت کی تصدیق کریں۔ پھر ایک
  علیحدہ محفوظ چینل کے ذریعے دستاویزات کی درخواست کریں۔ اشاعت سے
  پہلے ہر دستاویز کے SHA-256 کا مینیفیسٹ سے ملنا تصدیق کریں۔
  وسل بلوور کی رضامندی کے بغیر دستاویزات شائع نہ کریں؛ مہر شدہ
  مینیفیسٹ کے وجود اور اپنے تصدیقی نتیجے کو شائع کریں۔
- **ریگولیٹرز**: وہی تصدیقی عمل۔ مہر شدہ مینیفیسٹ کے ساتھ مطابقت
  پذیر دستاویزات رسمی چین آف کسٹڈی کلیکشن کے باہر دستیاب سب سے
  مضبوط ثبوتی شکل ہے۔
- **وکلاء**: مہر شدہ مینیفیسٹ دعویٰ کیے گئے وقت دستاویزات کے وجود
  کے لیے فرانزک اینکرنگ فراہم کرتا ہے۔ کیا یہ کسی مخصوص دائرہ
  اختیار کے شواہد کے قواعد کو پورا کرتا ہے، یہ آپ کا فیصلہ کرنا ہے۔

## یہ کِٹ کیا نہیں ہے

- قانونی مشورے کا متبادل نہیں۔
- یہ ضمانت نہیں کہ کوئی آپ کے انکشاف پر عمل کرے گا۔
- انتقام کے خلاف تحفظ نہیں۔
- دستاویزات کے مندرجات کی تصدیق نہیں (صرف ان کے مہر لگانے کے وقت
  کے بائٹس کی)۔
- کسی کے ساتھ گمنام طور پر بات چیت کرنے کا طریقہ نہیں — مہر عوامی
  ہے؛ اگر آپ کا signing-principal نام حقیقی ہے، تو آپ کی شناخت
  ظاہر ہے۔

## اختتام

ایک مہر شدہ مینیفیسٹ ایک چھوٹی چیز ہے۔ یہ ایک بائٹ سٹریم ہے جو
ثابت کرتی ہے کہ بائٹ سٹریمز کا ایک بڑا مجموعہ ایک خاص شکل میں ایک
خاص وقت میں موجود تھا۔ یہ چھوٹا پن اس کی طاقت ہے: یہ صرف ایک کام
کرتا ہے، اور وہ ایک کام Bitcoin بلاکچین کی پوری زندگی میں کرتا ہے۔

اگر آپ یہ اس لیے پڑھ رہے ہیں کہ آپ وسل بلوور بننے پر غور کر رہے
ہیں، براہِ کرم: پہلے ایک وکیل سے بات کریں۔ اوپر دیے گئے آپریشنل
سیکیورٹی گائیڈز استعمال کریں۔ پھر، اگر اور صرف اگر وہ تمام اوپری
دھارے کے غور و فکر ترتیب میں ہوں، تو اپنے انکشاف کی کرپٹوگرافک
ریڑھ کی ہڈی کے طور پر اس کِٹ پر غور کریں۔ کرپٹو آسان حصہ ہے۔ کرپٹو
سے اوپر کی ہر چیز وہ ہے جہاں حقیقی خطرہ زندگی گزارتا ہے۔
