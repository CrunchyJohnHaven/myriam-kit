# ערכת חתימה למניפסט חושפי שחיתות

דוגמת myriam-kit מוכנה לשימוש עבור חושפי שחיתות המתמודדים עם
מצב הכשל הספציפי שבו **ראיות נקברות, נערכות או מוטלות במחלוקת
על ידי מתווכים בין החושף לציבור**. המקרים ההיסטוריים הקנוניים
שמפניהם הערכה הזו מגינה:

- Boeing 737 MAX (דיווחי בטיחות פנימיים הופחתו/נקברו)
- Theranos (חששות לגבי בדיקות מעבדה הודחקו במשך שנים)
- דיווחי מפקחי FDA נערכו לאחר ההגשה
- Frances Haugen / Facebook (מניפסט חתום היה מוכיח
  שהמסמכים אותנטיים והתקיימו בזמן הנטען)

מפני מה הערכה הזו מגינה (במונחים פרוטוקולריים):

- T1–T3 ב-[FACT 05](facts/05_threat_model.txt): מחלוקת ציבורית על
  תוכן מסמכים, טענות שהמסמכים לא נחתמו בזמן הנטען, טענות לזיוף
  רטרואקטיבי.
- T9: צדדים שלישיים המעבירים לעיתונאים עותקים שעברו שינוי.

מפני מה הערכה הזו **לא** מגינה:

- נקמה אישית בחושף השחיתות (T7).
- לחץ על החושף לחזור בו (T4, מצומצם חלקית על ידי
  FACT 07 המבליט לציבור חזרה שקטה).
- צעדים משפטיים נגד החושף עצמו בשל החשיפה (T5).
- הכפשת אופיו של החושף (T8).
- מתווכים המשליכים בשקט את הסיפור (T10).

אם אתם חושפי שחיתות השוקלים להשתמש בערכה הזו, **קראו את כל
ה-README הזה תחילה**. הפרוטוקול הוא כלי. הפרוטוקול אינו תחליף
לייעוץ משפטי, לעיתונאי, לרגולטור או לאבטחה תפעולית קפדנית.

## מה הערכה הזו מייצרת

*מניפסט* חתום קריפטוגרפית של ראיות (לא הראיות עצמן) המוכיח:

1. שהבייטים הספציפיים של כל קובץ ראיה התקיימו ברגע
   החתימה.
2. שהחתימה התרחשה בזמן מסוים או לפניו (באמצעות אישור Bitcoin
   OTS).
3. שהמקור, הטענה, מודל האיומים, פרוטוקול הנמען וההתחייבות
   לאי-חזרה-שקטה הוצהרו יחד.

המסמכים הבסיסיים נשארים ברשותכם עד שתשלחו אותם
לנמען. נמענים מאמתים שהם קיבלו את אותם בייטים שחתמתם.

## מתי להשתמש בערכה הזו

עליכם לשקול חתימת מניפסט **לפני** העברת ראיות
לעיתונאי, רגולטור או עורך דין, אם כל התנאים הבאים מתקיימים:

- יש לכם ראיות בצורה דיגיטלית (PDFs, אימיילים, תמונות וכו') שאתם
  רוצים לשמר כאותנטיות-בייט-אחר-בייט.
- אתם מצפים שמתווכים עשויים לשנות, לערער או לקבור את
  הראיות.
- אתם יכולים להשאיר את המסמכים הבסיסיים ברשותכם עד
  ההעברה.
- אתם יכולים לפרסם את המניפסט בפומבי **מבלי** לחשוף את
  תוכן המסמכים (רק הגיבובים פומביים).
- שקלתם את הסיכונים של הפיכה לזיהוי-פומבי כחושפי
  שחיתות (שדה signing-principal). אם אנונימיים, השתמשו בשם בדוי
  באופן עקבי בכל הפרסומים הקשורים.
- התייעצתם עם עורך דין בנוגע לחוק להגנת חושפי שחיתות
  בתחום השיפוט שלכם.

## מתי לא להשתמש בערכה הזו

אל תשתמשו בערכה הזו אם:

- המסמכים עצמם מכילים מידע אישי על צדדים
  שלישיים (מטופלים, לקוחות, עובדים) שלא צריך להפוך אותם
  לפומביים-לגיבוב. גבבו גרסה ערוכה, לא את המקור.
- עצם פרסום המניפסט יזהה את
  חושף השחיתות בדרך שהוא לא הסכים לה.
- עדיין לא התייעצתם עם עורך דין. הפרוטוקול הוא כלי;
  חוק להגנת חושפי שחיתות הוא ספציפי לתחום שיפוט וחי
  מחוץ לפרוטוקול.
- אתם עשויים לרצות לחזור בכם מטענה כלשהי בעתיד. החתימה
  היא קבועה; תיקונים חתומים חדשים הם הדרך הישרה היחידה לעדכן.
- אתם עושים זאת תחת כפייה או לחץ זמן. הפרוטוקול
  מגביר כפייה; שער הסירוב (myriam-kit `SPEC §8`) חל
  על חושפי שחיתות כמו על מפרסמים.

## איך לבצע fork לערכה לחשיפה אמיתית

```bash
# 1. העתיקו את הדוגמה הזו לתיקייה חדשה
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. הסירו את התוכן הבדיוני; החליפו בעובדות האמיתיות שלכם.
# כל קובץ תחת facts/ הוא עובדה אחת. קראו כל אחד והחליפו.
$EDITOR facts/01_example_notice.txt   # שנו להצהרת כיסוי אמיתית
$EDITOR facts/02_provenance.txt       # מקור אמיתי של הראיות שלכם
$EDITOR facts/03_evidence_manifest.txt # גבבו את המסמכים האמיתיים ורשמו אותם
$EDITOR facts/04_allegation_summary.txt # הטענה האמיתית שלכם
$EDITOR facts/05_threat_model.txt     # התגובות הצפויות האמיתיות שלכם
$EDITOR facts/06_recipient_protocol.txt # התאימו לנמען שלכם
$EDITOR facts/07_no_retraction_commitment.txt # הצהרת אי-חזרה שלכם
$EDITOR facts/08_signing_principal.txt # זהות החתימה האמיתית שלכם (או שם בדוי)

# 3. עדכנו את site/index.html כדי לשקף את העובדות האמיתיות.
# סקריפט seal.sh לא מאכלס את ה-HTML באופן אוטומטי; העריכה הידנית
# היא שלב הסקירה האנושית.
$EDITOR site/index.html

# 4. חתמו.
bash build/seal.sh
# פלטים: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. אמתו מקומית.
python3 build/verify.py

# 6. פרסמו את המניפסט (לא את המסמכים) באחת מהאפשרויות:
#    - מארח סטטי שאתם שולטים בו (Cloudflare Pages, GitHub Pages וכו')
#    - רשת IPFS תחת ה-CID המודפס
#    - שניהם (מומלץ)

# 7. בנפרד, העבירו את המסמכים הבסיסיים לנמען
#    דרך ערוץ מאובטח לבחירתכם (Signal, SecureDrop וכו').
#    הנמען מאמת את גיבובי הקבצים מול המניפסט שלכם.
```

## חישוב גיבובים עבור קבצי הראיות האמיתיים

לגיבוב קובץ מסמך עם SHA-256:

```bash
shasum -a 256 my_document.pdf
```

64 התווים הראשונים בהקסדצימלי הם ה-SHA-256. הכניסו אותו במניפסט.

אפשר לגבב קבצים רבים בו-זמנית:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

זה נותן לכם את הנתונים עבור `facts/03_evidence_manifest.txt`.

## הערות על אבטחה תפעולית

פרוטוקול MYRIAM **אינו** מספק אבטחה תפעולית. הוא מספק
*עמידות ראייתית*. אתם עדיין צריכים:

- ערוץ תקשורת מאובטח לעורך הדין שלכם
  ([Signal](https://signal.org/), פנים אל פנים, או הערוץ המאובטח
  המבוסס של עורך הדין שלכם)
- מקום אחסון מאובטח למסמכים הבסיסיים (מוצפן
  במנוחה, לא בשירותי ענן שהמעסיק שלכם יכול לגשת אליהם)
- מכשיר נקי לחתימה (מחשב נייד אישי שאינו משויך
  לחשבונות המעסיק; רצוי כזה המשמש רק למטרה זו)
- רשת נקייה לפרסום (לא רשת המעסיק שלכם, לא
  כתובת ה-IP הביתית שלכם אם המעסיק עשוי לזמן את ספק האינטרנט שלכם)

קריאה מומלצת על אבטחה תפעולית לחושפי שחיתות:

- [תיעוד SecureDrop של The Intercept](https://securedrop.org/)
- [משאבי Freedom of the Press Foundation](https://freedom.press/)
- [מדריך ההגנה העצמית של EFF לפיקוח](https://ssd.eff.org/)

התייעצו בהם **לפני** שאתם מתחילים בתהליך החתימה. הפרוטוקול
אינו מגן מפני בחירות אבטחה תפעולית גרועות שנעשו לפני שהפרוטוקול הופעל.

## מה שעל הנמענים לדעת

שלחו לנמענים את ה-README הזה יחד עם המניפסט החתום. המניפסט
עצמו מכיל את פרוטוקול הנמען (FACT 06), אך ה-README
מרחיב את תפקיד הנמען:

- **עיתונאים**: ראשית אמתו את שלמותו של המניפסט עצמו. לאחר מכן
  בקשו את המסמכים דרך ערוץ מאובטח נפרד. אמתו
  שה-SHA-256 של כל מסמך תואם למניפסט לפני הפרסום.
  **אל** תפרסמו את המסמכים ללא הסכמת חושף השחיתות;
  כן פרסמו את קיום המניפסט החתום ואת מסקנת
  האימות שלכם.
- **רגולטורים**: אותו תהליך אימות. המניפסט החתום בתוספת
  מסמכים תואמים הוא הצורה הראייתית החזקה ביותר הזמינה
  מחוץ לאיסוף פורמלי של שרשרת-משמורת.
- **עורכי דין**: המניפסט החתום מספק עיגון פורנזי
  לקיום המסמכים בזמן הנטען. האם הוא
  עומד בכללי הראיות של תחום שיפוט ספציפי הוא דבר שעליכם
  לקבוע.

## מה הערכה הזו אינה

- אינה תחליף לייעוץ משפטי.
- אינה ערובה שמישהו יפעל על סמך החשיפה שלכם.
- אינה הגנה מפני נקמה.
- אינה אימות של תוכן המסמכים (רק של
  הבייטים-בזמן-החתימה שלהם).
- אינה דרך לתקשר באנונימיות עם איש — החתימה היא
  פומבית; אם שם signing-principal שלכם אמיתי, אתם מזוהים.

## סיכום

מניפסט חתום הוא דבר קטן. הוא זרם בייטים אחד שמוכיח כי
מערך גדול יותר של זרמי בייטים התקיים בצורה מסוימת בזמן
מסוים. הקטנות הזו היא חוזקו: הוא עושה רק דבר אחד, והוא
עושה את אותו דבר אחד לאורך כל חיי הבלוקצ'יין של Bitcoin.

אם אתם קוראים זאת מפני שאתם שוקלים להפוך לחושפי
שחיתות, אנא: דברו תחילה עם עורך דין. השתמשו במדריכי
האבטחה התפעולית לעיל. ואז, אם ורק אם כל השיקולים
המקדמיים הללו במקום, שקלו את הערכה הזו בתור עמוד השדרה
הקריפטוגרפי של החשיפה שלכם. הקריפטו הוא החלק הקל. כל מה
שמקדים את הקריפטו הוא המקום שבו חי הסיכון האמיתי.
