# Whistleblower Manifest Seal Kit

*پښتو ژباړه — د اصلي انګلیسي نسخې لپاره، [README.md](README.md) وګورئ*

د افشاکوونکو لپاره د myriam-kit یوه چمتو نمونه چې د هغې ځانګړې ناکامۍ بڼې سره مخامخ کیږي چېرته چې **ثبوتونه د افشاکوونکي او ولس ترمنځ د منځګړو لخوا ښخیږي، سمیږي، یا د جنجال وړ ګرځي**. هغه تاریخي قضیې چې د دې پروتوکول دفاع کوي:

- بوینګ ۷۳۷ MAX (داخلي د خوندیتوب راپورونه کم ارزښت یا ښخ شوي)
- تېرانوس (د لابراتوار ازموینې اندیښنې د کلونو لپاره ځپل شوې)
- د FDA پلټونکو راپورونه چې له سپارلو وروسته سم شوي
- فرانسس هاوګن / فېسبوک (یوه مهر شوې فهرست به ثابته کړې وای چې اسناد اصلي وو او په ادعا شوي وخت کې شته وو)

د څه شی په وړاندې دا کټ دفاع کوي (د پروتوکول په اصطلاحاتو کې):

- په [FACT 05](facts/05_threat_model.txt) کې T1–T3: د اسنادو منځپانګې عامه جنجال، ادعاوې چې اسناد په ادعا شوي وخت کې مهر شوي نه وو، د وروسته-حقیقي جوړولو ادعاوې.
- T9: درېیمې خواوې ژورنالیستانو ته بدل شوي کاپي ورکوي.

د څه شی په وړاندې دا کټ دفاع **نه** کوي:

- د افشاکوونکي په وړاندې شخصي غچ اخیستنه (T7).
- د افشاکوونکي پر بېرته اخیستلو فشار (T4، چې د FACT 07 له لارې جزوي کم شوی چې خاموش بېرته اخیستنه په عامه توګه د کشف وړ کوي).
- د افشاکولو لپاره د افشاکوونکي په وړاندې قانوني اقدام (T5).
- د افشاکوونکي د شخصیت بدنامول (T8).
- منځګړي چې خاموش پر کیسې پاتې راځي (T10).

که تاسو یو افشاکوونکی یاست چې د دې کټ کارولو په اړه فکر کوئ، **لومړی دا ټوله README ولولئ**. پروتوکول یوه وسیله ده. پروتوکول د قانوني سلاکار، ژورنالیست، تنظیم کوونکي، یا کرکجن عملیاتي خوندیتوب بدیل نه دی.

## دا کټ څه شی تولیدوي

د ثبوتونو یوه کریپټوګرافي مهر شوې *فهرست* (نه پخپله ثبوت) چې ثابتوي:

1. د هرې ثبوتي فایل ځانګړي بایټونه د مهر کولو په شیبه کې شته وو.
2. مهر کول د یوې ځانګړې وخت په وخت یا له هغې مخکې وشول (د Bitcoin OTS تصدیق له لارې).
3. سرچینه، تور، د ګواښ ماډل، د اخیستونکي پروتوکول، او د هېڅ-خاموش-بېرته-اخیستلو ژمنه ټول یوځای اعلان شول.

اصلي اسناد ستاسو په لاس کې پاتې کیږي تر هغه چې تاسو یې اخیستونکي ته ولېږئ. اخیستونکي تایید کوي چې هغوی هغه ورته بایټونه ترلاسه کړي چې تاسو مهر کړي وو.

## دا کټ کله وکاروئ

تاسو باید د یوې فهرست د مهر کولو په اړه فکر وکړئ **مخکې** د ثبوت د ژورنالیست، تنظیم کوونکي، یا وکیل ته د لیږلو څخه، که د لاندې ټول رښتیا وي:

- تاسو په ډیجیټل بڼه کې ثبوت لرئ (PDFs، بریښنالیکونه، انځورونه، نور) چې غواړئ په بایټ-وار ډول اصلي وساتل شي.
- تاسو تمه لرئ چې منځګړي ممکن ثبوت بدل کړي، جنجال وکړي، یا ښخ یې کړي.
- تاسو کولی شئ اصلي اسناد د لیږد تر وخته خپل په لاس کې وساتئ.
- تاسو کولی شئ فهرست په عامه توګه خپاره کړئ **پرته** له دې چې د اسنادو منځپانګې څرګندې کړئ (یوازې هش عامه دي).
- تاسو د افشاکوونکي په توګه د عامه پېژندنې خطرونه ارزولي دي (د لاسلیک کوونکي اصلي ساحه). که نوم پټ یاست، یوه پسوډونیم په ټولو اړوندو خپرونو کې په کلکه وکاروئ.
- تاسو خپل د حوزې د افشاکوونکو-ساتنې قانون په اړه له وکیل سره مشوره کړې ده.

## دا کټ کله **مه** کاروئ

دا کټ مه کاروئ که:

- اصلي اسناد د درېیمو خواوو (ناروغان، پیرودونکي، کارمندان) په اړه شخصي معلومات لري چې باید عامه-هش نه شي. د یوې سمونې شوې نسخې هش وکاروئ، نه اصلي.
- د فهرست د خپرولو عمل به افشاکوونکی په داسې ډول وپېژني چې هغه ته یې رضایت نه دی ورکړی.
- تاسو تر اوسه له وکیل سره مشوره نه ده کړې. پروتوکول یوه وسیله ده؛ د افشاکوونکو-ساتنې قانون د حوزې سره ځانګړی دی او د پروتوکول له بهر ژوند کوي.
- تاسو ممکن وروسته یو ځانګړی تور بېرته واخلئ. مهر دایمي دی؛ نوي مهر شوي اصلاحات د یوازینۍ صادقې لارې دي د اصلاح کولو لپاره.
- تاسو دا د فشار یا د وخت تر فشار لاندې کوئ. پروتوکول جبر زیاتوي؛ د ردولو دروازه (د myriam-kit `SPEC §8`) د خپروونکو په څېر په افشاکوونکو هم پلې کیږي.

## دا کټ څنګه د حقیقي افشا لپاره فورک کړو

```bash
# 1. Copy this example into a new directory
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. Remove the fictional content; replace with your real facts.
# Each file under facts/ is one fact. Read each and replace.
$EDITOR facts/01_example_notice.txt   # change to a real cover statement
$EDITOR facts/02_provenance.txt       # real provenance of your evidence
$EDITOR facts/03_evidence_manifest.txt # hash your real documents and list them
$EDITOR facts/04_allegation_summary.txt # your real allegation
$EDITOR facts/05_threat_model.txt     # your real expected responses
$EDITOR facts/06_recipient_protocol.txt # adjust for your recipient
$EDITOR facts/07_no_retraction_commitment.txt # your no-retraction declaration
$EDITOR facts/08_signing_principal.txt # your real signing identity (or pseudonym)

# 3. Update site/index.html to reflect the real facts.
# The seal.sh script does NOT auto-populate the HTML; the manual edit
# is the human-review step.
$EDITOR site/index.html

# 4. Seal.
bash build/seal.sh
# Outputs: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. Verify locally.
python3 build/verify.py

# 6. Publish the manifest (NOT the documents) to any of:
#    - A static host you control (Cloudflare Pages, GitHub Pages, etc.)
#    - The IPFS network at the printed CID
#    - Both (recommended)

# 7. Separately, transmit the underlying documents to the recipient
#    via a secure channel of your choice (Signal, SecureDrop, etc.).
#    The recipient verifies file hashes against your manifest.
```

## د حقیقي ثبوت فایلونو لپاره د هش محاسبه

د یوې سند فایل د SHA-256 سره د هش کولو لپاره:

```bash
shasum -a 256 my_document.pdf
```

لومړي ۶۴ هیکس کرکترونه SHA-256 دي. هغه په فهرست کې واچوئ.

تاسو کولی شئ په یو وخت کې ډېرې فایلونه هش کړئ:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

دا تاسو ته د `facts/03_evidence_manifest.txt` لپاره ډاټا درکوي.

## د عملیاتي خوندیتوب یادښتونه

د MYRIAM پروتوکول عملیاتي خوندیتوب نه چمتو کوي. دا د *ثبوتي دوام* چمتو کوي. تاسو لاهم اړتیا لرئ:

- خپل وکیل ته خوندي اړیکې کانال
  ([Signal](https://signal.org/)، مخامخ، یا ستاسو د وکیل ټاکل شوې خوندي کانال)
- د اصلي اسنادو لپاره خوندي ذخیره (په پاتې حالت کې کوډ شوي، نه په هغو کلاوډ خدمتونو کې چې ستاسو کارفرما لاسرسي ولري)
- د مهر کولو لپاره پاک وسیله (یو شخصي لپتاپ چې د کارفرما د حسابونو سره تړلی نه وي؛ غوره به وي چې یوازې د دې هدف لپاره وکارول شي)
- د خپرولو لپاره پاک شبکه (نه ستاسو د کارفرما شبکه، نه ستاسو د کور IP که ستاسو کارفرما کولی شي ستاسو ISP وغواړي)

د افشاکوونکو لپاره د عملیاتي خوندیتوب وړاندیز شوي لوستل:

- [د Intercept د SecureDrop اسناد](https://securedrop.org/)
- [د مطبوعاتو د ازادۍ بنسټ سرچینې](https://freedom.press/)
- [د EFF د څارنې د خپل-دفاع لارښود](https://ssd.eff.org/)

د مهر کولو پروسې پیل کولو څخه مخکې له هغوی سره مشوره وکړئ. پروتوکول د بدو opsec پریکړو په وړاندې دفاع نه کوي چې د پروتوکول له بلنې مخکې شوي وي.

## اخیستونکي باید څه پوه شي

دا README د مهر شوي فهرست سره اخیستونکو ته ولېږئ. فهرست پخپله د اخیستونکي پروتوکول (FACT 06) لري، خو README د اخیستونکي رول پراخوي:

- **ژورنالیستان**: لومړی د فهرست پخپله بشپړتیا تایید کړئ. بیا اسناد د یوې جلا خوندي کانال له لارې وغواړئ. د خپرولو څخه مخکې د هرې سند SHA-256 د فهرست سره مطابقت تایید کړئ. د افشاکوونکي د رضایت پرته اسناد مه خپروئ؛ خو د مهر شوي فهرست شتون او د خپل تایید پایله خپره کړئ.
- **تنظیم کوونکي**: د تایید همدا پروسه. مهر شوې فهرست او ورته مطابقي اسناد د رسمي چین-آف-کسټوډي راټولولو څخه پرته ترلاسه کیدونکي ترټولو پیاوړې ثبوتي بڼه ده.
- **وکیلان**: مهر شوې فهرست د اسنادو په ادعا شوي وخت د شتون لپاره فورنزیک ثبت چمتو کوي. دا چې ایا د کومې ځانګړې حوزې د ثبوت-قواعدو سره مطابقت لري، ستاسو د ټاکلو لپاره دی.

## دا کټ څه شی **نه** دی

- د قانوني سلاکار بدیل نه دی.
- دا تضمین نه دی چې څوک به ستاسو په افشا کولو عمل وکړي.
- د غچ اخیستنې په وړاندې ساتنه نه ده.
- د اسنادو د منځپانګې تایید نه دی (یوازې د مهر کولو په وخت د بایټونو).
- له هیچا سره د بې نومه اړیکې لاره نه ده — مهر عامه دی؛ که ستاسو د لاسلیک کوونکي اصلي نوم رښتینی وي، تاسو پېژندل شوي یاست.

## پای

یوه مهر شوې فهرست یو وړوکی شی دی. دا د یو بایټ جریان دی چې ثابتوي د بایټ جریانونو یوه لویه ټولګه په یوې ځانګړې بڼه کې په یوې ځانګړې وخت کې شتون درلود. دا وړوکی والی یې قوت دی: دا یوازې یو شی کوي، او هغه یو شی د Bitcoin بلاکچین د ژوند په اوږدو کې کوي.

که تاسو دا د دې لپاره لولئ چې غواړئ افشاکوونکی شئ، مهرباني وکړئ: لومړی له وکیل سره خبرې وکړئ. د عملیاتي خوندیتوب پورته لارښوونې وکاروئ. بیا، که او یوازې که ټول هغه پورتني ملاحظات په کرکجن ډول حل شي، دا کټ د خپل افشا د کریپټوګرافي ملا تیر په توګه په پام کې ونیسئ. کریپټو اسانه برخه ده. د کریپټو څخه پورته هرڅه چېرته ژوند کوي چې حقیقي خطر دی.
