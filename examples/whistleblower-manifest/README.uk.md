# Набір запечатування маніфесту викривача

Готовий до використання приклад myriam-kit для викривачів, які стикаються
зі специфічним режимом відмови, при якому **докази хоронять, редагують
або оскаржують посередники між викривачем і громадськістю**. Канонічні
історичні випадки, проти яких захищає цей набір:

- Boeing 737 MAX (внутрішні звіти про безпеку відкидалися/ховалися)
- Theranos (занепокоєння щодо лабораторних тестів роками замовчувалися)
- Звіти інспекторів FDA редагувалися після подачі
- Frances Haugen / Facebook (запечатаний маніфест довів би, що
  документи були автентичними та існували в зазначений час)

Від чого цей набір захищає (у термінах протоколу):

- T1–T3 у [FACT 05](facts/05_threat_model.txt): публічне оскарження
  змісту документів, заяви про те, що документи не були запечатані в
  заявлений час, заяви про ретроспективну фабрикацію.
- T9: треті сторони, що передають журналістам змінені копії.

Від чого цей набір **не** захищає:

- Особистих репресій проти викривача (T7).
- Тиску на викривача задля відкликання (T4, частково пом'якшується
  FACT 07, що робить тихе відкликання публічно виявленим).
- Юридичних дій проти викривача за саме розкриття (T5).
- Дискредитації характеру викривача (T8).
- Посередників, які тихо припиняють розголос (T10).

Якщо ви викривач, який розглядає можливість використання цього набору,
**спершу прочитайте цей README повністю**. Протокол — це інструмент.
Протокол не є замінником юридичного консультанта, журналіста, регулятора
або ретельної операційної безпеки.

## Що цей набір продукує

Криптографічно запечатаний *маніфест* доказів (а не самі докази), що
доводить:

1. Конкретні байти кожного файлу доказів існували в момент запечатування.
2. Запечатування відбулося в певний момент або раніше (через
   засвідчення Bitcoin OTS).
3. Походження, звинувачення, модель загроз, протокол отримувача та
   зобов'язання не відкликати тихо — усе було заявлено разом.

Самі документи залишаються у вашому розпорядженні, доки ви не надішлете
їх отримувачу. Отримувачі перевіряють, що отримали ті самі байти, які
ви запечатали.

## Коли використовувати цей набір

Слід розглянути запечатування маніфесту **перед** передачею доказів
журналісту, регулятору або юристу, якщо всі наведені нижче умови
істинні:

- У вас є докази в цифровій формі (PDF, електронні листи, зображення
  тощо), які ви хочете зберегти як побайтово автентичні.
- Ви очікуєте, що посередники можуть змінити, оскаржити або
  поховати докази.
- Ви можете тримати документи у себе аж до передачі.
- Ви можете опублікувати маніфест публічно **без** розкриття вмісту
  документів (публічні лише хеші).
- Ви зважили ризики стати публічно ідентифікованим викривачем
  (поле signing-principal). Якщо анонімно, послідовно використовуйте
  псевдонім у всіх пов'язаних публікаціях.
- Ви проконсультувалися з юристом щодо законодавства про захист
  викривачів у вашій юрисдикції.

## Коли **не** використовувати цей набір

Не використовуйте цей набір, якщо:

- Самі документи містять персональну інформацію про третіх осіб
  (пацієнтів, клієнтів, працівників), яку не слід робити публічно
  хешованою. Хешуйте відредаговану версію, а не оригінал.
- Сама дія опублікування маніфесту ідентифікувала б викривача в
  спосіб, на який він не дав згоди.
- Ви ще не консультувалися з юристом. Протокол — це інструмент;
  законодавство про захист викривачів специфічне для юрисдикції і
  перебуває поза протоколом.
- Ви, можливо, захочете пізніше відкликати будь-яке окреме
  звинувачення. Печатка є постійною; нові запечатані виправлення —
  єдиний чесний спосіб ревізії.
- Ви робите це під примусом або під часовим тиском. Протокол
  посилює примус; ворота відмови (myriam-kit `SPEC §8`) стосуються
  викривачів так само, як і видавців.

## Як форкнути цей набір для реального розкриття

```bash
# 1. Скопіюйте цей приклад у нову директорію
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. Видаліть вигаданий вміст; замініть на ваші реальні факти.
# Кожен файл під facts/ — це один факт. Прочитайте кожен і замініть.
$EDITOR facts/01_example_notice.txt   # змініть на реальну супровідну заяву
$EDITOR facts/02_provenance.txt       # реальне походження ваших доказів
$EDITOR facts/03_evidence_manifest.txt # хешуйте ваші реальні документи і перелічіть їх
$EDITOR facts/04_allegation_summary.txt # ваше реальне звинувачення
$EDITOR facts/05_threat_model.txt     # ваші реальні очікувані відповіді
$EDITOR facts/06_recipient_protocol.txt # відкоригуйте під вашого отримувача
$EDITOR facts/07_no_retraction_commitment.txt # ваша декларація про невідкликання
$EDITOR facts/08_signing_principal.txt # ваша реальна підписувальна особа (або псевдонім)

# 3. Оновіть site/index.html, щоб відобразити реальні факти.
# Скрипт seal.sh НЕ автоматично заповнює HTML; ручне редагування
# є кроком людської ревізії.
$EDITOR site/index.html

# 4. Запечатайте.
bash build/seal.sh
# Виводи: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. Перевірте локально.
python3 build/verify.py

# 6. Опублікуйте маніфест (НЕ документи) на будь-якому з:
#    - Статичний хостинг, який ви контролюєте (Cloudflare Pages, GitHub Pages тощо)
#    - Мережа IPFS за надрукованим CID
#    - Обидва (рекомендовано)

# 7. Окремо передайте самі документи отримувачу через безпечний
#    канал на ваш вибір (Signal, SecureDrop тощо).
#    Отримувач перевіряє хеші файлів проти вашого маніфесту.
```

## Обчислення хешів для реальних файлів доказів

Щоб хешувати файл документа за допомогою SHA-256:

```bash
shasum -a 256 my_document.pdf
```

Перші 64 шістнадцяткові символи — це SHA-256. Вставте його в маніфест.

Ви можете хешувати багато файлів одночасно:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

Це дає дані для `facts/03_evidence_manifest.txt`.

## Примітки щодо операційної безпеки

Протокол MYRIAM **не** забезпечує операційної безпеки. Він забезпечує
*доказову довговічність*. Вам все ще потрібно:

- Безпечний канал зв'язку з вашим юристом
  ([Signal](https://signal.org/), особисто або встановлений
  безпечний канал вашого юриста)
- Безпечне місце зберігання для документів (зашифровані в стані
  спокою, не в хмарних сервісах, до яких ваш роботодавець має доступ)
- Чистий пристрій для запечатування (особистий ноутбук, не пов'язаний
  з обліковими записами роботодавця; бажано той, що використовується
  лише для цієї мети)
- Чисту мережу для публікації (не мережу вашого роботодавця, не вашу
  домашню IP, якщо ваш роботодавець може запитати у вашого ISP)

Рекомендоване читання з операційної безпеки для викривачів:

- [Документація SecureDrop від The Intercept](https://securedrop.org/)
- [Ресурси Freedom of the Press Foundation](https://freedom.press/)
- [Посібник із самозахисту від спостереження EFF](https://ssd.eff.org/)

Зверніться до них ДО того, як почати процес запечатування. Протокол не
захищає від поганих рішень з операційної безпеки, прийнятих до того, як
було задіяно протокол.

## Що повинні знати отримувачі

Надсилайте отримувачам цей README разом із запечатаним маніфестом.
Сам маніфест містить протокол отримувача (FACT 06), але README
розширює роль отримувача:

- **Журналісти**: спершу перевірте власну цілісність маніфесту. Потім
  запросіть документи через окремий безпечний канал. Перед публікацією
  переконайтеся, що SHA-256 кожного документа відповідає маніфесту.
  Не публікуйте документи без згоди викривача; публікуйте існування
  запечатаного маніфесту та ваш висновок щодо перевірки.
- **Регулятори**: той самий процес перевірки. Запечатаний маніфест
  плюс відповідні документи — це найсильніша доказова форма,
  доступна поза формальним збором доказів за ланцюгом зберігання.
- **Юристи**: запечатаний маніфест забезпечує криміналістичне
  закріплення існування документів у заявлений час. Чи задовольняє
  він правила доказів конкретної юрисдикції, вирішувати вам.

## Чим цей набір **не** є

- Не замінник юридичної консультації.
- Не гарантія того, що хтось діятиме на основі вашого розкриття.
- Не захист від репресій.
- Не автентифікація змісту документів (тільки байтів у момент запечатування).
- Не спосіб анонімного спілкування з ким-небудь — печатка є публічною;
  якщо ваше ім'я підписувача справжнє, ви ідентифіковані.

## Завершення

Запечатаний маніфест — невелика річ. Це один потік байтів, який доводить,
що більший набір потоків байтів існував у певній формі в певний час. Ця
малість і є його силою: він робить лише одну річ, і він робить цю одну
річ протягом усього часу існування блокчейну Bitcoin.

Якщо ви читаєте це, бо розглядаєте можливість стати викривачем, будь
ласка: спершу поговоріть з юристом. Скористайтеся посібниками з
операційної безпеки вище. Потім, якщо і тільки якщо всі ті висхідні
міркування в порядку, розгляньте цей набір як криптографічний хребет
вашого розкриття. Криптографія — це легка частина. Усе, що виходить
за межі криптографії, — там, де живе реальний ризик.
