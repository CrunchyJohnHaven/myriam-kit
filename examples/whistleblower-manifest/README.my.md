# Whistleblower Manifest Seal Kit

*မြန်မာဘာသာပြန် — မူရင်းအင်္ဂလိပ်ဗားရှင်းအတွက် [README.md](README.md) ကြည့်ပါ*

ထောက်လှမ်းရေးထုတ်ဖော်သူ (whistleblower) နှင့် အများပြည်သူ ကြားရှိ ကြားသူများကြောင့် **အထောက်အထားများ မြှုပ်နှံခြင်း၊ ပြင်ဆင်ခြင်း သို့မဟုတ် ငြင်းခုံခြင်း** ဖြစ်စဉ်ပျက်ကွက်မှု ပုံစံကို ရင်ဆိုင်နေသော ထောက်လှမ်းရေးထုတ်ဖော်သူများအတွက် အသင့်သုံး myriam-kit ဥပမာ။ ဤကိရိယာက ကာကွယ်ပေးသော သမိုင်းဝင် ဖြစ်ရပ်များ —

- Boeing 737 MAX (အတွင်းပိုင်း ဘေးကင်းရေး အစီရင်ခံစာများကို လျှော့ပေါ့ဆခြင်း/မြှုပ်နှံခြင်း)
- Theranos (ဓာတ်ခွဲခန်း စမ်းသပ်မှု စိုးရိမ်စရာများကို နှစ်ပေါင်းများစွာ ဖိနှိပ်ထားခြင်း)
- FDA စစ်ဆေးရေးမှူး အစီရင်ခံစာများ — တင်သွင်းပြီးနောက် ပြင်ဆင်ခြင်း
- Frances Haugen / Facebook (တံဆိပ်ခတ်ထားသော manifest ရှိခဲ့ပါက စာရွက်စာတမ်းများသည် မှန်ကန်ပြီး တောင်းဆိုထားသော အချိန်တွင် တည်ရှိခဲ့သည်ဟု သက်သေပြနိုင်ခဲ့မည်)

ဤကိရိယာက ကာကွယ်ပေးသည် (protocol စကားလုံးအသုံးအနှုန်းဖြင့်) —

- [FACT 05](facts/05_threat_model.txt) ရှိ T1–T3: စာရွက်စာတမ်း အကြောင်းအရာများကို လူသိရှင်ကြား ငြင်းခုံခြင်း၊ စာရွက်စာတမ်းများကို တောင်းဆိုထားသော အချိန်တွင် တံဆိပ်မခတ်ထားဟု တောင်းဆိုခြင်း၊ နောက်ပိုင်း လုပ်ကြံဖန်တီးထားဟု တောင်းဆိုခြင်း။
- T9: တတိယပါတီများက သတင်းသမားများထံ ပြောင်းလဲထားသော မိတ္တူများ ပေးအပ်ခြင်း။

ဤကိရိယာက ကာကွယ်**မပေး**သည် —

- ထောက်လှမ်းရေးထုတ်ဖော်သူအပေါ် ပုဂ္ဂိုလ်ရေး လက်တုံ့ပြန်မှု (T7)။
- ထောက်လှမ်းရေးထုတ်ဖော်သူအား ပြန်လည်ရုပ်သိမ်းရန် ဖိအား (T4၊ FACT 07 က တိတ်တိတ်တိုးတိုး ပြန်လည်ရုပ်သိမ်းခြင်းကို လူသိရှင်ကြားသိနိုင်စေသဖြင့် တစ်စိတ်တစ်ပိုင်း လျှော့ပါးထားသည်)။
- ထောက်လှမ်းရေးထုတ်ဖော်ခြင်း ကိုယ်တိုင်အပေါ် တရားရေး လုပ်ဆောင်ချက် (T5)။
- ထောက်လှမ်းရေးထုတ်ဖော်သူ၏ ကိုယ်ရည်ကိုယ်သွေးကို ဖျက်ဆီးခြင်း (T8)။
- ကြားသူများက ဇာတ်လမ်းကို တိတ်တိတ်တိုးတိုး ဖယ်ထုတ်ခြင်း (T10)။

သင်သည် ဤကိရိယာအသုံးပြုရန် စဉ်းစားနေသော ထောက်လှမ်းရေးထုတ်ဖော်သူဆိုပါက **ဤ README တစ်ခုလုံးကို ဦးစွာ ဖတ်ပါ**။ protocol သည် ကိရိယာတစ်ခုဖြစ်သည်။ protocol သည် ရှေ့နေ၊ သတင်းသမား၊ ထိန်းချုပ်ရေး အာဏာပိုင် သို့မဟုတ် ဂရုတစိုက် လုပ်ဆောင်ထားသော လုပ်ငန်းခွင် လုံခြုံရေး၏ အစားထိုးမဟုတ်ပါ။

## ဤကိရိယာက ထုတ်ပေးသည်များ

အောက်ပါတို့ကို သက်သေပြသော အထောက်အထား *manifest* (အထောက်အထား ကိုယ်တိုင် မဟုတ်ပါ) တစ်ခုကို cryptographic နည်းဖြင့် တံဆိပ်ခတ်ထားသည် —

1. တံဆိပ်ခတ်ချိန်တွင် အထောက်အထား ဖိုင်တစ်ခုစီ၏ တိကျသော bytes များ တည်ရှိခဲ့ခြင်း။
2. တံဆိပ်ခတ်ခြင်းသည် တိကျသော အချိန်တွင် သို့မဟုတ် ထိုအချိန်မတိုင်မီ ဖြစ်ပျက်ခြင်း (Bitcoin OTS attestation ဖြင့်)။
3. မူရင်းနေရာ၊ စွပ်စွဲချက်၊ ခြိမ်းခြောက်မှု မော်ဒယ်၊ လက်ခံသူ protocol နှင့် တိတ်တိတ်တိုးတိုး ပြန်လည်ရုပ်သိမ်းမည်မဟုတ်ဟု ကတိကဝတ်တို့ ပါဝင်သည်များကို အတူတကွ ထုတ်ဖော်ခြင်း။

မူရင်း စာရွက်စာတမ်းများသည် လက်ခံသူထံ မပို့မီအထိ သင်၏ လက်ဝယ်တွင် ဆက်လက်တည်ရှိနေသည်။ လက်ခံသူများသည် သင်တံဆိပ်ခတ်ထားသော bytes များနှင့် တူညီသော bytes များ လက်ခံရရှိခြင်းကို စစ်ဆေးသည်။

## ဤကိရိယာကို မည်သည့်အခါ သုံးသင့်သနည်း

အောက်ပါအချက် အားလုံး မှန်ကန်လျှင် သတင်းသမား၊ ထိန်းချုပ်ရေး အာဏာပိုင် သို့မဟုတ် ရှေ့နေထံ အထောက်အထား **မပို့မီ** manifest တံဆိပ်ခတ်ရန် စဉ်းစားသင့်သည် —

- သင်တွင် bytewise စစ်မှန်ကြောင်း ထိန်းသိမ်းလိုသော ဒစ်ဂျစ်တယ်ပုံစံ အထောက်အထား (PDFs၊ emails၊ images စသည်) ရှိသည်။
- ကြားသူများက အထောက်အထားကို ပြောင်းလဲခြင်း၊ ငြင်းခုံခြင်း သို့မဟုတ် မြှုပ်နှံခြင်း ပြုလုပ်နိုင်သည်ဟု သင်မျှော်လင့်ထားသည်။
- မူရင်း စာရွက်စာတမ်းများကို ပို့ဆောင်ချိန်အထိ သင်၏ လက်ဝယ်တွင် ထိန်းသိမ်းနိုင်သည်။
- စာရွက်စာတမ်းများ၏ အကြောင်းအရာ မဖော်ပြဘဲ (hash များသာ အများပြည်သူသိရှိနိုင်) manifest ကို လူသိရှင်ကြား ထုတ်ဝေနိုင်သည်။
- ထောက်လှမ်းရေးထုတ်ဖော်သူအဖြစ် လူသိရှင်ကြား ဖော်ထုတ်ခံရနိုင်သည့် အန္တရာယ်များကို သင် ချိန်ဆပြီးပြီ (signing-principal ပိုင်း)။ အမည်မဖော်လိုပါက ဆက်နွှယ်ထုတ်ဝေချက်များ အားလုံးတွင် pseudonym တစ်ခုကို တသမတ်တည်း သုံးပါ။
- သင်၏ နယ်နိမိတ်တွင်း ထောက်လှမ်းရေးထုတ်ဖော်သူ ကာကွယ်ရေး ဥပဒေ အကြောင်း ရှေ့နေတစ်ဦးနှင့် တိုင်ပင်ပြီးပြီ။

## ဤကိရိယာကို မည်သည့်အခါ မသုံးသင့်သနည်း

အောက်ပါအခြေအနေများတွင် ဤကိရိယာကို **မသုံးပါနှင့်** —

- စာရွက်စာတမ်းများ ကိုယ်တိုင်တွင် တတိယပါတီများ (လူနာများ၊ ဖောက်သည်များ၊ ဝန်ထမ်းများ) ၏ public-hashable ဖြစ်စေသင့်သည့် ပုဂ္ဂိုလ်ရေး အချက်အလက်များ ပါဝင်နေသည်။ မူရင်းအစား ပြန်လည် တည်းဖြတ်ထားသော ဗားရှင်းကို hash လုပ်ပါ။
- manifest ထုတ်ဝေခြင်း ကိုယ်တိုင်က ထောက်လှမ်းရေးထုတ်ဖော်သူကို သူခွင့်ပြုမထားသော နည်းလမ်းဖြင့် ဖော်ထုတ်နိုင်သည်။
- ရှေ့နေတစ်ဦးနှင့် တိုင်ပင်မပြီးသေးပါ။ protocol သည် ကိရိယာဖြစ်ပြီး၊ ထောက်လှမ်းရေးထုတ်ဖော်သူ ကာကွယ်ရေး ဥပဒေသည် နယ်နိမိတ်အလိုက် ဖြစ်သည်၊ protocol ၏ ပြင်ပတွင် တည်ရှိသည်။
- နောက်ပိုင်းတွင် စွပ်စွဲချက်တစ်ခုခုကို ပြန်လည်ရုပ်သိမ်းချင်ကောင်း ရုပ်သိမ်းချင်နိုင်သည်။ တံဆိပ်ခတ်ခြင်းသည် အပြီးအပိုင်ဖြစ်သည်၊ တံဆိပ်ခတ်ထားသော ပြင်ဆင်ချက်အသစ်များသာ ပြန်လည်ပြင်ဆင်ရန် ရိုးသားသော တစ်ခုတည်းသော နည်းလမ်းဖြစ်သည်။
- သင်သည် အတင်းအဓမ္မ သို့မဟုတ် အချိန်ဖိအားအောက်တွင် ဤအလုပ်ကို လုပ်နေသည်။ protocol သည် အတင်းအဓမ္မပြုခြင်းကို တိုးပွားစေသည်၊ refusal gate (myriam-kit `SPEC §8`) သည် ထုတ်ဝေသူများကဲ့သို့ ထောက်လှမ်းရေးထုတ်ဖော်သူများအတွက်လည်း သက်ဆိုင်သည်။

## ဤကိရိယာကို အမှန်တကယ် ထုတ်ဖော်ချက်အတွက် မည်သို့ fork လုပ်မည်နည်း

```bash
# 1. Copy this example into a new directory
cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. Remove the fictional content; replace with your real facts.
# Each file under facts/ is one fact. Read each and replace.
$EDITOR facts/01_example_notice.txt   # change to a real cover statement
$EDITOR facts/02_provenance.txt       # real provenance of your evidence
$EDITOR facts/03_evidence_manifest.txt # hash your real documents and list them
$EDITOR facts/04_allegation_summary.txt # your real allegation
$EDITOR facts/05_threat_model.txt     # your real expected responses
$EDITOR facts/06_recipient_protocol.txt # adjust for your recipient
$EDITOR facts/07_no_retraction_commitment.txt # your no-retraction declaration
$EDITOR facts/08_signing_principal.txt # your real signing identity (or pseudonym)

# 3. Update site/index.html to reflect the real facts.
# The seal.sh script does NOT auto-populate the HTML; the manual edit
# is the human-review step.
$EDITOR site/index.html

# 4. Seal.
bash build/seal.sh
# Outputs: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. Verify locally.
python3 build/verify.py

# 6. Publish the manifest (NOT the documents) to any of:
#    - A static host you control (Cloudflare Pages, GitHub Pages, etc.)
#    - The IPFS network at the printed CID
#    - Both (recommended)

# 7. Separately, transmit the underlying documents to the recipient
#    via a secure channel of your choice (Signal, SecureDrop, etc.).
#    The recipient verifies file hashes against your manifest.
```

## အမှန်တကယ် အထောက်အထား ဖိုင်များအတွက် hash များ တွက်ချက်ခြင်း

SHA-256 ဖြင့် စာရွက်စာတမ်း ဖိုင်တစ်ခုကို hash လုပ်ရန် —

```bash
shasum -a 256 my_document.pdf
```

ပထမ hex စာလုံး ၆၄ လုံးသည် SHA-256 ဖြစ်သည်။ ၎င်းကို manifest ထဲတွင် ထည့်ပါ။

ဖိုင်အများကို တစ်ပြိုင်တည်း hash လုပ်နိုင်သည် —

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

ထို့ဖြင့် `facts/03_evidence_manifest.txt` အတွက် ဒေတာ ရရှိသည်။

## လုပ်ငန်းခွင် လုံခြုံရေး မှတ်ချက်များ

MYRIAM protocol သည် လုပ်ငန်းခွင် လုံခြုံရေးကို မပေးပါ။ ၎င်းသည် *အထောက်အထား ကြာရှည်ခံမှု* ကိုသာ ပေးသည်။ သင်လိုအပ်နေသေးသည်များ —

- သင်၏ ရှေ့နေထံ လုံခြုံသော ဆက်သွယ်ရေး လမ်းကြောင်း ([Signal](https://signal.org/) ၊ မျက်နှာချင်းဆိုင် တွေ့ဆုံခြင်း သို့မဟုတ် သင်၏ ရှေ့နေ၏ တည်ထောင်ပြီးသား လုံခြုံသော လမ်းကြောင်း)
- မူရင်း စာရွက်စာတမ်းများအတွက် လုံခြုံသော သိုလှောင်နေရာ (encrypt လုပ်ထား၊ သင်၏ အလုပ်ရှင်က ဝင်ရောက်ကြည့်ရှုနိုင်သော cloud ဝန်ဆောင်မှုများပေါ်တွင် မဟုတ်)
- တံဆိပ်ခတ်ခြင်းအတွက် သန့်ရှင်းသော စက် (အလုပ်ရှင် အကောင့်များနှင့် မပတ်သက်သော ပုဂ္ဂိုလ်ရေး laptop ၊ ဤလုပ်ငန်းအတွက်သာ သုံးသော တစ်လုံးက ပိုကောင်းသည်)
- ထုတ်ဝေခြင်းအတွက် သန့်ရှင်းသော ကွန်ရက် (သင်၏ အလုပ်ရှင်၏ ကွန်ရက် မဟုတ်၊ သင်၏ အလုပ်ရှင်က သင်၏ ISP ကို subpoena ထုတ်နိုင်ပါက သင်၏ အိမ်တွင်း IP မဟုတ်)

ထောက်လှမ်းရေးထုတ်ဖော်သူများအတွက် လုပ်ငန်းခွင် လုံခြုံရေး ဖတ်စရာ အကြံပြုချက်များ —

- [The Intercept's SecureDrop documentation](https://securedrop.org/)
- [Freedom of the Press Foundation's resources](https://freedom.press/)
- [EFF's surveillance self-defense guide](https://ssd.eff.org/)

တံဆိပ်ခတ်ခြင်း လုပ်ငန်းစဉ်ကို မစတင်မီ ၎င်းတို့ကို တိုင်ပင်ပါ။ protocol စတင်ခြင်းမတိုင်မီ ပြုလုပ်ထားသော လုပ်ငန်းခွင် လုံခြုံရေး ဆုံးဖြတ်ချက် မှားယွင်းမှုများကို protocol က မကာကွယ်ပါ။

## လက်ခံသူများ သိရှိသင့်သည်များ

တံဆိပ်ခတ်ထားသော manifest နှင့်အတူ ဤ README ကို လက်ခံသူများထံ ပို့ပါ။ manifest ကိုယ်တိုင်တွင် လက်ခံသူ protocol (FACT 06) ပါဝင်သော်လည်း README က လက်ခံသူ၏ အခန်းကဏ္ဍကို ပိုမို ချဲ့ထွင်ထားသည် —

- **သတင်းသမားများ**: manifest ၏ ကိုယ်ပိုင် တိကျမှန်ကန်မှုကို ဦးစွာ စစ်ဆေးပါ။ ထို့နောက် သီးခြား လုံခြုံသော လမ်းကြောင်းမှ စာရွက်စာတမ်းများကို တောင်းခံပါ။ ထုတ်ဝေခြင်းမတိုင်မီ စာရွက်စာတမ်းတစ်ခုစီ၏ SHA-256 သည် manifest နှင့် ကိုက်ညီခြင်းကို စစ်ဆေးပါ။ ထောက်လှမ်းရေးထုတ်ဖော်သူ၏ သဘောတူညီချက် မရှိဘဲ စာရွက်စာတမ်းများကို **မထုတ်ဝေပါနှင့်**၊ တံဆိပ်ခတ်ထားသော manifest တည်ရှိခြင်းနှင့် သင်၏ စစ်ဆေးမှု ကောက်ချက်ကို ထုတ်ဝေပါ။
- **ထိန်းချုပ်ရေး အာဏာပိုင်များ**: တူညီသော စစ်ဆေးမှု လုပ်ငန်းစဉ်။ တံဆိပ်ခတ်ထားသော manifest နှင့် ကိုက်ညီသော စာရွက်စာတမ်းများ ပေါင်းစပ်ထားခြင်းသည် တရားဝင် chain-of-custody စုဆောင်းမှု ၏ ပြင်ပတွင် ရရှိနိုင်သော အခိုင်မာဆုံး အထောက်အထား ပုံစံဖြစ်သည်။
- **ရှေ့နေများ**: တံဆိပ်ခတ်ထားသော manifest သည် တောင်းဆိုထားသော အချိန်တွင် စာရွက်စာတမ်းများ တည်ရှိခြင်းအတွက် forensic anchor ပေးသည်။ မည်သည့် တိကျသော နယ်နိမိတ်၏ အထောက်အထား စည်းမျဉ်းများကို ဖြည့်ဆည်းနိုင်သည် မဖြည့်ဆည်းနိုင်သည်ကို သင် ဆုံးဖြတ်ရန် ဖြစ်သည်။

## ဤကိရိယာက မဟုတ်သည်များ

- ဥပဒေဆိုင်ရာ ဆွေးနွေးတိုင်ပင်မှု၏ အစားထိုး မဟုတ်ပါ။
- သင်၏ ထုတ်ဖော်မှု အပေါ် မည်သူမဆို လုပ်ဆောင်လိမ့်မည်ဟု အာမခံချက် မဟုတ်ပါ။
- လက်တုံ့ပြန်မှုကို ကာကွယ်မှု မဟုတ်ပါ။
- စာရွက်စာတမ်းများ၏ အကြောင်းအရာ စစ်မှန်ကြောင်း သက်သေပြခြင်း မဟုတ်ပါ (၎င်းတို့၏ တံဆိပ်ခတ်ချိန်ရှိ bytes များကိုသာ သက်သေပြသည်)။
- မည်သူနှင့်မဆို အမည်မဖော်ဘဲ ဆက်သွယ်ရန် နည်းလမ်း မဟုတ်ပါ — တံဆိပ်ခတ်ခြင်းသည် အများပြည်သူဆိုင်ရာ၊ သင်၏ signing-principal အမည်သည် အမှန်တကယ်ဖြစ်ပါက သင်ကို ဖော်ထုတ်ပြီးသား။

## နိဂုံး

တံဆိပ်ခတ်ထားသော manifest သည် သေးငယ်သော အရာတစ်ခုဖြစ်သည်။ ၎င်းသည် byte stream များ ပိုကြီးသော အစုသည် တိကျသော အချိန်တွင် တိကျသော ပုံစံဖြင့် တည်ရှိခဲ့ကြောင်း သက်သေပြသော byte stream တစ်ခုဖြစ်သည်။ ထို သေးငယ်မှုသည် ၎င်း၏ ခွန်အားဖြစ်သည် — ၎င်းသည် တစ်ခုသာ လုပ်ဆောင်ပြီး၊ ထို တစ်ခုကို Bitcoin blockchain ၏ သက်တမ်းအတွင်း လုပ်ဆောင်သည်။

သင်သည် ထောက်လှမ်းရေးထုတ်ဖော်သူ ဖြစ်ရန် စဉ်းစားနေ၍ ဤစာကို ဖတ်နေသည်ဆိုပါက ကျေးဇူးပြု၍ — ဦးစွာ ရှေ့နေတစ်ဦးနှင့် ဆွေးနွေးပါ။ အထက်ပါ လုပ်ငန်းခွင် လုံခြုံရေး လမ်းညွှန်ချက်များကို သုံးပါ။ ထို့နောက် အထက်ပိုင်း စဉ်းစားရမည့်များ အားလုံး အဆင်ပြေပါက သာလျှင် ဤကိရိယာအစုံကို သင်၏ ထုတ်ဖော်ချက် ၏ cryptographic ကျောရိုးအဖြစ် စဉ်းစားပါ။ cryptography သည် လွယ်ကူသော အပိုင်းဖြစ်သည်။ cryptography ၏ အထက်ပိုင်းရှိ အရာအားလုံးတွင် အမှန်တကယ် အန္တရာယ်များ တည်ရှိသည်။
