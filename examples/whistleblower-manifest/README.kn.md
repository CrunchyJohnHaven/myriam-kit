# ವಿಸಲ್‌ಬ್ಲೋವರ್ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಸೀಲ್ ಕಿಟ್

*ಕನ್ನಡ ಅನುವಾದ — ಮೂಲ ಇಂಗ್ಲಿಷ್ ಆವೃತ್ತಿಗಾಗಿ, ನೋಡಿ [README.md](README.md)*

**ವಿಸಲ್‌ಬ್ಲೋವರ್ ಮತ್ತು ಸಾರ್ವಜನಿಕರ ನಡುವಿನ ಮಧ್ಯವರ್ತಿಗಳಿಂದ ಸಾಕ್ಷ್ಯವನ್ನು ಹೂತುಹಾಕುವ, ಸಂಪಾದಿಸುವ ಅಥವಾ ವಿವಾದಿಸುವ** ನಿರ್ದಿಷ್ಟ ವೈಫಲ್ಯ ಪ್ರಕಾರವನ್ನು ಎದುರಿಸುತ್ತಿರುವ ವಿಸಲ್‌ಬ್ಲೋವರ್‌ಗಳಿಗಾಗಿ ಸಿದ್ಧವಾಗಿರುವ myriam-kit ಉದಾಹರಣೆ. ಈ ಕಿಟ್ ರಕ್ಷಿಸುವ ಪ್ರಮಾಣಭೂತ ಐತಿಹಾಸಿಕ ಪ್ರಕರಣಗಳು:

- ಬೋಯಿಂಗ್ 737 MAX (ಆಂತರಿಕ ಸುರಕ್ಷತೆ ವರದಿಗಳನ್ನು ಕಡೆಗಣಿಸಲಾಗಿದೆ/ಹೂತುಹಾಕಲಾಗಿದೆ)
- Theranos (ಲ್ಯಾಬ್-ಪರೀಕ್ಷಾ ಕಾಳಜಿಗಳನ್ನು ವರ್ಷಗಳವರೆಗೆ ನಿಗ್ರಹಿಸಲಾಗಿದೆ)
- FDA ಪರಿವೀಕ್ಷಕ ವರದಿಗಳನ್ನು ಸಲ್ಲಿಸಿದ ನಂತರ ಸಂಪಾದಿಸಲಾಗಿದೆ
- Frances Haugen / Facebook (ಮುದ್ರಿತ ಮ್ಯಾನಿಫೆಸ್ಟ್ ದಾಖಲೆಗಳು ಅಧಿಕೃತವಾಗಿದ್ದವು ಮತ್ತು ಪ್ರತಿಪಾದಿತ ಸಮಯದಲ್ಲಿ ಅಸ್ತಿತ್ವದಲ್ಲಿದ್ದವು ಎಂಬುದನ್ನು ಸಾಬೀತುಪಡಿಸುತ್ತಿತ್ತು)

ಪ್ರೋಟೋಕಾಲ್ ಭಾಷೆಯಲ್ಲಿ ಈ ಕಿಟ್ ಏನನ್ನು ರಕ್ಷಿಸುತ್ತದೆ:

- [FACT 05](facts/05_threat_model.txt) ನಲ್ಲಿನ T1–T3: ದಾಖಲೆ ವಿಷಯಗಳ ಸಾರ್ವಜನಿಕ ವಿವಾದ, ದಾಖಲೆಗಳು ಪ್ರತಿಪಾದಿತ ಸಮಯದಲ್ಲಿ ಮುದ್ರಿಸಲಾಗಿಲ್ಲ ಎಂಬ ಹಕ್ಕು, ಹಿಂದಿರುಗಿಸುವಿಕೆಯ ರಚನೆಯ ಹಕ್ಕು.
- T9: ಮೂರನೇ ಪಕ್ಷಗಳು ಪತ್ರಕರ್ತರಿಗೆ ಬದಲಾಯಿಸಲ್ಪಟ್ಟ ಪ್ರತಿಗಳನ್ನು ಹಸ್ತಾಂತರಿಸುವುದು.

ಈ ಕಿಟ್ **ರಕ್ಷಿಸದ** ವಿಷಯಗಳು:

- ವಿಸಲ್‌ಬ್ಲೋವರ್ ವಿರುದ್ಧ ವೈಯಕ್ತಿಕ ಪ್ರತೀಕಾರ (T7).
- ವಿಸಲ್‌ಬ್ಲೋವರ್ ಮೇಲೆ ಹಿಂದೆಗೆದುಕೊಳ್ಳುವಂತೆ ಒತ್ತಡ (T4, FACT 07 ಮೂಲಕ ಭಾಗಶಃ ತಗ್ಗಿಸಲಾಗಿದೆ, ಇದು ಮೌನ ಹಿಂದೆಗೆದುಕೊಳ್ಳುವಿಕೆಯನ್ನು ಸಾರ್ವಜನಿಕವಾಗಿ ಪತ್ತೆಹಚ್ಚಬಹುದಾಗಿಸುತ್ತದೆ).
- ಬಹಿರಂಗಪಡಿಸುವಿಕೆಗಾಗಿಯೇ ವಿಸಲ್‌ಬ್ಲೋವರ್ ವಿರುದ್ಧ ಕಾನೂನು ಕ್ರಮ (T5).
- ವಿಸಲ್‌ಬ್ಲೋವರ್‌ರ ಚಾರಿತ್ರ್ಯವನ್ನು ಅಪಖ್ಯಾತಿಗೊಳಿಸುವುದು (T8).
- ಮಧ್ಯವರ್ತಿಗಳು ಕಥೆಯನ್ನು ಮೌನವಾಗಿ ಬಿಡುವುದು (T10).

ಈ ಕಿಟ್‌ನ ಬಳಕೆಯನ್ನು ಪರಿಗಣಿಸುತ್ತಿರುವ ವಿಸಲ್‌ಬ್ಲೋವರ್ ಆಗಿದ್ದರೆ, **ಮೊದಲು ಈ ಸಂಪೂರ್ಣ README ಅನ್ನು ಓದಿ**. ಪ್ರೋಟೋಕಾಲ್ ಒಂದು ಸಾಧನವಾಗಿದೆ. ಪ್ರೋಟೋಕಾಲ್ ಕಾನೂನು ಸಲಹೆ, ಪತ್ರಕರ್ತ, ನಿಯಂತ್ರಕ ಅಥವಾ ಎಚ್ಚರಿಕೆಯ ಕಾರ್ಯಾಚರಣಾ ಭದ್ರತೆಗೆ ಬದಲಿಯಲ್ಲ.

## ಈ ಕಿಟ್ ಏನನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ

ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕಲ್ ಆಗಿ ಮುದ್ರಿಸಲಾದ ಸಾಕ್ಷ್ಯದ *ಮ್ಯಾನಿಫೆಸ್ಟ್* (ಸಾಕ್ಷ್ಯವಲ್ಲ) ಇದನ್ನು ಸಾಬೀತುಪಡಿಸುತ್ತದೆ:

1. ಪ್ರತಿ ಸಾಕ್ಷ್ಯ ಫೈಲ್‌ನ ನಿರ್ದಿಷ್ಟ ಬೈಟ್‌ಗಳು ಮುದ್ರಣದ ಕ್ಷಣದಲ್ಲಿ ಅಸ್ತಿತ್ವದಲ್ಲಿದ್ದವು.
2. ಮುದ್ರಣವು ನಿರ್ದಿಷ್ಟ ಸಮಯದಲ್ಲಿ ಅಥವಾ ಮೊದಲು ಸಂಭವಿಸಿತು (Bitcoin OTS ದೃಢೀಕರಣದ ಮೂಲಕ).
3. ಮೂಲ, ಆರೋಪ, ಬೆದರಿಕೆ ಮಾದರಿ, ಸ್ವೀಕರಿಸುವ ಪ್ರೋಟೋಕಾಲ್ ಮತ್ತು ಮೌನ-ಹಿಂದೆಗೆದುಕೊಳ್ಳುವಿಕೆಯಿಲ್ಲದ ಬದ್ಧತೆಯನ್ನು ಎಲ್ಲವನ್ನೂ ಒಟ್ಟಿಗೆ ಘೋಷಿಸಲಾಗಿದೆ.

ಮೂಲ ದಾಖಲೆಗಳು ನೀವು ಅವುಗಳನ್ನು ಸ್ವೀಕರಿಸುವವರಿಗೆ ಕಳುಹಿಸುವವರೆಗೆ ನಿಮ್ಮ ಬಳಿಯೇ ಇರುತ್ತವೆ. ಸ್ವೀಕರಿಸುವವರು ನೀವು ಮುದ್ರಿಸಿದ ಅದೇ ಬೈಟ್‌ಗಳನ್ನು ಅವರು ಸ್ವೀಕರಿಸಿದ್ದಾರೆಂದು ಪರಿಶೀಲಿಸುತ್ತಾರೆ.

## ಈ ಕಿಟ್ ಅನ್ನು ಯಾವಾಗ ಬಳಸಬೇಕು

ಪತ್ರಕರ್ತ, ನಿಯಂತ್ರಕ ಅಥವಾ ವಕೀಲರಿಗೆ ಸಾಕ್ಷ್ಯವನ್ನು ರವಾನಿಸುವ **ಮೊದಲು** ಮ್ಯಾನಿಫೆಸ್ಟ್ ಅನ್ನು ಮುದ್ರಿಸುವುದನ್ನು ನೀವು ಪರಿಗಣಿಸಬೇಕು, ಈ ಎಲ್ಲ ಷರತ್ತುಗಳು ನಿಜವಾಗಿದ್ದರೆ:

- ನೀವು ಬೈಟ್‌ವೈಸ್ ಅಧಿಕೃತವಾಗಿ ಸಂರಕ್ಷಿಸಲು ಬಯಸುವ ಡಿಜಿಟಲ್ ರೂಪದಲ್ಲಿ (PDFs, ಇಮೇಲ್‌ಗಳು, ಚಿತ್ರಗಳು, ಇತ್ಯಾದಿ) ಸಾಕ್ಷ್ಯವಿದೆ.
- ಮಧ್ಯವರ್ತಿಗಳು ಸಾಕ್ಷ್ಯವನ್ನು ಬದಲಾಯಿಸಬಹುದು, ವಿವಾದಿಸಬಹುದು ಅಥವಾ ಹೂತುಹಾಕಬಹುದು ಎಂದು ನೀವು ನಿರೀಕ್ಷಿಸುತ್ತೀರಿ.
- ಪ್ರಸರಣದವರೆಗೆ ಮೂಲ ದಾಖಲೆಗಳನ್ನು ನಿಮ್ಮ ಬಳಿಯೇ ಇಟ್ಟುಕೊಳ್ಳಬಹುದು.
- ದಾಖಲೆಗಳ ವಿಷಯಗಳನ್ನು ಬಹಿರಂಗಪಡಿಸದೆ ಸಾರ್ವಜನಿಕವಾಗಿ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಅನ್ನು ಪ್ರಕಟಿಸಬಹುದು (ಕೇವಲ ಹ್ಯಾಶ್‌ಗಳು ಸಾರ್ವಜನಿಕ).
- ವಿಸಲ್‌ಬ್ಲೋವರ್ ಆಗಿ ಸಾರ್ವಜನಿಕವಾಗಿ ಗುರುತಿಸಲ್ಪಡುವ ಅಪಾಯಗಳನ್ನು ನೀವು ತೂಗಿನೋಡಿದ್ದೀರಿ (ಸಹಿ-ಪ್ರಧಾನ ಕ್ಷೇತ್ರ). ಅನಾಮಧೇಯವಾಗಿದ್ದರೆ, ಎಲ್ಲ ಸಂಬಂಧಿತ ಪ್ರಕಟಣೆಗಳಲ್ಲಿ ಸ್ಥಿರವಾಗಿ ಗುಪ್ತನಾಮವನ್ನು ಬಳಸಿ.
- ನಿಮ್ಮ ನ್ಯಾಯವ್ಯಾಪ್ತಿಯಲ್ಲಿ ವಿಸಲ್‌ಬ್ಲೋವರ್-ರಕ್ಷಣಾ ಕಾನೂನಿನ ಬಗ್ಗೆ ವಕೀಲರನ್ನು ನೀವು ಸಂಪರ್ಕಿಸಿದ್ದೀರಿ.

## ಈ ಕಿಟ್ ಅನ್ನು ಯಾವಾಗ ಬಳಸಬಾರದು

ಈ ಸಂದರ್ಭಗಳಲ್ಲಿ ಈ ಕಿಟ್ ಅನ್ನು ಬಳಸಬೇಡಿ:

- ದಾಖಲೆಗಳಲ್ಲಿಯೇ ಮೂರನೇ ಪಕ್ಷಗಳ (ರೋಗಿಗಳು, ಗ್ರಾಹಕರು, ಉದ್ಯೋಗಿಗಳು) ಬಗ್ಗೆ ವೈಯಕ್ತಿಕ ಮಾಹಿತಿ ಇದ್ದು, ಅದನ್ನು ಸಾರ್ವಜನಿಕವಾಗಿ ಹ್ಯಾಶ್ ಮಾಡಬಾರದು. ಮೂಲದ ಬದಲು ಸಂಪಾದಿತ ಆವೃತ್ತಿಯನ್ನು ಹ್ಯಾಶ್ ಮಾಡಿ.
- ಮ್ಯಾನಿಫೆಸ್ಟ್ ಅನ್ನು ಪ್ರಕಟಿಸುವ ಕ್ರಿಯೆಯೇ ವಿಸಲ್‌ಬ್ಲೋವರ್‌ರನ್ನು ಅವರು ಒಪ್ಪದ ರೀತಿಯಲ್ಲಿ ಗುರುತಿಸುತ್ತದೆ.
- ನೀವು ಇನ್ನೂ ವಕೀಲರನ್ನು ಸಂಪರ್ಕಿಸಿಲ್ಲ. ಪ್ರೋಟೋಕಾಲ್ ಒಂದು ಸಾಧನವಾಗಿದೆ; ವಿಸಲ್‌ಬ್ಲೋವರ್-ರಕ್ಷಣಾ ಕಾನೂನು ನ್ಯಾಯವ್ಯಾಪ್ತಿ-ನಿರ್ದಿಷ್ಟವಾಗಿದೆ ಮತ್ತು ಪ್ರೋಟೋಕಾಲ್‌ನ ಹೊರಗಿರುತ್ತದೆ.
- ಯಾವುದೇ ಒಂದು ಆರೋಪವನ್ನು ನಂತರ ಹಿಂದೆಗೆದುಕೊಳ್ಳಲು ನೀವು ಬಯಸಬಹುದು. ಸೀಲ್ ಶಾಶ್ವತವಾಗಿದೆ; ಪ್ರಾಮಾಣಿಕವಾಗಿ ಪರಿಷ್ಕರಿಸುವ ಏಕೈಕ ಮಾರ್ಗ ಹೊಸ ಮುದ್ರಿತ ತಿದ್ದುಪಡಿಗಳು.
- ನೀವು ಒತ್ತಡ ಅಥವಾ ಸಮಯದ ಒತ್ತಡದಲ್ಲಿ ಇದನ್ನು ಮಾಡುತ್ತಿದ್ದೀರಿ. ಪ್ರೋಟೋಕಾಲ್ ಬಲಾತ್ಕಾರವನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ; ನಿರಾಕರಣಾ ಗೇಟ್ (myriam-kit `SPEC §8`) ಪ್ರಕಾಶಕರಿಗೆ ಎಷ್ಟು ಅನ್ವಯವಾಗುತ್ತದೆಯೋ ಅಷ್ಟೇ ವಿಸಲ್‌ಬ್ಲೋವರ್‌ಗಳಿಗೂ ಅನ್ವಯವಾಗುತ್ತದೆ.

## ನಿಜವಾದ ಬಹಿರಂಗಪಡಿಸುವಿಕೆಗಾಗಿ ಈ ಕಿಟ್ ಅನ್ನು ಫೋರ್ಕ್ ಮಾಡುವುದು ಹೇಗೆ

```bash

cp -r ~/Genesis/myriam-kit/examples/whistleblower-manifest ~/my-disclosure
cd ~/my-disclosure

# 2. ಕಾಲ್ಪನಿಕ ವಿಷಯವನ್ನು ತೆಗೆದುಹಾಕಿ; ನಿಮ್ಮ ನಿಜವಾದ ಸಂಗತಿಗಳಿಂದ ಬದಲಾಯಿಸಿ.

$EDITOR facts/02_provenance.txt       # ನಿಮ್ಮ ಸಾಕ್ಷ್ಯದ ನಿಜವಾದ ಮೂಲ
$EDITOR facts/03_evidence_manifest.txt # ನಿಮ್ಮ ನಿಜವಾದ ದಾಖಲೆಗಳನ್ನು ಹ್ಯಾಶ್ ಮಾಡಿ ಮತ್ತು ಪಟ್ಟಿ ಮಾಡಿ
$EDITOR facts/04_allegation_summary.txt # ನಿಮ್ಮ ನಿಜವಾದ ಆರೋಪ
$EDITOR facts/05_threat_model.txt     # ನಿಮ್ಮ ನಿಜವಾದ ನಿರೀಕ್ಷಿತ ಪ್ರತಿಕ್ರಿಯೆಗಳು

# 3. ನಿಜವಾದ ಸಂಗತಿಗಳನ್ನು ಪ್ರತಿಬಿಂಬಿಸಲು site/index.html ಅನ್ನು ನವೀಕರಿಸಿ.

$EDITOR site/index.html

bash build/seal.sh
# ಔಟ್‌ಪುಟ್‌ಗಳು: merkle_root.txt, merkle_root.txt.ots, IPFS CID

# 5. ಸ್ಥಳೀಯವಾಗಿ ಪರಿಶೀಲಿಸಿ.
```

## ನಿಜವಾದ ಸಾಕ್ಷ್ಯ ಫೈಲ್‌ಗಳಿಗಾಗಿ ಹ್ಯಾಶ್‌ಗಳನ್ನು ಲೆಕ್ಕಿಸುವುದು

SHA-256 ನೊಂದಿಗೆ ದಾಖಲೆ ಫೈಲ್ ಅನ್ನು ಹ್ಯಾಶ್ ಮಾಡಲು:

```bash
shasum -a 256 my_document.pdf
```

ಮೊದಲ 64 ಹೆಕ್ಸ್ ಅಕ್ಷರಗಳು SHA-256 ಆಗಿವೆ. ಅದನ್ನು ಮ್ಯಾನಿಫೆಸ್ಟ್‌ನಲ್ಲಿ ಇರಿಸಿ.

ನೀವು ಒಂದು ಬಾರಿಗೆ ಹಲವು ಫೈಲ್‌ಗಳನ್ನು ಹ್ಯಾಶ್ ಮಾಡಬಹುದು:

```bash
shasum -a 256 *.pdf *.eml *.jpg > MANIFEST.txt
```

ಅದು `facts/03_evidence_manifest.txt` ಗಾಗಿ ಡೇಟಾವನ್ನು ನೀಡುತ್ತದೆ.

## ಕಾರ್ಯಾಚರಣಾ ಭದ್ರತಾ ಟಿಪ್ಪಣಿಗಳು

MYRIAM ಪ್ರೋಟೋಕಾಲ್ ಕಾರ್ಯಾಚರಣಾ ಭದ್ರತೆಯನ್ನು ಒದಗಿಸುವುದಿಲ್ಲ. ಇದು *ಸಾಕ್ಷ್ಯಾಧಾರಿತ ಬಾಳಿಕೆ* ಯನ್ನು ಒದಗಿಸುತ್ತದೆ. ನಿಮಗೆ ಇನ್ನೂ ಬೇಕು:

- ನಿಮ್ಮ ವಕೀಲರಿಗೆ ಸುರಕ್ಷಿತ ಸಂವಹನ ಚಾನಲ್ ([Signal](https://signal.org/), ಪ್ರತ್ಯಕ್ಷ, ಅಥವಾ ನಿಮ್ಮ ವಕೀಲರ ಸ್ಥಾಪಿತ ಸುರಕ್ಷಿತ ಚಾನಲ್)
- ಮೂಲ ದಾಖಲೆಗಳಿಗಾಗಿ ಸುರಕ್ಷಿತ ಸಂಗ್ರಹಣಾ ಸ್ಥಳ (ವಿಶ್ರಾಂತಿಯಲ್ಲಿ ಎನ್‌ಕ್ರಿಪ್ಟ್ ಮಾಡಲಾಗಿದೆ, ನಿಮ್ಮ ಉದ್ಯೋಗದಾತರು ಪ್ರವೇಶಿಸಬಹುದಾದ ಕ್ಲೌಡ್ ಸೇವೆಗಳಲ್ಲಿ ಅಲ್ಲ)
- ಮುದ್ರಣಕ್ಕಾಗಿ ಸ್ವಚ್ಛ ಸಾಧನ (ಉದ್ಯೋಗದಾತರ ಖಾತೆಗಳೊಂದಿಗೆ ಸಂಬಂಧವಿಲ್ಲದ ವೈಯಕ್ತಿಕ ಲ್ಯಾಪ್‌ಟಾಪ್; ಮೇಲಾಗಿ ಈ ಉದ್ದೇಶಕ್ಕಾಗಿ ಮಾತ್ರ ಬಳಸಲಾಗಿರುವುದು)
- ಪ್ರಕಟಣೆಗಾಗಿ ಸ್ವಚ್ಛ ನೆಟ್‌ವರ್ಕ್ (ನಿಮ್ಮ ಉದ್ಯೋಗದಾತರ ನೆಟ್‌ವರ್ಕ್ ಅಲ್ಲ, ನಿಮ್ಮ ಉದ್ಯೋಗದಾತರು ನಿಮ್ಮ ISP ಅನ್ನು ಸಬ್‌ಪೋನ ಮಾಡಬಹುದಾದರೆ ನಿಮ್ಮ ಮನೆಯ IP ಅಲ್ಲ)

ವಿಸಲ್‌ಬ್ಲೋವರ್‌ಗಳಿಗಾಗಿ ಕಾರ್ಯಾಚರಣಾ ಭದ್ರತೆಯ ಬಗ್ಗೆ ಶಿಫಾರಸು ಮಾಡಿದ ಓದುವಿಕೆ:

- [The Intercept's SecureDrop documentation](https://securedrop.org/)
- [Freedom of the Press Foundation's resources](https://freedom.press/)
- [EFF's surveillance self-defense guide](https://ssd.eff.org/)

ಸೀಲಿಂಗ್ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಅವುಗಳನ್ನು ಸಂಪರ್ಕಿಸಿ. ಪ್ರೋಟೋಕಾಲ್ ಆಹ್ವಾನಿಸುವ ಮೊದಲು ಮಾಡಿದ ಕೆಟ್ಟ ಆಪ್‌ಸೆಕ್ ಆಯ್ಕೆಗಳ ವಿರುದ್ಧ ಪ್ರೋಟೋಕಾಲ್ ರಕ್ಷಿಸುವುದಿಲ್ಲ.

## ಸ್ವೀಕರಿಸುವವರು ಏನನ್ನು ತಿಳಿದಿರಬೇಕು

ಮುದ್ರಿತ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಜೊತೆಗೆ ಸ್ವೀಕರಿಸುವವರಿಗೆ ಈ README ಅನ್ನು ಕಳುಹಿಸಿ. ಮ್ಯಾನಿಫೆಸ್ಟ್ ಸ್ವತಃ ಸ್ವೀಕರಿಸುವ ಪ್ರೋಟೋಕಾಲ್ ಅನ್ನು ಹೊಂದಿರುತ್ತದೆ (FACT 06), ಆದರೆ README ಸ್ವೀಕರಿಸುವ ಪಾತ್ರವನ್ನು ವಿಸ್ತರಿಸುತ್ತದೆ:

- **ಪತ್ರಕರ್ತರು**: ಮೊದಲು ಮ್ಯಾನಿಫೆಸ್ಟ್‌ನ ಸ್ವಂತ ಸಮಗ್ರತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ. ನಂತರ ಪ್ರತ್ಯೇಕ ಸುರಕ್ಷಿತ ಚಾನಲ್ ಮೂಲಕ ದಾಖಲೆಗಳನ್ನು ವಿನಂತಿಸಿ. ಪ್ರಕಟಣೆಯ ಮೊದಲು ಪ್ರತಿ ದಾಖಲೆಯ SHA-256 ಮ್ಯಾನಿಫೆಸ್ಟ್‌ಗೆ ಹೊಂದಿಕೆಯಾಗುತ್ತದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ. ವಿಸಲ್‌ಬ್ಲೋವರ್‌ರ ಒಪ್ಪಿಗೆಯಿಲ್ಲದೆ ದಾಖಲೆಗಳನ್ನು ಪ್ರಕಟಿಸಬೇಡಿ; ಮುದ್ರಿತ ಮ್ಯಾನಿಫೆಸ್ಟ್‌ನ ಅಸ್ತಿತ್ವ ಮತ್ತು ನಿಮ್ಮ ಪರಿಶೀಲನಾ ತೀರ್ಮಾನವನ್ನು ಪ್ರಕಟಿಸಿ.
- **ನಿಯಂತ್ರಕರು**: ಅದೇ ಪರಿಶೀಲನಾ ಪ್ರಕ್ರಿಯೆ. ಔಪಚಾರಿಕ ಕಸ್ಟಡಿ-ಚೈನ್ ಸಂಗ್ರಹಣೆಯ ಹೊರತಾಗಿ ಲಭ್ಯವಿರುವ ಪ್ರಬಲವಾದ ಸಾಕ್ಷ್ಯಾಧಾರಿತ ರೂಪವೆಂದರೆ ಮುದ್ರಿತ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಮತ್ತು ಹೊಂದಾಣಿಕೆಯ ದಾಖಲೆಗಳು.
- **ವಕೀಲರು**: ಮುದ್ರಿತ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಪ್ರತಿಪಾದಿತ ಸಮಯದಲ್ಲಿ ದಾಖಲೆಗಳ ಅಸ್ತಿತ್ವಕ್ಕಾಗಿ ಫೋರೆನ್ಸಿಕ್ ಲಂಗರು ಒದಗಿಸುತ್ತದೆ. ಇದು ಯಾವುದೇ ನಿರ್ದಿಷ್ಟ ನ್ಯಾಯವ್ಯಾಪ್ತಿಯ ಸಾಕ್ಷ್ಯ-ನಿಯಮಗಳನ್ನು ತೃಪ್ತಿಪಡಿಸುತ್ತದೆಯೇ ಎಂಬುದನ್ನು ನೀವು ನಿರ್ಧರಿಸಬೇಕು.

## ಈ ಕಿಟ್ ಏನಲ್ಲ

- ಕಾನೂನು ಸಲಹೆಗೆ ಬದಲಿಯಲ್ಲ.
- ಯಾರೂ ನಿಮ್ಮ ಬಹಿರಂಗಪಡಿಸುವಿಕೆಯ ಮೇಲೆ ಕ್ರಮ ಕೈಗೊಳ್ಳುತ್ತಾರೆ ಎಂಬುದರ ಗ್ಯಾರಂಟಿಯಲ್ಲ.
- ಪ್ರತೀಕಾರದ ವಿರುದ್ಧ ರಕ್ಷಣೆಯಲ್ಲ.
- ದಾಖಲೆಗಳ ವಿಷಯಗಳ ದೃಢೀಕರಣವಲ್ಲ (ಕೇವಲ ಸೀಲಿಂಗ್-ಸಮಯದ ಬೈಟ್‌ಗಳದು ಮಾತ್ರ).
- ಯಾರೊಂದಿಗಾದರೂ ಅನಾಮಧೇಯವಾಗಿ ಸಂವಹನ ಮಾಡುವ ಮಾರ್ಗವಲ್ಲ — ಸೀಲ್ ಸಾರ್ವಜನಿಕ; ನಿಮ್ಮ ಸಹಿ-ಪ್ರಧಾನ ಹೆಸರು ನಿಜವಾಗಿದ್ದರೆ, ನೀವು ಗುರುತಿಸಲ್ಪಡುತ್ತೀರಿ.

## ಮುಕ್ತಾಯ

ಮುದ್ರಿತ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಒಂದು ಸಣ್ಣ ವಸ್ತು. ಇದು ಒಂದು ಬೈಟ್ ಸ್ಟ್ರೀಮ್ ಆಗಿದ್ದು, ನಿರ್ದಿಷ್ಟ ಸಮಯದಲ್ಲಿ ನಿರ್ದಿಷ್ಟ ರೂಪದಲ್ಲಿ ಬೈಟ್ ಸ್ಟ್ರೀಮ್‌ಗಳ ದೊಡ್ಡ ಸೆಟ್ ಅಸ್ತಿತ್ವದಲ್ಲಿತ್ತು ಎಂಬುದನ್ನು ಸಾಬೀತುಪಡಿಸುತ್ತದೆ. ಆ ಸಣ್ಣತನವೇ ಅದರ ಶಕ್ತಿ: ಅದು ಕೇವಲ ಒಂದು ಕೆಲಸವನ್ನು ಮಾಡುತ್ತದೆ, ಮತ್ತು Bitcoin ಬ್ಲಾಕ್‌ಚೈನ್‌ನ ಜೀವಿತಾವಧಿಯಾದ್ಯಂತ ಆ ಒಂದು ಕೆಲಸವನ್ನು ಮಾಡುತ್ತದೆ.

ವಿಸಲ್‌ಬ್ಲೋವರ್ ಆಗುವುದನ್ನು ಪರಿಗಣಿಸುತ್ತಿರುವ ಕಾರಣ ನೀವು ಇದನ್ನು ಓದುತ್ತಿದ್ದರೆ, ದಯವಿಟ್ಟು: ಮೊದಲು ವಕೀಲರೊಂದಿಗೆ ಮಾತನಾಡಿ. ಮೇಲಿನ ಕಾರ್ಯಾಚರಣಾ ಭದ್ರತಾ ಮಾರ್ಗದರ್ಶಿಗಳನ್ನು ಬಳಸಿ. ನಂತರ, ಆ ಎಲ್ಲಾ ಮೇಲ್ಮಟ್ಟದ ಪರಿಗಣನೆಗಳು ಸರಿಯಾಗಿದ್ದರೆ ಮಾತ್ರ, ಈ ಕಿಟ್ ಅನ್ನು ನಿಮ್ಮ ಬಹಿರಂಗಪಡಿಸುವಿಕೆಯ ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಬೆನ್ನೆಲುಬು ಎಂದು ಪರಿಗಣಿಸಿ. ಕ್ರಿಪ್ಟೋ ಸುಲಭ ಭಾಗ. ಕ್ರಿಪ್ಟೋದ ಮೇಲ್ಮಟ್ಟದ ಎಲ್ಲವೂ ನಿಜವಾದ ಅಪಾಯ ವಾಸಿಸುವ ಸ್ಥಳ.
