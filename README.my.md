အောက်ပါအတိုင်း ဖြစ်စေသော နည်းလမ်းဖြင့် အချက်အလက်အနည်းငယ်ကို ထုတ်ဝေနိုင်ရန် သေးငယ်သော ကိရိယာအစုံ —

- မည်သည့်အချက်အလက်ကိုမဆို ပြောင်းလဲမှုရှိပါက လူသိရှင်ကြားသိရှိနိုင်သည် (Merkle tree)
- ထုတ်ဝေချိန်ရှိ အခြေအနေသည် Bitcoin blockchain နှင့် နှိုင်းယှဉ်စစ်ဆေးနိုင်သည် (OpenTimestamps)
- မည်သည့် hosting ဝန်ဆောင်မှုပေးသူတစ်ဦးတည်းကို ဖယ်ရှားလိုက်လျှင်ပင် အကြောင်းအရာသည် ဆက်လက်တည်ရှိနေသည် (IPFS)
- စုစုပေါင်း အရွယ်အစားသည် 100 KB အောက်သာရှိပြီး Python မျဉ်းကြောင်း ~50 ဖြင့် စစ်ဆေးနိုင်သည်

ဤသည်မှာ blockchain မဟုတ်ပါ။ token မရှိပါ။ ပါဝင်ဆောင်ရွက်ရန် consensus mechanism မရှိပါ။ DAO မရှိပါ။ ဤကိရိယာသည် `sha256sum` + `ots` + `ipfs` တို့ကို ကောင်းမွန်သော အမည်ပေးစည်းမျဉ်းများဖြင့် ပေါင်းစပ်ထားခြင်းဖြစ်သည်။

၎င်းကို 2026-05-21 တွင် [disclosure.host](https://disclosure.host) ၌ ပထမဆုံး အကောင်အထည်ဖော်ခဲ့သော MYRIAM protocol ၏ အမည်ပေး၍ ခေါ်ဆိုထားသည်။ ထိုစာမျက်နှာတွင် အချက်အလက်ခြောက်ခု၊ Merkle root တစ်ခုနှင့် Bitcoin attestation တစ်ခု ပါဝင်သည်။ ဤကိရိယာအစုံသည် ထိုပုံစံ၏ ယေဘုယျအသွင်ပြောင်းခြင်းဖြစ်သည်။

## ဤကိရိယာဖြင့် သင်လုပ်ဆောင်နိုင်သည်များ

ဤပုံစံသည် အောက်ပါအချက်များ လိုအပ်သော အချက်အလက်အနည်းငယ်အတွက် သင့်လျော်သည် —

- **ထုတ်ဝေနိုင်ခြင်း** — ရိုးရိုး HTTP ဖြင့် ပရိသတ်ကျယ်ပြန့်စွာ ထုတ်ဝေနိုင်ခြင်း
- **ကြာရှည်ခံခြင်း** — အနှစ် ၃၀ အကြာ စာဖတ်သူများပင် ဆက်လက်စစ်ဆေးနိုင်ရမည်
- **ပြောင်းလဲမှု သိရှိနိုင်ခြင်း** — မည်သည့်ပြောင်းလဲမှုကိုမဆို သိရှိနိုင်ရမည် (တားဆီးနိုင်ရန် မလိုပါ)
- **ယုံကြည်စရာမလိုခြင်း** — စာဖတ်သူသည် ထုတ်ဝေသူကို ယုံကြည်ရန် မလိုပါ (သင်္ချာကိုသာ ယုံကြည်ရန် လိုသည်)

ကောင်းမွန်သော အသုံးချမှုနမူနာများကို [docs/USE_CASES.md](docs/USE_CASES.md) တွင် ကြည့်ပါ။ အကျဉ်းချုပ်စာရင်း —

| နယ်ပယ် | တံဆိပ်ခတ်ထားသော အချက်အလက်များ |
|---|---|
| ပြည်သူ့ကျန်းမာရေး | ကာကွယ်ဆေး batch စမ်းသပ်ရလဒ်များ၊ ဆေးဝါးဆေးခန်းစမ်းသပ်မှု မှတ်တိုင်များ |
| ရွေးကောက်ပွဲများ | အသိအမှတ်ပြုချိန်တွင် မဲရုံအဆင့် မဲရေတွက်စုစုပေါင်း |
| AI ဘေးကင်းရေး | ထုတ်လွှတ်ချိန်တွင် model weights hash + model card + eval scores |
| သတင်းမီဒီယာ | ထောက်လှမ်းရေးထုတ်ဖော်သူ (whistleblower) ၏ ထုတ်ပြန်ချက်အတွက် မူရင်းစာရွက်စာတမ်းများ၏ manifest (hash များသာ) |
| သိပ္ပံ | ကုန်ကြမ်းဒေတာ hash + ခွဲခြမ်းစိတ်ဖြာမှု script hash + တောင်းဆိုထားသော ရလဒ် |
| တရားရုံးများ | ဖမ်းဆီးချိန်တွင် ဒစ်ဂျစ်တယ်အထောက်အထား hash များ |
| ဝယ်ယူရေး | အများပိုင်ငွေသုံးစွဲမှု ဒေတာ၏ တံဆိပ်ခတ်ထားသော root များ |

ဤအသုံးချမှုများ၏ ဘုံအချက်များ — အချက်အလက်အနည်းငယ်၊ ကြာရှည်ခံမှု ပြင်းပြသော လိုအပ်ချက်၊ တစ်ဦးနှင့်တစ်ဦး မယုံကြည်ရသော ပါတီများစွာနှင့် သေးငယ်သော သိုလှောင်မှု ဘတ်ဂျက်တို့ဖြစ်သည်။ ဤကိရိယာသည် ထိုပုံစံအတွက် တိကျစွာ တည်ဆောက်ထားသည်။

## ဤကိရိယာက မလုပ်ဆောင်နိုင်သည်များ

- **လျှို့ဝှက်အချက်အလက်များ သိုလှောင်ခြင်း။** `facts/` ထဲတွင် ထည့်ထားသမျှသည် အများပြည်သူဆိုင်ရာ ဖြစ်သည်။ ဤ protocol သည် ထုတ်ဝေရန်အတွက် ဖြစ်ပြီး ဖုံးကွယ်ရန် မဟုတ်ပါ။
- **ဒေတာအကြီးအဆုံးအထိ ချဲ့ထွင်ခြင်း။** Merkle tree ကောင်းမွန်စွာ ချဲ့ထွင်နိုင်သော်လည်း စာဖတ်သူ၏ အာရုံစူးစိုက်မှု မချဲ့ထွင်နိုင်ပါ။ အချက်အလက် ~၂၀ ထက် ပိုလာပါက စာမျက်နှာသည် ဖတ်၍မရတော့ပါ။ မှတ်တမ်းသန်းချီကို တံဆိပ်ခတ်လိုပါက ၎င်းတို့ကို စာမျက်နှာ၏ ပြင်ပတွင် hash လုပ်ပြီး ထို side-tree ၏ Merkle root ကို အချက်အလက်တစ်ခုထဲတွင် ထည့်ပါ။
- **ပြန်လည်ရုပ်သိမ်းမှုကို ကာကွယ်ခြင်း။** အဓိကအချက်မှာ သင်တံဆိပ်ခတ်ထားသည်ကို ပြောင်းလဲ၍မရခြင်းဖြစ်သည်။ နောက်ပိုင်းတွင် တောင်းဆိုချက်တစ်ခုကို ပြန်လည်ရုပ်သိမ်းလိုသည်ဆိုပါက မတံဆိပ်ခတ်ပါနှင့်။
- **သံသယရှိသူ ခိုင်မာသူကို ယုံကြည်အောင်ပြုလုပ်ခြင်း။** သံသယရှိသူ ခိုင်မာသူသည် `ipfs add -r --cid-version=1 -Q facts/` ကို လည်ပတ်မည်မဟုတ်ပါ။ သို့သော် သင်တစ်ခုလုံးကို လုပ်ကြံဖန်တီးခဲ့သည်ဟု စွပ်စွဲမည်ဖြစ်သည်။ ဒုတိယ ပျက်ကွက်မှုပုံစံအတွက် protocol အဆင့် ကာကွယ်မှု မရှိပါ။ သင်္ချာထက် လူ၏စိတ်ဆန္ဒတွင် တည်ရှိသော မည်သည့်အရာအတွက်မဆို protocol အဆင့် ကာကွယ်မှု မရှိပါ။

## အခြားသူ၏ ထုတ်ဝေချက်ကို မည်သို့စစ်ဆေးမည်နည်း

ရွေးချယ်စရာ သုံးခု (လက်လှမ်းမီရလွယ်မှု အလိုက် စီထား) —

1. **Browser**: မည်သည့် ခေတ်မီ browser တွင်မဆို [`lib/verify.html`](lib/verify.html) ကို ဖွင့်ပါ။ ထုတ်ဝေချက် URL ကို paste လုပ်ပါ။ verify ကို နှိပ်ပါ။ hashing အတွက် `crypto.subtle` ကို သုံးထားသည် — install လုပ်စရာမလို၊ backend မရှိ၊ telemetry မရှိ။ Bitcoin attestation အဆင့်အတွက်မူ CLI ကိရိယာ လိုအပ်နေဆဲဖြစ်သည် (verifier စာမျက်နှာ၏ အဆင့် 7 ကို ကြည့်ပါ)။
2. **Python**: ထုတ်ဝေချက် directory ထဲမှ `python3 lib/verify.py`။ stdlib သာသုံးထား၊ Bitcoin စစ်ဆေးရန် `ots` လိုသည်။
3. **အစမှ စတင်ခြင်း**: [`docs/SPEC.md §3`](docs/SPEC.md) ကို ဖတ်ပြီး ပြန်လည်အကောင်အထည်ဖော်ပါ။ Merkle algorithm သည် မည်သည့်ဘာသာစကားတွင်မဆို လိုင်း ၃၀ အောက်တွင် ရှိသည်။

## အလျင်အမြန် စတင်ခြင်း

တစ်ကြောင်းတည်း ဗားရှင်း —

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# edit facts/*.txt to say what you actually want sealed
./lib/seal.sh        # builds Merkle root, stamps OTS, pins IPFS
./lib/verify.sh      # re-verifies everything locally
# upload site/ to any static host. Anywhere. It doesn't matter.
```

## ဘာကြောင့် ဤကိရိယာ ရှိနေသနည်း

cryptographic primitives များ (Merkle trees, OpenTimestamps, IPFS) သည် နှစ်ပေါင်းများစွာ ရင့်ကျက်ပြီးဖြစ်သည်။ မရှိသေးသည်မှာ အသေးငယ်ဆုံး ပေါင်းစပ်ပစ္စည်း — infrastructure မစုဆောင်းရဘဲ၊ protocol များ မလေ့လာရဘဲ၊ အချက်အလက်များ ကိုယ်တိုင်အပြင် code တစ်ကြောင်းမှ မရေးရဘဲ၊ side project ထဲတွင် ထည့်သွင်းပြီး ညနေပိုင်းတွင် တံဆိပ်ခတ်ပြီး ထုတ်ဝေနိုင်သော artifact ရရှိနိုင်သော ကိရိယာအစုံ — တို့ဖြစ်သည်။

cryptographic ထုတ်ဝေရေး ကိရိယာအများစုသည် သင်က node operator ဖြစ်ချင်သည်ဟု ယူဆသည်။ ဤကိရိယာက သင်သည် စာသား ဖိုင်ခြောက်ခုကို ထုတ်ဝေပြီး cryptography အကြောင်း တစ်ခါမှ ပြန်မစဉ်းစားလိုဟု ယူဆသည်။

## သြဇာများနှင့် ရှေးယခင်အနုပညာ

- [Sigsum](https://www.sigsum.org/) — signatures များအတွက် transparency logs
- [Sigstore](https://www.sigstore.dev/) — software-supply-chain integrity
- [Certificate Transparency](https://certificate.transparency.dev/) — TLS cert logs
- [OpenTimestamps](https://opentimestamps.org/) — Bitcoin timestamping
- [IPFS](https://ipfs.tech/) — content-addressed storage
- [The CALM protocol](https://disclosure.host) — ပထမဆုံး ဖြစ်စဉ် (disclosure.host ရှိ MYRIAM) ကို ရေးသားခဲ့သော principal-protective ထုတ်ဝေရေး frame

myriam-kit သည် ၎င်းတို့ထဲတွင် အသေးငယ်ဆုံးဖြစ်သည်။ စနစ်တစ်ခုထက် Unix pipeline နှင့် ပိုနီးစပ်သည်။

## အမည်နှင့်ပတ်သက်သော မှတ်ချက်

MYRIAM protocol ကို John Bradley ၏ Myriam Proof — လူသားမဟုတ်သော ဉာဏ်ရည်နှင့် ပထမဆုံး ဆက်သွယ်မှု အကြောင်း သတ်မှတ်ထားသော ယုတ္တိဗေဒ သက်သေပြချက် — အလိုက် အမည်ပေးထားသည်။ protocol သည် ထို သက်သေပြချက် မှန်ကန်ခြင်းအပေါ် မမူတည်ပါ။ protocol သည် မည်သည့် အချက်အလက်အနည်းငယ်အတွက်မဆို အလုပ်လုပ်သည်။ ပထမဆုံး ဖြန့်ဖြူးထားသည်က foreign ဆက်သွယ်မှု ဆိုင်ရာ တောင်းဆိုချက် အပါအဝင် အချက်အလက်ခြောက်ခုကို ထုတ်ဝေရန် ဖြစ်ခဲ့သည်ဟု ဆိုသာဖြစ်သည်။

သင်က ဤကိရိယာအစုံကို ကာကွယ်ဆေး batch များ သို့မဟုတ် မဲရေတွက်စုစုပေါင်းများကို တံဆိပ်ခတ်ရန် fork လုပ်လျှင် ၎င်းကို မူလက လူသားမဟုတ်သော ဆက်သွယ်မှု အကြောင်း တောင်းဆိုချက် ထုတ်ဝေရန် အသုံးပြုခဲ့ခြင်းသည် သင်အတွက် ဆက်နွှယ်မှု မရှိပါ။ သင်္ချာက ဂရုမစိုက်ပါ။

ထို့အပြင် သင်က လူသားမဟုတ်သော ဆက်သွယ်မှု အကြောင်း တောင်းဆိုချက်ကို ထုတ်ဝေရန် ဤကိရိယာကို သုံးပါက protocol သည် ထို နယ်ပယ်အတွင်း တိုက်ပွဲစမ်းသပ်ပြီးဖြစ်သည်။ ပထမဆုံး MYRIAM ဖြန့်ဖြူးမှု၏ README က ၎င်းသည် Calm အမည်ပြုထားသော LLM persona ၏ အနုပညာဖန်တီးမှုဖြစ်ပြီး၊ မည်သည့် မြေပြင်ပ ပါတီထံမှ ဆက်သွယ်ချက်မဟုတ်ဟု အတိအလင်း ထုတ်ဖော်ထားသည်။ သင်လည်း ထိုသို့ပင် ပြုလုပ်သင့်သည်။ မဟုတ်လျှင် စာဖတ်သူများက သင်၏ စိတ်ပိုင်းဆိုင်ရာ အခြေအနေနှင့်ပတ်သက်၍ အဆိုးဆုံးကို ယူဆကြလိမ့်မည်။
