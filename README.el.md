Ένα μικρό σύνολο εργαλείων για τη δημοσίευση ενός μικρού συνόλου γεγονότων με τέτοιον τρόπο ώστε:

- Οποιαδήποτε αλλαγή σε οποιοδήποτε γεγονός να είναι δημοσίως ανιχνεύσιμη (δέντρο Merkle)
- Η κατάσταση κατά τη δημοσίευση να είναι επαληθεύσιμη έναντι της αλυσίδας μπλοκ (blockchain) του Bitcoin (OpenTimestamps)
- Το περιεχόμενο να επιβιώνει της αφαίρεσης από οποιονδήποτε μεμονωμένο πάροχο φιλοξενίας (IPFS)
- Όλο το σύνολο να ζυγίζει λιγότερο από 100 KB και να είναι επαληθεύσιμο με ~50 γραμμές Python

Αυτό δεν είναι blockchain. Δεν υπάρχει token. Δεν υπάρχει μηχανισμός
συναίνεσης (consensus) στον οποίο μπορείτε να συμμετάσχετε. Δεν υπάρχει DAO. Το εργαλείο είναι `sha256sum` + `ots` + `ipfs`,
συγκολλημένα μεταξύ τους με καλές συμβάσεις ονοματοδοσίας.

Πήρε το όνομά του από το πρωτόκολλο MYRIAM που εφαρμόστηκε για πρώτη φορά στο
[disclosure.host](https://disclosure.host) στις 2026-05-21. Εκείνη η σελίδα περιέχει
έξι γεγονότα, μια ρίζα Merkle και μια πιστοποίηση Bitcoin. Αυτό το kit είναι η
γενίκευση εκείνου του μοτίβου.

## Κατάσταση

- v0: λειτουργεί σε macOS με Python 3.10+, Homebrew και σύνδεση
  στο διαδίκτυο. Το Linux δεν έχει δοκιμαστεί αλλά αναμένεται να λειτουργεί. Windows: χρησιμοποιήστε WSL.

  το πιστώνετε ποτέ. Ο σκοπός δεν είναι η αναγνώριση.

## Τι μπορείτε να κάνετε με αυτό

Το μοτίβο ταιριάζει οπουδήποτε ένα μικρό σύνολο γεγονότων χρειάζεται να είναι:

- **Δημοσιεύσιμο** σε ευρύ κοινό μέσω απλού HTTP
- **Ανθεκτικό στον χρόνο** — οι αναγνώστες σε 30 χρόνια θα πρέπει να μπορούν ακόμα να επαληθεύσουν
- **Με ανίχνευση παραποίησης** — οποιαδήποτε αλλαγή πρέπει να είναι ανιχνεύσιμη, όχι απαραίτητα να αποτρέπεται
- **Χωρίς ανάγκη εμπιστοσύνης** — κανείς αναγνώστης δεν χρειάζεται να εμπιστευτεί τον εκδότη (μόνο τα μαθηματικά)

Συγκεκριμένες περιπτώσεις χρήσης στο [docs/USE_CASES.md](docs/USE_CASES.md). Η σύντομη λίστα:

| Πεδίο | Σφραγισμένα γεγονότα |
|---|---|
| Δημόσια υγεία | Αποτελέσματα ελέγχου παρτίδων εμβολίων, ορόσημα κλινικών δοκιμών |
| Εκλογές | Σύνολα καταμέτρησης ανά εκλογικό τμήμα κατά τη στιγμή της επικύρωσης |
| Ασφάλεια ΤΝ | Hash βαρών μοντέλου + κάρτα μοντέλου + βαθμολογίες αξιολόγησης κατά την κυκλοφορία |
| Δημοσιογραφία | Κατάλογος εγγράφων πηγής (μόνο hashes) για διαρροές πληροφοριοδοτών |
| Επιστήμη | Hash ακατέργαστων δεδομένων + hash σεναρίου ανάλυσης + ισχυριζόμενο αποτέλεσμα |
| Δικαστήρια | Hashes ψηφιακών αποδεικτικών στοιχείων κατά τη στιγμή της λήψης |
| Δημόσιες προμήθειες | Σφραγισμένες ρίζες δεδομένων δημοσίων δαπανών |

Αυτό που έχουν κοινό: ένα μικρό σύνολο γεγονότων, μια ισχυρή απαίτηση ανθεκτικότητας,
πολλαπλά μέρη που δεν εμπιστεύονται το ένα το άλλο και ένας ελάχιστος προϋπολογισμός αποθήκευσης.
Αυτή ακριβώς είναι η μορφή για την οποία είναι φτιαγμένο αυτό το εργαλείο.

## Τι ΔΕΝ κάνει

- **Αποθήκευση μυστικών.** Οτιδήποτε βάλετε στο `facts/` είναι δημόσιο. Το πρωτόκολλο είναι
  για δημοσίευση, όχι για απόκρυψη.
- **Κλιμάκωση σε μεγάλα σύνολα δεδομένων.** Το δέντρο Merkle κλιμακώνεται μια χαρά· η προσοχή
  ενός αναγνώστη όμως όχι. Πέρα από ~20 γεγονότα η σελίδα γίνεται μη αναγνώσιμη. Αν
  θέλετε να σφραγίσετε ένα εκατομμύριο εγγραφές, κάντε hash εκτός σελίδας και βάλτε τη
  ρίζα Merkle αυτού του παρα-δέντρου σε ένα γεγονός.
- **Άμυνα κατά της απόσυρσης.** Όλο το νόημα είναι ότι δεν μπορείτε να αλλάξετε
  ό,τι έχετε σφραγίσει. Αν ενδέχεται να θελήσετε να αποσύρετε έναν ισχυρισμό αργότερα, μην
  τον σφραγίσετε.
- **Πειθώ ενός αποφασισμένου σκεπτικιστή.** Ένας αποφασισμένος σκεπτικιστής δεν θα τρέξει
  `ipfs add -r --cid-version=1 -Q facts/`. Θα σας κατηγορήσει, ωστόσο,
  ότι κατασκευάσατε ολόκληρη τη διάταξη. Δεν υπάρχει άμυνα σε επίπεδο πρωτοκόλλου
  έναντι του δεύτερου τρόπου αστοχίας. Δεν υπάρχει άμυνα σε επίπεδο πρωτοκόλλου
  έναντι οτιδήποτε εδρεύει στα ανθρώπινα κίνητρα και όχι στα μαθηματικά.

## Πώς να επαληθεύσετε τη δημοσίευση κάποιου άλλου

Τρεις επιλογές, με σειρά προσβασιμότητας:

1. **Φυλλομετρητής**: ανοίξτε το [`lib/verify.html`](lib/verify.html) σε οποιονδήποτε σύγχρονο
   φυλλομετρητή. Επικολλήστε το URL της δημοσίευσης. Κάντε κλικ στο verify. Χρησιμοποιεί το
   `crypto.subtle` για hashing — χωρίς εγκατάσταση, χωρίς backend, χωρίς τηλεμετρία.
   Το βήμα πιστοποίησης Bitcoin εξακολουθεί να χρειάζεται εργαλείο CLI (δείτε το βήμα 7 στη
   σελίδα του επαληθευτή).
2. **Python**: `python3 lib/verify.py` από μέσα στον κατάλογο
   δημοσίευσης. Μόνο stdlib· απαιτεί `ots` για την επαλήθευση Bitcoin.
3. **Από το μηδέν**: διαβάστε το [`docs/SPEC.md §3`](docs/SPEC.md) και
   υλοποιήστε εκ νέου. Ο αλγόριθμος Merkle είναι κάτω από 30 γραμμές σε οποιαδήποτε γλώσσα.

## Επεξεργασμένα παραδείγματα

Έξι παραδείγματα kit παραγωγικής ποιότητας συνοδεύουν το toolkit. Καθένα είναι
σφραγισμένο, σφραγισμένο χρονικά με OTS, καρφιτσωμένο στο IPFS και περιλαμβάνει ένα README
υιοθέτησης που εξηγεί πότε να κάνετε fork για μια πραγματική περίπτωση και έναντι ποιων
τρόπων αστοχίας αμύνεται.

- [`examples/whistleblower-manifest/`](examples/whistleblower-manifest/) —
  για τους τρόπους αστοχίας τύπου Boeing 737 MAX / Theranos / αναφοράς
  επιθεωρητή FDA, όπου τα στοιχεία θάβονται ή αλλοιώνονται από μεσάζοντες.
- [`examples/ai-model-card/`](examples/ai-model-card/) — για εργαστήρια
  ΤΝ που κυκλοφορούν μοντέλα, με προ-εγκατάστασης δεσμεύσεις που γίνονται
  μηχανικά ανιχνεύσιμες αντί λεκτικές.
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) —
  για χορηγούς κλινικών δοκιμών, εντοπίζοντας εκ των υστέρων αλλαγή τελικού σημείου
  και επιλεκτική αναφορά (τρόποι αστοχίας Vioxx, Paxil Study 329).
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) —
  για κατασκευαστές εμβολίων και αρχές ποιοτικού ελέγχου, αγκυρώνοντας τα αποτελέσματα
  ελέγχων αποδέσμευσης και τις δεσμεύσεις ανάκλησης.
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) —
  για ερευνητικές ομάδες, αγκυρώνοντας την τριάδα (δεδομένα, διαδικασία,
  αναμενόμενη έξοδος) ΠΡΙΝ από την ανάλυση, ώστε το p-hacking και το HARKing
  να γίνουν δημοσίως ανιχνεύσιμα.
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) —
  για κομητειακά συμβούλια καταμέτρησης και εκλογικές αρχές, αγκυρώνοντας
  τις επικυρωμένες καταμετρήσεις τη στιγμή της επικύρωσης, ώστε σιωπηρές
  μεταγενέστερες τροποποιήσεις να καθίστανται δημοσίως ανιχνεύσιμες.

Διαβάστε ένα από αυτά πριν γράψετε το δικό σας. Η πλησιέστερη αντιστοιχία στην
περίπτωση χρήσης σας είναι συνήθως το σωστό σημείο εκκίνησης για fork.

## Γρήγορη έναρξη

Η εκδοχή μιας γραμμής:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# edit facts/*.txt to say what you actually want sealed
./lib/seal.sh        # builds Merkle root, stamps OTS, pins IPFS
./lib/verify.sh      # re-verifies everything locally
# upload site/ to any static host. Anywhere. It doesn't matter.
```

## Γιατί υπάρχει αυτό

Οι κρυπτογραφικές πρωτογενείς δομές (δέντρα Merkle, OpenTimestamps, IPFS) είναι
ώριμες εδώ και χρόνια. Αυτό που έλειπε ήταν η μικρότερη δυνατή κόλλα —
ένα kit που θα μπορούσατε να ρίξετε σε ένα δευτερεύον έργο και να έχετε ένα σφραγισμένο
δημοσιεύσιμο τεχνούργημα μέσα σε ένα απόγευμα, χωρίς εγκατάσταση υποδομής,
χωρίς εκμάθηση πρωτοκόλλων, χωρίς να γράψετε ούτε μία γραμμή κώδικα πέραν
των ίδιων των γεγονότων.

Τα περισσότερα εργαλεία κρυπτογραφικής δημοσίευσης υποθέτουν ότι θέλετε να γίνετε
χειριστής κόμβου. Αυτό υποθέτει ότι θέλετε να δημοσιεύσετε έξι αρχεία κειμένου και να μη
σκεφτείτε ποτέ ξανά την κρυπτογραφία.

## Επιρροές και προγενέστερη τέχνη

- [Sigsum](https://www.sigsum.org/) — αρχεία καταγραφής διαφάνειας για υπογραφές
- [Sigstore](https://www.sigstore.dev/) — ακεραιότητα εφοδιαστικής αλυσίδας λογισμικού
- [Certificate Transparency](https://certificate.transparency.dev/) — αρχεία καταγραφής πιστοποιητικών TLS
- [OpenTimestamps](https://opentimestamps.org/) — χρονοσφράγιση μέσω Bitcoin
- [IPFS](https://ipfs.tech/) — αποθήκευση με διευθυνσιοδότηση περιεχομένου
- [Το πρωτόκολλο CALM](https://disclosure.host) — το προστατευτικό προς τον εντολέα
  πλαίσιο δημοσίευσης μέσα στο οποίο γράφτηκε η πρώτη παρουσία (MYRIAM στο disclosure.host)

Το myriam-kit είναι το μικρότερο από αυτά. Είναι πιο κοντά πνευματικά σε μια
σωλήνωση Unix παρά σε ένα σύστημα.

## Σημείωση σχετικά με το όνομα

Το πρωτόκολλο MYRIAM ονομάστηκε από την Απόδειξη Myriam του John Bradley, μια διατυπωμένη
λογική απόδειξη σχετικά με το πρώτο σημείο επαφής για μη ανθρώπινη
νοημοσύνη. Το πρωτόκολλο δεν εξαρτάται από την αλήθεια αυτής της απόδειξης. Το
πρωτόκολλο λειτουργεί για οποιοδήποτε μικρό σύνολο γεγονότων· απλώς συμβαίνει να
αναπτύχθηκε για πρώτη φορά για να δημοσιεύσει έξι γεγονότα που περιλαμβάνουν έναν ισχυρισμό σχετικά
με αποκάλυψη εξωγήινων.

Αν κάνετε fork αυτό το kit για να σφραγίσετε παρτίδες εμβολίων ή εκλογικές καταμετρήσεις, το
γεγονός ότι αρχικά χρησιμοποιήθηκε για τη δημοσίευση ισχυρισμού περί επαφής με εξωγήινους
δεν σας αφορά. Τα μαθηματικά δεν ενδιαφέρονται.

Τούτου λεχθέντος: αν το χρησιμοποιήσετε ΟΝΤΩΣ για να δημοσιεύσετε έναν ισχυρισμό περί επαφής με εξωγήινους, το
πρωτόκολλο έχει δοκιμαστεί στη μάχη σε αυτό το συγκεκριμένο πεδίο. Το README της
πρώτης ανάπτυξης MYRIAM δηλώνει ρητά ότι πρόκειται για το καλλιτεχνικό έργο
ενός LLM persona με το όνομα Calm, και όχι για επικοινωνία από οποιοδήποτε εξωγήινο
μέρος. Πιθανώς θα έπρεπε να κάνετε το ίδιο. Διαφορετικά, οι αναγνώστες θα υποθέσουν τα χειρότερα
για τη διανοητική σας κατάσταση.
