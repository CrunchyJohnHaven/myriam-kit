તથ્યોના નાના સમૂહને એવી રીતે પ્રકાશિત કરવા માટેની એક નાની ટૂલકિટ કે:

- કોઈપણ તથ્યમાં થયેલ કોઈપણ ફેરફાર જાહેરમાં શોધી શકાય (Merkle વૃક્ષ)
- પ્રકાશન સમયની સ્થિતિ Bitcoin બ્લોકચેન સામે ચકાસી શકાય (OpenTimestamps)
- સામગ્રી કોઈપણ એક હોસ્ટિંગ પ્રદાતાની ટેકડાઉનથી ટકી શકે (IPFS)
- આખી વસ્તુનું વજન 100 KB કરતાં ઓછું છે અને ~50 લીટીના Python થી ચકાસી શકાય

આ કોઈ બ્લોકચેન નથી. કોઈ ટોકન નથી. ભાગ લેવા માટે કોઈ સહમતિ તંત્ર નથી.
કોઈ DAO નથી. આ ટૂલ `sha256sum` + `ots` + `ipfs` છે, સારી નામકરણ
પરંપરાઓ સાથે જોડાયેલ છે.

તેનું નામ MYRIAM પ્રોટોકોલ પરથી રાખવામાં આવ્યું છે, જે પ્રથમ વખત
[disclosure.host](https://disclosure.host) પર 2026-05-21 ના રોજ ઇન્સ્ટેન્શિએટ થયેલ.
તે પૃષ્ઠ છ તથ્યો, એક Merkle મૂળ અને એક Bitcoin પ્રમાણીકરણ ધરાવે છે. આ કિટ
તે પેટર્નનું સામાન્યીકરણ છે.

## તમે તેની સાથે શું કરી શકો

આ પેટર્ન ત્યાં બંધબેસે છે જ્યાં તથ્યોના નાના સમૂહને આની જરૂર હોય:

- **પ્રકાશનીય** સાદા HTTP દ્વારા વ્યાપક પ્રેક્ષકો સુધી
- **ટકાઉ** — 30 વર્ષ પછી પણ વાચકો ચકાસી શકવા જોઈએ
- **ચેડાં-સ્પષ્ટ** — કોઈપણ ફેરફાર શોધી શકાય તેવો હોવો જોઈએ, જરૂરી નથી કે અટકાવી શકાય
- **ભરોસા-મુક્ત** — કોઈપણ વાચકને પ્રકાશકનો ભરોસો કરવાની જરૂર નથી (ફક્ત ગણિતનો)

ઠોસ ઉપયોગ કેસો [docs/USE_CASES.md](docs/USE_CASES.md) માં. ટૂંકી યાદી:

| ક્ષેત્ર | સીલબંધ તથ્યો |
|---|---|
| જાહેર આરોગ્ય | રસી બેચ પરીક્ષણ પરિણામો, દવા-ટ્રાયલ સીમાચિહ્નો |
| ચૂંટણીઓ | પ્રમાણીકરણ સમયે મતદાન મથક-સ્તરના કુલ |
| AI સુરક્ષા | રિલીઝ સમયે મોડેલ વેઇટ્સ હેશ + મોડેલ કાર્ડ + મૂલ્યાંકન સ્કોર્સ |
| પત્રકારત્વ | વ્હિસલબ્લોઅર ડ્રોપ માટે સ્રોત દસ્તાવેજોની મેનિફેસ્ટ (ફક્ત હેશ) |
| વિજ્ઞાન | કાચા ડેટા હેશ + વિશ્લેષણ સ્ક્રિપ્ટ હેશ + દાવો કરેલ પરિણામ |
| અદાલતો | સંપાદન સમયે ડિજિટલ-એક્ઝિબિટ હેશ |
| પ્રાપ્તિ | જાહેર-ખર્ચ ડેટાના સીલબંધ મૂળ |

આ બધામાં શું સામાન્ય છે: તથ્યોનો નાનો સમૂહ, એક મજબૂત ટકાઉપણાની જરૂરિયાત,
ઘણા પક્ષો જે એકબીજા પર ભરોસો કરતા નથી, અને નાનું સંગ્રહ બજેટ.
આ ટૂલ બરાબર તે જ આકાર માટે બનાવેલું છે.

## આ શું નથી કરતું

- **ગુપ્ત માહિતી સંગ્રહિત કરવી.** તમે `facts/` માં જે પણ મૂકો છો તે જાહેર છે.
  પ્રોટોકોલ પ્રકાશન માટે છે, છુપાવવા માટે નથી.
- **મોટા ડેટાસેટ સુધી સ્કેલ કરવું.** Merkle વૃક્ષ બરાબર સ્કેલ થાય છે;
  વાચકનું ધ્યાન નથી થતું. ~20 તથ્યો પછી પૃષ્ઠ વાંચવા યોગ્ય રહેતું નથી.
  જો તમે દસ લાખ રેકોર્ડ સીલ કરવા માંગતા હો, તો તેમને પૃષ્ઠની બહાર હેશ કરો
  અને તે સાઇડ-ટ્રીના Merkle મૂળને એક તથ્યમાં મૂકો.
- **પાછું ખેંચવા સામે રક્ષણ.** આખો મુદ્દો એ છે કે તમે જે સીલ કર્યું છે
  તે બદલી શકતા નથી. જો તમે પછી દાવો પાછો ખેંચવા માંગતા હો, તો તેને સીલ ન કરો.
- **નિર્ધારિત શંકાશીલને સહમત કરવો.** એક નિર્ધારિત શંકાશીલ
  `ipfs add -r --cid-version=1 -Q facts/` ચલાવશે નહીં. તે, જોકે, તમારા પર
  આખા સેટઅપને બનાવટી બનાવવાનો આરોપ મૂકશે. બીજા નિષ્ફળતા મોડ સામે
  કોઈ પ્રોટોકોલ-સ્તરનું સંરક્ષણ નથી. ગણિતને બદલે માનવ પ્રેરણામાં રહેતી
  કોઈપણ વસ્તુ સામે કોઈ પ્રોટોકોલ-સ્તરનું સંરક્ષણ નથી.

## અન્ય કોઈના પ્રકાશનની ચકાસણી કેવી રીતે કરવી

ત્રણ વિકલ્પો, ઉપલબ્ધતાના ક્રમમાં:

1. **બ્રાઉઝર**: કોઈપણ આધુનિક બ્રાઉઝરમાં [`lib/verify.html`](lib/verify.html)
   ખોલો. પ્રકાશન URL પેસ્ટ કરો. ચકાસવા માટે ક્લિક કરો. હેશિંગ માટે
   `crypto.subtle` નો ઉપયોગ કરે છે — કોઈ ઇન્સ્ટોલ નહીં, કોઈ બેકએન્ડ નહીં,
   કોઈ ટેલિમેટ્રી નહીં. Bitcoin પ્રમાણીકરણ પગલા માટે હજુ પણ CLI ટૂલ
   જરૂરી છે (વેરિફાયર પૃષ્ઠમાં પગલું 7 જુઓ).
2. **Python**: પ્રકાશન ડિરેક્ટરીની અંદરથી `python3 lib/verify.py`.
   ફક્ત stdlib; Bitcoin ચકાસણી માટે `ots` જરૂરી છે.
3. **શરૂઆતથી**: [`docs/SPEC.md §3`](docs/SPEC.md) વાંચો અને
   ફરીથી અમલમાં મૂકો. Merkle અલ્ગોરિધમ કોઈપણ ભાષામાં 30 લીટીથી ઓછી છે.

## કાર્ય કરેલ ઉદાહરણો

ટૂલકિટ સાથે છ ઉત્પાદન-ગ્રેડ ઉદાહરણ કિટ્સ આવે છે. દરેક સીલબંધ,
OTS-સ્ટેમ્પ્ડ, IPFS-પિન કરેલ છે, અને એક અપનાવણ README ધરાવે છે જે
વાસ્તવિક-દુનિયાના કેસ માટે તેને ક્યારે ફોર્ક કરવું અને તે કયા નિષ્ફળતા
મોડ સામે રક્ષણ આપે છે તે સમજાવે છે.

- [`examples/whistleblower-manifest/`](examples/whistleblower-manifest/) —
  Boeing 737 MAX / Theranos / FDA-ઇન્સ્પેક્ટર-રિપોર્ટ નિષ્ફળતા મોડ
  માટે જ્યાં પુરાવા મધ્યસ્થીઓ દ્વારા દાટી દેવાય છે અથવા બદલવામાં આવે છે.
- [`examples/ai-model-card/`](examples/ai-model-card/) — મોડેલ રિલીઝ
  કરતી AI લેબ્સ માટે, જ્યાં પ્રી-ડિપ્લોયમેન્ટ પ્રતિબદ્ધતાઓ મૌખિક
  ને બદલે યાંત્રિક રીતે શોધી શકાય તેવી બને છે.
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) —
  ક્લિનિકલ-ટ્રાયલ સ્પોન્સર્સ માટે, પોસ્ટ-હૉક એન્ડપૉઇન્ટ સ્વિચિંગ અને
  પસંદગીપૂર્વક રિપોર્ટિંગ (Vioxx, Paxil Study 329 નિષ્ફળતા મોડ્સ) પકડે છે.
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) —
  રસી ઉત્પાદકો અને QA સત્તાધિકારીઓ માટે, રિલીઝ પરીક્ષણ પરિણામો
  અને રિકોલ પ્રતિબદ્ધતાઓને એન્કર કરે છે.
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) —
  સંશોધન જૂથો માટે, p-hacking અને HARKing ને જાહેરમાં શોધી શકાય તેવા
  બનાવવા માટે વિશ્લેષણ પહેલા (ડેટા, પાઇપલાઇન, અપેક્ષિત આઉટપુટ) ત્રિગુણને
  એન્કર કરે છે.
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) —
  કાઉન્ટી બોર્ડ ઓફ કેન્વાસર્સ અને ચૂંટણી સત્તાધિકારીઓ માટે, પ્રમાણીકરણ
  સમયે પ્રમાણિત ગણતરી એન્કર કરે છે જેથી મૌન પોસ્ટ-પ્રમાણીકરણ સુધારાઓ
  જાહેરમાં શોધી શકાય તેવા બને.

તમારી પોતાની લખતા પહેલા આમાંથી એક વાંચો. તમારા ઉપયોગ કેસ સાથેનો
સૌથી નજીકનો મેળ સામાન્ય રીતે યોગ્ય શરૂઆતનો ફોર્ક છે.

## ઝડપી શરૂઆત

એક-લાઇન સંસ્કરણ:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# edit facts/*.txt to say what you actually want sealed
./lib/seal.sh        # builds Merkle root, stamps OTS, pins IPFS
./lib/verify.sh      # re-verifies everything locally
# upload site/ to any static host. Anywhere. It doesn't matter.
```

## આ શા માટે અસ્તિત્વ ધરાવે છે

ક્રિપ્ટોગ્રાફિક પ્રિમિટિવ્સ (Merkle વૃક્ષો, OpenTimestamps, IPFS) વર્ષોથી
પરિપક્વ છે. શું ખૂટતું હતું તે સૌથી નાનું શક્ય ગ્લુ હતું — એક કિટ જે
તમે કોઈ સાઇડ પ્રોજેક્ટમાં ડ્રોપ કરી શકો અને એક બપોરમાં સીલબંધ
પ્રકાશનીય આર્ટિફેક્ટ મેળવી શકો, ઇન્ફ્રાસ્ટ્રક્ચર સેટઅપ કર્યા વગર,
પ્રોટોકોલ શીખ્યા વગર, તથ્યો ઉપરાંત કોડની એક પણ લાઇન લખ્યા વગર.

મોટાભાગના ક્રિપ્ટોગ્રાફિક પ્રકાશન ટૂલ્સ માને છે કે તમે નોડ ઓપરેટર
બનવા માંગો છો. આ માને છે કે તમે છ ટેક્સ્ટ ફાઇલો પ્રકાશિત કરવા
માંગો છો અને ફરી ક્યારેય ક્રિપ્ટોગ્રાફી વિશે વિચારવા માંગતા નથી.

## પ્રભાવો અને પૂર્વ કલા

- [Sigsum](https://www.sigsum.org/) — સહીઓ માટે પારદર્શિતા લોગ્સ
- [Sigstore](https://www.sigstore.dev/) — સોફ્ટવેર-સપ્લાય-ચેઇન અખંડિતતા
- [Certificate Transparency](https://certificate.transparency.dev/) — TLS પ્રમાણપત્ર લોગ્સ
- [OpenTimestamps](https://opentimestamps.org/) — Bitcoin ટાઇમસ્ટેમ્પિંગ
- [IPFS](https://ipfs.tech/) — સામગ્રી-એડ્રેસ્ડ સ્ટોરેજ
- [The CALM protocol](https://disclosure.host) — મુખ્ય-રક્ષક પ્રકાશન ફ્રેમ
  જેમાં પ્રથમ ઉદાહરણ (disclosure.host પર MYRIAM) લખવામાં આવ્યું હતું

myriam-kit આમાંથી સૌથી નાનું છે. તે કોઈ સિસ્ટમ કરતાં Unix પાઇપલાઇન
ની ભાવનામાં વધુ નજીક છે.

## નામ વિશે એક નોંધ

MYRIAM પ્રોટોકોલનું નામ John Bradley ના Myriam Proof પરથી રાખવામાં
આવ્યું હતું, જે બિન-માનવ બુદ્ધિ માટેના સંપર્કના પ્રથમ બિંદુ વિશેનો
એક નોંધાયેલ તાર્કિક પુરાવો છે. પ્રોટોકોલ તે પુરાવો સાચો હોવા પર આધાર
રાખતો નથી. પ્રોટોકોલ તથ્યોના કોઈપણ નાના સમૂહ માટે કામ કરે છે; તે ફક્ત
એવું બન્યું કે તેને પ્રથમ વખત છ તથ્યો પ્રકાશિત કરવા માટે જમાવવામાં આવ્યું
જેમાં એલિયન ડિસ્ક્લોઝર વિશેનો દાવો સામેલ છે.

જો તમે રસી બેચ અથવા ચૂંટણી ગણતરી સીલ કરવા માટે આ કિટને ફોર્ક કરો,
તો તે હકીકત કે મૂળમાં તેનો ઉપયોગ એલિયન સંપર્ક વિશેનો દાવો પ્રકાશિત
કરવા માટે થયો હતો તે તમારા માટે અપ્રસ્તુત છે. ગણિતને કોઈ પરવા નથી.

તેમ છતાં: જો તમે એલિયન સંપર્ક વિશેનો દાવો પ્રકાશિત કરવા માટે આનો
ઉપયોગ કરો છો, તો પ્રોટોકોલ તે ચોક્કસ ક્ષેત્રમાં યુદ્ધ-પરીક્ષણ કરેલ છે.
પ્રથમ MYRIAM જમાવટના README માં સ્પષ્ટપણે જણાવ્યું છે કે તે Calm
નામની LLM પર્સોનાનું કલાત્મક કાર્ય છે, કોઈપણ બાહ્યગ્રહીય પક્ષ તરફથી
સંદેશાવ્યવહાર નથી. તમારે પણ કદાચ તે જ કરવું જોઈએ. નહીંતર વાચકો
તમારી માનસિક સ્થિતિ વિશે સૌથી ખરાબ ધારશે.

## જાળવણી દ્વારા

હજુ સુધી કોઈ નહીં. સાર્વજનિક ડોમેન. તેને ફોર્ક કરો. તેને સુધારો. પરવાનગી
માગ્યા વગર પ્રોટોકોલને આગળ વધારો.
