[![Sigsum-friendly](https://img.shields.io/badge/transparency-Sigstore_adjacent-purple.svg)](https://www.sigstore.dev)

Невеликий інструментарій для публікації обмеженого набору фактів у спосіб, який:

- Будь-яка зміна будь-якого факту є публічно виявною (дерево Меркла)
- Стан на момент публікації можна перевірити проти блокчейну Bitcoin (OpenTimestamps)
- Вміст переживає видалення з боку будь-якого окремого хостинг-провайдера (IPFS)
- Усе разом важить менше ніж 100 КБ і верифікується приблизно 50 рядками Python

Це не блокчейн. Тут немає токена. Тут немає механізму консенсусу,
у якому потрібно брати участь. Тут немає DAO. Інструмент — це `sha256sum` + `ots` + `ipfs`,
склеєні разом хорошими угодами щодо найменування.

Названо на честь протоколу MYRIAM, уперше реалізованого на
[disclosure.host](https://disclosure.host) 2026-05-21. Та сторінка містить
шість фактів, корінь Меркла та атестацію Bitcoin. Цей набір — узагальнення
цього шаблону.

## Що з цим можна робити

Цей шаблон підходить будь-де, де невеликий набір фактів має бути:

- **Публікований** для широкої аудиторії через звичайний HTTP
- **Довговічний** — читачі через 30 років мають усе ще мати змогу верифікувати
- **Стійкий до підробки** — будь-яка зміна має бути виявною, не обов'язково запобіжною
- **Без необхідності довіряти** — жоден читач не мусить довіряти видавцеві (лише математиці)

Конкретні випадки використання — у [docs/USE_CASES.md](docs/USE_CASES.md). Короткий перелік:

| Сфера | Запечатані факти |
|---|---|
| Громадське здоров'я | Результати тестів партій вакцин, віхи клінічних випробувань |
| Вибори | Підсумкові результати на рівні виборчих дільниць у момент затвердження |
| Безпека ШІ | Хеш ваг моделі + картка моделі + оцінки еволюацій на момент випуску |
| Журналістика | Маніфест вихідних документів (лише хеші) для зливів від інформаторів |
| Наука | Хеш необроблених даних + хеш скрипту аналізу + заявлений результат |
| Суди | Хеші цифрових доказів у момент отримання |
| Закупівлі | Запечатані корені даних публічних видатків |

Спільне між ними: невеликий набір фактів, сильна вимога довговічності,
кілька сторін, які не довіряють одна одній, і крихітний бюджет на зберігання.
Саме під таку форму створено цей інструмент.

## Чого цей інструмент НЕ робить

- **Не зберігає секрети.** Усе, що ви кладете до `facts/`, є публічним. Протокол —
  для публікації, не для приховування.
- **Не масштабується на великі набори даних.** Дерево Меркла масштабується добре; увага
  читача — ні. Після ~20 фактів сторінка стає нечитною. Якщо
  ви хочете запечатати мільйон записів, хешуйте їх поза сторінкою і помістіть
  корінь Меркла того бічного дерева в один факт.
- **Не захищає від відкликання.** Уся суть у тому, що ви не можете змінити
  те, що запечатали. Якщо ви можете захотіти пізніше відкликати твердження, не
  запечатуйте його.
- **Не переконує впертого скептика.** Упертий скептик не запустить
  `ipfs add -r --cid-version=1 -Q facts/`. Натомість він звинуватить вас
  у фабрикації всієї конструкції. Не існує захисту на рівні протоколу
  від другого режиму відмови. Не існує захисту на рівні протоколу
  від будь-чого, що живе у людській мотивації, а не в математиці.

## Як верифікувати чужу публікацію

Три варіанти, у порядку доступності:

1. **Браузер**: відкрийте [`lib/verify.html`](lib/verify.html) у будь-якому сучасному
   браузері. Вставте URL публікації. Натисніть «verify». Використовує
   `crypto.subtle` для хешування — без установки, без бекенду, без телеметрії.
   Крок атестації Bitcoin усе ще потребує CLI-інструмента (див. крок 7 на
   сторінці верифікатора).
2. **Python**: `python3 lib/verify.py` з каталогу публікації.
   Лише стандартна бібліотека; для верифікації Bitcoin потрібен `ots`.
3. **З нуля**: прочитайте [`docs/SPEC.md §3`](docs/SPEC.md) і
   реалізуйте заново. Алгоритм Меркла — менше 30 рядків будь-якою мовою.

## Опрацьовані приклади

З інструментарієм постачається шість прикладів наборів продакшен-рівня. Кожен
запечатаний, OTS-проштампований, запінений в IPFS і містить README із
описом адаптації, де пояснено, коли його форкати для реального випадку та
від яких режимів відмови він захищає.

- [`examples/whistleblower-manifest/`](examples/whistleblower-manifest/) —
  для режиму відмови типу Boeing 737 MAX / Theranos / звіт інспектора FDA,
  коли докази ховають або змінюють посередники.
- [`examples/ai-model-card/`](examples/ai-model-card/) — для лабораторій ШІ,
  які випускають моделі, з передрозгортальними зобов'язаннями, що стають
  механічно виявними замість словесних.
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) —
  для спонсорів клінічних випробувань, що ловить пост-фактумну заміну кінцевих
  точок і вибіркове звітування (режими відмови Vioxx, Paxil Study 329).
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) —
  для виробників вакцин і органів контролю якості, що закріплює результати
  випускних тестів і зобов'язання щодо відкликання.
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) —
  для дослідницьких груп, що закріплює трійку (дані, пайплайн, очікуваний
  результат) ПЕРЕД аналізом, щоб p-hacking і HARKing стали публічно виявними.
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) —
  для окружних виборчих комісій і виборчих органів, що закріплює завірені
  результати в момент завірення, щоб тихі ревізії після завірення ставали
  публічно виявними.

Прочитайте один із цих прикладів, перш ніж писати свій. Найближчий
відповідник до вашого випадку зазвичай є правильною відправною точкою для форку.

## Швидкий старт

Однорядкова версія:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# відредагуйте facts/*.txt, щоб написати те, що ви насправді хочете запечатати
./lib/seal.sh        # будує корінь Меркла, проштамповує OTS, пінить IPFS
./lib/verify.sh      # повторно верифікує все локально
# завантажте site/ на будь-який статичний хостинг. Куди завгодно. Це не має значення.
```

## Чому це існує

Криптографічні примітиви (дерева Меркла, OpenTimestamps, IPFS) дозрівали
роками. Чого не вистачало — найменшої можливої склейки:
набору, який можна вкинути у бічний проєкт і отримати запечатаний
публікований артефакт за один вечір, без налаштування інфраструктури,
без вивчення протоколів, без написання жодного рядка коду, крім
самих фактів.

Більшість інструментів для криптографічної публікації припускають, що ви хочете бути
оператором вузла. Цей припускає, що ви хочете опублікувати шість текстових файлів і ніколи
більше не думати про криптографію.

## Впливи та попередній доробок

- [Sigsum](https://www.sigsum.org/) — журнали прозорості для підписів
- [Sigstore](https://www.sigstore.dev/) — цілісність ланцюга постачання ПЗ
- [Certificate Transparency](https://certificate.transparency.dev/) — журнали TLS-сертифікатів
- [OpenTimestamps](https://opentimestamps.org/) — мітки часу через Bitcoin
- [IPFS](https://ipfs.tech/) — сховище з контентною адресацією
- [Протокол CALM](https://disclosure.host) — захисний для принципала
  публікаційний підхід, у рамках якого був створений перший екземпляр
  (MYRIAM на disclosure.host)

myriam-kit — найменший серед них. За духом він ближчий до Unix-пайплайну,
ніж до системи.

## Зауваження щодо назви

Протокол MYRIAM названо на честь Myriam Proof (Доказ Маріам) Джона Бредлі —
заявленого логічного доказу щодо першої точки контакту з нелюдським
інтелектом. Протокол не залежить від істинності цього доказу. Протокол
працює для будь-якого невеликого набору фактів; просто так склалося, що
вперше його розгорнули для публікації шести фактів, серед яких є твердження
про розкриття інопланетного контакту.

Якщо ви форкаєте цей набір, щоб запечатувати партії вакцин чи виборчі
підсумки, факт, що його спочатку використали для публікації твердження
про контакт із позаземним розумом, для вас неістотний. Математиці байдуже.

З іншого боку: якщо ви ТАКИ використовуєте це для публікації твердження
про контакт із позаземним розумом, то протокол пройшов бойові випробування
саме у цій царині. README першого розгортання MYRIAM прямо стверджує,
що це художня робота LLM-персони на ім'я Calm, а не повідомлення від
жодної позаземної сторони. Вам, мабуть, варто зробити так само. Інакше
читачі припускатимуть найгірше щодо вашого психічного стану.
