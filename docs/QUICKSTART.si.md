## 2. කරුණු සංස්කරණය කරන්න

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

සෑම ගොනුවක්ම එක් කරුණකි. UTF-8. LF පේළි අවසානයන්. ගොනු නාම රටාව `NN_short_name.txt` වන අතර, NN යනු ශුන්‍ය-පුරවන ලද ය. ඒවා Merkle ගසට ශබ්ද කෝෂ අනුව වර්ග කරනු ලැබේ, එබැවින් 02 සැමවිටම 01 ට පසු පැමිණේ.

ඉඟිය: සෑම කරුණක්ම ලෝකය පිළිබඳ ඔන්ටොලොජිකල් ප්‍රකාශයක් ලෙස නොව *නම් කිරීමේ හෝ ප්‍රකාශන සිදුවීමක්* ලෙස ප්‍රකාශ කරන්න. "Acme Corp විසින් කණ්ඩායම X, Y පරීක්ෂණය සමත් වූ බව ප්‍රකාශ කර ඇත" යනු සත්‍යාපන කළ හැකි ය. "කණ්ඩායම X ආරක්ෂිතයි" යනු නොවේ.

## 3. IPFS daemon ආරම්භ කරන්න (යන්ත්‍රයකට වරක්)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. මුද්‍රා කරන්න

```bash
bash build/seal.sh
```

මෙය:
1. සෑම කරුණක්ම හැෂ් කරයි (SHA-256), Bitcoin-විලාසයේ Merkle මූලය ගණනය කරයි.
2. ස්වාධීන දින දර්ශන දෙකක් හරහා (පෙරනිමියෙන් alice + finney — `MYRIAM_CAL_A`/`MYRIAM_CAL_B` හරහා ඉක්මවා යන්න) OpenTimestamps සමඟ මූලය මුද්‍රාංකනය කරයි.
3. `site/` නාමාවලිය IPFS වෙත පින් කර CID මුද්‍රණය කරයි.

මුළු: උණුසුම් IPFS daemon එකක් සමඟ තත්පර ~10ක්.

## 5. `site/index.html` සංස්කරණය කරන්න

ඔබේ සැබෑ කරුණු, හැෂ්, සහ Merkle මූලය සමඟ අච්චු ස්ථානධාරක ප්‍රතිස්ථාපනය කරන්න. පෙන්වන ලද එක්-කරුණ හැෂ් අගයන් එක්-කරුණ ගොනු සමඟ ගැලපිය යුතු ය, නැතහොත් පිටුව තමන්ගේම ස්වයං-සත්‍යාපනයෙන් අසමත් වේ.

සෑම කරුණක් සඳහාම, .txt ගොනුවේ ශරීරය `<div class="fact">` බ්ලොක් එකකට පිටපත් කර එයට යටින් කෙලින්ම `<div class="hash">` තුළ SHA-256 එක දමන්න. Merkle මූලය ක්‍රිප්ටෝග්‍රැෆික්-මුද්‍රා කොටසට යයි.

(ඔව්, මෙය ස්වයං-ජනනය කළ හැකි ය. එය හිතාමතා නැත: අතින් කරන පියවර මිනිස් සමාලෝචන පියවරයි. ඔබ නොකියවූ දෙයක් ප්‍රකාශයට පත් කිරීමෙන් ප්‍රොටොකෝලය ආරක්ෂා නොකරයි.)

## 6. දේශීයව සත්‍යාපනය කරන්න

```bash
python3 build/verify.py
```

අපේක්ෂිත ප්‍රතිදානය:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

"Pending" සාමාන්‍ය ය — OTS දින දර්ශනය පැය කිහිපයක් ඇතුළත සහතිකය සම්පූර්ණ Bitcoin සහතිකයකට උත්ශ්‍රේණි කරනු ඇත.

## 7. යෙදවුම් කරන්න

මේවායින් කුමක් හෝ තෝරන්න:

- **Cloudflare Pages**: dashboard → Pages → Direct upload → `site/` ඇද දමන්න
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `site/` තුළ `git init`, repo වෙත push කරන්න, Pages සක්‍රීය කරන්න
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (static hosting on)
- **සාමාන්‍ය VPS එකක්**: `scp -r site/* user@host:/var/www/html/`

පියවර 4 හි IPFS CID ඔබට අතිරේක අන්තර්ගත-ලිපිනය කරන ලද URL එකක් ලබා දෙයි: `https://<CID>.ipfs.dweb.link/`. එය භාවිතා කරන්න.

## 8. OTS රිසිට්පත උත්ශ්‍රේණි කරන්න (පසුව)

මුද්‍රා කිරීමෙන් පැය කිහිපයක් ඇතුළත, ක්‍රියාත්මක කරන්න:

```bash
ots upgrade build/merkle_root.txt.ots
```

මෙය සෑම දින දර්ශනයක්ම පොරොත්තුවෙන් සිටින සහතිකය නිශ්චිත බ්ලොක් එකක් සඳහන් කරන සම්පූර්ණ Bitcoin සහතිකයකට පරිවර්තනය කරයි. උත්ශ්‍රේණි කිරීමෙන් පසු `.ots` ගොනුව ඔබේ සත්කාරක වෙත නැවත යෙදවුම් කරන්න.

## 9. එච්චරයි

ඔබ කරුණු කුඩා සමූහයක් ඕනෑම තෙවන පාර්ශ්වයකට පහත දේ කළ හැකි ආකාරයෙන් ප්‍රකාශයට පත් කර ඇත:

- ඔබේ සත්කාරකයෙන් සරල HTTP හරහා සියලුම කරුණු ලබා ගන්න
- කරුණ බයිට් වලින් Merkle මූලය නැවත ගණනය කරන්න
- ඔබේ ප්‍රකාශිත මූලයට සසඳන්න
- මූලය OTS-මුද්‍රාංකනය කරන ලද වේලාවේදී Bitcoin වෙත කැප කර ඇති බව සත්‍යාපනය කරන්න
- ප්‍රකාශිත CID හි IPFS හරහා එම කරුණු ස්වාධීනව ලබා ගන්න
- සියලුම හැෂ් ගැලපෙන බව තහවුරු කරන්න

එම පියවරයන්ගෙන් කිසිවක් අසාර්ථක වුවහොත්, මැදිහත් වීමක් සිදුව ඇත. පාඨකයාට ඔබව විශ්වාස කිරීමට අවශ්‍ය නැත. ඔවුන්ට යළිමම් ගණිතය කිරීමට ඇත්තේ ය.
