# త్వరిత ప్రారంభం

*తెలుగు అనువాదం — అసలు ఆంగ్ల వెర్షన్ కోసం, చూడండి [QUICKSTART.md](QUICKSTART.md)*

క్లోన్ నుండి సీలు చేయబడిన ప్రచురణ వరకు ఐదు నిమిషాలు. macOS-పరీక్షించబడింది; Linux పనిచేయాలి;
Windows వినియోగదారులకు WSL అవసరం అవుతుంది.

## 0. ముందస్తు అవసరాలు

```bash
brew install python3 opentimestamps-client ipfs
python3 -m pip install --user certifi  # for macOS SSL
```

## 1. స్కాఫోల్డ్

```bash
git clone <wherever-this-lives> myriam-kit
cd myriam-kit
bash lib/myriam-init.sh my-first-publication
cd my-first-publication
```

## 2. వాస్తవాలను సవరించండి

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

ప్రతి ఫైల్ ఒక వాస్తవం. UTF-8. LF లైన్ ఎండింగ్‌లు. ఫైల్‌నేమ్ నమూనా
`NN_short_name.txt`, ఇక్కడ NN అనేది zero-padded. అవి Merkle ట్రీలోకి
లెక్సికోగ్రాఫికల్‌గా క్రమబద్ధీకరించబడతాయి, కాబట్టి 02 ఎల్లప్పుడూ 01 తర్వాత వస్తుంది.

చిట్కా: ప్రతి వాస్తవాన్ని ప్రపంచం గురించి ఒక *ఆంటోలాజికల్ దావాగా* కాకుండా, ఒక
*నామకరణ లేదా ప్రకటన సంఘటనగా* పదబంధించండి. "Acme Corp బ్యాచ్ X పరీక్ష Y లో
ఉత్తీర్ణత పొందిందని ప్రకటించింది" అనేది సరిచూడదగినది. "బ్యాచ్ X సురక్షితమైనది" అనేది కాదు.

## 3. IPFS డేమన్‌ను ప్రారంభించండి (ఒక యంత్రానికి ఒక్కసారి)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. సీలు

```bash
bash build/seal.sh
```

ఇది ఈ క్రింది వాటిని చేస్తుంది:
1. ప్రతి వాస్తవాన్ని హ్యాష్ చేస్తుంది (SHA-256), Bitcoin-శైలి Merkle రూట్‌ను లెక్కిస్తుంది.
2. రెండు స్వతంత్ర క్యాలెండర్‌ల ద్వారా OpenTimestamps తో రూట్‌ను స్టాంప్ చేస్తుంది
   (డిఫాల్ట్‌గా alice + finney — `MYRIAM_CAL_A`/`MYRIAM_CAL_B` ద్వారా ఓవర్‌రైడ్ చేయండి).
3. `site/` డైరెక్టరీని IPFS కు పిన్ చేస్తుంది మరియు CID ను ముద్రిస్తుంది.

మొత్తం: వెచ్చని IPFS డేమన్‌తో ~10 సెకన్లు.

## 5. `site/index.html` ను సవరించండి

టెంప్లేట్ ప్లేస్‌హోల్డర్‌లను మీ వాస్తవ వాస్తవాలు, హ్యాష్‌లు, మరియు Merkle రూట్‌తో
భర్తీ చేయండి. ప్రదర్శించబడిన ప్రతి-వాస్తవ హ్యాష్‌లు తప్పనిసరిగా ప్రతి-వాస్తవ ఫైళ్లతో
సరిపోలాలి, లేకుంటే పేజీ తన స్వంత స్వీయ-ధృవీకరణలో విఫలమవుతుంది.

ప్రతి వాస్తవం కోసం, .txt ఫైల్ యొక్క బాడీని ఒక `<div class="fact">` బ్లాక్‌లోకి కాపీ
చేయండి మరియు దానికి సరిగ్గా క్రింద ఉన్న ఒక `<div class="hash">` లో SHA-256 ను
ఉంచండి. Merkle రూట్ క్రిప్టోగ్రాఫిక్-సీలు విభాగంలోకి వెళుతుంది.

(అవును, ఇది ఆటో-జనరేట్ చేయబడగలదు. ఇది ఉద్దేశపూర్వకంగా చేయలేదు: మాన్యువల్ దశ
అనేది మానవ సమీక్షా దశ. మీరు చదవని దాన్ని ప్రచురించడానికి వ్యతిరేకంగా ప్రోటోకాల్
రక్షణ ఇవ్వదు.)

## 6. స్థానికంగా సరిచూడండి

```bash
python3 build/verify.py
```

ఆశించిన అవుట్‌పుట్:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

"Pending" అనేది సాధారణం — OTS క్యాలెండర్ కొన్ని గంటల్లోపు అటెస్టేషన్‌ను పూర్తి
Bitcoin అటెస్టేషన్‌గా అప్‌గ్రేడ్ చేస్తుంది.

## 7. డిప్లాయ్

వీటిలో ఏదైనా ఎంచుకోండి:

- **Cloudflare Pages**: dashboard → Pages → Direct upload → `site/` ను డ్రాగ్ చేయండి
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `site/` లో `git init`, రిపోకు పుష్ చేయండి, Pages ను ఎనేబుల్ చేయండి
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (static hosting on)
- **ఏదైనా సాదా VPS**: `scp -r site/* user@host:/var/www/html/`

దశ 4 నుండి వచ్చిన IPFS CID మీకు ఒక అదనపు కంటెంట్-అడ్రస్డ్ URL ను ఇస్తుంది:
`https://<CID>.ipfs.dweb.link/`. దాన్ని ఉపయోగించండి.

## 8. OTS రసీదును అప్‌గ్రేడ్ చేయండి (తర్వాత)

సీలు చేసిన ~కొన్ని గంటల్లోపు, నడపండి:

```bash
ots upgrade build/merkle_root.txt.ots
```

ఇది ప్రతి క్యాలెండర్ యొక్క పెండింగ్ అటెస్టేషన్‌ను ఒక నిర్దిష్ట బ్లాక్‌ను సూచించే
పూర్తి Bitcoin అటెస్టేషన్‌గా మారుస్తుంది. అప్‌గ్రేడ్ చేసిన తర్వాత `.ots` ఫైల్‌ను
మీ హోస్ట్‌లకు తిరిగి డిప్లాయ్ చేయండి.

## 9. అంతే

మీరు ఒక చిన్న వాస్తవాల సమూహాన్ని ఒక మార్గంలో ప్రచురించారు, ఇక్కడ ఏదైనా మూడవ పక్షం:

- మీ హోస్ట్ నుండి సాదా HTTP ద్వారా అన్ని వాస్తవాలను తీసుకోగలదు
- వాస్తవ బైట్‌ల నుండి Merkle రూట్‌ను తిరిగి లెక్కించగలదు
- మీ ప్రచురించబడిన రూట్‌తో పోల్చగలదు
- OTS-స్టాంప్ చేయబడిన సమయంలో Bitcoin కు రూట్ నిబద్ధం అయిందని సరిచూడగలదు
- ప్రచురించబడిన CID వద్ద IPFS ద్వారా అదే వాస్తవాలను స్వతంత్రంగా తీసుకోగలదు
- అన్ని హ్యాష్‌లు సరిపోలుతున్నాయని నిర్ధారించగలదు

ఆ దశల్లో ఏదైనా విఫలమైతే, తారుమారు జరిగింది. పాఠకుడు మిమ్మల్ని నమ్మాల్సిన అవసరం
లేదు. వారు కేవలం గణితాన్ని చేయాలి.
