# விரைவான தொடக்கம் (Quickstart)

*தமிழ் மொழிபெயர்ப்பு — மூல ஆங்கில பதிப்புக்கு, [QUICKSTART.md](QUICKSTART.md) காண்க*

க்ளோன் செய்வதிலிருந்து முத்திரை வைக்கப்பட்ட வெளியீடு வரை ஐந்து நிமிடங்கள். macOS-இல் சோதிக்கப்பட்டது; Linux செயல்பட வேண்டும்; Windows பயனர்களுக்கு WSL தேவைப்படும்.

## 0. முன்நிபந்தனைகள்

```bash
brew install python3 opentimestamps-client ipfs
python3 -m pip install --user certifi  # for macOS SSL
```

## 1. சாரக்கட்டு (Scaffold)

```bash
git clone <wherever-this-lives> myriam-kit
cd myriam-kit
bash lib/myriam-init.sh my-first-publication
cd my-first-publication
```

## 2. உண்மைகளை திருத்துங்கள்

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

ஒவ்வொரு கோப்பும் ஒரு உண்மை. UTF-8. LF வரி முடிவுகள். கோப்புப் பெயர் வடிவம் `NN_short_name.txt`, இங்கு NN என்பது சுழியால் நிரப்பப்பட்டது. அவை Merkle மரத்திற்குள் அகராதி வரிசையில் (lexicographically) வரிசைப்படுத்தப்படுகின்றன, எனவே 02 எப்போதும் 01-க்குப் பிறகே வரும்.

குறிப்பு: ஒவ்வொரு உண்மையையும் உலகத்தைப் பற்றிய ஒரு ஆன்டாலஜிக்கல் (ontological) கூற்றாக அல்ல, ஒரு *பெயரிடல் அல்லது அறிவிப்பு நிகழ்வாக* (naming or declaration event) கூறுங்கள். "Acme Corp ஆனது X தொகுதி Y சோதனையில் தேர்ச்சி பெற்றது என்று அறிவித்துள்ளது" சரிபார்க்கக்கூடியது. "X தொகுதி பாதுகாப்பானது" சரிபார்க்கக்கூடியதல்ல.

## 3. IPFS டீமனை தொடங்குங்கள் (ஒரு இயந்திரத்திற்கு ஒருமுறை)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. முத்திரை (Seal)

```bash
bash build/seal.sh
```

இது செய்யும்:
1. ஒவ்வொரு உண்மையையும் ஹாஷ் செய்யும் (SHA-256), Bitcoin-பாணி Merkle ரூட்டை கணக்கிடும்.
2. இரண்டு சுயாதீன காலெண்டர்கள் வழியாக OpenTimestamps மூலம் ரூட்டை முத்திரையிடும்
   (இயல்பாக alice + finney — `MYRIAM_CAL_A`/`MYRIAM_CAL_B` வழியாக மாற்றியமைக்கலாம்).
3. `site/` கோப்பகத்தை IPFS-இல் பின் செய்து CID-ஐ அச்சிடும்.

மொத்தம்: சூடான IPFS டீமனுடன் ~10 வினாடிகள்.

## 5. `site/index.html` ஐ திருத்துங்கள்

வார்ப்புரு (template) placeholders-ஐ உங்கள் உண்மையான உண்மைகள், ஹாஷ்கள், மற்றும் Merkle ரூட்டுடன் மாற்றவும். காட்டப்படும் ஒவ்வொரு-உண்மை ஹாஷ்களும் ஒவ்வொரு-உண்மை கோப்புகளுடன் பொருந்த வேண்டும், இல்லையெனில் பக்கம் தனது சொந்த சுய-சரிபார்ப்பில் தோல்வியடையும்.

ஒவ்வொரு உண்மைக்கும், .txt கோப்பின் உள்ளடக்கத்தை `<div class="fact">` பகுதிக்குள் நகலெடுத்து, SHA-256-ஐ அதற்கு கீழே நேரடியாக `<div class="hash">`-இல் வைக்கவும். Merkle ரூட் கிரிப்டோகிராஃபிக்-முத்திரை பகுதியில் செல்கிறது.

(ஆம், இதை தானாக உருவாக்க முடியும். வேண்டுமென்றே செய்யவில்லை: கைமுறை படியே மனித மறுபார்வை படி. நீங்கள் வாசிக்காத ஒன்றை வெளியிடுவதிலிருந்து நெறிமுறை பாதுகாக்காது.)

## 6. உள்ளூரில் சரிபார்க்கவும்

```bash
python3 build/verify.py
```

எதிர்பார்க்கப்படும் வெளியீடு:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

"Pending" என்பது இயல்பானது — OTS காலெண்டர் சில மணி நேரங்களுக்குள் சான்றளிப்பை முழு Bitcoin சான்றளிப்பாக மேம்படுத்தும்.

## 7. பயன்படுத்தவும் (Deploy)

இவற்றில் எதையும் தேர்வு செய்யவும்:

- **Cloudflare Pages**: டாஷ்போர்டு → Pages → Direct upload → `site/`-ஐ இழுக்கவும்
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `site/`-இல் `git init`, ரெப்போவுக்கு push செய்யவும், Pages-ஐ இயக்கவும்
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (நிலையான ஹோஸ்டிங் இயக்கப்பட்டது)
- **எந்த எளிய VPS-ம்**: `scp -r site/* user@host:/var/www/html/`

படி 4-இல் இருந்து வரும் IPFS CID உங்களுக்கு கூடுதலான உள்ளடக்கம்-முகவரியிடப்பட்ட URL-ஐ வழங்குகிறது: `https://<CID>.ipfs.dweb.link/`. அதை பயன்படுத்துங்கள்.

## 8. OTS ரசீதை மேம்படுத்துங்கள் (பின்னர்)

முத்திரை வைத்த ~சில மணி நேரங்களுக்குள், இயக்கவும்:

```bash
ots upgrade build/merkle_root.txt.ots
```

இது ஒவ்வொரு காலெண்டரின் நிலுவையில் உள்ள சான்றளிப்பையும் ஒரு குறிப்பிட்ட பிளாக்கைக் குறிப்பிடும் முழு Bitcoin சான்றளிப்பாக மாற்றுகிறது. மேம்படுத்திய பிறகு உங்கள் ஹோஸ்ட்களுக்கு `.ots` கோப்பை மீண்டும் பயன்படுத்தவும்.

## 9. அவ்வளவுதான்

நீங்கள் ஒரு சிறிய தொகுப்பு உண்மைகளை வெளியிட்டுள்ளீர்கள், எந்தவொரு மூன்றாம் தரப்பினரும் இவற்றை செய்ய முடியும்:

- உங்கள் ஹோஸ்டிலிருந்து சாதாரண HTTP வழியாக அனைத்து உண்மைகளையும் பெறவும்
- உண்மை பைட்டுகளிலிருந்து Merkle ரூட்டை மீண்டும் கணக்கிடவும்
- உங்கள் வெளியிடப்பட்ட ரூட்டுடன் ஒப்பிடவும்
- OTS-முத்திரை வைக்கப்பட்ட நேரத்தில் Bitcoin-க்கு ரூட் உறுதிமொழியளிக்கப்பட்டது என்பதை சரிபார்க்கவும்
- வெளியிடப்பட்ட CID-இல் IPFS வழியாக அதே உண்மைகளை சுயாதீனமாக பெறவும்
- அனைத்து ஹாஷ்களும் பொருந்துகின்றன என்பதை உறுதிப்படுத்தவும்

அவற்றில் ஏதேனும் ஒரு படி தோல்வியடைந்தால், சேதம் ஏற்பட்டுள்ளது. வாசகர் உங்களை நம்ப வேண்டியதில்லை. அவர்கள் வெறும் கணிதத்தை செய்ய வேண்டும்.
