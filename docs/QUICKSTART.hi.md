# त्वरित शुरुआत

*हिंदी अनुवाद — मूल अंग्रेजी संस्करण के लिए [QUICKSTART.md](QUICKSTART.md) देखें*

क्लोन से सीलबंद प्रकाशन तक पाँच मिनट। macOS पर परीक्षित; Linux काम करना चाहिए; Windows उपयोगकर्ताओं को WSL की आवश्यकता होगी।

## 0. पूर्वापेक्षाएँ

```bash
brew install python3 opentimestamps-client ipfs
python3 -m pip install --user certifi  # for macOS SSL
```

## 1. स्कैफ़ोल्ड

```bash
git clone <wherever-this-lives> myriam-kit
cd myriam-kit
bash lib/myriam-init.sh my-first-publication
cd my-first-publication
```

## 2. तथ्यों का संपादन

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

प्रत्येक फ़ाइल एक तथ्य है। UTF-8। LF पंक्ति-अंत। फ़ाइलनाम पैटर्न `NN_short_name.txt` है जहाँ NN शून्य-पैड किया हुआ है। उन्हें शब्दकोशीय (lexicographic) क्रम में Merkle ट्री में सॉर्ट किया जाता है, इसलिए 02 हमेशा 01 के बाद आता है।

युक्ति: प्रत्येक तथ्य को विश्व के बारे में किसी सत्तामीमांसीय (ontological) दावे के रूप में नहीं, बल्कि *नामकरण या घोषणा घटना* के रूप में अभिव्यक्त करें। "Acme Corp ने घोषित किया है कि बैच X ने परीक्षण Y उत्तीर्ण किया" सत्यापन योग्य है। "बैच X सुरक्षित है" नहीं है।

## 3. IPFS डेमन प्रारंभ करें (प्रति मशीन एक बार)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. सील

```bash
bash build/seal.sh
```

यह करेगा:
1. प्रत्येक तथ्य को हैश करना (SHA-256), Bitcoin-शैली Merkle रूट की गणना करना।
2. दो स्वतंत्र कैलेंडरों के माध्यम से रूट पर OpenTimestamps के साथ मुहर लगाना
   (डिफ़ॉल्ट रूप से alice + finney — `MYRIAM_CAL_A`/`MYRIAM_CAL_B` के माध्यम से ओवरराइड)।
3. `site/` निर्देशिका को IPFS पर पिन करना और CID प्रिंट करना।

कुल: गर्म IPFS डेमन के साथ ~10 सेकंड।

## 5. `site/index.html` का संपादन

टेम्पलेट प्लेसहोल्डर को अपने वास्तविक तथ्यों, हैश और Merkle रूट से बदलें। प्रदर्शित प्रति-तथ्य हैश प्रति-तथ्य फ़ाइलों से मेल खाने चाहिए, अन्यथा पृष्ठ अपने स्वयं के स्व-सत्यापन में विफल हो जाएगा।

प्रत्येक तथ्य के लिए, .txt फ़ाइल की मुख्य सामग्री को `<div class="fact">` ब्लॉक में कॉपी करें और उसके ठीक नीचे `<div class="hash">` में SHA-256 डालें। Merkle रूट क्रिप्टोग्राफिक-सील अनुभाग में जाता है।

(हाँ, यह स्वचालित रूप से उत्पन्न किया जा सकता है। यह जानबूझकर ऐसा नहीं है: मैन्युअल चरण ही मानवीय समीक्षा का चरण है। प्रोटोकॉल आपको ऐसी कोई चीज़ प्रकाशित करने से नहीं बचाता जिसे आपने नहीं पढ़ा है।)

## 6. स्थानीय रूप से सत्यापन

```bash
python3 build/verify.py
```

अपेक्षित आउटपुट:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

"Pending" सामान्य है — OTS कैलेंडर कुछ ही घंटों के भीतर सत्यापन को पूर्ण Bitcoin सत्यापन में उन्नत कर देगा।

## 7. परिनियोजन

इनमें से कोई भी चुनें:

- **Cloudflare Pages**: डैशबोर्ड → Pages → Direct upload → `site/` को खींचें
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `site/` में `git init`, रेपो पर पुश करें, Pages सक्षम करें
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (स्टैटिक होस्टिंग चालू)
- **कोई भी सादा VPS**: `scp -r site/* user@host:/var/www/html/`

चरण 4 का IPFS CID आपको एक अतिरिक्त सामग्री-संबोधित URL देता है: `https://<CID>.ipfs.dweb.link/`। उसका उपयोग करें।

## 8. OTS रसीद को अपग्रेड करें (बाद में)

सीलिंग के ~कुछ घंटों के भीतर, चलाएँ:

```bash
ots upgrade build/merkle_root.txt.ots
```

यह प्रत्येक कैलेंडर के लंबित सत्यापन को एक पूर्ण Bitcoin सत्यापन में परिवर्तित करता है जो एक विशिष्ट ब्लॉक का संदर्भ देता है। अपग्रेड के बाद `.ots` फ़ाइल को अपने होस्ट पर पुनः परिनियोजित करें।

## 9. बस इतना ही

आपने तथ्यों का एक छोटा समूह इस तरह प्रकाशित किया है कि कोई भी तीसरा पक्ष यह कर सकता है:

- आपके होस्ट से सादे HTTP के माध्यम से सभी तथ्य प्राप्त करना
- तथ्य बाइट्स से Merkle रूट की पुनर्गणना करना
- आपके प्रकाशित रूट से तुलना करना
- सत्यापित करना कि रूट को OTS-स्टांप वाले समय पर Bitcoin पर प्रतिबद्ध किया गया था
- प्रकाशित CID पर IPFS के माध्यम से समान तथ्यों को स्वतंत्र रूप से प्राप्त करना
- पुष्टि करना कि सभी हैश मेल खाते हैं

यदि उनमें से कोई भी चरण विफल होता है, तो छेड़छाड़ हुई है। पाठक को आप पर भरोसा नहीं करना है। उन्हें बस गणित करना है।
