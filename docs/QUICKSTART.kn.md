# Quickstart

*ಕನ್ನಡ ಅನುವಾದ — ಮೂಲ ಇಂಗ್ಲಿಷ್ ಆವೃತ್ತಿಗಾಗಿ, ನೋಡಿ [QUICKSTART.md](QUICKSTART.md)*

ಕ್ಲೋನ್‌ನಿಂದ ಮುದ್ರೆಯೊತ್ತಿದ ಪ್ರಕಟಣೆಯವರೆಗೆ ಐದು ನಿಮಿಷಗಳು. macOS-ಪರೀಕ್ಷಿತ; Linux
ಕೆಲಸ ಮಾಡಬೇಕು; Windows ಬಳಕೆದಾರರಿಗೆ WSL ಬೇಕಾಗುತ್ತದೆ.

## 0. ಪೂರ್ವಾಗತ್ಯತೆಗಳು

```bash
brew install python3 opentimestamps-client ipfs
python3 -m pip install --user certifi  # for macOS SSL
```

## 1. ಚೌಕಟ್ಟು ರಚಿಸಿ

```bash
git clone <wherever-this-lives> myriam-kit
cd myriam-kit
bash lib/myriam-init.sh my-first-publication
cd my-first-publication
```

## 2. ಸತ್ಯಾಂಶಗಳನ್ನು ಸಂಪಾದಿಸಿ

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

ಪ್ರತಿಯೊಂದು ಫೈಲ್ ಒಂದು ಸತ್ಯಾಂಶ. UTF-8. LF ಸಾಲಿನ ಕೊನೆಗಳು. ಕಡತದ-ಹೆಸರಿನ ಮಾದರಿಯು
`NN_short_name.txt` ಆಗಿದೆ, ಇಲ್ಲಿ NN ಶೂನ್ಯ-ಪ್ಯಾಡ್ ಆಗಿರುತ್ತದೆ. ಅವುಗಳನ್ನು ಶಬ್ದಕೋಶೀಯವಾಗಿ
Merkle ವೃಕ್ಷದಲ್ಲಿ ವಿಂಗಡಿಸಲಾಗುತ್ತದೆ, ಆದ್ದರಿಂದ 02 ಯಾವಾಗಲೂ 01 ರ ನಂತರ ಬರುತ್ತದೆ.

ಸಲಹೆ: ಪ್ರತಿ ಸತ್ಯಾಂಶವನ್ನು ಜಗತ್ತಿನ ಕುರಿತಾದ ಆಂಟೋಲಾಜಿಕಲ್ ಹೇಳಿಕೆಯಾಗಿ ಅಲ್ಲ, ಒಂದು
*ಹೆಸರಿಸುವಿಕೆ ಅಥವಾ ಘೋಷಣಾ ಘಟನೆ*ಯಾಗಿ ರೂಪಿಸಿ. "Acme Corp ಬ್ಯಾಚ್ X ಪರೀಕ್ಷೆ Y
ಯಲ್ಲಿ ಉತ್ತೀರ್ಣವಾಗಿದೆ ಎಂದು ಘೋಷಿಸಿದೆ" ಎಂಬುದು ಪರಿಶೀಲಿಸಬಹುದಾದದ್ದು. "ಬ್ಯಾಚ್ X ಸುರಕ್ಷಿತ"
ಎಂಬುದು ಅಲ್ಲ.

## 3. IPFS ಡೀಮನ್ ಪ್ರಾರಂಭಿಸಿ (ಪ್ರತಿ ಯಂತ್ರಕ್ಕೆ ಒಂದು-ಬಾರಿ)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. ಮುದ್ರೆಯೊತ್ತಿ

```bash
bash build/seal.sh
```

ಇದು ಮಾಡುತ್ತದೆ:
1. ಪ್ರತಿ ಸತ್ಯಾಂಶವನ್ನು ಹ್ಯಾಶ್ ಮಾಡಿ (SHA-256), Bitcoin-ಶೈಲಿಯ Merkle ಮೂಲವನ್ನು ಗಣಿಸಿ.
2. ಮೂಲವನ್ನು ಎರಡು ಸ್ವತಂತ್ರ ಕ್ಯಾಲೆಂಡರ್‌ಗಳ ಮೂಲಕ OpenTimestamps ನೊಂದಿಗೆ ಸ್ಟ್ಯಾಂಪ್ ಮಾಡಿ
   (ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ alice + finney — `MYRIAM_CAL_A`/`MYRIAM_CAL_B` ಮೂಲಕ ಪುನರಾವರ್ತಿಸಿ).
3. `site/` ಡೈರೆಕ್ಟರಿಯನ್ನು IPFS ಗೆ ಪಿನ್ ಮಾಡಿ ಮತ್ತು CID ಮುದ್ರಿಸಿ.

ಒಟ್ಟು: ಬೆಚ್ಚಗಿನ IPFS ಡೀಮನ್‌ನೊಂದಿಗೆ ~10 ಸೆಕೆಂಡ್.

## 5. `site/index.html` ಸಂಪಾದಿಸಿ

ಟೆಂಪ್ಲೇಟ್ ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್‌ಗಳನ್ನು ನಿಮ್ಮ ನಿಜವಾದ ಸತ್ಯಾಂಶಗಳು, ಹ್ಯಾಶ್‌ಗಳು ಮತ್ತು Merkle
ಮೂಲದೊಂದಿಗೆ ಬದಲಾಯಿಸಿ. ಪ್ರದರ್ಶಿತ ಪ್ರತಿ-ಸತ್ಯಾಂಶ ಹ್ಯಾಶ್‌ಗಳು ಪ್ರತಿ-ಸತ್ಯಾಂಶ ಫೈಲ್‌ಗಳಿಗೆ
ಹೊಂದಾಣಿಕೆಯಾಗಬೇಕು, ಇಲ್ಲದಿದ್ದರೆ ಪುಟವು ತನ್ನದೇ ಸ್ವಯಂ-ಪರಿಶೀಲನೆಯಲ್ಲಿ ವಿಫಲವಾಗುತ್ತದೆ.

ಪ್ರತಿ ಸತ್ಯಾಂಶಕ್ಕೂ, .txt ಫೈಲ್‌ನ ದೇಹವನ್ನು ಒಂದು `<div class="fact">` ಬ್ಲಾಕ್‌ಗೆ ನಕಲಿಸಿ
ಮತ್ತು SHA-256 ಅನ್ನು ನೇರವಾಗಿ ಅದರ ಕೆಳಗೆ ಒಂದು `<div class="hash">` ನಲ್ಲಿ ಹಾಕಿ.
Merkle ಮೂಲವು ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್-ಸೀಲ್ ವಿಭಾಗಕ್ಕೆ ಹೋಗುತ್ತದೆ.

(ಹೌದು, ಇದನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ-ಉತ್ಪಾದಿಸಬಹುದಿತ್ತು. ಅದನ್ನು ಉದ್ದೇಶಪೂರ್ವಕವಾಗಿ ಮಾಡಲಾಗಿಲ್ಲ:
ಕೈಯಾರೆ ಮಾಡುವ ಹಂತವೇ ಮಾನವ ಪರಿಶೀಲನಾ ಹಂತ. ನೀವು ಓದದ್ದನ್ನು ಪ್ರಕಟಿಸುವುದರ ವಿರುದ್ಧ
ಪ್ರೋಟೋಕಾಲ್ ರಕ್ಷಿಸುವುದಿಲ್ಲ.)

## 6. ಸ್ಥಳೀಯವಾಗಿ ಪರಿಶೀಲಿಸಿ

```bash
python3 build/verify.py
```

ನಿರೀಕ್ಷಿತ ಔಟ್‌ಪುಟ್:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

"Pending" ಸಾಮಾನ್ಯ — OTS ಕ್ಯಾಲೆಂಡರ್ ಕೆಲವು ಗಂಟೆಗಳ ಒಳಗೆ ದೃಢೀಕರಣವನ್ನು ಪೂರ್ಣ Bitcoin
ದೃಢೀಕರಣಕ್ಕೆ ಉನ್ನತೀಕರಿಸುತ್ತದೆ.

## 7. ನಿಯೋಜಿಸಿ

ಯಾವುದಾದರೂ ಒಂದನ್ನು ಆರಿಸಿ:

- **Cloudflare Pages**: dashboard → Pages → Direct upload → drag `site/`
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `git init` in `site/`, push to repo, enable Pages
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (static hosting on)
- **Any plain VPS**: `scp -r site/* user@host:/var/www/html/`

ಹಂತ 4 ರ IPFS CID ನಿಮಗೆ ಹೆಚ್ಚುವರಿ ವಿಷಯ-ವಿಳಾಸಿತ URL ಒದಗಿಸುತ್ತದೆ:
`https://<CID>.ipfs.dweb.link/`. ಅದನ್ನು ಬಳಸಿ.

## 8. OTS ರಸೀದಿಯನ್ನು ಉನ್ನತೀಕರಿಸಿ (ನಂತರ)

ಮುದ್ರೆಯೊತ್ತಿದ ~ಕೆಲವು ಗಂಟೆಗಳ ಒಳಗೆ, ಚಲಾಯಿಸಿ:

```bash
ots upgrade build/merkle_root.txt.ots
```

ಇದು ಪ್ರತಿ ಕ್ಯಾಲೆಂಡರ್‌ನ ಬಾಕಿ-ಇರುವ ದೃಢೀಕರಣವನ್ನು ಒಂದು ನಿರ್ದಿಷ್ಟ ಬ್ಲಾಕ್ ಅನ್ನು
ಉಲ್ಲೇಖಿಸುವ ಪೂರ್ಣ Bitcoin ದೃಢೀಕರಣಕ್ಕೆ ಪರಿವರ್ತಿಸುತ್ತದೆ. ಉನ್ನತೀಕರಣದ ನಂತರ `.ots`
ಫೈಲ್ ಅನ್ನು ನಿಮ್ಮ ಆತಿಥೇಯಗಳಿಗೆ ಮರು-ನಿಯೋಜಿಸಿ.

## 9. ಅಷ್ಟೇ

ನೀವು ಒಂದು ಚಿಕ್ಕ ಗುಂಪಿನ ಸತ್ಯಾಂಶಗಳನ್ನು ಯಾವುದೇ ಮೂರನೆಯ ಪಕ್ಷವು ಈ ಕೆಳಗಿನವುಗಳನ್ನು
ಮಾಡಬಹುದಾದ ರೀತಿಯಲ್ಲಿ ಪ್ರಕಟಿಸಿದ್ದೀರಿ:

- ನಿಮ್ಮ ಆತಿಥೇಯದಿಂದ ಸಾಮಾನ್ಯ HTTP ಮೂಲಕ ಎಲ್ಲಾ ಸತ್ಯಾಂಶಗಳನ್ನು ಪಡೆಯಬಹುದು
- ಸತ್ಯಾಂಶದ ಬೈಟ್‌ಗಳಿಂದ Merkle ಮೂಲವನ್ನು ಮರು-ಗಣಿಸಬಹುದು
- ನಿಮ್ಮ ಪ್ರಕಟಿತ ಮೂಲಕ್ಕೆ ಹೋಲಿಸಬಹುದು
- OTS-ಸ್ಟ್ಯಾಂಪ್ ಮಾಡಿದ ಸಮಯದಲ್ಲಿ ಮೂಲವು Bitcoin ಗೆ ಬದ್ಧಗೊಳಿಸಲ್ಪಟ್ಟಿತು ಎಂಬುದನ್ನು
  ಪರಿಶೀಲಿಸಬಹುದು
- ಪ್ರಕಟಿತ CID ನಲ್ಲಿ IPFS ಮೂಲಕ ಸ್ವತಂತ್ರವಾಗಿ ಅದೇ ಸತ್ಯಾಂಶಗಳನ್ನು ಪಡೆಯಬಹುದು
- ಎಲ್ಲಾ ಹ್ಯಾಶ್‌ಗಳು ಹೊಂದಾಣಿಕೆಯಾಗುತ್ತವೆ ಎಂದು ದೃಢೀಕರಿಸಬಹುದು

ಈ ಯಾವುದೇ ಹಂತಗಳು ವಿಫಲವಾದರೆ, ಛೇಡಿಸುವಿಕೆ ಸಂಭವಿಸಿದೆ. ಓದುಗನು ನಿಮ್ಮನ್ನು ನಂಬುವ
ಅಗತ್ಯವಿಲ್ಲ. ಅವರು ಕೇವಲ ಗಣಿತವನ್ನು ಮಾಡಬೇಕು.
