# ઝડપી શરૂઆત

*ગુજરાતી અનુવાદ — મૂળ અંગ્રેજી સંસ્કરણ માટે, જુઓ [QUICKSTART.md](QUICKSTART.md)*

ક્લોનથી સીલબંધ પ્રકાશન સુધી પાંચ મિનિટ. macOS પર પરીક્ષણ કરેલ; Linux પર
કામ કરવું જોઈએ; Windows વપરાશકર્તાઓને WSL ની જરૂર પડશે.

## 0. પૂર્વ-જરૂરિયાતો

```bash
brew install python3 opentimestamps-client ipfs
python3 -m pip install --user certifi  # for macOS SSL
```

## 1. સ્કેફોલ્ડ

```bash
git clone <wherever-this-lives> myriam-kit
cd myriam-kit
bash lib/myriam-init.sh my-first-publication
cd my-first-publication
```

## 2. તથ્યો સંપાદિત કરો

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

દરેક ફાઇલ એક તથ્ય છે. UTF-8. LF લાઇન અંત. ફાઇલનામ પેટર્ન
`NN_short_name.txt` છે જ્યાં NN શૂન્ય-પૅડેડ છે. તેઓ Merkle વૃક્ષમાં
શબ્દકોશીય ક્રમે સૉર્ટ થાય છે, તેથી 02 હંમેશા 01 પછી આવે છે.

ટિપ: દરેક તથ્યને *નામકરણ અથવા ઘોષણા ઘટના* તરીકે વ્યક્ત કરો, વિશ્વ વિશેના
ઑન્ટોલોજિકલ દાવા તરીકે નહીં. "Acme Corp એ ઘોષણા કરી છે કે બેચ X પરીક્ષણ
Y પાસ થયું" ચકાસી શકાય તેવું છે. "બેચ X સુરક્ષિત છે" નથી.

## 3. IPFS ડેમન શરૂ કરો (દરેક મશીન દીઠ એક વખત)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. સીલ કરો

```bash
bash build/seal.sh
```

આ કરશે:
1. દરેક તથ્યને હેશ કરો (SHA-256), Bitcoin-શૈલીના Merkle મૂળની ગણતરી કરો.
2. બે સ્વતંત્ર કૅલેન્ડર દ્વારા OpenTimestamps વડે મૂળ પર સ્ટેમ્પ લગાવો
   (મૂળભૂત રીતે alice + finney — `MYRIAM_CAL_A`/`MYRIAM_CAL_B` દ્વારા
   ઓવરરાઇડ કરો).
3. `site/` ડિરેક્ટરીને IPFS પર પિન કરો અને CID છાપો.

કુલ: ગરમ IPFS ડેમન સાથે ~10 સેકન્ડ.

## 5. `site/index.html` સંપાદિત કરો

ટેમ્પ્લેટ પ્લેસહોલ્ડર્સને તમારા વાસ્તવિક તથ્યો, હેશ અને Merkle મૂળ વડે
બદલો. દર્શાવેલ પ્રતિ-તથ્ય હેશ પ્રતિ-તથ્ય ફાઇલો સાથે મેળ ખાવા જોઈએ, અથવા
પૃષ્ઠ તેની પોતાની સ્વ-ચકાસણીમાં નિષ્ફળ જાય છે.

દરેક તથ્ય માટે, .txt ફાઇલના મુખ્ય ભાગને `<div class="fact">` બ્લોકમાં
કૉપિ કરો અને SHA-256 ને તેની સીધી નીચે `<div class="hash">` માં મૂકો.
Merkle મૂળ ક્રિપ્ટોગ્રાફિક-સીલ વિભાગમાં જાય છે.

(હા, આ આપમેળે જનરેટ થઈ શકે છે. તે ઇરાદાપૂર્વક નથી: મેન્યુઅલ પગલું
માનવ સમીક્ષા પગલું છે. પ્રોટોકોલ તમે ન વાંચ્યું હોય તે પ્રકાશિત કરવા
સામે રક્ષણ આપતો નથી.)

## 6. સ્થાનિક રીતે ચકાસો

```bash
python3 build/verify.py
```

અપેક્ષિત આઉટપુટ:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

"Pending" સામાન્ય છે — OTS કૅલેન્ડર થોડા કલાકોમાં પ્રમાણીકરણને
સંપૂર્ણ Bitcoin પ્રમાણીકરણ માટે અપગ્રેડ કરશે.

## 7. ડિપ્લોય

કોઈપણ પસંદ કરો:

- **Cloudflare Pages**: dashboard → Pages → Direct upload → `site/` ખેંચો
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `site/` માં `git init`, repo માં પુશ કરો, Pages સક્ષમ કરો
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (સ્ટેટિક હોસ્ટિંગ ચાલુ)
- **કોઈપણ સાદું VPS**: `scp -r site/* user@host:/var/www/html/`

પગલું 4 માંથી IPFS CID તમને એક વધારાનું સામગ્રી-એડ્રેસ્ડ URL આપે છે:
`https://<CID>.ipfs.dweb.link/`. તેનો ઉપયોગ કરો.

## 8. OTS રસીદ અપગ્રેડ કરો (પછી)

સીલ કર્યાના ~થોડા કલાકોમાં, ચલાવો:

```bash
ots upgrade build/merkle_root.txt.ots
```

આ દરેક કૅલેન્ડરના પેન્ડિંગ પ્રમાણીકરણને સંપૂર્ણ Bitcoin પ્રમાણીકરણમાં
પરિવર્તિત કરે છે જે ચોક્કસ બ્લોકનો સંદર્ભ આપે છે. અપગ્રેડ કર્યા પછી
`.ots` ફાઇલને તમારા હોસ્ટ્સ પર ફરીથી ડિપ્લોય કરો.

## 9. બસ આટલું જ

તમે તથ્યોના નાના સમૂહને એવી રીતે પ્રકાશિત કર્યો છે કે કોઈપણ ત્રીજો પક્ષ:

- તમારા હોસ્ટથી સાદા HTTP દ્વારા બધા તથ્યો મેળવી શકે
- તથ્ય બાઇટ્સમાંથી Merkle મૂળની ફરીથી ગણતરી કરી શકે
- તમારા પ્રકાશિત મૂળ સાથે તુલના કરી શકે
- ચકાસી શકે કે મૂળ OTS-સ્ટેમ્પ કરેલ સમયે Bitcoin માટે પ્રતિબદ્ધ કરવામાં આવ્યું હતું
- પ્રકાશિત CID પર IPFS દ્વારા સ્વતંત્ર રીતે તે જ તથ્યો મેળવી શકે
- બધા હેશ મેળ ખાય છે તેની પુષ્ટિ કરી શકે

જો આમાંથી કોઈપણ પગલું નિષ્ફળ જાય, તો ચેડાં થયા છે. વાચકને તમારો
ભરોસો કરવાની જરૂર નથી. તેમને ફક્ત ગણિત કરવાનું છે.
