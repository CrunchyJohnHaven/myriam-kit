# Γρήγορη έναρξη

Πέντε λεπτά από το κλωνοποίηση (clone) μέχρι τη σφραγισμένη δημοσίευση. Δοκιμασμένο σε macOS·
το Linux αναμένεται να λειτουργεί· οι χρήστες Windows θα χρειαστούν WSL.

## 0. Προαπαιτούμενα

```bash
brew install python3 opentimestamps-client ipfs
python3 -m pip install --user certifi  # for macOS SSL
```

## 1. Σκαλωσιά

```bash
git clone <wherever-this-lives> myriam-kit
cd myriam-kit
bash lib/myriam-init.sh my-first-publication
cd my-first-publication
```

## 2. Επεξεργαστείτε τα γεγονότα

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

Κάθε αρχείο είναι ένα γεγονός. UTF-8. Τέλη γραμμής LF. Το μοτίβο ονόματος αρχείου είναι
`NN_short_name.txt`, όπου NN είναι ένδειξη με πρόθεμα μηδενικού. Ταξινομούνται
λεξικογραφικά μέσα στο δέντρο Merkle, οπότε το 02 έρχεται πάντα μετά το 01.

Συμβουλή: διατυπώστε κάθε γεγονός ως *συμβάν ονοματοδοσίας ή δήλωσης*, όχι ως
οντολογικό ισχυρισμό για τον κόσμο. Το «Η Acme Corp δήλωσε ότι η παρτίδα X
πέρασε τον έλεγχο Y» είναι επαληθεύσιμο. Το «Η παρτίδα X είναι ασφαλής» δεν είναι.

## 3. Ξεκινήστε τον δαίμονα (daemon) του IPFS (μία φορά ανά μηχάνημα)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. Σφραγίστε

```bash
bash build/seal.sh
```

Αυτό θα κάνει:
1. Hash σε κάθε γεγονός (SHA-256), υπολογισμός της ρίζας Merkle τύπου Bitcoin.
2. Σφράγιση της ρίζας με OpenTimestamps μέσω δύο ανεξάρτητων ημερολογίων
   (alice + finney από προεπιλογή — παράκαμψη μέσω `MYRIAM_CAL_A`/`MYRIAM_CAL_B`).
3. Καρφίτσωμα (pin) του καταλόγου `site/` στο IPFS και εκτύπωση του CID.

Σύνολο: ~10 δευτερόλεπτα με προθερμασμένο δαίμονα IPFS.

## 5. Επεξεργαστείτε το `site/index.html`

Αντικαταστήστε τα placeholders του προτύπου με τα πραγματικά σας γεγονότα, hashes και
τη ρίζα Merkle. Τα εμφανιζόμενα hashes ανά γεγονός πρέπει να ταιριάζουν με τα αρχεία γεγονότων,
αλλιώς η σελίδα αποτυγχάνει στην ίδια της την αυτο-επαλήθευση.

Για κάθε γεγονός, αντιγράψτε το σώμα του αρχείου .txt σε ένα μπλοκ
`<div class="fact">` και βάλτε το SHA-256 σε ένα `<div class="hash">` ακριβώς από κάτω.
Η ρίζα Merkle πάει στην ενότητα κρυπτογραφικής σφραγίδας.

(Ναι, αυτό θα μπορούσε να παράγεται αυτόματα. Δεν παράγεται, εσκεμμένα: το
χειροκίνητο βήμα είναι το βήμα ανθρώπινης αναθεώρησης. Το πρωτόκολλο δεν προστατεύει
από τη δημοσίευση κάποιου πράγματος που δεν διαβάσατε.)

## 6. Επαληθεύστε τοπικά

```bash
python3 build/verify.py
```

Αναμενόμενη έξοδος:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

Το «Pending» είναι φυσιολογικό — το ημερολόγιο OTS θα αναβαθμίσει την πιστοποίηση
σε πλήρη πιστοποίηση Bitcoin εντός λίγων ωρών.

## 7. Ανάπτυξη

Επιλέξτε οποιοδήποτε από τα παρακάτω:

- **Cloudflare Pages**: dashboard → Pages → Direct upload → drag `site/`
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `git init` στο `site/`, push στο repo, ενεργοποιήστε Pages
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (με ενεργό στατικό hosting)
- **Οποιοδήποτε απλό VPS**: `scp -r site/* user@host:/var/www/html/`

Το IPFS CID από το βήμα 4 σάς δίνει ένα πρόσθετο URL με διευθυνσιοδότηση
περιεχομένου: `https://<CID>.ipfs.dweb.link/`. Χρησιμοποιήστε το.

## 8. Αναβαθμίστε την απόδειξη OTS (αργότερα)

Εντός λίγων ωρών από τη σφράγιση, τρέξτε:

```bash
ots upgrade build/merkle_root.txt.ots
```

Αυτό μετατρέπει την εκκρεμή πιστοποίηση κάθε ημερολογίου σε πλήρη πιστοποίηση
Bitcoin που αναφέρεται σε συγκεκριμένο μπλοκ. Επαναναπτύξτε το αρχείο `.ots`
στους hosts σας μετά την αναβάθμιση.

## 9. Αυτό είναι

Έχετε δημοσιεύσει ένα μικρό σύνολο γεγονότων με τρόπο που οποιοδήποτε τρίτο μέρος μπορεί:

- Να ανακτήσει όλα τα γεγονότα μέσω απλού HTTP από τον host σας
- Να ξαναϋπολογίσει τη ρίζα Merkle από τα bytes των γεγονότων
- Να συγκρίνει με τη δημοσιευμένη σας ρίζα
- Να επαληθεύσει ότι η ρίζα δεσμεύτηκε στο Bitcoin κατά τη σφραγισμένη ώρα OTS
- Να ανακτήσει ανεξάρτητα τα ίδια γεγονότα μέσω IPFS στο δημοσιευμένο CID
- Να επιβεβαιώσει ότι όλα τα hashes ταιριάζουν

Αν οποιοδήποτε από αυτά τα βήματα αποτύχει, έχει συμβεί παραποίηση. Ο αναγνώστης δεν
χρειάζεται να σας εμπιστευτεί. Χρειάζεται απλώς να κάνει τα μαθηματικά.
