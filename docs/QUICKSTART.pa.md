# ਜਲਦੀ ਸ਼ੁਰੂਆਤ

ਕਲੋਨ ਤੋਂ ਮੁਹਰਬੰਦ ਪ੍ਰਕਾਸ਼ਨ ਤੱਕ ਪੰਜ ਮਿੰਟ। macOS 'ਤੇ ਟੈਸਟ ਕੀਤਾ ਗਿਆ; Linux ਨੂੰ
ਕੰਮ ਕਰਨਾ ਚਾਹੀਦਾ ਹੈ; Windows ਉਪਭੋਗਤਾਵਾਂ ਨੂੰ WSL ਦੀ ਲੋੜ ਹੋਵੇਗੀ।

## 0. ਪੂਰਵ-ਲੋੜਾਂ

```bash
brew install python3 opentimestamps-client ipfs
python3 -m pip install --user certifi  # for macOS SSL
```

## 1. ਸਕੈਫੋਲਡ

```bash
git clone <wherever-this-lives> myriam-kit
cd myriam-kit
bash lib/myriam-init.sh my-first-publication
cd my-first-publication
```

## 2. ਤੱਥ ਸੰਪਾਦਿਤ ਕਰੋ

```
facts/
├── 01_what_this_is.txt
├── 02_what_im_claiming.txt
└── 03_who_signs.txt
```

ਹਰ ਫਾਈਲ ਇੱਕ ਤੱਥ ਹੈ। UTF-8। LF ਲਾਈਨ ਅੰਤ। ਫਾਈਲ ਨਾਮ ਦਾ ਪੈਟਰਨ
`NN_short_name.txt` ਹੈ ਜਿੱਥੇ NN ਜ਼ੀਰੋ-ਪੈਡਡ ਹੈ। ਇਹਨਾਂ ਨੂੰ Merkle ਟਰੀ ਵਿੱਚ
ਸ਼ਬਦ-ਕੋਸ਼ ਦੇ ਕ੍ਰਮ ਵਿੱਚ ਛਾਂਟਿਆ ਜਾਂਦਾ ਹੈ, ਇਸ ਲਈ 02 ਹਮੇਸ਼ਾ 01 ਤੋਂ ਬਾਅਦ ਆਉਂਦਾ ਹੈ।

ਸੁਝਾਅ: ਹਰ ਤੱਥ ਨੂੰ ਇੱਕ *ਨਾਮਕਰਣ ਜਾਂ ਘੋਸ਼ਣਾ ਘਟਨਾ* ਦੇ ਰੂਪ ਵਿੱਚ ਪੇਸ਼ ਕਰੋ, ਨਾ ਕਿ
ਸੰਸਾਰ ਬਾਰੇ ਕਿਸੇ ਆਨਟੋਲੋਜੀਕਲ ਦਾਅਵੇ ਦੇ ਰੂਪ ਵਿੱਚ। "Acme Corp ਨੇ ਘੋਸ਼ਣਾ ਕੀਤੀ ਹੈ ਕਿ
ਬੈਚ X ਟੈਸਟ Y ਪਾਸ ਹੋ ਗਿਆ" ਤਸਦੀਕਯੋਗ ਹੈ। "ਬੈਚ X ਸੁਰੱਖਿਅਤ ਹੈ" ਨਹੀਂ ਹੈ।

## 3. IPFS daemon ਸ਼ੁਰੂ ਕਰੋ (ਪ੍ਰਤੀ ਮਸ਼ੀਨ ਇੱਕ-ਵਾਰ)

```bash
ipfs init                 # one-time per user
ipfs daemon &             # background, leave running
```

## 4. ਮੁਹਰਬੰਦ ਕਰੋ

```bash
bash build/seal.sh
```

ਇਹ ਕਰੇਗਾ:
1. ਹਰ ਤੱਥ ਨੂੰ ਹੈਸ਼ ਕਰੋ (SHA-256), Bitcoin-ਸ਼ੈਲੀ ਦੀ Merkle ਰੂਟ ਦੀ ਗਣਨਾ ਕਰੋ।
2. ਦੋ ਸੁਤੰਤਰ ਕੈਲੰਡਰਾਂ ਰਾਹੀਂ OpenTimestamps ਨਾਲ ਰੂਟ ਨੂੰ ਸਟੈਂਪ ਕਰੋ
   (ਡਿਫਾਲਟ ਤੌਰ 'ਤੇ alice + finney — `MYRIAM_CAL_A`/`MYRIAM_CAL_B` ਰਾਹੀਂ ਓਵਰਰਾਈਡ ਕਰੋ)।
3. `site/` ਡਾਇਰੈਕਟਰੀ ਨੂੰ IPFS ਉੱਤੇ ਪਿੰਨ ਕਰੋ ਅਤੇ CID ਪ੍ਰਿੰਟ ਕਰੋ।

ਕੁੱਲ: ਇੱਕ ਗਰਮ IPFS daemon ਨਾਲ ~10 ਸਕਿੰਟ।

## 5. `site/index.html` ਸੰਪਾਦਿਤ ਕਰੋ

ਟੈਮਪਲੇਟ ਪਲੇਸਹੋਲਡਰਾਂ ਨੂੰ ਆਪਣੇ ਅਸਲ ਤੱਥਾਂ, ਹੈਸ਼ਾਂ, ਅਤੇ Merkle ਰੂਟ ਨਾਲ ਬਦਲੋ।
ਪ੍ਰਦਰਸ਼ਿਤ ਪ੍ਰਤੀ-ਤੱਥ ਹੈਸ਼ ਪ੍ਰਤੀ-ਤੱਥ ਫਾਈਲਾਂ ਨਾਲ ਮੇਲ ਖਾਣੇ ਚਾਹੀਦੇ ਹਨ, ਨਹੀਂ ਤਾਂ
ਪੰਨਾ ਆਪਣੀ ਖੁਦ ਦੀ ਸਵੈ-ਤਸਦੀਕ ਵਿੱਚ ਅਸਫਲ ਹੋ ਜਾਂਦਾ ਹੈ।

ਹਰ ਤੱਥ ਲਈ, .txt ਫਾਈਲ ਦੇ ਮੁੱਖ ਭਾਗ ਨੂੰ ਇੱਕ `<div class="fact">` ਬਲਾਕ ਵਿੱਚ
ਕਾਪੀ ਕਰੋ ਅਤੇ SHA-256 ਨੂੰ ਸਿੱਧੇ ਇਸਦੇ ਹੇਠਾਂ ਇੱਕ `<div class="hash">` ਵਿੱਚ ਪਾਓ।
Merkle ਰੂਟ ਕ੍ਰਿਪਟੋਗ੍ਰਾਫਿਕ-ਮੁਹਰ ਸੈਕਸ਼ਨ ਵਿੱਚ ਜਾਂਦਾ ਹੈ।

(ਹਾਂ, ਇਹ ਆਪਣੇ-ਆਪ ਤਿਆਰ ਕੀਤਾ ਜਾ ਸਕਦਾ ਸੀ। ਜਾਣ-ਬੁੱਝ ਕੇ ਨਹੀਂ ਕੀਤਾ ਗਿਆ: ਹੱਥੀਂ
ਕਦਮ ਮਨੁੱਖੀ ਸਮੀਖਿਆ ਦਾ ਕਦਮ ਹੈ। ਪ੍ਰੋਟੋਕੋਲ ਇਸ ਦੇ ਵਿਰੁੱਧ ਬਚਾਅ ਨਹੀਂ ਕਰਦਾ ਕਿ ਤੁਸੀਂ
ਕੁਝ ਅਜਿਹਾ ਪ੍ਰਕਾਸ਼ਿਤ ਕਰੋ ਜੋ ਤੁਸੀਂ ਨਹੀਂ ਪੜ੍ਹਿਆ।)

## 6. ਸਥਾਨਕ ਤੌਰ 'ਤੇ ਤਸਦੀਕ ਕਰੋ

```bash
python3 build/verify.py
```

ਉਮੀਦ ਕੀਤਾ ਆਉਟਪੁੱਟ:
```
OK  merkle root matches: <hex>
OK  N leaf hashes match
--- ots verify merkle_root.txt.ots ---
Pending confirmation in Bitcoin blockchain
```

"Pending" ਆਮ ਹੈ — OTS ਕੈਲੰਡਰ ਕੁਝ ਘੰਟਿਆਂ ਅੰਦਰ ਪ੍ਰਮਾਣਨ ਨੂੰ ਇੱਕ ਪੂਰੇ
Bitcoin ਪ੍ਰਮਾਣਨ ਵਿੱਚ ਅੱਪਗ੍ਰੇਡ ਕਰ ਦੇਵੇਗਾ।

## 7. ਡਿਪਲੋਏ ਕਰੋ

ਇਹਨਾਂ ਵਿੱਚੋਂ ਕੋਈ ਵੀ ਚੁਣੋ:

- **Cloudflare Pages**: dashboard → Pages → Direct upload → `site/` ਨੂੰ ਖਿੱਚੋ
- **Netlify**: `netlify deploy --dir=site --prod`
- **GitHub Pages**: `site/` ਵਿੱਚ `git init`, repo ਉੱਤੇ ਪੁਸ਼ ਕਰੋ, Pages ਚਾਲੂ ਕਰੋ
- **S3**: `aws s3 sync site/ s3://your-bucket/ --delete` (ਸਟੈਟਿਕ ਹੋਸਟਿੰਗ ਚਾਲੂ)
- **ਕੋਈ ਵੀ ਸਾਦਾ VPS**: `scp -r site/* user@host:/var/www/html/`

ਕਦਮ 4 ਤੋਂ IPFS CID ਤੁਹਾਨੂੰ ਇੱਕ ਵਾਧੂ ਸਮੱਗਰੀ-ਸੰਬੋਧਿਤ URL ਦਿੰਦਾ ਹੈ:
`https://<CID>.ipfs.dweb.link/`। ਇਸਨੂੰ ਵਰਤੋ।

## 8. OTS ਰਸੀਦ ਨੂੰ ਅੱਪਗ੍ਰੇਡ ਕਰੋ (ਬਾਅਦ ਵਿੱਚ)

ਮੁਹਰਬੰਦ ਕਰਨ ਦੇ ~ਕੁਝ ਘੰਟਿਆਂ ਅੰਦਰ, ਚਲਾਓ:

```bash
ots upgrade build/merkle_root.txt.ots
```

ਇਹ ਹਰ ਕੈਲੰਡਰ ਦੇ ਲੰਬਿਤ ਪ੍ਰਮਾਣਨ ਨੂੰ ਇੱਕ ਪੂਰੇ Bitcoin ਪ੍ਰਮਾਣਨ ਵਿੱਚ ਬਦਲ ਦਿੰਦਾ ਹੈ
ਜੋ ਇੱਕ ਖਾਸ ਬਲਾਕ ਨੂੰ ਸੰਦਰਭ ਕਰਦਾ ਹੈ। ਅੱਪਗ੍ਰੇਡ ਕਰਨ ਤੋਂ ਬਾਅਦ ਆਪਣੇ ਹੋਸਟਾਂ
ਉੱਤੇ `.ots` ਫਾਈਲ ਨੂੰ ਮੁੜ-ਡਿਪਲੋਏ ਕਰੋ।

## 9. ਬਸ ਇੰਨਾ ਹੀ

ਤੁਸੀਂ ਤੱਥਾਂ ਦੇ ਇੱਕ ਛੋਟੇ ਸਮੂਹ ਨੂੰ ਇਸ ਤਰੀਕੇ ਨਾਲ ਪ੍ਰਕਾਸ਼ਿਤ ਕੀਤਾ ਹੈ ਕਿ ਕੋਈ ਵੀ ਤੀਜੀ ਧਿਰ:

- ਤੁਹਾਡੇ ਹੋਸਟ ਤੋਂ ਸਾਦੇ HTTP 'ਤੇ ਸਾਰੇ ਤੱਥ ਲੈ ਸਕਦੀ ਹੈ
- ਤੱਥ ਬਾਈਟਸ ਤੋਂ Merkle ਰੂਟ ਮੁੜ-ਗਣਨਾ ਕਰ ਸਕਦੀ ਹੈ
- ਤੁਹਾਡੇ ਪ੍ਰਕਾਸ਼ਿਤ ਰੂਟ ਨਾਲ ਤੁਲਨਾ ਕਰ ਸਕਦੀ ਹੈ
- ਤਸਦੀਕ ਕਰ ਸਕਦੀ ਹੈ ਕਿ ਰੂਟ OTS-ਸਟੈਂਪ ਕੀਤੇ ਸਮੇਂ Bitcoin ਉੱਤੇ ਪ੍ਰਤੀਬੱਧ ਕੀਤਾ ਗਿਆ ਸੀ
- ਪ੍ਰਕਾਸ਼ਿਤ CID ਉੱਤੇ IPFS ਰਾਹੀਂ ਸੁਤੰਤਰ ਤੌਰ 'ਤੇ ਉਹੀ ਤੱਥ ਲੈ ਸਕਦੀ ਹੈ
- ਪੁਸ਼ਟੀ ਕਰ ਸਕਦੀ ਹੈ ਕਿ ਸਾਰੇ ਹੈਸ਼ ਮੇਲ ਖਾਂਦੇ ਹਨ

ਜੇ ਇਹਨਾਂ ਵਿੱਚੋਂ ਕੋਈ ਵੀ ਕਦਮ ਅਸਫਲ ਹੁੰਦਾ ਹੈ, ਤਾਂ ਛੇੜਛਾੜ ਹੋਈ ਹੈ। ਪਾਠਕ ਨੂੰ
ਤੁਹਾਡੇ 'ਤੇ ਭਰੋਸਾ ਕਰਨ ਦੀ ਲੋੜ ਨਹੀਂ ਹੈ। ਉਹਨਾਂ ਨੂੰ ਸਿਰਫ਼ ਗਣਿਤ ਕਰਨੀ ਹੁੰਦੀ ਹੈ।
