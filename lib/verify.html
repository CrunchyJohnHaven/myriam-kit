MYRIAM verifier

Browser-side cryptographic verification · no install, no backend, no telemetry

Paste a MYRIAM publication URL (e.g. https://disclosure.host) or an IPFS gateway URL (e.g. https://ipfs.io/ipfs/<CID>/) and click Verify. This page will:

  1. Fetch the published Merkle root from <URL>/merkle_root.txt.
  2. Fetch the leaf-hash list from <URL>/build/leaf_hashes.txt.
  3. Fetch each fact file from <URL>/facts/<filename>.
  4. Recompute per-fact SHA-256 using your browser's crypto.subtle.digest.
  5. Recompute the Bitcoin-style Merkle root over those leaves.
  6. Compare to the published root.

No data leaves your browser. The OpenTimestamps Bitcoin attestation must still be verified externally (see Step 7 below) because the OTS protocol requires reaching a Bitcoin node or calendar server, which is out of scope for a self-contained browser verifier.

How this works (and how to verify the verifier itself)

This page uses only browser-native APIs: fetch, crypto.subtle.digest, and DOM manipulation. The entire verifier is in a single HTML file. To audit it:

  1. View source: in your browser, right-click → View Source (or Cmd-U). The Merkle and SHA-256 code is under 100 lines.
  2. Compare hashing: hash the same input with shasum -a 256 on the command line. Browser crypto.subtle.digest("SHA-256", ...) must produce the same bytes for any input.
  3. Cross-check Merkle: the algorithm is in SPEC §3. Both Python (build.py) and this JavaScript MUST reproduce identical roots for the same leaves.

If you don't trust this verifier, run ../lib/verify.py (the Python verifier in this kit). Or read the SPEC and write your own — it's about 30 lines of code.

Step 7: how to also verify the Bitcoin commitment

The browser cannot verify the OpenTimestamps receipt against Bitcoin directly (no built-in Bitcoin client; OTS calendar protocols require non-browser-friendly request paths). To complete the cryptographic verification: 

brew install opentimestamps-client
curl -O <URL>/merkle_root.txt
curl -O <URL>/merkle_root.txt.ots
ots verify merkle_root.txt.ots

If the receipt's pending-attestation status has upgraded to a full Bitcoin attestation, this confirms the Merkle root was committed to Bitcoin at or before the time of the referenced block.
