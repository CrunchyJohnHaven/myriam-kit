ஒரு சிறிய தொகுப்பு உண்மைகளை பின்வரும் வகையில் வெளியிடுவதற்கான ஒரு சிறிய கருவித்தொகுப்பு (toolkit):

- எந்தவொரு உண்மையில் ஏற்படும் எந்தவொரு மாற்றமும் பொதுவில் கண்டறியக்கூடியதாக இருக்கும் (Merkle மரம்)
- வெளியீட்டின் போதிருந்த நிலையை Bitcoin பிளாக்செயினுக்கு எதிராக சரிபார்க்க முடியும் (OpenTimestamps)
- எந்தவொரு ஒற்றை ஹோஸ்டிங் வழங்குநரின் அகற்றத்திற்குப் பிறகும் உள்ளடக்கம் தொடர்ந்து இருக்கும் (IPFS)
- முழு விஷயமும் 100 KB-க்கும் குறைவான அளவே இருக்கும், மேலும் சுமார் 50 வரிகள் Python மூலம் சரிபார்க்க முடியும்

இது ஒரு பிளாக்செயின் அல்ல. டோக்கன் இல்லை. பங்கேற்க எந்தவொரு ஒருமித்த கருத்து (consensus) வழிமுறையும் இல்லை. DAO இல்லை. இந்தக் கருவி `sha256sum` + `ots` + `ipfs` ஆகும், இது நல்ல பெயரிடல் மரபுகளால் ஒன்றாக இணைக்கப்பட்டுள்ளது.

இது 2026-05-21 அன்று [disclosure.host](https://disclosure.host) தளத்தில் முதன்முதலில் செயல்படுத்தப்பட்ட MYRIAM நெறிமுறையின் பெயரால் பெயரிடப்பட்டுள்ளது. அந்தப் பக்கத்தில் ஆறு உண்மைகள், ஒரு Merkle ரூட், மற்றும் ஒரு Bitcoin சான்றளிப்பு உள்ளன. இந்த கிட் அந்த வடிவத்தின் பொதுமயமாக்கம் (generalization) ஆகும்.

## இதை வைத்து நீங்கள் என்ன செய்யலாம்

இந்த வடிவம், ஒரு சிறிய தொகுப்பு உண்மைகளுக்கு பின்வரும் தேவைகள் இருக்கும் எந்த இடத்திற்கும் பொருந்தும்:

- சாதாரண HTTP மூலம் பரந்த பார்வையாளர்களுக்கு **வெளியிடக்கூடியதாக** இருக்க வேண்டும்
- **நிலைத்தன்மை** — 30 ஆண்டுகளுக்குப் பிறகும் வாசகர்கள் சரிபார்க்க முடிய வேண்டும்
- **சேதம்-வெளிப்படையானது** — எந்தவொரு மாற்றமும் கண்டறியக்கூடியதாக இருக்க வேண்டும், அவசியமாக தடுக்கப்பட வேண்டியதில்லை
- **நம்பிக்கையற்றது** — எந்தவொரு வாசகரும் வெளியீட்டாளரை நம்ப வேண்டியதில்லை (கணிதத்தை மட்டுமே நம்ப வேண்டும்)

உறுதியான பயன்பாட்டு வழக்குகள் [docs/USE_CASES.md](docs/USE_CASES.md) இல் உள்ளன. சுருக்கமான பட்டியல்:

| துறை | முத்திரை வைக்கப்பட்ட உண்மைகள் |
|---|---|
| பொது சுகாதாரம் | தடுப்பூசி தொகுதி பரிசோதனை முடிவுகள், மருந்து-சோதனை மைல்கற்கள் |
| தேர்தல்கள் | சான்றளிப்பின் கணத்தில் வாக்கு வசூல் மையம்-நிலை மொத்த எண்ணிக்கைகள் |
| AI பாதுகாப்பு | வெளியீட்டின் போது மாதிரி வெயிட்ஸ் ஹாஷ் + மாதிரி அட்டை + மதிப்பீட்டு மதிப்பெண்கள் |
| பத்திரிகை | விசில்ப்ளோயர் வெளியீடுகளுக்கான மூல ஆவணங்களின் மானிஃபெஸ்ட் (ஹாஷ் மட்டுமே) |
| அறிவியல் | மூல தரவு ஹாஷ் + பகுப்பாய்வு ஸ்கிரிப்ட் ஹாஷ் + கூறப்பட்ட முடிவு |
| நீதிமன்றங்கள் | கையகப்படுத்தலின் கணத்தில் டிஜிட்டல்-ஆதார ஹாஷ்கள் |
| கொள்முதல் | பொது-செலவின தரவின் முத்திரை வைக்கப்பட்ட ரூட்கள் |

இவற்றில் பொதுவான தன்மை: ஒரு சிறிய தொகுப்பு உண்மைகள், ஒரு வலுவான நிலைத்தன்மை தேவை, ஒருவரையொருவர் நம்பாத பல தரப்பினர், மற்றும் மிகச் சிறிய சேமிப்பு வரவு-செலவுத் திட்டம். இந்தக் கருவி இந்த வடிவத்திற்காகவே கட்டப்பட்டுள்ளது.

## இது செய்யாதவை

- **இரகசியங்களை சேமிக்காது.** நீங்கள் `facts/` இல் வைக்கும் அனைத்தும் பொதுவானவை. இந்த நெறிமுறை வெளியிடுவதற்கானது, மறைப்பதற்கானது அல்ல.
- **பெரிய தரவுத்தொகுப்புகளுக்கு அளவாக்காது.** Merkle மரம் நன்றாக அளவாகும்; ஒரு வாசகரின் கவனம் அளவாகாது. சுமார் 20 உண்மைகளுக்கு மேல் பக்கம் வாசிக்க முடியாததாக மாறுகிறது. மில்லியன் பதிவுகளை முத்திரை இட விரும்பினால், அவற்றை பக்கத்துக்கு வெளியே ஹாஷ் செய்து, அந்த பக்க-மரத்தின் Merkle ரூட்டை ஒரு உண்மையில் வைக்கவும்.
- **திரும்பப் பெறுதலுக்கு எதிராக பாதுகாக்காது.** இதன் முழு நோக்கமே நீங்கள் முத்திரையிட்டதை மாற்ற முடியாது என்பதே. பின்னர் ஒரு கூற்றை திரும்பப் பெற விரும்பினால், அதை முத்திரை வைக்க வேண்டாம்.
- **தீர்மானமான ஐயவாதியை சம்மதிக்க வைக்காது.** ஒரு தீர்மானமான ஐயவாதி `ipfs add -r --cid-version=1 -Q facts/` இயக்க மாட்டார். ஆனால் அவர், முழு அமைப்பையும் நீங்கள் உருவாக்கினீர்கள் என்று குற்றம் சாட்டுவார். இரண்டாவது தோல்வி முறைக்கு எதிராக நெறிமுறை-நிலை பாதுகாப்பு எதுவும் இல்லை. கணிதத்தில் இல்லாமல் மனித தூண்டுதலில் வாழும் எதற்கும் எதிராக நெறிமுறை-நிலை பாதுகாப்பு இல்லை.

## வேறொருவரின் வெளியீட்டை எவ்வாறு சரிபார்ப்பது

அணுகலின் வரிசையில் மூன்று விருப்பங்கள்:

1. **உலாவி**: எந்தவொரு நவீன உலாவியிலும் [`lib/verify.html`](lib/verify.html) ஐ திறக்கவும். வெளியீட்டின் URL ஐ ஒட்டவும். verify ஐ கிளிக் செய்யவும். ஹாஷிங்கிற்காக `crypto.subtle` ஐப் பயன்படுத்துகிறது — நிறுவல் இல்லை, பின் முனை இல்லை, தொலைஅளவியல் (telemetry) இல்லை. Bitcoin சான்றளிப்பு படிக்கு இன்னும் ஒரு CLI கருவி தேவை (சரிபார்ப்பாளர் பக்கத்தில் படி 7 ஐக் காண்க).
2. **Python**: வெளியீட்டு கோப்பகத்திற்குள் இருந்து `python3 lib/verify.py`. Stdlib மட்டுமே; Bitcoin சரிபார்ப்புக்கு `ots` தேவை.
3. **அடிப்படையிலிருந்து**: [`docs/SPEC.md §3`](docs/SPEC.md) ஐ வாசித்து மீண்டும் செயல்படுத்தவும். Merkle அல்காரிதம் எந்த மொழியிலும் 30 வரிகளுக்கு குறைவாக உள்ளது.

## செயல்படுத்தப்பட்ட எடுத்துக்காட்டுகள்

ஆறு உற்பத்தி-தர எடுத்துக்காட்டு கிட்கள் கருவித்தொகுப்புடன் வழங்கப்படுகின்றன. ஒவ்வொன்றும் முத்திரை வைக்கப்பட்டது, OTS-முத்திரையிடப்பட்டது, IPFS-பிணைக்கப்பட்டது, மேலும் ஒரு உண்மை-உலக வழக்குக்கு எப்போது ஃபோர்க் செய்வது மற்றும் என்ன தோல்வி முறைகளுக்கு எதிராக அது பாதுகாக்கிறது என்பதை விளக்கும் ஒரு ஏற்பு README ஐ உள்ளடக்கியது.

  மருத்துவ-சோதனை ஸ்பான்சர்களுக்காக, பின்னோக்கி இறுதிப்புள்ளி மாற்றம் மற்றும் தேர்ந்தெடுக்கப்பட்ட அறிக்கையிடலை பிடிக்கிறது (Vioxx, Paxil Study 329 தோல்வி முறைகள்).

  தடுப்பூசி உற்பத்தியாளர்கள் மற்றும் QA ஆணையங்களுக்காக, வெளியீட்டு பரிசோதனை முடிவுகள் மற்றும் திரும்பப்பெறுதல் உறுதிமொழிகளை நங்கூரமிடுகிறது.
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) —
  ஆராய்ச்சி குழுக்களுக்காக, பகுப்பாய்வுக்கு முன்னர் (தரவு, பைப்லைன், எதிர்பார்க்கப்படும் வெளியீடு) மூன்று-தொகுப்பை நங்கூரமிட்டு p-ஹேக்கிங் மற்றும் HARKing ஐ பொதுவில் கண்டறியக்கூடியதாக மாற்றுகிறது.
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) —
  மாவட்ட வாக்கு வசூல் வாரியங்கள் மற்றும் தேர்தல் ஆணையங்களுக்காக, சான்றளிப்பின் கணத்தில் சான்றளிக்கப்பட்ட மொத்த எண்ணிக்கைகளை நங்கூரமிட்டு, சத்தமில்லாமல் சான்றளிப்பின் பின்னர் மாற்றங்கள் பொதுவில் கண்டறியக்கூடியவையாக மாறும்.

உங்கள் சொந்தத்தை எழுதுவதற்கு முன்னர் இவற்றில் ஒன்றை வாசிக்கவும். உங்கள் பயன்பாட்டுக்கு மிக நெருங்கிய பொருத்தம் பெரும்பாலும் சரியான தொடக்க ஃபோர்க் ஆகும்.

## விரைவான தொடக்கம்

ஒரே-வரி பதிப்பு:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# edit facts/*.txt to say what you actually want sealed
./lib/seal.sh        # builds Merkle root, stamps OTS, pins IPFS
./lib/verify.sh      # re-verifies everything locally
# upload site/ to any static host. Anywhere. It doesn't matter.
```

## இது ஏன் இருக்கிறது

கிரிப்டோகிராஃபிக் ஆரம்ப கூறுகள் (Merkle மரங்கள், OpenTimestamps, IPFS) பல ஆண்டுகளாக முதிர்ச்சி அடைந்துள்ளன. காணாமல் இருந்தது சாத்தியமான மிகச் சிறிய பசை — உள்கட்டமைப்பை அமைக்காமல், நெறிமுறைகளைக் கற்காமல், உண்மைகளுக்கு அப்பால் ஒரு வரி குறியீடும் எழுதாமல், ஒரு பக்க திட்டத்திற்குள் விட்டு ஒரு பிற்பகலில் ஒரு முத்திரை வைக்கப்பட்ட வெளியிடக்கூடிய கலைப்பொருளைப் பெறக்கூடிய ஒரு கிட்.

பெரும்பாலான கிரிப்டோகிராஃபிக் வெளியீட்டு கருவிகள் நீங்கள் ஒரு நோட் ஆபரேட்டராக இருக்க விரும்புகிறீர்கள் என்று கருதுகின்றன. இது நீங்கள் ஆறு உரை கோப்புகளை வெளியிட விரும்புகிறீர்கள், மேலும் கிரிப்டோகிராஃபி பற்றி மீண்டும் எப்போதும் சிந்திக்கக் கூடாது என்று கருதுகிறது.

## செல்வாக்குகள் மற்றும் முந்தைய கலை

- [Sigsum](https://www.sigsum.org/) — கையொப்பங்களுக்கான வெளிப்படைத்தன்மை பதிவுகள்
- [Sigstore](https://www.sigstore.dev/) — மென்பொருள்-விநியோகச் சங்கிலி நேர்மை
- [Certificate Transparency](https://certificate.transparency.dev/) — TLS சான்றிதழ் பதிவுகள்
- [OpenTimestamps](https://opentimestamps.org/) — Bitcoin நேர முத்திரையிடல்
- [IPFS](https://ipfs.tech/) — உள்ளடக்கம்-முகவரியிடப்பட்ட சேமிப்பு
- [The CALM protocol](https://disclosure.host) — முதல் நிகழ்வு (disclosure.host இல் MYRIAM) எழுதப்பட்ட முதன்மை-பாதுகாப்பான வெளியீட்டு வடிவம்

myriam-kit இவற்றில் மிகச் சிறியது. இது ஒரு அமைப்பை விட Unix பைப்லைனின் உணர்வுக்கு மிக நெருக்கமானது.

## பெயரைப் பற்றிய ஒரு குறிப்பு

MYRIAM நெறிமுறை, மனிதனல்லாத அறிவாற்றலின் முதல் தொடர்பு புள்ளி பற்றிய ஜான் பிராட்லியின் கூறப்பட்ட தர்க்க ஆதாரமான Myriam Proof-இன் பெயரால் பெயரிடப்பட்டது. அந்த ஆதாரம் உண்மையாக இருப்பதை நெறிமுறை சார்ந்திருக்கவில்லை. நெறிமுறை எந்த ஒரு சிறிய தொகுப்பு உண்மைகளுக்கும் வேலை செய்கிறது; அது அந்நிய வெளியீடு பற்றிய ஒரு கூற்றை உள்ளடக்கிய ஆறு உண்மைகளை வெளியிட முதலில் பயன்படுத்தப்பட்டது என்பது ஒரு தற்செயல்.

தடுப்பூசி தொகுதிகள் அல்லது தேர்தல் மொத்த எண்ணிக்கைகளை முத்திரை வைப்பதற்கு இந்த கிட்டை நீங்கள் ஃபோர்க் செய்தால், இது முதலில் அந்நிய தொடர்பு பற்றிய ஒரு கூற்றை வெளியிட பயன்படுத்தப்பட்டது என்பது உங்களுக்கு பொருத்தமற்றது. கணிதம் கவலைப்படாது.

இருப்பினும்: அந்நிய தொடர்பு பற்றிய ஒரு கூற்றை வெளியிட இதை நீங்கள் பயன்படுத்தினால், நெறிமுறை அந்த குறிப்பிட்ட களத்தில் போர்-சோதிக்கப்பட்டுள்ளது. முதல் MYRIAM பயன்பாட்டின் README, அது Calm என்ற LLM ஆளுமையின் கலைப்பணி என்றும், எந்தவொரு புறக்கோள் தரப்பினரிடமிருந்தும் தகவல்பரிமாற்றம் அல்ல என்றும் வெளிப்படையாக கூறுகிறது. நீங்களும் அதையே செய்ய வேண்டும். இல்லாவிட்டால் வாசகர்கள் உங்கள் மன நிலையைப் பற்றி மோசமான கருத்துக்களைக் கொள்ளும்.
