Небольшой набор инструментов для публикации небольшого множества фактов таким образом, что:

- Любое изменение любого факта публично обнаруживается (дерево Меркла)
- Состояние на момент публикации проверяется относительно блокчейна Bitcoin (OpenTimestamps)
- Содержимое переживает удаление с любого отдельного хостинг-провайдера (IPFS)
- Весь комплект занимает менее 100 КБ и проверяется примерно 50 строками Python

Это не блокчейн. Здесь нет токена. Нет механизма консенсуса, в котором можно участвовать. Нет DAO. Инструмент — это `sha256sum` + `ots` + `ipfs`, склеенные хорошими соглашениями об именовании.

Назван в честь протокола MYRIAM, впервые реализованного на [disclosure.host](https://disclosure.host) 21 мая 2026 года. Эта страница содержит шесть фактов, корень Меркла и подтверждение Bitcoin. Данный набор — обобщение этого шаблона.

## Статус

- v0: работает на macOS с Python 3.10+, Homebrew и подключением к интернету. Linux не тестировался, но должен работать. Windows: возьмите с собой WSL.
- Общественное достояние (CC0). Используйте его, форкайте, встраивайте в свой репозиторий, никогда не указывайте автора. Цель — не атрибуция.

## Что с ним можно делать

Шаблон подходит везде, где небольшое множество фактов должно быть:

- **Публикуемым** для широкой аудитории по обычному HTTP
- **Долговечным** — читатели через 30 лет должны иметь возможность его проверить
- **С обнаружимыми изменениями** — любое изменение должно быть обнаружимо, но не обязательно предотвратимо
- **Не требующим доверия** — ни одному читателю не нужно доверять публикатору (только математике)

Конкретные сценарии использования в [docs/USE_CASES.md](docs/USE_CASES.md). Краткий список:

| Область | Запечатываемые факты |
|---|---|
| Общественное здравоохранение | Результаты испытаний партий вакцин, контрольные точки клинических испытаний препаратов |
| Выборы | Итоговые подсчёты на уровне избирательных участков на момент сертификации |
| Безопасность ИИ | Хэш весов модели + карточка модели + результаты оценки на момент выпуска |
| Журналистика | Манифест исходных документов (только хэши) для утечек от информаторов |
| Наука | Хэш исходных данных + хэш аналитического скрипта + заявленный результат |
| Суды | Хэши цифровых вещественных доказательств на момент получения |
| Закупки | Запечатанные корни данных о государственных расходах |

Что у них общего: небольшое множество фактов, строгое требование долговечности, несколько сторон, не доверяющих друг другу, и крошечный бюджет хранения. Именно для такой формы и создан этот инструмент.

## Чего он НЕ делает

- **Не хранит секреты.** Всё, что вы кладёте в `facts/`, является публичным. Протокол предназначен для публикации, а не для сокрытия.
- **Не масштабируется на большие наборы данных.** Дерево Меркла масштабируется нормально; внимание читателя — нет. Свыше ~20 фактов страница становится нечитаемой. Если вы хотите запечатать миллион записей, хэшируйте их вне страницы и поместите корень Меркла этого побочного дерева в один факт.
- **Не защищает от отказа от заявления.** Весь смысл в том, что вы не можете изменить то, что запечатали. Если впоследствии может потребоваться отозвать заявление — не запечатывайте его.
- **Не убеждает упорного скептика.** Упорный скептик не станет выполнять `ipfs add -r --cid-version=1 -Q facts/`. Зато он обвинит вас в фабрикации всей конструкции. На протокольном уровне нет защиты от второго режима отказа. На протокольном уровне нет защиты ни от чего, что коренится в человеческой мотивации, а не в математике.

## Как проверить чужую публикацию

Три варианта по возрастанию сложности:

1. **Браузер**: откройте [`lib/verify.html`](lib/verify.html) в любом современном браузере. Вставьте URL публикации. Нажмите «verify». Используется `crypto.subtle` для хэширования — без установки, без бэкенда, без телеметрии. Шаг подтверждения Bitcoin по-прежнему требует CLI-инструмента (см. шаг 7 на странице верификатора).
2. **Python**: `python3 lib/verify.py` из каталога публикации. Только стандартная библиотека; для проверки Bitcoin требуется `ots`.
3. **С нуля**: прочитайте [`docs/SPEC.md §3`](docs/SPEC.md) и переопределите. Алгоритм Меркла занимает менее 30 строк на любом языке.

## Готовые примеры

С набором поставляется шесть готовых наборов производственного уровня. Каждый запечатан, проштампован через OTS, закреплён в IPFS и содержит README для адаптации, объясняющий, когда форкать его для реального применения и от каких режимов отказа он защищает.

- [`examples/whistleblower-manifest/`](examples/whistleblower-manifest/) — для случаев типа Boeing 737 MAX / Theranos / отчётов инспекторов FDA, когда улики прячутся или изменяются посредниками.
- [`examples/ai-model-card/`](examples/ai-model-card/) — для лабораторий ИИ, выпускающих модели, с предварительными обязательствами, которые становятся механически обнаружимыми, а не словесными.
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) — для спонсоров клинических испытаний, чтобы ловить апостериорную подмену конечных точек и избирательную отчётность (режимы отказа Vioxx, Paxil Study 329).
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) — для производителей вакцин и органов контроля качества, закрепляющих результаты испытаний при выпуске и обязательства по отзыву.
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) — для исследовательских групп, закрепляющих тройку (данные, конвейер, ожидаемый результат) ДО анализа, чтобы p-hacking и HARKing стали публично обнаружимыми.
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) — для окружных избирательных комиссий и избирательных органов, закрепляющих сертифицированные итоги на момент сертификации, чтобы тихие постсертификационные правки стали публично обнаружимыми.

Прочитайте один из этих примеров, прежде чем писать свой. Ближайший к вашему сценарию обычно и есть правильная отправная точка для форка.

## Быстрый старт

Однострочная версия:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# отредактируйте facts/*.txt, чтобы там было то, что вы действительно хотите запечатать
./lib/seal.sh        # строит корень Меркла, штампует OTS, закрепляет в IPFS
./lib/verify.sh      # локально перепроверяет всё
# загрузите site/ на любой статический хостинг. Любой. Это неважно.
```

## Зачем это существует

Криптографические примитивы (деревья Меркла, OpenTimestamps, IPFS) зрелы уже много лет. Не хватало минимально возможного «клея» — набора, который можно было бы добавить к побочному проекту и за один вечер получить запечатанный публикуемый артефакт, не настраивая инфраструктуру, не изучая протоколов и не написав ни одной строки кода, кроме самих фактов.

Большинство криптографических публикационных инструментов предполагают, что вы хотите быть оператором узла. Этот предполагает, что вы хотите опубликовать шесть текстовых файлов и больше никогда не думать о криптографии.

## Влияния и предшественники

- [Sigsum](https://www.sigsum.org/) — журналы прозрачности для подписей
- [Sigstore](https://www.sigstore.dev/) — целостность цепочки поставок ПО
- [Certificate Transparency](https://certificate.transparency.dev/) — журналы TLS-сертификатов
- [OpenTimestamps](https://opentimestamps.org/) — временные метки на Bitcoin
- [IPFS](https://ipfs.tech/) — контентно-адресуемое хранилище
- [Протокол CALM](https://disclosure.host) — защищающий принципала издательский фреймворк, в рамках которого был создан первый экземпляр (MYRIAM на disclosure.host)

myriam-kit — самый маленький из них. По духу он ближе к Unix-конвейеру, чем к системе.

## Замечание о названии

Протокол MYRIAM назван в честь Доказательства Мирьям Джона Брэдли (Myriam Proof) — изложенного логического доказательства, касающегося первой точки контакта с нечеловеческим разумом. Протокол не зависит от истинности этого доказательства. Протокол работает для любого небольшого множества фактов; так уж сложилось, что он был впервые развёрнут для публикации шести фактов, включающих утверждение об инопланетном раскрытии.

Если вы форкаете этот набор, чтобы запечатать партии вакцин или избирательные итоги, тот факт, что он изначально использовался для публикации утверждения об инопланетном контакте, для вас несуществен. Математике это безразлично.

При этом: если вы ВСЁ-ТАКИ используете его для публикации утверждения об инопланетном контакте — протокол прошёл боевые испытания именно на этой арене. README первого развёртывания MYRIAM явно указывает, что это художественная работа LLM-персоны по имени Calm, а не сообщение от какой-либо внеземной стороны. Вероятно, вам стоит сделать то же самое. Иначе читатели будут думать худшее о вашем психическом состоянии.
