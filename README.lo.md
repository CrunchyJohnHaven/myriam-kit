ຊຸດເຄື່ອງມືນ້ອຍສຳລັບການເຜີຍແຜ່ຊຸດຂໍ້ມູນຄວາມຈິງຂະໜາດນ້ອຍໃນລັກສະນະທີ່:

- ການປ່ຽນແປງໃດໆ ຕໍ່ຂໍ້ມູນຄວາມຈິງໃດໜຶ່ງສາມາດກວດພົບໄດ້ໂດຍສາທາລະນະ (ຕົ້ນໄມ້ Merkle)
- ສະຖານະໃນເວລາເຜີຍແຜ່ສາມາດກວດສອບໄດ້ກັບບລັອກເຊນ Bitcoin (OpenTimestamps)
- ເນື້ອຫາຍັງຄົງຢູ່ໄດ້ເຖິງວ່າຜູ້ໃຫ້ບໍລິການໂຮດສະຕິງໃດໜຶ່ງຈະຖືກປິດກໍຕາມ (IPFS)
- ທັງໝົດມີຂະໜາດຕ່ຳກວ່າ 100 KB ແລະສາມາດກວດສອບໄດ້ດ້ວຍ Python ປະມານ 50 ແຖວ

ນີ້ບໍ່ແມ່ນບລັອກເຊນ. ບໍ່ມີໂທເຄນ. ບໍ່ມີກົນໄກສ້າງຄວາມເຫັນພ້ອມໃຫ້ເຂົ້າຮ່ວມ. ບໍ່ມີ DAO. ເຄື່ອງມືນີ້ຄື `sha256sum` + `ots` + `ipfs`, ປະກອບເຂົ້າກັນດ້ວຍຫຼັກການຕັ້ງຊື່ທີ່ດີ.

ມັນຖືກຕັ້ງຊື່ຕາມໂປຣໂຕຄອນ MYRIAM ທີ່ໄດ້ຖືກນຳໃຊ້ຄັ້ງທຳອິດທີ່ [disclosure.host](https://disclosure.host) ໃນວັນທີ 2026-05-21. ໜ້ານັ້ນປະກອບດ້ວຍຫົກຂໍ້ມູນຄວາມຈິງ, ຮາກ Merkle, ແລະຄຳຢັ້ງຢືນ Bitcoin. ຊຸດເຄື່ອງມືນີ້ແມ່ນການຂະຫຍາຍຮູບແບບດັ່ງກ່າວໃຫ້ເປັນທົ່ວໄປ.

## ສິ່ງທີ່ທ່ານສາມາດເຮັດໄດ້ກັບມັນ

ຮູບແບບນີ້ເໝາະສົມໃນທຸກບ່ອນທີ່ຊຸດຂໍ້ມູນຄວາມຈິງຂະໜາດນ້ອຍຈຳເປັນຕ້ອງເປັນ:

- **ສາມາດເຜີຍແຜ່ໄດ້** ໃຫ້ຜູ້ຊົມໃນວົງກວ້າງຜ່ານ HTTP ທຳມະດາ
- **ທົນທານ** — ຜູ້ອ່ານໃນອີກ 30 ປີຄວນຍັງສາມາດກວດສອບໄດ້
- **ກວດພົບການແກ້ໄຂໄດ້** — ການປ່ຽນແປງໃດໆ ຕ້ອງສາມາດກວດພົບໄດ້, ບໍ່ຈຳເປັນຕ້ອງປ້ອງກັນໄດ້
- **ບໍ່ຕ້ອງເຊື່ອໃຈ** — ຜູ້ອ່ານບໍ່ຕ້ອງເຊື່ອໃຈຜູ້ເຜີຍແຜ່ (ເຊື່ອພຽງແຕ່ຄະນິດສາດ)

ກໍລະນີການນຳໃຊ້ທີ່ເປັນຮູບປະທຳຢູ່ທີ່ [docs/USE_CASES.md](docs/USE_CASES.md). ລາຍການສັ້ນ:

| ຂົງເຂດ | ຂໍ້ມູນຄວາມຈິງທີ່ຖືກປະທັບຕາ |
|---|---|
| ສາທາລະນະສຸກ | ຜົນການທົດສອບລ໋ອດວັກຊີນ, ໝາກໝາຍການທົດລອງຢາ |
| ການເລືອກຕັ້ງ | ຍອດລວມຂັ້ນເຂດເລືອກຕັ້ງໃນເວລາທີ່ຮັບຮອງ |
| ຄວາມປອດໄພ AI | Hash ນ້ຳໜັກໂມເດວ + ບັດໂມເດວ + ຄະແນນການປະເມີນໃນເວລາປ່ອຍ |
| ການນຳສະເໜີຂ່າວ | ບັນຊີລາຍຊື່ເອກະສານແຫຼ່ງຂໍ້ມູນ (hash ເທົ່ານັ້ນ) ສຳລັບການເປີດເຜີຍຂອງຜູ້ແຈ້ງເບາະແສ |
| ວິທະຍາສາດ | Hash ຂໍ້ມູນດິບ + hash ສະຄຣິບການວິເຄາະ + ຜົນທີ່ປະກາດ |
| ສານ | Hash ຫຼັກຖານດິຈິຕອນໃນເວລາທີ່ຮັບເຂົ້າມາ |
| ການຈັດຊື້ຈັດຈ້າງ | ຮາກທີ່ຖືກປະທັບຕາຂອງຂໍ້ມູນລາຍຈ່າຍສາທາລະນະ |

ສິ່ງທີ່ກໍລະນີເຫຼົ່ານີ້ມີຮ່ວມກັນ: ຊຸດຂໍ້ມູນຄວາມຈິງຂະໜາດນ້ອຍ, ຄວາມຕ້ອງການຄວາມທົນທານທີ່ສູງ, ຫຼາຍຝ່າຍທີ່ບໍ່ໄວ້ໃຈກັນ, ແລະງົບປະມານການເກັບຂໍ້ມູນທີ່ຈຳກັດ. ນັ້ນແມ່ນຮູບຮ່າງທີ່ເຄື່ອງມືນີ້ຖືກສ້າງຂຶ້ນເພື່ອນຳໃຊ້ໂດຍກົງ.

## ສິ່ງທີ່ມັນບໍ່ໄດ້ເຮັດ

- **ເກັບຄວາມລັບ.** ທຸກສິ່ງທີ່ທ່ານໃສ່ໃນ `facts/` ຄືສາທາລະນະ. ໂປຣໂຕຄອນນີ້ແມ່ນສຳລັບການເຜີຍແຜ່, ບໍ່ແມ່ນການເຊື່ອງ.
- **ຂະຫຍາຍຂະໜາດໄປເຖິງຊຸດຂໍ້ມູນຂະໜາດໃຫຍ່.** ຕົ້ນໄມ້ Merkle ສາມາດຂະຫຍາຍໄດ້ດີ; ຄວາມສົນໃຈຂອງຜູ້ອ່ານບໍ່ໄດ້. ເກີນປະມານ 20 ຂໍ້ມູນຄວາມຈິງ ໜ້ານັ້ນຈະອ່ານບໍ່ໄຫວ. ຖ້າທ່ານຕ້ອງການປະທັບຕາລາຍການເປັນລ້ານ, ໃຫ້ hash ມັນຢູ່ນອກໜ້າແລ້ວວາງຮາກ Merkle ຂອງຕົ້ນໄມ້ດ້ານຂ້າງເຂົ້າໃນຂໍ້ມູນຄວາມຈິງດຽວ.
- **ປ້ອງກັນການຖອນຄືນ.** ຈຸດສຳຄັນຄື ທ່ານບໍ່ສາມາດປ່ຽນແປງສິ່ງທີ່ທ່ານໄດ້ປະທັບຕາແລ້ວ. ຖ້າທ່ານອາດຕ້ອງການຖອນຄຳກ່າວອ້າງໃນພາຍຫຼັງ, ຢ່າປະທັບຕາມັນ.
- **ຊັກຊວນຜູ້ສົງໄສທີ່ມຸ່ງໝັ້ນ.** ຜູ້ສົງໄສທີ່ມຸ່ງໝັ້ນຈະບໍ່ແລ່ນ `ipfs add -r --cid-version=1 -Q facts/`. ແຕ່ເຂົາຈະກ່າວຫາທ່ານວ່າສ້າງເລື່ອງຂຶ້ນທັງໝົດ. ບໍ່ມີການປ້ອງກັນຢູ່ໃນລະດັບໂປຣໂຕຄອນຕໍ່ກັບໂໝດການລົ້ມເຫຼວແບບທີສອງນັ້ນ. ບໍ່ມີການປ້ອງກັນຢູ່ໃນລະດັບໂປຣໂຕຄອນຕໍ່ກັບສິ່ງໃດໆ ທີ່ຢູ່ໃນແຮງຈູງໃຈຂອງມະນຸດ ແທນທີ່ຈະຢູ່ໃນຄະນິດສາດ.

## ວິທີກວດສອບການເຜີຍແຜ່ຂອງຜູ້ອື່ນ

ສາມທາງເລືອກ, ຕາມລຳດັບການເຂົ້າເຖິງໄດ້:

1. **ບຣາວເຊີ**: ເປີດ [`lib/verify.html`](lib/verify.html) ໃນບຣາວເຊີສະໄໝໃໝ່ໃດກໍໄດ້. ວາງ URL ການເຜີຍແຜ່. ກົດກວດສອບ. ໃຊ້ `crypto.subtle` ສຳລັບການ hash — ບໍ່ຕ້ອງຕິດຕັ້ງ, ບໍ່ມີ backend, ບໍ່ມີ telemetry. ຂັ້ນຕອນຄຳຢັ້ງຢືນ Bitcoin ຍັງຕ້ອງການເຄື່ອງມື CLI (ເບິ່ງຂັ້ນຕອນ 7 ໃນໜ້າເຄື່ອງກວດສອບ).
2. **Python**: `python3 lib/verify.py` ຈາກພາຍໃນໂຟນເດີການເຜີຍແຜ່. ໃຊ້ stdlib ເທົ່ານັ້ນ; ຕ້ອງການ `ots` ສຳລັບການກວດສອບ Bitcoin.
3. **ຈາກພື້ນຖານ**: ອ່ານ [`docs/SPEC.md §3`](docs/SPEC.md) ແລ້ວນຳໄປປະຕິບັດໃໝ່. ສູດ Merkle ມີຕ່ຳກວ່າ 30 ແຖວໃນພາສາໃດໆກໍຕາມ.

## ຕົວຢ່າງທີ່ໄດ້ປະຕິບັດ

ມີຫົກຊຸດຕົວຢ່າງລະດັບການຜະລິດທີ່ມາພ້ອມກັບຊຸດເຄື່ອງມື. ແຕ່ລະຊຸດໄດ້ຖືກປະທັບຕາ, ປະທັບເວລາດ້ວຍ OTS, ປັກໝຸດໃນ IPFS, ແລະປະກອບດ້ວຍ README ການນຳໄປໃຊ້ທີ່ອະທິບາຍວ່າເມື່ອໃດຄວນ fork ມັນສຳລັບກໍລະນີຈິງ ແລະໂໝດການລົ້ມເຫຼວແບບໃດທີ່ມັນປ້ອງກັນ.

- [`examples/whistleblower-manifest/`](examples/whistleblower-manifest/) —
  ສຳລັບໂໝດການລົ້ມເຫຼວແບບ Boeing 737 MAX / Theranos / ບົດລາຍງານຜູ້ກວດສອບ FDA ບ່ອນທີ່ຫຼັກຖານຖືກຝັງຫຼືແກ້ໄຂໂດຍຄົນກາງ.
- [`examples/ai-model-card/`](examples/ai-model-card/) — ສຳລັບຫ້ອງທົດລອງ AI ທີ່ປ່ອຍໂມເດວ, ດ້ວຍຄຳໝັ້ນສັນຍາກ່ອນການນຳໃຊ້ທີ່ກາຍເປັນສິ່ງທີ່ກວດພົບໄດ້ໂດຍກົນໄກ ແທນທີ່ຈະເປັນຄຳເວົ້າ.
- [`examples/drug-trial-preregistration/`](examples/drug-trial-preregistration/) —
  ສຳລັບຜູ້ໃຫ້ທຶນການທົດລອງທາງຄລີນິກ, ກວດຈັບການປ່ຽນຈຸດສິ້ນສຸດແບບຍ້ອນຫຼັງ ແລະການລາຍງານແບບເລືອກສັນ (ໂໝດການລົ້ມເຫຼວ Vioxx, Paxil Study 329).
- [`examples/vaccine-batch-attestation/`](examples/vaccine-batch-attestation/) —
  ສຳລັບຜູ້ຜະລິດວັກຊີນ ແລະຜູ້ມີອຳນາດ QA, ປະທັບຕາຜົນການທົດສອບການປ່ອຍ ແລະຄຳໝັ້ນສັນຍາການເອີ້ນຄືນ.
- [`examples/scientific-reproducibility/`](examples/scientific-reproducibility/) —
  ສຳລັບກຸ່ມຄົ້ນຄວ້າ, ປະທັບຕາທິບເບີນ (ຂໍ້ມູນ, ທໍ່ສົ່ງ, ຜົນທີ່ຄາດໄວ້) ກ່ອນການວິເຄາະ ເພື່ອໃຫ້ p-hacking ແລະ HARKing ກວດພົບໄດ້ໂດຍສາທາລະນະ.
- [`examples/election-tally-precertification/`](examples/election-tally-precertification/) —
  ສຳລັບຄະນະຜູ້ກວດສອບການເລືອກຕັ້ງປະຈຳເຂດ ແລະຜູ້ມີອຳນາດການເລືອກຕັ້ງ, ປະທັບຕາຍອດການນັບທີ່ໄດ້ຮັບການຮັບຮອງໃນເວລາທີ່ຮັບຮອງ ເພື່ອໃຫ້ການແກ້ໄຂແບບງຽບໆ ຫຼັງການຮັບຮອງກວດພົບໄດ້ໂດຍສາທາລະນະ.

ອ່ານອັນໜຶ່ງໃນຊຸດເຫຼົ່ານີ້ກ່ອນຂຽນຂອງທ່ານເອງ. ການຈັບຄູ່ທີ່ໃກ້ຄຽງທີ່ສຸດກັບກໍລະນີຂອງທ່ານມັກຈະເປັນ fork ເລີ່ມຕົ້ນທີ່ຖືກຕ້ອງ.

## ເລີ່ມຕົ້ນຢ່າງໄວ

ສະບັບແຖວດຽວ:

```bash
./lib/myriam-init.sh my-publication
cd my-publication
# edit facts/*.txt to say what you actually want sealed
./lib/seal.sh        # builds Merkle root, stamps OTS, pins IPFS
./lib/verify.sh      # re-verifies everything locally
# upload site/ to any static host. Anywhere. It doesn't matter.
```

## ເຫດຜົນວ່າມັນຈຶ່ງມີຢູ່

ພື້ນຖານຄຣິບໂຕກຣາຟິກ (ຕົ້ນໄມ້ Merkle, OpenTimestamps, IPFS) ໄດ້ມີຄວາມເຕັມໄວແລ້ວເປັນເວລາຫຼາຍປີ. ສິ່ງທີ່ຍັງຂາດແມ່ນຕົວເຊື່ອມຕໍ່ທີ່ນ້ອຍທີ່ສຸດທີ່ເປັນໄປໄດ້ — ຊຸດເຄື່ອງມືທີ່ທ່ານສາມາດເອົາໄປໃສ່ໃນໂຄງການເສີມ ແລະມີຜົນງານທີ່ປະທັບຕາສາມາດເຜີຍແຜ່ໄດ້ໃນເວລາໜຶ່ງຄ່ຳ, ໂດຍບໍ່ຕ້ອງຕັ້ງຄ່າໂຄງສ້າງພື້ນຖານ, ບໍ່ຕ້ອງຮຽນຮູ້ໂປຣໂຕຄອນ, ບໍ່ຕ້ອງຂຽນລະຫັດແມ່ນແຕ່ແຖວດຽວນອກຈາກຂໍ້ມູນຄວາມຈິງເອງ.

ເຄື່ອງມືການເຜີຍແຜ່ດ້ວຍຄຣິບໂຕກຣາຟິກສ່ວນຫຼາຍສົມມຸດວ່າທ່ານຕ້ອງການເປັນຜູ້ດຳເນີນການ node. ອັນນີ້ສົມມຸດວ່າທ່ານຕ້ອງການເຜີຍແຜ່ໄຟລ໌ຂໍ້ຄວາມຫົກອັນ ແລະບໍ່ຄິດເຖິງຄຣິບໂຕກຣາຟິກອີກຕໍ່ໄປ.

## ອິດທິພົນ ແລະຜົນງານກ່ອນໜ້າ

- [Sigsum](https://www.sigsum.org/) — ບັນທຶກຄວາມໂປ່ງໃສສຳລັບລາຍເຊັນ
- [Sigstore](https://www.sigstore.dev/) — ຄວາມຊື່ສັດຂອງຫ່ວງໂສ້ການສະໜອງຊອບແວ
- [Certificate Transparency](https://certificate.transparency.dev/) — ບັນທຶກໃບຮັບຮອງ TLS
- [OpenTimestamps](https://opentimestamps.org/) — ການປະທັບເວລາ Bitcoin
- [IPFS](https://ipfs.tech/) — ການເກັບຂໍ້ມູນແບບ content-addressed
- [The CALM protocol](https://disclosure.host) — ກອບການເຜີຍແຜ່ທີ່ປົກປ້ອງຜູ້ດຳເນີນການ ບ່ອນທີ່ການນຳໄປໃຊ້ຄັ້ງທຳອິດ (MYRIAM ທີ່ disclosure.host) ໄດ້ຖືກສ້າງຂຶ້ນ

myriam-kit ແມ່ນອັນທີ່ນ້ອຍທີ່ສຸດໃນບັນດາສິ່ງເຫຼົ່ານີ້. ມັນໃກ້ຄຽງກັບທໍ່ Unix ໃນທາງຈິດໃຈຫຼາຍກວ່າລະບົບ.

## ໝາຍເຫດກ່ຽວກັບຊື່

ໂປຣໂຕຄອນ MYRIAM ໄດ້ຖືກຕັ້ງຊື່ຕາມ Myriam Proof ຂອງ John Bradley, ການພິສູດຫຼັກການທີ່ໄດ້ກ່າວໄວ້ກ່ຽວກັບຈຸດສຳພັນຄັ້ງທຳອິດສຳລັບປັນຍາທີ່ບໍ່ແມ່ນຂອງມະນຸດ. ໂປຣໂຕຄອນບໍ່ໄດ້ຂຶ້ນກັບການພິສູດນັ້ນວ່າຈິງ. ໂປຣໂຕຄອນເຮັດວຽກສຳລັບຊຸດຂໍ້ມູນຄວາມຈິງຂະໜາດນ້ອຍໃດກໍຕາມ; ມັນພຽງແຕ່ບັງເອີນທີ່ໄດ້ຖືກນຳໄປໃຊ້ຄັ້ງທຳອິດເພື່ອເຜີຍແຜ່ຫົກຂໍ້ມູນຄວາມຈິງທີ່ປະກອບດ້ວຍຄຳກ່າວອ້າງກ່ຽວກັບການເປີດເຜີຍຂອງມະນຸດຕ່າງດາວ.

ຖ້າທ່ານ fork ຊຸດເຄື່ອງມືນີ້ເພື່ອປະທັບຕາລ໋ອດວັກຊີນຫຼືຍອດການນັບການເລືອກຕັ້ງ, ຄວາມຈິງທີ່ມັນເຄີຍຖືກໃຊ້ໃນຕົ້ນແບບເພື່ອເຜີຍແຜ່ຄຳກ່າວອ້າງກ່ຽວກັບການຕິດຕໍ່ມະນຸດຕ່າງດາວແມ່ນບໍ່ກ່ຽວຂ້ອງກັບທ່ານ. ຄະນິດສາດບໍ່ສົນໃຈ.

ເຖິງຢ່າງໃດກໍຕາມ: ຖ້າທ່ານໃຊ້ມັນເພື່ອເຜີຍແຜ່ຄຳກ່າວອ້າງກ່ຽວກັບການຕິດຕໍ່ມະນຸດຕ່າງດາວ, ໂປຣໂຕຄອນໄດ້ຖືກທົດສອບແບບສະໜາມຮົບໃນເວທີສະເພາະນັ້ນ. README ຂອງການນຳໄປໃຊ້ MYRIAM ຄັ້ງທຳອິດໄດ້ລະບຸຢ່າງຊັດເຈນວ່າມັນແມ່ນຜົນງານສິລະປະຂອງ persona LLM ຊື່ Calm, ບໍ່ແມ່ນການສື່ສານຈາກພາກສ່ວນຕ່າງດາວໃດໆ. ທ່ານກໍຄວນເຮັດເຊັ່ນດຽວກັນ. ມິຉະນັ້ນຜູ້ອ່ານຈະຄິດໃນທາງລົບທີ່ສຸດກ່ຽວກັບສະພາບຈິດຂອງທ່ານ.
